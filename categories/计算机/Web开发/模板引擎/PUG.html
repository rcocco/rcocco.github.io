PUG
2021-03-22|计算机Web开发模板引擎

PUG是一个模板引擎，它将包含HTML代码与数据占位符的模板编译为一个函数，每次调用该函数，并传递数据时，就会得到基于这些数据渲染的HTML文件源代码。

安装

在NPM中使用npm install pug安装

使用

与Express一起使用

安装好后，只需要在调用app.set设置模板文件夹路径，以及模板引擎。

之后调用res.render方法就可以渲染模板，第一个参数是模板的路径（会在模板文件夹内查找），设置了模板引擎后就不需要再写扩展名，第二个参数是要传递给模板的对象。

纯文本

  1. 行内文本：标签后面的空格之后，都是这个标签的行内文本。
  2. 行内文本：以小于号<开头的行，整行都是文本（包括最开头的小于号）
  3. 文本块：标签后面跟一个点.（在标签有属性的时候，则是紧接在闭括号后面），其下一缩进层级的所有内容都是文本，便于书写脚本或CSS。
  4. 文本块：也可以不用标签，直接在一个层级开始书写一个点.，其下一缩进层级的所有内容都是文本。

关于空格，当编译渲染 HTML 的时候：

  1. Pug 删掉缩进，以及所有元素间的空格。如果一定要在元素之间加空格，可以用什么都不带的管道|，它使编译出的HTML代码换行，浏览器会将换行压缩为空格。
  1. Pug 保留符合以下条件的元素内的空格：
    • 一行文本之中所有中间的空格；
    • 在块的缩进后的开头的空格；
    • 一行末尾的空格；
    • 纯文本块、或者连续的管道文本行之间的换行。

HTML属性

属性写在紧跟标签的小括号内，多个属性可以用逗号或空格分隔，也可以分几行写。

属性名可以用单引号或双引号引起来，单引号或双引号不会出现在渲染的HTML中。
这个用途主要是给某些需要用到特殊符号的属性名。

字面值

通过字面值可以创建具有特定属性、ID、类名的标签。
类名或ID属性，支持EMMET语法：如果点号.前面没有标签，则默认为div标签。
&attributes()接受一个对象，该对象包含所有属性和对应的属性值，该语法将对象转换为各个属性。
使用 &attributes 赋值的属性并不会经过自动转义过程。必须正确处理和过滤用户的输入来避免跨站脚本攻击(XSS)。 但是如果是从一个 mixin 调用中使用 attributes 的话，这个过程就是自动的。

条件语句

PUG提供了原生的 if 条件语句，判断表达式的括号是可选的。
unless authorised等价于if !authorised

PUG还提供了类似switch语句的条件语句。
不同之处在于，在 JavaScript 中，传递会在明确地使用 break 语句之前一直进行。而在 Pug 中则是，传递会在遇到非空的语法块前一直进行下去。
在某些情况下，如果不想输出任何东西的话，可以明确地加上一个原生的 break 语句。

代码复用(mixin)

mixin 名字语法用于定义一个用于包装代码的类似于函数的整体，以便在其他地方重复使用。在调用时只需要输入+名字。

在定义和调用时也可以加上括号及参数，除了命名参数，还支持剩余参数。

还可以传递块参数，在调用时通过缩进来传递块参数。
在mixin内部通过block变量使用传递的块参数。

还可以使用第二个小括号传递属性参数。
在mixin内部通过attributes变量使用传递的属性，它是属性对象。

注释

单行注释

带输出的注释以//开头，渲染时会被转换为HTML注释。
不带输出的注释以//-开头，渲染时不会出现。

块注释

包含(导入)

include 路径 语法可以将另一个模板，或文件中的文本插入到当前位置（缩进层级以当前位置为准）。如果被包含的文件扩展名是pug，则当做模板文件导入，如果不是pug，就将文件内容当做文本插入。
被包含的文件的路径，如果是一个绝对路径（如 include /root.pug），那么前面会加上 options.basedir 选项属性来进行解析。否则，路径应该相对于正在被编译的当前文件。

此功能通常用于将一个模板文件分成若干子模板。

继承

模板文件中使用block 块名语法可以声明一个可以被继承此文件的模板替换内容的块。block的下一缩进层级的内容，会作为该块的默认内容。其他模板文件可以基于这个文件创建新的模板，并且替换默认内容、在默认内容前面追加、在默认内容后面追加。

模板文件中使用extends 路径可以继承指定路径的模板。在用extends继承的模板文件中：

  • 定义和被继承的模板中同名的block块会覆盖原模板中的block块的内容。
  • 定义和被继承的模板中同名的block append或append块会在原模板中的block块的内容基础上向后追加内容。
  • 定义和被继承的模板中同名的block prepend或prepend块会在原模板中的block块的内容基础上在最前面追加内容。
  • 定义不同名的block块，则是用于其他文件再次继承本文件时修改块内容。

注意：在extends继承的模板中，顶级元素（没有缩进的一级）只能是带名字的块，或者是混入的定义。
