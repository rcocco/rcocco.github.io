安装

使用NPM安装Hexo：npm install -g hexo-cli

命令行

选一个合适的位置，输入hexo init <folder>来创建一个Hexo项目目录，它会自动创建初始项目结构。
输入cd <folder>进入目录。
最后输入npm install安装所有依赖。

其他常用命令如下：

  • hexo new [scaffold] <title>：创建一篇使用scaffold模板，标题为title的文档，例如hexo new post "Hello World"。
  • hexo generate或hexo g：基于当前源文件生成博客网站，默认放到public目录下。
  • hexo server：启动Hexo内置的服务器，以便你能查看网站效果，默认网址为http://localhost:4000/

项目结构

默认创建的Hexo项目结构如下：

node_modules：存放Hexo项目的所有依赖模块
package-lock.json：描述模块信息的文件
package.json：描述模块信息的文件
scaffolds：存放hexo new命令使用的模板，模板会作为新文件的默认内容
source：存放所有源文件的文件夹。里面的Mardown和HTML文件会被渲染，除了以_和.开头的文件和文件夹外，其他文件和文件夹会被相对于source目录原封不动的复制到输出路径（_post是唯一的例外，它里面的内容也会被处理）
themes：存放主题文件的文件夹，主题可以让你控制生成的网站的布局和样式。
yarn.lock：描述模块信息的文件
_config.yml：本Hexo项目的配置文件
.gitignore：描述Git应该忽略哪些内容的文件
_config.landscape.yml：默认主题landscape的配置文件

源文件

source目录是最主要的目录，文章或者单独的页面都存放在这里。Hexo中文章是以Markdown文件的形式存在的。

  • 想要作为最终博客的文章的md文件放在source/_posts目录中
  • 暂时不想发布为文章的md文件放在source_drafts目录中
  • 其他在source目录内的文件，在生成网站时，它相对于source的位置，也是它最终相对于public的位置，例如source/a/Hello.md，最终生成网站时它会在public/a/Hello.html

主题

在Hexo中，如果你想改变最终生成页面的显示效果，唯一的办法就是自定义主题。
在自定义主题中，你只能写6个页面，它们的文件名（不包括扩展名）及用途如下：

  • index：首页，必须有
  • post：文章页面，如果没有则用index
  • page：非文章页面，如果没有则用index
  • archive：归档页面，如果没有则用index
  • category：分类页面，如果没有则用archive
  • tag：标签页面，如果没有则用archive

这些页面需要通过模板引擎（例如PUG）来创建，Hexo会提供文章正文等数据，由你负责在页面中安排这些数据显示的位置和效果。

插件

Hexo有插件系统，它提供了很多接口，可以让你扩展Hexo的功能，而不必直接去修改Hexo的源代码。
最简单的编写插件的方法的方法，就是在项目目录创建一个名为scripts的文件夹。任何在此文件夹内的.js文件都将在Hexo运行时被执行。

处理器

处理器对source_dir目录下的文件进行处理，当处理的文件的路径匹配pattern时，才会执行回调函数fn。如果没有提供pattern，则处理所有文件时都会执行回调函数fn。
使用hexo.extend.processor.register()注册一个处理器，内置的处理器会比你注册的处理器优先执行。

当pattern是一个函数时，如果不匹配传入的路径str，它应该返回undefined。如果匹配，它应该返回一个对象，其对象的各属性可在file.params中得到。
回调函数fn的参数file具有以下属性：

  • source：文件的完整路径
  • path：文件相对于source的路径
  • type：文件类型。有create, update, skip, delete。
  • params：由pattern返回的各项数据。

生成器

生成器根据处理完成的文件来创建最终的网站文件。
生成器的名字不能相同，如果相同会覆盖，因此你可以覆盖Hexo的内置生成器，当然你可能需要把内置生成器的行为重新实现一遍，否则某些功能会缺失。

path是相对于输出根目录的，不包括前缀/。
layout是可选的，如果不用布局应该忽略它。
data是要创建的数据，可以是一个可读流。

对于Hexo内置数据库的Asset表下的文件（例如图片），内置生成器会根据数据的source（是一个get访问器，等于当前工作目录base_dir加_id）创建可读流，然后根据数据的path字段输出文件。因此修改path字段可以改变输出路径。

数据库

Hexo使用了warehouse这个JSON格式的数据库来管理内部数据。在hexo/lib/models中可以看到Hexo中使用的数据模型（数据表），在脚本文件中通过模型的名字就可以访问到，例如hexo.model("Asset")。
模板中使用的变量都是数据库中的数据，有时你无法在模板中正确输出变量的内容时，不妨到hexo/lib/models中看看变量的类型。

由Vnote笔记本生成网站

我一直使用Vnote 2.8.2来书写Markdown笔记，以及进行笔记的整理，它支持的MathJax数学公式显示，以及代码高亮等功能很好的满足了我的需求。
Vnote中一个笔记本对应一个目录，笔记本中可以有很多个文件夹，用于存放.md格式的笔记，每个文件夹也可以有子文件夹。

在使用Hexo时，我希望的是能够不改动Vnote笔记本内的任何文件结构就能生成博客。这样既不影响我继续使用Vnote软件管理笔记，也可以将笔记输出为博客中的文章。

自动化构建

为了让社区Actions有权限将内容推送到你指定的仓库，你需要生成自己的个人访问Token：

  1. 在Github页面，点击右上角你自己头像，在弹出的列表中选择 Settings
  2. 在左侧面板中选择 Developer Settings
  3. 在左侧面板中选择 Personal access tokens
  4. 点击右侧的 Generate new token 按钮
  5. Note部分随便写，这是你给这个Token的注释，Select scopes部分，勾选 repo (Full control of private repositories)，点击按钮 Generate token
  6. 生成成功，复制这个Token

为了让你自己的Actions能够访问到这个token，还需要添加为 Secrets：

  1. 进到你用来保存笔记以及Hexo项目的仓库（也就是这个Actions所属的仓库）
  2. 点击上方靠右的三个点，选择 Settings
  3. 在左侧面板中选择 Secrets
  4. 点击右侧的 New repository secret 按钮
  5. Name 一栏输入API_TOKEN_GITHUB，Value一栏输入你刚复制的Token，点击 Add secret按钮

大功告成，现在你每次写完笔记后，只要push到你用来保存Hexo项目和笔记的仓库，它就会自动构建并推送到你Github Pages所在的仓库。

注意，上面的自动化脚本中使用actions/setup-node@v1社区Actions来在虚拟机上安装Node.js，但它安装的Node.js目前似乎不支持npm v7生成的package-lock.json，你可能需要将本地的npm降级到npm v6后重新生成package-lock.json。
