安装

首先安装hexo命令行工具：npm install -g hexo-cli

命令行

选一个合适的位置，输入hexo init <folder>来创建一个Hexo项目目录，它会自动创建初始项目结构。
输入cd <folder>进入目录。
最后输入npm install安装所有依赖。

hexo new [layout] <title>：创建一篇使用layout布局，标题为title的文档。
hexo generate或hexo g：基于当前源文件生成网站。
hexo server：启动静态文件服务器，默认网址为http://localhost:4000/
hexo publish：将draft目录下的文件移出放到post目录中

模板引擎

在Hexo项目目录下输入命令npm install hexo-renderer-pug --save安装PUG模板引擎。

项目结构

默认的Hexo项目结构如下：

node_modules：npm模块相关文件
scaffolds：存放模板的文件夹
source：存放所有源文件（文章、图片等）的文件夹。此文件夹内的Mardown和HTML文件会被渲染，除了以_和.开头的文件和文件夹外，其他文件和文件夹会被相对于source目录原封不动的复制到输出路径（_post是唯一的例外，它里面的内容也会被处理）
_config.yml：hexo的配置文件
.gitignore：用于配置Git忽略哪些文件
package.json：node项目的信息，包括依赖等。

源文件目录

每个layout对应一个_post、_draft文件夹，存放特定模板的文件
hexo new page c将创建一个名为c的文件夹和c/index.html文件

post_asset_folder设为true可以使用asset目录，它在_posts下，每当hexo new aaa创建文章时，都会自动创建aaa文件夹和a.md文件
在md文件中输入{% asset_img mypic.jpg %}会引用对应的图片

页面前件

scaffolds中放的都是md文件，它用于创建默认信息，包括默认前件和默认内容。
它只在hexo new [layout] <title>的时候有用。
前件中的{{ title }}是用户输入的

在Hexo中，如果你想改变文章的显示效果，唯一的办法就是自定义主题。
在自定义主题中，你只能写6个页面，它们的文件名（不包括扩展名）及用途如下：

  • index：首页，必须有
  • post：文章页面，如果没有则用index
  • page：非文章页面，如果没有则用index
  • archive：归档页面，如果没有则用index
  • category：分类页面，如果没有则用archive
  • tag：标签页面，如果没有则用archive

将网上的主题git clone到项目的主题文件夹。

处理器

处理器对source_dir目录下的文件进行处理，当处理的文件的路径匹配pattern时，才会交给由fn处理。
若pattern是一个函数时，如果不匹配，它返回undefined。
若匹配，它应该返回一个对象，其对象的各参数可在file.params中得到。
回调函数的参数file具有以下属性
source 文件完整路径
path 文件相对于 Box 的路径
type 文件类型。有 create, update, skip, delete。
params

内置的处理器先执行。
参考hexo/lib/plugins/processor

生成器

先处理器再生成器

path是相对于输出根目录的，不包括前缀/。
layout是可选的，如果不用布局应该忽略它。
data是要创建的数据，可以是一个可读流。

对于Asset表下的文件（例如图片），它会根据模型的source（是一个get访问器，等于当前工作目录base_dir + _id）创建可读流，然后根据模型的path字段输出文件。
因此修改path字段可以改变输出路径。
