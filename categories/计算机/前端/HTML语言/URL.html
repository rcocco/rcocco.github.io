URL概述

URL的全称是统一资源定位符（Uniform Resource Locator），它的实质是一个特定的资源（如HTML页面、js脚本、图片、视频等等）在Web上的地址，一个URL由不同部分组成，这些部分有些是必须有的，有些则是可选的。以下面的URL为例：
http://www.example.com:80/path/to/myfile.html?key1=value1&key2=value2#SomewhereInTheDocument

  1. http://是协议，它表明了浏览器必须用哪种协议去访问它，对于网页常用的有http://和https://，另外还有诸如邮件客户端mailto:，处理文件传输的ftp://等协议。这些协议有些需要加双斜杠//，有些不需要，根据互联网标准RFC3986，如果一个URI存在授权组件，那么在授权组件之前必须加//，不必强记，用到一种新协议之前去查阅相关规则即可。
  2. www.example.com是域名，它表明正在请求哪个Web服务器，也可以直接使用IP，但IP难以记忆且不太方便，人们通常还是会用域名。
  3. :80是端口，域名只能定位到网络上的某台主机，但主机上有很多执行不同任务的程序，端口就负责定位到主机上特定的程序，http://协议的默认端口是80，https://协议的默认端口是443，只要服务器不搞特殊改变了默认端口号，那么在URL中这两个常见协议的端口可以省略。
  4. /path/to/myfile.html是Web服务器上资源的路径，类似自己电脑中某个文件的路径。在早期，这样的路径就是服务器上文件的实际位置，但如今很多Web服务器已经能够采用虚拟路径，不一定和文件实际存放位置相对应。
  5. ?key1=value1&key2=value2是发送给Web服务器的额外参数，这些参数的格式是关键字=值，中间用&号分隔。Web服务器自己会决定处理或不处理这些参数，以及如何理解每个参数的含义。
  6. #SomewhereInTheDocument是资源本身的锚点，用来定位到资源的某个部分，例如在HTML文档中#SomewhereInTheDocument会使浏览器滚动到当前页面中id属性为SomewhereInTheDocument的元素所在位置。

绝对URL

绝对URL的路径部分以斜杠/开头，通常有三种类型：

  1. 完整网址
  • 假设当前文档的URL是https://example.com/path/rel，那么上面URL的完整地址是https://example.com/path/Skills/under
  • 假设当前文档的URL是https://example.com/path/rel/，那么上面URL的完整地址是https://example.com/path/rel/Skills/under

Data URL

Data URLs 由四个部分组成：前缀(data:)、指示数据类型的MIME类型、如果非文本则为可选的base64标记、数据本身：
data:[<mediatype>][;base64],<data>
mediatype 是个 MIME 类型的字符串，例如 “image/jpeg” 表示 JPEG 图像文件。如果被省略，则默认值为 text/plain;charset=US-ASCII
如果数据是文本类型，你可以直接将文本嵌入 (根据文档类型，使用合适的实体字符或转义字符)。如果是二进制数据，你可以将数据进行base64编码之后再进行嵌入。
