head概述

<head>标签用于存放元数据信息，而用来表示元数据的元素有六个：<title>，<base>，<link>，<style>，<script>，和<meta>，对于前五个元素不能表示的任何元数据信息，都应该用<meta>元素表示。
所有能放在<head>元素中的内容参见：joshbuchea/HEAD

根URL

<base>元素用来指定网页中所有相对URL的根路径，以及所有链接的默认浏览上下文。这个根路径既可以是绝对URL也可以是相对URL，但必须是一个目录而不能是一个具体文件，它应该放在所有含有URL相关属性的元素（例如具有href属性的<link>元素）之前。一个网页只有第一个<base>元素是有效的，常见用法是

  • href属性指定页面中所有相对URL的根路径
  • target属性指定页面中所有链接的默认浏览上下文，例如在当前窗口打开还是在新窗口打开，这个属性对所有链接都有效，与链接本身是相对URL还是绝对URL无关。可以被链接自己的target属性覆盖。

添加内部样式

<style>元素用来包含样式表，可以把CSS样式写在<style>元素的内容区域，不推荐使用，因为这种做法将HTML和CSS代码混合在了一起，想修改样式信息也必须同时修改HTML文件，难以维护，所以应该使用外部样式表。

行内脚本

写在元素的内容区域的脚本称为行内脚本

在当前<script>标签内的代码执行完毕之前，浏览器不会继续解析页面。注意，代码中如果出现类似"</script>"的字符串会被认为是结束标签，要避免这个问题，需要对斜线进行转义："<\/script>"

外部脚本

使用src属性引用外部文件的脚本称为外部脚本

内容部分不应该再书写代码，如果书写了则会被忽略。

type属性一般忽略，因为按照惯例它始终是text/javascript，如果设为其他值，可以使得这段代码不执行。如果这个值是 module，则代码会被当成 ES6 模块，而且只有这时候代码中才能出现 import 和 export 关键字。

  • 如果不加async和defer属性，则按以下步骤执行脚本文件：
    1. 开始解析 HTML
    2. 遇到 <script ...> 时，暂停解析 HTML 并开始下载脚本
    3. 脚本下载完毕后，立刻执行脚本，
    4. 待脚本执行完毕后继续解析 HTML 直至完成。
  • 如果加了async属性，则按以下步骤执行脚本文件：
    1. 开始解析 HTML
    2. 遇到 <script async ... > 时开始下载脚本，下载脚本的过程不影响 HTML 的解析
    3. 脚本下载完毕后，暂停解析 HTML 并开始执行脚本，
    4. 待脚本执行完毕后继续解析 HTML 直至完成。
    5. 由于下载完成后立即执行，该属性不能保证按照下载顺序执行脚本，只能保证会在页面的 load 事件前执行，但可能会在 DOMContentLoaded 之前或之后。
  • 如果加了defer属性，则按以下步骤执行脚本文件：
    1. 开始解析 HTML
    2. 遇到 <script defer ...> 时开始下载脚本，下载脚本的过程不影响 HTML 的解析，
    3. 脚本在下载完毕后并不立即执行，而是等待 HTML 解析完毕后再执行。
    4. 由于等待解析HTML完毕后再执行，该属性可以保证按照下载顺序执行脚本，并且会在DOMContentLoaded 事件之前执行。不过在实际当中，推迟执行的脚本不一定总会按顺序执行或者在 DOMContentLoaded事件之前执行，因此最好只包含一个这样的脚本。

网页编码

<meta>元素的charset属性表示这个页面所使用的字符编码，这一条是必须的，而且必须和HTML文件所使用的编码一致，因为浏览器会根据这个值来用对应的编码解析网页，不匹配的编码强行解析将导致乱码。鼓励使用utf-8，例如

刷新或重定向

<meta>元素的http-equiv属性有很多值，这些值是子属性的名称，子属性的具体值由content属性给出，两者配合可以改变服务器和客户端的行为。例如refresh子属性，如果content是数字则表示本页面将在几秒后刷新，如果是数字加链接则表示本页面将在几秒后重定向到指定链接

其他元数据

用<meta>元素表示元数据最基本的形式就是利用name和content属性，name指明元数据的名称，content指明元数据的值。
