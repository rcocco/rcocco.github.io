1.7 设计说明

本节为非规范性内容。

必须承认，HTML的许多方面乍一看似乎是荒谬和不一致的。

HTML及其支持的DOM API，以及许多支持的技术，都是在数十年间由一系列具有不同优先级的人开发的，在许多情况下，他们并不知道对方的存在。

因此，功能来自于许多方面，而且并不总是以特别一致的方式设计。此外，由于网络的特殊性，实现错误往往成为事实上的标准，而现在则是法律上的标准，因为内容往往是在无意中写成的，在它们被修复之前就依赖于它们。

尽管这样，人们还是努力坚持某些设计目标。这些将在接下来的几个小节中描述。

1.7.3 可扩展性

本节为非规范性内容。

HTML具有广泛的可扩展性机制，可用于以安全的方式添加语义。

  • 作者可以使用class属性来扩展元素，有效地创建他们自己的元素，同时使用最适用的现有 "真正的 "HTML元素，这样浏览器和其他不知道该扩展的工具仍然可以在一定程度上支持它。例如，这就是微格式(microformats)所使用的方法。
  • 作者可以使用data-*=""属性为内联的客户端脚本或服务器端的网站脚本提供数据处理。这些属性保证不会被浏览器触及，并允许脚本在HTML元素上包含数据，然后脚本可以查找和处理这些数据。
  • 作者可以使用<meta name="" content="">机制来包含整个页面的元数据。
  • 作者可以使用rel=""机制，通过注册扩展到预定义的链接类型集，来为链接注释特定的含义。这也被微格式所使用。
  • 作者可以使用<script type="">机制嵌入原始数据，并使用自定义类型，以便由内联或服务器端的脚本进一步处理。
  • 作者可以创建插件并使用embed元素来调用它们。这就是Flash的工作原理。
  • 作者可以使用JavaScript原型设计机制来扩展API。例如，这被脚本库所广泛使用。
  • 作者可以使用微数据功能 （itemscope=""和itemprop=""属性）来嵌入嵌套的数据名-值对，以便与其他应用程序和网站共享。

1.8 HTML vs XML 语法

本节为非规范性内容。

HTML规范定义了一种用于描述文档和应用的抽象语言，以及一些用于与使用这种语言的资源的内存表示进行交互的应用编程接口(API)。

内存中的表示被称为DOM HTML，简称DOM

有各种具体的语法可以用来传输使用这种抽象语言的资源，本规范中定义了其中的两种。

第一个这样的具体语法是HTML语法。这是大多数作者建议使用的格式。它与大多数旧版 Web 浏览器兼容。如果一个文档以text/html MIME类型传输，那么它将被Web浏览器当作HTML文档来处理。本规范定义了最新的HTML语法，简称为 “HTML”。

第二种具体的语法是XML。当一个文档以XML MIME类型（如application/xhtml+xml）传输时，那么它就会被Web浏览器当作XML文档来处理，由XML处理器进行解析。请作者注意，对XML和HTML的处理是不同的；特别是，即使是轻微的语法错误，也会阻止标记为XML的文档被完全呈现，而在HTML语法中，这些错误会被忽略。

注：HTML的XML语法以前被称为 “XHTML”，但本规范没有使用这个术语（除其他原因外，还因为MathML和SVG的HTML语法没有使用这个术语）。

DOM、HTML语法和XML语法不能都表示相同的内容。例如，命名空间不能用HTML语法表示，但在DOM和XML语法中都支持。同样，使用noscript功能的文档可以用HTML语法表示，但不能用DOM或XML语法表示。包含字符串"-->"的注释只能用DOM表示，不能用HTML和XML语法表示。

1.9 本规范的结构

本节为非规范性内容。

  1. 简介
    • 为HTML标准提供背景的非规范性材料。
  2. 通用基础设施
    • 一致性类、算法、定义和本规范其他部分的共同基础。
  3. HTML文档的语义、结构和API
    • 文档是由元素构建的。这些元素使用DOM形成一棵树。本节定义了这个DOM的特征，并介绍了所有元素的共同特征，以及定义元素时使用的概念。
  4. HTML的元素
    • 每个元素都有预定的含义，本节将对此进行解释。还给出了作者如何使用该元素的规则，以及用户代理对如何处理每个元素的要求。这包括HTML的大型签名功能，如视频播放和字幕、表单控制和表单提交，以及被称为HTML画布的二维图形API。
  5. 微数据
    • 本规范引入了一种向文档添加机器可读注释的机制，这样工具就可以从文档中提取名-值对的树。本节描述了这种机制和一些可以用来将HTML文档转换成其他格式的算法。本节还定义了一些用于联系信息、日历事件和许可作品的Microdata词汇表样本。
  6. 用户互动
    • HTML文档可以为用户提供一些互动和修改内容的机制，本节对这些机制进行了描述，如聚焦如何工作，以及拖放等。
  7. 加载网页
    • HTML文档并不是凭空存在的的——本节定义了许多影响处理多个页面的环境的功能，如Web浏览器。
  8. Web应用程序API
    • 本节介绍了在HTML中编写应用程序脚本的基本功能。
  9. Web Worker
    • 本节定义了JavaScript中后台线程的API。
  10. Worklet
    • 本节定义了需要与主JavaScript执行环境分开运行JavaScript的API的基础结构。
  11. 通信API
    • 本节描述了一些机制，用HTML编写的应用程序可以用来与运行在同一客户端的不同领域的其他应用程序进行通信。它还介绍了一种被称为 "服务器发送的事件 "或 "EventSource "的服务器推送事件流机制，以及一种被称为Web Sockets的脚本双向全双工套接字协议。
  12. 网络存储
    • 本节定义了一种基于名-值对的客户端存储机制。
  13. HTML语法
  14. XML语法
    • 如果不能以序列化的形式表示并发送给其他人，那么所有这些功能都是徒劳的，因此这些部分定义了HTML和XML的语法，以及如何使用这些语法解析内容的规则。
  15. 渲染
    • 本节定义了网络浏览器的默认渲染规则。
