bootstrap
2021-03-10|计算机前端

Bootstrap是用来建立响应式、移动优先网站的前端框架。
Bootstrap提供了一系列默认样式，每个样式关联到一个类名，通过组合这些类名就将各种样式组合在了一起，形成较好的效果。

js支持

除了引入CSS文件外，Bootstarp中的部分组件还需要js支持。
下拉菜单，气泡提示，弹窗依赖 Popper.js.

浏览器支持

Bootstrap4支持以下版本的浏览器：

  • Chrome >= 45
  • Firefox >= 38
  • Edge >= 12
  • Explorer >= 10
  • iOS >= 9
  • Safari >= 9
  • Android >= 4.4
  • Opera >= 30

如果要支持IE8-9，使用Bootstrap3

注意事项

  • 所有元素的box-sizing被设置为border-box。如果有需要，可以在组件上单独设置为contet-box。
  • 常见元素的默认样式被重置，以便修正不同浏览器之间和设备之间的差异。

布局

容器

容器是Bootstrap中最基本的布局元素，并且当你使用默认的栅格布局时，容器是必须的。
容器用来包含、pad、居中它里面的内容。
容器也可以嵌套，但大多数布局不需要嵌套容器。
Bootstrap提供了三种容器：

  • .container：响应式定宽容器。每个响应式断点都设置了max-width。
  • .container-fluid：全宽容器。每个响应式断点都是width: 100%
  • .container-{breakpoint}：在特定的响应式断点之前是全宽容器，之后是响应式定宽容器。width: 100%一直到特定的响应式断点。

响应式断点

类名中的响应式断点有四个，实际范围有五个。类名中虽然没有xs，但它是默认行为。

  • sm：视口宽度大于等于576px
  • md：视口宽度大于等于768px
  • lg：视口宽度大于等于992px
  • xl：视口宽度大于等于1200px

z-index

bootstrap定义了一组变量来设置z-index的值，以便正确处理元素重叠时的顺序，不鼓励修改这些值，如果你一定要改，你可能需要改很多地方。

栅格系统

Bootstrap的栅格系统使用一系列容器、行、列，来布局和对齐内容。
Bootstrap4的栅格系统基于flexbox，并且是完全响应式的。

基本结构如下：

  • 栅格布局有12列，5种默认响应式层(tier)
  • 容器用来给内容居中并提供一些横向内边距
  • 行用来包装列，每一列都有横向内边距（叫做gutter）来控制列之间的空白。内边距之后会被行的负外边距抵消。这样，列中的所有内容都会左对齐。
  • 栅格布局中，你想展示的内容必须放在列中，并且只有列是行的直接子元素。
  • 由于flexbox，没有指定width的列会自动等宽。例如四个.col-sm从sm断点开始是1/4=25%的宽度。
  • 列类名指示你想占用每行12列中的几列。例如.col-4。
  • 每个列之间有横向内边距来创建间隙，可以在行上加.no-gutters来删除行上的外边距，以及列上的内边距。
  • 要创建响应式栅格，可以用五个栅格断点。
  • 栅格断点会应用到大于等于某个视口宽度的值。
  • 你可以使用预定义栅格类，或者sass中的mixin。

  • 所有类名都有一个宽度为100%的样式，这意味着如果没有到达响应式断点的宽度，所有列都会垂直排列。当达到响应式断点时，会被满足媒体查询而应用的.col-sm等类重写宽度或弹性基本尺寸。
  • 类名中的断点是指，只有视口宽度大于等于该响应式断点时才启用。

等宽列

  • 效果：各列宽度相等。有几列，元素的宽度就是几分之一。
  • 使用：只需要给列加上.col或者.col-sm等，不需要加数字。
  • 实现：.col { flex: 1 1 0; }

等宽多行

  • 效果：在不定义新的row的情况下，让一行变成多行
  • 使用：在需要换行的列后面增加一个元素和类名w-100
  • 实现：它的作用是增加一个宽度为100%但高度0的弹性元素，flex默认是0 1 auto，等价于flex: 0 1 100%。 到该元素时弹性基本尺寸是100%，当前行一定放不下，从而生成新的弹性行，并占用一个完整的弹性行，使得前后的弹性元素分开。

一列定宽多列等宽

  • 效果：一列设置为定宽，其他列设置为等宽
  • 使用：只要给定宽列加上.col-5即配上数字
  • 实现：原理是.col-5 { flex: 0 0 calc(5 / 12 * 100%);}。该定宽列是冻结弹性元素，剩余空间是主轴减去冻结元素宽度。

宽度基于内容的列

  • 效果：该列的宽度取决于内容宽度
  • 使用：只要给该列加上col-auto
  • 实现：.col-auto { flex: 0 0 auto; width: auto; }

移动端和PC端不同布局

  • 效果：在移动端和PC端使用不同的布局
  • 使用：通过结合.col与带有响应式断点的.col-{breakpoint}，又或者只书写带有响应式断点的.col-{breakpoint}结合其自带的默认width: 100%来实现不同视口宽度的不同布局。

间距

bootstrap使用形如px(-breakpoint)?-[n]?[1-5]的格式来表示内边距，这里px表示padding in x水平方向的内边距，即同时设置左内边距和右内边距，类似的还有py、pt、pb、pl、pr表示垂直方向、上、下、左、右内边距。
也有mx、my等等来表示外边距。
数字1-5是五个不同级别的大小，分别是0.25rem、0.5rem、1rem、1.5rem和3rem。对于外边距，数字前可选的n表示负，内边距前面不能有n。
栅格系统默认的间距是15px，不是五个级别的任意一种。
要改变栅格系统的间距并维持良好的对齐显示，需要同时改变三个地方：容器的内边距、行的负外边距、列的内边距。且三者的级别要相同。

折叠

折叠JavaScript插件用于显示或隐藏内容。
折叠功能需要有两个元素，一个用于点击（可以是按钮或a），一个是被折叠的元素。
折叠元素会使其高度从当前值变为0，因此不要在被折叠元素上设置padding。

用于点击的元素必须设置的属性：data-toggle="collapse"。
如果是按钮，还需要有data-target="被折叠元素的JQuery选择器"。
如果是a标签，还需要有href="被折叠元素的JQuery选择器"。
data-target或href中可以选择多个被折叠元素。

被折叠元素类名：.collapse（隐藏）或.collapse.show（显示）。
其类名在动画过程中会变化，
隐藏状态类名是.collapse，
动画过程中类名变成.collapsing，
显示状态类名是.collapse.show。

导航栏

通常用于nav元素。
主类：.navbar
响应式断点：.navbar-expand{-sm|-md|-lg|-xl}
背景配色方案（深色或浅色）：navbar-{light|dark}
具体配色：bg-{light|dark|primary}和style="background-color: #e3f2fd;"
导航栏及其内容默认是fluid的，占整个屏幕宽度，要限制水平宽度需要包围一层container

.navbar的直接子元素使用弹性布局，并且默认是justify-content: space-between的，要改变默认行为需要添加其他弹性类名。

导航栏LOGO

任何元素都可以作为LOGO，可以是图片也可以是文字。通常是a{text}或a>img或a>img+{text}
主类：.navbar-brand

折叠

用于点击的元素类名：.navbar-toggler
被折叠元素类名：.collapse.navbar-collapse

没有触发响应式断点时，不显示用于点击的元素，显示被折叠的元素。
触发响应式断点时，显示用于点击的元素，不显示被折叠的元素。

对于永远不折叠的导航栏，给.navbar添加.navbar-expand类。
对于总是折叠的导航栏，不要添加.navbar-expand类。

通过调整元素在HTML中的位置，可以改变折叠后元素的位置。

如果使用列表，类名就是三层结构、

表单

如果要在导航栏里添加表单和控件，例如搜索框。
给表单添加类名.form-inline

文本

如果要给导航栏添加垂直居中的文本，使用类名.navbar-text

容器

如果用.container包裹.navbar，会使导航栏在页面中居中。
如果用.navbar包裹.container，只要低于指定的响应式断点.navbar-expand，容器的水平padding就会被删除

控件

控件默认为display: block和width: 100%，因此默认垂直排列。
文本形式的控件（如input、select和textarea）设置类名：.form-control
设置了readonly的input:text，使用类名.form-control-plaintext删除文本框。
文件上传input:file设置类名：.form-control-file
范围input:range设置类名：.form-control-range
单选input:radio和多选input:checkbox：
label和控件需要是兄弟元素，分别设置类名。包装元素设置form-check。

分组

控件分组，为filedset或div元素加上类名.from-group。
要使分组内控件水平布局，再加类名.row。分组内的元素使用.col-*-*调整宽度，分组内的label使用.col-form-label来使它垂直居中。

大小

调整控件大小：不写是默认值，更大或更小使用类名.form-control-{lg|sm}。如果调整了大小，也要同时调整label或legend的大小，加上.col-form-label-{lg|sm}。

文本提示

提示文本经常配合淡灰色类使用：.text-muted。
一般是行内元素（如span或div）。如果想让提示文本变成块级元素，使用.form-text，它会添加上外边距，并设置display:block。
