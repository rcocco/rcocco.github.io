Document结点

Document结点简称为文档。
每个文档都有关联的编码、内容类型(content type)、URL、origin、类型、模式。
一个类型为xml的文档称为XML文档，类型为html的文档称为HTML文档。
模式为no-quirks的文档称为非怪异模式的，模式为quirks的文档称为怪异模式的，模式为limited-quirks称为有限怪异模式。
非怪异模式最初被称为"标准模式"，有限怪异模式曾被称为"准标准模式"。由于它们的细节现在由标准定义，因此被重新命名了（同时因为 Ian Hickson 否决了它们的原名，理由是它们是无意义的）

文档的URL

documentURI和URL属性均返回文档的URL

文档模式

compatMode属性返回文档模式。

  • 当文档模式为quirks（怪异模式）时，该属性返回字符串"BackCompat"
  • 当文档模式为no-quirks（非怪异模式）或limited-quirks（有限怪异模式）时，该属性返回字符串"CSS1Compat"

文档编码

characterSet、charset和inputEncoding属性均返回文档的编码，通常是"UTF-8"。charset和inputEncoding是旧版本遗留的别名。

文档的MIME类型

contentType属性返回文档的MIME类型，常见的例如"text/html"、"application/xml"、"application/xhtml+xml"、"image/svg+xml"。

子结点

根据结点树的规定，Document结点的子结点只能是以下几种（按深度优先顺序）：

  1. 零个或多个ProcessingInstruction或Comment结点
  2. 一个可选的DocumentType结点
  3. 零个或多个ProcessingInstruction或Comment结点
  4. 一个可选的Element结点
  5. 零个或多个ProcessingInstruction或Comment结点

获取DocumentType结点

doctype属性返回文档的DocumentType子结点，如果不存在则返回null

获取Element结点

文档的唯一Element子结点通常称为文档元素，在HTML文档中是html元素。

  • documentElement、firstElementChild、lastElementChild属性返回文档的Element子结点，如果不存在则返回null。
  • children属性返回一个动态的HTMLCollection集合，它匹配文档的Element子结点。
  • childElementCount属性返回文档的Element子结点的数量，一般为1。

添加子结点

prepend方法将nodes所有结点插入到this的第一个子节点之前，参数中的字符串被替换为Text结点。
append方法将nodes所有结点插入到this的最后一个子节点之后，参数中的字符串被替换为Text结点。
replaceChildren方法将this的所有子结点移除，然后将nodes所有结点作为this的子结点，参数中的字符串被替换为Text结点。

查找后代元素结点

根据标签名查找

返回一个动态的HTMLCollection集合。

  • getElementsByTagName：当参数qualifiedName为"*"时，集合匹配的是this后代结点中所有的Element结点。
    • 对于通常的HTML文档，集合的根为this，过滤器为匹配this的后代结点中：
      • 位于HTML命名空间，标签名为qualifiedName（不区分大小写）的Element结点。
      • 不在HTML命名空间，标签名为qualifiedName（区分大小写）的Element结点。
    • 例如getElementsByTagName('SVG')就获取不到元素，因为不在HTML命名空间，标签名为svg在区分大小写的情况下不等于参数SVG。
  • getElementsByTagNameNS：当参数namespace和localName都为"*"时，集合匹配的是this后代结点中所有的Element结点。
    • 如果参数namespace不为"*"，则匹配的结点要求namespaceURI等于namespace
    • 如果参数localName不为"*"，则匹配的结点要求localName属性等于参数localName

由于大多数情况都不需要考虑命名空间，因此使用getElementsByTagName更合适。

根据类名查找

返回一个动态的HTMLCollection集合。
参数classNames会根据空白符（[\t\n\f\r ]+）切分成类名，集合会匹配this后代结点中，具有classNames中全部类名的Element结点。
也就是集合中的结点会包含所有类名，它可以有比classNames更多的类名，但不能缺少className中的类名。
quirk（怪异模式）的文档在类名匹配时不区分大小写，标准模式的文档进行类名匹配时区分大小写。

根据ID查找

返回this的后代中（按树顺序，即深度优先遍历顺序）第一个ID为参数elementId的Element结点，如果没有这样的结点则返回null。

拷贝结点

该方法拷贝当前结点并返回一个新的副本结点。不支持拷贝影子根ShadowRoot。
若参数deep为false则不会拷贝子结点。
若参数deep为true则会拷贝子结点以及子结点的后代。
同时被拷贝的每个结点的结点文档(doucment node)都是this。

node.cloneNode(deep)和document.importNode(node, deep)的唯一区别就是前者会同时拷贝结点文档信息，而后者会将结点文档改为document。
从另一个文档拷贝结点时通常用importNode()，从相同文档拷贝结点时通常用cloneNode。

修改结点文档

此方法会将node以及其后代结点的结点文档(node document)修改为this。
如果node目前的结点文档和this不同，则会将node从当前结点文档所属DOM中移除。

创建结点

Document

DocumentType

创建一个DocumentType结点，由于HTML5的doctype为<!DOCTYPE html>，因为最常见的用法是createDocumentType('html', "", "")。后两个参数一般为空串。

DocumentFragment

创建一个DocumentFragment结点。

Element

创建一个标签名为参数localName的Element结点。

  • createElement：
    • 在HTML文档中，
      • 参数localName无所谓大小写，因为最终都会被转为小写。
      • 结点的namespace被设为HTML的命名空间
      • 结点的prefix为null
    • 其他情况：
      • 参数localName区分大小写
      • 结点的namespace和prefix都为null
  • createElementNS：
    • 参数qualifiedName如果是prefix:localName的格式则分割它，否则就是localName。
    • 结点的namespace被设为参数namespace
    • 结点的prefix是null或者分割参数qualifiedName得到的值

在HTML文档中通常使用createElement方法，注意参数无所谓大小写。
参数options一般用不到，它用于自定义元素。

Attr

创建一个属性名为参数localName的Attr结点。

  • createAttribute：
    • 在HTML文档中参数localName无所谓大小写，因为最终都会被转为小写。
    • 结点的namespace和prefix被设为null
  • createElementNS：
    • 参数qualifiedName如果是prefix:localName的格式则分割它，否则qualifiedName就是localName。
    • 结点的namespace被设为参数namespace
    • 结点的prefix是null或者分割参数qualifiedName得到的值

在HTML文档中通常使用createAttribute方法，注意参数无所谓大小写。

Text

创建一个文本数据为data的Text结点。
两种方法区别在于：
createTextNode创建的结点的结点文档(node document)是this。
new Text创建的结点的结点文档(node document)是globalThis.document

CDATASection

创建一个文本数据为data的CDATASection结点。
在HTML文档中调用这个方法会抛出异常。

Comment

创建一个文本数据为data的Comment结点。
两种方法区别在于：
createTextNode创建的结点的结点文档(node document)是this。
new Text创建的结点的结点文档(node document)是globalThis.document

ProcessingInstruction

创建一个target和data为给定参数的ProcessingInstruction结点。

创建事件

此方法是旧版遗留方法，不要使用它，你应该用事件构造函数例如Event来创建事件。

创建范围Range

创建一个实时范围Range对象。
createRange()和new Range()的唯一区别是：
createRange()的开始结点和结束结点都是this，偏移都是0。
new Range()的开始结点和结束结点都是globalThis.document，偏移都是0。
