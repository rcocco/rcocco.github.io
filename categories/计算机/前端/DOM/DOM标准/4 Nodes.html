4 结点

4.1 DOM介绍

在其原始意义上，"The DOM"是一个用于访问和操作文档（特别是HTML和XML文档）的API。在本规范中，术语 "文档"被用于任何基于标记的资源，从短的静态文档到长的文章或带有丰富多媒体的报告，以及完全成熟的互动应用程序。

每个这样的文档都被表示为一个结点树。树中的一些结点可以有子结点，而另一些结点则始终是叶子结点。

4.2 结点树

Document、DocumentType、DocumentFragment、Element、Text、ProcessingInstruction和Comment对象（简称为结点）参与到一棵树中，简称为结点树。

结点树的约束如下，表示为结点类型和其允许的子结点之间的关系。

  • Document：按树顺序
    1. 零个或多个ProcessingInstruction或Comment结点
    2. 一个可选的DocumentType结点
    3. 零个或多个ProcessingInstruction或Comment结点
    4. 一个可选的Element结点
    5. 零个或多个ProcessingInstruction或Comment结点
  • DocumentFragment
  • Element：
    1. 零个或多个Element,Text, ProcessingInstruction或Comment结点
  • DocumentType
  • Text
  • ProcessingInstruction
  • Comment
    1. 无

要确定一个结点node的长度，请打开节点：

  • DocumentType
    1. 零
  • Text
  • ProcessingInstruction
  • Comment
    1. 它的data的长度
  • 其他结点
    1. 它的孩子的数量

如果一个结点的长度为零，则视为空结点。

4.2.1 文档树

文档树是根为Document结点的结点树。

一个Document结点的文档元素是父结点为此Document结点的Element结点，如果存在的话。否则为null

注意：根据结点树的限制，只能有一个这样的元素。

如果一个Element结点的根是Document结点，则称这个Element结点在一个文档树中。

如果一个Element结点在一个文档树中，则称这个Element结点在文档中。注意：术语【在文档中】不应该再被使用，它表明使用它的标准还没有更新影子树。

4.2.2 影子树

影子树是根为影子根的结点树

影子根总是通过它的宿主连接到另一个结点树。因此，一棵影子树永远不是单独的。影子根的宿主的结点树有时被称为光树。

注意：一棵影子树对应的光树本身也可以是一棵影子树。

如果一个Element结点的包含影子根是一个Document结点，那么它就是连接(connected)的。

4.2.2.1 槽

影子树包含零个或多个作为槽(slot)的Element结点。

注意：槽只能通过HTML的slot元素创建

槽有关联的name（字符串），除非另有说明，否则它是空串。

注意：影子树中按照树顺序的第一个name为空字符串的槽，有时被称为 “默认槽”。

一个槽有一个关联的分配结点（一个slottables列表）。 除非另有说明，否则为空。

4.2.2.2 Slottables

Element和Text结点是slottables

注意：一个槽也可以是slottable

slottable有关联的name（字符串），除非另有说明，否则它是空串。

slottable有关联的分配槽（null或slot）。 除非另有说明，否则为空。 如果分配的槽非null，则slottable是已分配的。

4.2.4 Mixin NonElementParentNode

Web兼容性阻止了getElementById()方法在Element元素上的暴露（因此也阻止了ParentNode）。

interface mixin类似于partial interface，都是用于组合多个接口。区别在于定义interface mixin后需要includes这个接口才生效，而partial interface是直接生效的。

——————
对于Web开发者：
node . getElementById(elementId)
返回node的后代中ID为elementID的第一个Element结点。
——————

getElementById(elementId)方法的步骤是，按树顺序，返回this的后代中第一个ID为elementId的元素；如果没有这样的元素，则为null。

4.2.6 Mixin ParentNode

为了将nodes转换为node，给定nodes和document，执行以下步骤：

  1. 令node为null
  2. 将nodes中的每个字符串替换为一个新的data为此字符串，node document为document的文本结点
  3. 如果nodes包含一个结点，将node设为此结点
  4. 否则，将node设为一个新的DocumentFragment，其node document为document，然后将nodes中的每个node（如果有）append到它里面。
  5. 返回node

如果[SameObject]扩展属性出现在一个只读属性上，那么它表明当在一个给定的对象上获得该属性的值时，必须始终返回相同的值。[SameObject] 扩展属性必须不接受任何参数。
[Unscopable]表示with它的时候该属性不会作为标识符

——————
对于Web开发者：
collection = node . children
返回子Element结点。

element = node . firstElementChild
返回第一个子Element结点，否则为null

element = node . lastElementChild
返回最后一个子Element结点，否则为null

node . prepend(nodes)
将nodes插入到node的第一个孩子之前，同时将nodes中的字符串替换为相同的Text结点。如果违反了节点树的约束，抛出一个 “HierarchyRequestError” DOMException异常

node . append(nodes)
将nodes插入到node的最后一个孩子之后，同时将nodes中的字符串替换为相同的Text结点。如果违反了节点树的约束，抛出一个 “HierarchyRequestError” DOMException异常

node . replaceChildren(nodes)
用nodes替换node的所有孩子，同时将nodes中的字符串替换为相同的Text结点。如果违反了节点树的约束，抛出一个 “HierarchyRequestError” DOMException异常

node . querySelector(selectors)
返回node后代中与选择器匹配的第一个Element结点

node . querySelectorAll(selectors)
返回node的后代中与选择器匹配的所有Element结点
——————
childrengetter会返回一个根植于this的HTMLCollection集合，该集合只匹配子Element结点

firstElementChildgetter返回第一个孩子，这个孩子是Element结点。否则为null

lastElementChildgetter返回最后一个孩子，这个孩子是Element结点。否则为null

childElementCountgetter返回this的孩子的中Element结点的数量

prepend(nodes)方法：

  1. 令node为用给定nodes和this的结点文档(node document)执行将nodes转为node的结果
  2. Pre-insert node into this before this’s first child.

replaceChildren(nodes)方法：

  1. 令node为用给定nodes和this的结点文档(node document)执行将nodes转为node的结果
  2. Ensure pre-insertion validity of node into this before null.
  3. Replace all with node within this

querySelector(selectors)方法会返回在this上用selectors执行scope-match a selectors string的第一个结果，如果结果不是空列表的话。否则，返回null

querySelectorAll(selectors)方法会返回在this上用selectors执行scope-match a selectors string的static结果。

4.2.10 旧式集合: NodeList和HTMLCollection

一个集合是表示结点的列表的对象。一个集合可以是live的，也可以是static的。除非另有说明，一个集合必须是live的。

如果一个集合是live的，那么该对象上的特性和方法必须对实际的基础数据，而不是数据的快照进行操作

当一个集合被创建时，一个过滤器和一个根与它相关联。

然后，集合表示以集合的根为根的子树的视图，只包含与给定的过滤器相匹配的结点。该视图是线性结构的。如果没有相反的具体要求，集合中的结点必须按树顺序排序。

4.2.10.1 NodeList接口

NodeList对象是一个结点的集合

4.2.10.2 HTMLCollection接口

HTMLCollection对象是一个Element结点的集合

注意：HTMLCollection是一个我们无法摆脱的历史文物。我们当然欢迎开发者继续使用它，但新的API标准设计者不应该使用它（使用IDL中的sequence<T>代替）。

4.3 变化观察者

每个similar-origin window agent都有一个变化观察者微任务排队(mutation observer microtask queued)（它是一个布尔值），最初是false。[HTML]

每个similar-origin window agent也有变化观察者(mutation observers)（一组零个或多个MutationObserver对象），最初为空。

4.3.1 MutationObserver接口

一个MutationObserver对象可以用于观察结点树的变化。

每个MutationObserver对象都有这些相关概念。

  • 一个创建时的callback集。
  • 一个结点列表（结点的列表），它最初为空。
  • 一个记录队列（由零个或多个MutationRecord对象组成的队列），最初为空。

4.4 Node接口

Node是一个抽象的接口，并不作为结点存在。它被所有结点（Document、DocumentType、DocumentFragment、Element、Text、ProcessingInstruction和Comment）使用。

每个结点都有一个关联的结点文档(node document)，它是一个Document结点，在创建时设置。

注意：一个结点的结点文档可以通过adopt算法来改变。

结点的get the parent算法，给定一个event，如果结点是已分配的，则返回结点的分配槽；否则返回结点的父节点。

注意：每个结点都有一个注册观察者列表(registered observer list)
