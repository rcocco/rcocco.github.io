视觉格式化模型
2020-11-06|计算机前端CSS样式语言

CSS中，视觉格式化模型(Visual Formatting Model)描述了用户代理如何处理文档树，以及如何在视觉媒体上显示。
在视觉格式化模型中，文档树中的每个元素根据盒模型生成零个或更多盒子（矩形框），这些盒子的布局受以下因素控制：

  • 盒子的尺寸和类型
  • 定位模式（常规流、浮动和绝对定位）
  • 文档树中元素之间的关系
  • 外部信息（例如视口大小、图片的固有尺寸等等）

视觉格式化模型中的大多数内容是在CSS2中定义的，CSS3也增加了一些内容，但你仍然可以在CSS3规范中看到许多对CSS2中定义的模型的引用。

名词解释

连续/分页媒体

连续媒体和分页媒体是CSS2定义的媒体组之一。
屏幕是典型的连续媒体，文档在屏幕上是连续的。
幻灯片，或者用打印模式查看的文档是典型的分页媒体，文档在这些媒体上是分成一页一页展示的。

视口

连续媒体的用户代理一般会给用户提供一个视口（屏幕上的一个窗口或者视图区域），用户通过它来浏览文档。如下图：橙色框中的是文档，黑色框是浏览器窗口，用户将黑色区域向下移动来浏览文档的全部内容。

当视口尺寸发生变化时，用户代理可能会改变文档的布局。
当视口比渲染文档的画布区域小时，用户代理应该提供一种滚动机制。一个画布最多对应一个视口，但用户代理可能会渲染到多个画布上（即提供同一文档的不同视图）

包含块

元素（生成的）盒的位置和大小通常是根据一个特定矩形计算的，这个矩形叫做该元素的包含块。尽管每个盒子都相对于包含块确定位置，但它并不会被限制在包含块里，它可以溢出。元素的包含块的具体定义如下：

  1. 根元素所在的包含块是一个被称为初始包含块的矩形。对于连续媒体，初始包含块的尺寸取自视口的尺寸，并且被固定在画布开始的位置（即把视口移到文档最前面时的位置）
  2. 对于其它元素，如果该元素的position属性是relative或static或sticky，包含块由其最近的块容器祖先的 content box 边界，又或者最近的建立了格式化上下文的祖先（如table容器、flex容器、grid容器）的 content box 边界形成。
  3. 如果元素的position属性是fixed，对于连续媒体，包含块由视口建立。
  4. 如果元素的position属性是absolute，包含块由最近的建立了绝对定位包含块的祖先元素的 padding box 组成。（position属性非static，或transform属性非none，或perspective属性非none，或will-change属性值为transform或perspective，或contain属性值为paint会建立绝对定位包含块）
  5. 如果没有这样的祖先元素，包含块就是初始包含块。

常规流

常规流指从左到右，从上到下的顺序显示。元素盒从左到右，从上到下摆放。多数元素都采用常规流动方式，除非元素浮动、定位、放入弹性盒或采用栅格布局了。
常规流中的盒属于一个格式化上下文，可能是块或是行内（格式化上下文），但不能两者都是。块级盒参与块格式化上下文。行内级盒参与行内格式化上下文。
格式化上下文是一块区域，这个区域内的内容会按照上下文规定的规则来进行布局。
通俗的说，常规流就是默认情况下文档中元素的布局。

块格式化上下文

在块格式化上下文中，盒在垂直方向一个接一个地放置，从包含块的顶部开始。两个兄弟盒之间的垂直距离由margin属性决定。同一个块格式化上下文中的相邻块级盒之间的垂直外边距会合并。
在块格式化上下文中，每个盒的左外边界（left outer edge）挨着包含块的左外边界（对于从右向左的格式化，右外边界挨着）。即使存在浮动（尽管一个盒的行框可能会因为浮动而收缩 译注：环绕浮动元素放置的行框比正常的行短一些），这也成立，除非该盒建立了一个新的块格式化上下文（这种情况下，该盒自身可能会因为浮动变窄，如下）

如果此时想令浮动元素和最高的兄弟元素（看起来）一样高，可以让父元素overflow: hidden同时建立新的块格式化上下文，然后给自己和所有兄弟元素加padding，最后给负外边距，使得父元素高度计算根据margin box。
下列情况会为其内容建立新的块格式化上下文：

  • 浮动元素
  • 绝对定位的元素
  • 非块盒的块容器（例如inline-blocks，table-cells和table-captions）
  • overflow不为visible的块盒（当该值已从根元素被传播到视口时除外）
  • display: flow-root

行内格式化上下文

在行内格式化上下文中，盒是从包含块的顶部开始一个挨一个水平放置的。这些盒之间的水平外边距，边框和内边距都有效。盒可能会以不同的方式垂直对齐：以它们的底部或者顶部对齐，或者以它们里面的文本的基线对齐。包含来自同一行的盒的矩形区域叫做行框。

行框的宽度由包含块和浮动情况决定，行框的高度由行高的计算小节给出的规则决定

行框总是足够高，能够容纳它包含的所有盒。然而，它可能比它所包含的最高的盒还要高（例如，如果盒是以基线对齐的）。当盒B的高度小于它所在的行框的高度时，行框中B的垂直对齐方式由vertical-align属性决定。当几个行内级盒在水平方向上不能共存于一个行框时，它们会被分到两个或多个垂直堆叠的 行框里。因此，一个段落就是一叠垂直堆叠的行框。行框没有垂直间隔地堆放（除非在其它地方有特别说明）并且它们不会重叠

一般来说，一个行框的左边界挨着其包含块的左边界，右边界挨着其包含块的右边界。然而，浮动盒可能会跑到包含块边界与行框边界之间。因此，尽管同一个行内格式化上下文中的行框一般都有相同的宽度（即包含块的宽度），如果可用的水平空间因为浮动而减少了的话，它们的宽度就可能不同。同一个行内格式化上下文中的行框一般高度各不相同（例如，一行可能含有一个高图片，而其它的只含文本）

当一行的行内级盒的总宽度小于它们所在的行框的宽度时，它们在行框里的水平分布由text-align属性决定。如果该属性值为justify，用户代理可能还会拉伸行内盒（不包括inline-table和inline-block盒）里的空格和单词

当行内盒超出行框宽度时，它会被分成几个盒，并且这些盒会跨多行框分布。如果一个行内块无法分割（例如，如果该行内盒（只）含单一字符，或者特定语言的单词分隔规则不允许在该行内盒里分隔，或该行内盒受到了一个值为nowrap或者pre的white-space的影响），那么该行内盒会从行框溢出

当一个行内盒被分割后，外边距，边框和内边距在发生分割的地方（或者在任何分割处，如果有多处的话）没有视觉效果

同一个行框里的行内盒也可能因为双向（bidirectional）文本处理而被分割成几个盒

当需要在行内格式化上下文中放置行内级内容时，创建一个行框。对于没有文本、没有保留空白符、没有（外边距、内边距和边框非0）的行内元素、，没有其它流内内容（例如，图片，行内块或行内表格），并且不以保留换行符结束的行框，在确定其内部元素的位置时必须被当做0高度行框；其他情况，必须当它不存在。

定位方案

CSS中，一个盒可能会根据三种定位方案来布局：

  1. 常规流。CSS中，常规流包括块级盒的块格式化、行内级盒的行内格式化、以及块级或行内级盒的相对或粘性定位。
  2. 浮动。在浮动模型中，盒先根据常规流放置，然后从常规流中取出来并尽可能远地向左或向右移动。其他内容可能沿着浮动（盒）的一侧排列。
  3. 绝对定位。在绝对定位模型中，一个盒会从常规流中全部移除（它不会影响后面的兄弟元素）并根据包含块确定位置。
