名词解释

  1. 流外(out of flow)：如果一个元素是浮动的，或者是绝对定位的，又或者是根元素，则称之为流外元素。流外元素的流，是由它本身以及所有（最近的流外祖先为它的）流内元素组成的集合。
  2. 流内(in flow)：如果一个元素不是流外的，则称之为流内元素。流内元素的流，只由它本身组成。

元素盒

CSS假定每个元素都生成一个矩形框，称之为元素盒(element box)，也翻译为元素框。根据元素盒类型的不同，它们的视觉格式化也略有不同。元素盒的类型取决于CSS中的display属性值。该属性值初在CSS2中定义，并且在CSS Display Module Level 3中扩充。

  1. 元素框的中心是内容区域(content area)，也叫 content box
  2. 内容区域外侧是一圈内边距区域(padding area)，内边距区域与内容区域合起来称为 padding box
  3. 内边距区域外侧是一圈边框区域(border area)，内边距区域、内容区域、边框区域合起来称为 border box
  4. 边框区域外侧是一圈外边距区域(margin area)，内边距区域、内容区域、边框区域、外边距区域合起来称为 margin box
  • 内边距、边框和外边距是可选的，它们的宽度可以为0，效果就是元素框上没有这部分区域。
  • 元素的文字、子元素等都是放置在 content box 中的。
  • 背景颜色占据整个 border box，包括边框区域，如果你把边框设为虚线，是可以看到背景颜色的。
  • 当鼠标进入 border box 时，被认为在元素上面，也就是触发:hover的范围。在外边距区域是不会触发:hover的。
  • 外边距区域总是透明的，可以透过它看见其他元素的背景或内容。
  • CSS中用width和height属性表示宽度和高度，这两个属性适用于行内非置换元素以外的几乎所有元素，但它们究竟是指元素框哪部分的宽度和高度，取决于box-sizing属性。这是一个不继承的属性，你只能为每一个元素单独设置。box-sizing的属性值有两种：content-box和border-box。
    • 属性值为content-box时，内容的宽度就是width，内容的高度就是height。
    • 属性值为border-box时，内容的宽度是width减去左右边框宽度与内边距。内容的高度是height减去上下边框宽度与内边距。宽度和高度计算值的下限是0。另外，auto的width和height不受box-sizing属性影响，并始终设置为内容的宽度和高度。

盒的类型

盒的类型对其在视觉格式化模型中的行为有一定影响。
display属性指定了盒的类型，元素盒的类型分为外部显示类型和内部显示类型：

  • 外部显示类型描述的是元素盒与页面上其他元素挨着时的行为，外部显示类型包括block和inline。
    • block：元素放置在流布局中，会生成一个块级盒。
    • inline：元素放置在流布局中，会生成一个行内级盒。
  • 内部显示类型描述的是它的子元素的布局行为。内部显示类型包括flow、flow-root、table、flex、grid和ruby。
    • flow：该元素使用流布局来放置它的内容。如果外部显示类型为inline，并且它参与块格式化上下文或者行内格式化上下文，则会生成行内盒。否则会生成块容器盒。根据其他属性的值（例如position、float和overflow）以及它本身是否参与了块或行内格式化上下文，它要么为其内容建立新的块格式化上下文，要么将它的内容融入它的父格式化上下文。
    • flow-root：该元素生成块容器盒，并且使用流布局来放置它的内容。它总是会建立一个新的块格式化上下文。
    • flex：该元素生成一个principal flex容器盒，并且建立flex格式化上下文。

旧的display属性没有分开描述外部显示类型和内部显示类型：

块级元素与块盒

  1. 块级元素：源文档中那些被格式化成视觉上的块的元素（例如段落）。display属性的属性值block、list-item、table能让一个元素变成块级的。
  2. 块级盒：参与块格式化上下文的盒。每个块级元素生成一个主块级盒(principal block-level box)，用来包含后代盒及生成的内容，并且任何定位方案都与该盒有关。有些块级元素可能会生成除主盒外的额外的盒，如list-item元素，这些额外的盒根据主盒来放置。
  3. 块容器： 一个块容器要么只包含参与了行内格式化上下文的行内级盒，要么只包含参与了块格式化上下文的块级盒。如果一个块容器只包含行内级内容，则它会建立新的行内格式化上下文。如果一个块容器的父格式化上下文不是块格式化上下文，则它会建立新的块格式化上下文；否则，当它自己参与块格式化上下文时，它要么为它的内容建立块格式化上下文，要么继续它所参与的块格式化上下文，取决于其他属性的限制，例如overflow或align-content。一个块容器可以既建立块格式化上下文，又建立行内格式化上下文，例如它只包含行内级内容，父格式化上下文又不是块格式化上下文。
  4. 块盒：既是块级盒又是块容器就叫做块盒。
  5. 块级盒(block-level box)、块容器盒(block container box)和块盒(block box)三个术语在没有歧义的时候简称为块(block)。

总结：

  • 块级盒=块级元素生成的框
  • 块容器=只包含块级盒或者行内级盒的元素，大致=(块级元素-替换元素-flex容器-grid容器)+(行内块级元素+非替换表格单元格元素)
  • 块盒=块级元素-替换元素-flex容器-grid容器

行内级元素与行内盒

  1. 行内级元素：源文档中那些不会形成新内容块的元素，内容分布于多行。display属性的属性值inline、inline-table、inline-block能让一个元素变成行内级的。
  2. 行内级盒：参与行内格式化上下文的盒。行内级元素生成行内级盒。
  3. 行内盒是内容参与了它的包含行内格式化上下文的行内级盒。

匿名行内盒

任何被直接包含在一个块容器元素中（不在行内元素里面）的文本，必须视为一个匿名行内元素。

<p>会生成一个块盒，里面还有几个行内盒。"emphasized"的盒是一个由行内元素<em>生成的行内盒，但其它盒"Some"和"text"都是由块级元素<p>生成的行内盒。后者叫做匿名行内盒，因为它们没有与之相关的行内级元素。
这种匿名行内盒从它们的父级块盒继承了可继承的属性，不可继承的属性取其初始值。
对于后续会根据white-space属性合并起来的空白字符内容，不会生成任何匿名行内盒。
如果含义明确，可以将匿名行内盒和匿名块盒简称为匿名盒。
在格式化表格时会出现更多的匿名盒类型

元素盒属性

宽度

width属性用于设置宽度值，宽度值可能是 content box 的宽度，也可能是 border box 的宽度，取决于box-sizing的属性值。
min-width属性用于设置元素的最小宽度值，当元素的width小于该值时，width就会被设置为min-width。
max-width属性用于设置元素的最大宽度值，当元素的width大于该值时，width就会被设置为max-width。
更具体的说，首先不考虑min-width和max-width，根据下面计算width与margin的步骤计算出width。如果产生的宽度大于max-width，就把max-width的计算值作为width的计算值，再应用一遍计算width与margin的步骤。如果产生的宽度小于min-width，就把min-width的计算值作为width的计算值，再应用一遍计算width与margin的步骤。
对于具有固有比例并且width和height都被指定为auto的替换元素使用其他算法计算宽度。

  • 取值语法：
  • 初始值：auto
  • 适用于：除了行内非替换元素、表格行、表格行组之外的所有元素
  • 百分数：相对于包含块的宽度
  • 计算值：auto和百分数值为指定的值，否则为绝对长度。
  • 继承性：否
  • 动画性：是
  1. 该属性不适用于非替换的行内元素。非替换的行内元素盒的内容宽度是它里面渲染的内容的宽度（在子级的任何相对偏移之前）。回想一下，行内盒都排列在行框里面。行框的宽度由它们的包含块决定，但可能因为浮动的出现而被缩短。
  2. 负值是非法的
  3. max-content是首选（最大）宽度，即计算不换行（显式换行除外）时的内容宽度
  4. min-content是首选最小宽度（preferred minimum width），即每个单词都换行时的内容宽度。
  5. fit-content()使用公式min(max-content, max(min-content, <length-percentage>))计算宽度。
  6. -webkit-fill-available：填满可用空间，可以使非块元素有块元素的占满空间的效果。

高度

height属性用于设置高度值，高度值可能是 content box 的高度，也可能是 border box 的高度，取决于box-sizing的属性值。
min-height属性用于设置元素的最小高度值，当元素的height小于该值时，height就会被设置为min-height。
max-height属性用于设置元素的最大高度值，当元素的height大于该值时，height就会被设置为max-height。
更具体的说，首先不考虑min-height和max-height，根据下面计算height与margin的步骤计算出height。如果产生的宽度大于max-height，就把max-height的计算值作为height的计算值，再应用一遍计算height与margin的步骤。如果产生的宽度小于min-height，就把min-height的计算值作为height的计算值，再应用一遍计算height与margin的步骤。
对于具有固有比例并且width和height都被指定为auto的替换元素使用其他算法计算高度。

  • 取值语法：
  • 初始值：auto
  • 适用于：除了行内非置换元素、表格列、表格列组之外的所有元素
  • 百分数：相对于包含块的高度。如果包含块高度没有显式指定（例如取决于内容高度），并且元素不是绝对定位的，则不论百分数值为多少，计算值都为auto。
  • 计算值：auto和百分数值为指定的值，否则为绝对长度。
  • 继承性：否
  • 动画性：是
  1. 注意，绝对定位元素的包含块的高度独立于该元素本身的大小，因此这种元素上的百分比高度都可以计算出来（can always be resolved）。然而，可能直到文档中在此之后的元素处理完毕后，才能获知该高度
  2. 负值是非法的
  3. max-content是首选（最大）宽度，即每个单词都换行时的内容宽度。
  4. min-content是首选最小宽度（preferred minimum width），即计算不换行（显式换行除外）时的内容宽度
  5. fit-content()使用公式min(max-content, max(min-content, <length-percentage>))计算宽度。2
  6. -webkit-fill-available：填满可用空间，可以使非块元素有块元素的占满空间的效果。

外边距

外边距是元素周围的空白，其他元素不能存在，并且这片区域中，父元素的背景是可见的。
margin属性用于同时设置上下左右四个外边距，它是margin-top、margin-right、margin-bottom和margin-left的简写属性。

  • 取值语法：
  • 初始值：和被简写的属性的初始值相同，即0 0 0 0
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于包含块的宽度。注意，即使是垂直方向的外边距百分数也相对于包含块的宽度而不是高度。
  • 计算值：百分数值为指定的值，否则为绝对长度。
  • 继承性：否
  • 动画性：是

如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定上、右、下、左（顺时针）四个方向的外边距。

  • 如果只有一个值，即只有上方向的外边距，则其他方向外边距的值也是它。

  • 如果只有两个值，即只有上、右方向的外边距，则下、左方向外边距等于它对边的外边距。

  • 如果只有三个值，即只有上、右、下方向的外边距，则左方向的外边距等于它对边的外边距。

  • 该简写属性永远会同时设置四个外边距，即便只写一个值也被扩展成四个。因此如果不想改变当前的外边距值，只能使用各外边距单独的属性。

  • 垂直外边距属性对行内非替换元素无效。

  • 元素默认没有外边距，段落之间的间隔常常只用外边距实现。

外边距折叠

CSS中，两个或多个盒（可以是兄弟也可以不是）的相邻 margin 会被合并成一个 margin。
水平外边距不会合并。
相邻的垂直外边距会合并，除了以下情况：

  • 根元素盒的 margin 不合并
  • 如果一个带有间隙（clearance，指clear属性导致元素位置移动形成的间隙）的元素的上外边距与下外边距相邻，它的外边距会和紧挨着的兄弟（元素）的相邻外边距合并，但合并后不会再和父级块的下外边距合并

两个margin是相邻的，当且仅当满足以下所有条件：

  • 1 都属于流内（in-flow）块级盒，处于同一个块格式化上下文
  • 2 没有行框（line box），空隙，内边距和边框把它们隔开（注意，因此某些0高度行框（见9.4.2）会被忽略）
  • 3 都属于垂直相邻框边界（vertically-adjacent box edges），即形成下列某一对
    • 3.1 盒的上边距与其第一个流内（in-flow）孩子的上边距
    • 3.2 盒的下边距与其下一个流内紧挨着的兄弟的上边距
    • 3.3 最后一个流内孩子的下边距与其height计算值为’auto’的父元素的下边距
    • 3.4 盒的上边距和下边距。要求该盒没有建立新的块格式化上下文，并且’min-height’计算值为0，‘height’计算值为0或’auto’，还没有流内孩子

示例：

  1. 浮动的盒或者绝对定位的盒和与任何其它盒之间、与流内孩子之间的margin，以及自己的上下边距都不会合并（因为它们不是流内块级盒，不满足条件1）
  2. 建立了新的块格式化上下文的元素的margin不会与它们的流内孩子合并（因为它自身和它的流内孩子处于不同的块格式化上下文，不满足条件1）；上下边距也不会合并（因为建立了块格式化上下文，不满足条件3.4）；但与其他盒之间的margin可能合并（例如两行的inline-block）
  3. 流内块级元素的bottom margin总会与它的下一个流内块级兄弟的top margin合并，除非该兄弟（元素）具有间隙（条件3.2所说的垂直相邻框边界）
  4. 流内块级元素的top margin会与它的第一个流内块级子级的top margin合并，条件是该元素没有上边框和上内边距，并且其孩子不具有间隙（条件3.1所说的垂直相邻框边界）
  5. 一个’height’为’auto’并且’min-height’为0的流内块级盒的bottom margin会与它的最后一个流内块级子级的bottom margin合并，条件是该盒没有下内边距和下边框，并且其孩子的下外边距没有与具有间隙的上外边距合并（外边距之间没有分隔，同时满足条件3.4）
  6. 盒自身的外边距也会合并，条件是’min-height’属性为0，既没有上下边框，也没有上下内边距，‘height’为0或’auto’，且不含行框的话，那么其所有流内孩子的外边距（如果存在的话）都会合并

当两个或者更多的margin合并时，合并产生的margin宽度为：

  • 如果 margin 都为正，结果为 margin 中最大的（绝对值最大的）
  • 如果 margin 都为负，结果为 margin 中最小的（绝对值最大的）
  • 如果 margin 有正有负，结果为最大的正 margin 与最小的负 margin之和。

，产生的margin宽度为被合并的外边距宽度中的最大值。至于负margin，就从正相邻margin的最大值中减去负相邻margin的绝对值的最大值。如果没有正margin，就用0减去相邻margin的绝对值的最大值

内边距

padding属性用于同时设置上下左右四个内边距，它是padding-top、padding-right、padding-bottom和padding-left的简写属性。

  • 取值语法：
  • 初始值：和被简写的属性的初始值相同，即0 0 0 0
  • 适用于：除了table-row-group、table-header-group、table-footer-group、table-row、table-column-group和table-column之外的所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于包含块的宽度。注意，即使是垂直方向的内边距百分数也相对于包含块的宽度而不是高度。
  • 计算值：百分数值为指定的值，否则为绝对长度。
  • 继承性：否
  • 动画性：是

如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定上、右、下、左（顺时针）四个方向的内边距。
  • 如果只有一个值，即只有上方向的内边距，则其他方向内边距的值也是它。
  • 如果只有两个值，即只有上、右方向的内边距，则下、左方向内边距等于它对边的内边距。
  • 如果只有三个值，即只有上、右、下方向的内边距，则左方向的内边距等于它对边的内边距。

该简写属性永远会同时设置四个内边距，即便只写一个值也被扩展成四个。因此如果不想改变当前的内边距值，只能使用各内边距单独的属性。

元素默认没有内边距，段落之间的间隔常常只用外边距实现。
如果没有内边距，元素的边框将紧贴着内容，因此如果添加了边框，最好再添加一些内边距。

边框

border属性用于同时设置上下左右四个边框。
边框是元素的内容和内边距周围的一条或多条线段。边框有三个要素：样式、宽度、颜色。

边框样式

边框的样式是指边框线条的类型，如单实线、双实线、虚线、雕刻风格线等等。
border-style属性用于同时设置上下左右四个边框的样式。它是border-top-style、border-right-style、border-bottom-style和border-left-style的简写属性。

  • 取值语法：
  • 初始值：和被简写的属性的初始值相同，即none none none none
  • 适用于：所有元素，还可以应用到伪元素::first-letter上
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定上、右、下、左（顺时针）四个方向的边框样式。
  • 如果只有一个值，即只有上方向的边框样式，则其他方向的边框样式也是它。
  • 如果只有两个值，即只有上、右方向的边框样式，则下、左方向的边框样式就是它对边的边框样式。
  • 如果只有三个值，即只有上、右、下方向的边框样式，则左方向的边框样式就是它对边的边框样式。

该简写属性永远会同时设置四个边框样式，即便只写一个值也被扩展成四个。因此如果不想改变当前的边框样式，只能使用各边框样式单独的属性。

十种属性值的效果如下：

hidden和none样式相同，只是在应用到表格上时，解决边框冲突的方式略有不同。

  • groove：边框看起来像是被刻进了画布（3D凹槽效果）
  • ridge：与groove相反，边框看起来像是要从画布里跑出来（3D凸槽效果）
  • inset：边框让盒看起来像是被嵌进画布了（没有3D效果）
  • outset：与inset相反：边框让盒看起来像是要从画布里出来了（没有3D效果）

单边简写属性

border-top、border-right、border-bottom、border-left属性用于设置边框某个方向的样式、宽度和颜色。它是对应边的样式、宽度和颜色三个属性的简写属性。例如border-top是border-top-style、border-top-width、border-top-color的简写属性。

  • 取值语法：
  • 初始值：和被简写的属性的初始值相同，即medium none currentColor
  • 适用于：所有元素，还可以应用到伪元素::first-letter上
  • 计算值：对于宽度，当样式为none或hidden时，计算值为0；否则为绝对长度；对于样式，计算值是指定的值；对于颜色，计算值是颜色值。
  • 继承性：否
  • 动画性：和各简写属性相同

样式、宽度和颜色的顺序并不重要，如果不声明，就会是默认值。

  • 继承性：否
  • 动画性：和各简写属性相同

单个圆角

border-top-left-radius属性用于将边框的左上角变成圆角。圆角实际上是椭圆的四分之一弧，椭圆的左顶点和上顶点分别位于直角边框的左边框和上边框上。

  • 取值语法：
  • 初始值：0
  • 适用于：除了表格内元素的所有元素，还可以应用到伪元素::first-letter上
  • 计算值：两个绝对长度或百分数值
  • 百分数：水平半轴相对于 border box 的宽度计算；垂直半轴相对于 border box 的高度计算
  • 继承性：否

  • 如果提供了所有两个值，则两个值分别指定椭圆的水平半轴和垂直半轴（注意焦点未必在水平轴上，因此这里没有用术语半长轴与半短轴）。
  • 如果只有一个值，即只有水平半轴的长度，则垂直半轴的长度也是它。

注意百分数是相对于 border box 的宽度和高度计算的。因此一个50%的椭圆就是 border box 的内切椭圆。

所有圆角

border-radius属性用于同时设置左上、右上、右下、左下四个角的圆角。它是border-top-left-radius、border-top-right-radius、border-bottom-right-radius、border-bottom-left-radius四个属性的简写属性。

  • 初始值：和被简写的属性的初始值相同，即0 0 0 0 / 0 0 0 0

  • 计算值：每个角都是两个绝对长度或百分数值
  • 百分数：相对于 border box 的尺寸计算
  • 继承性：否
  • 动画性：是

属性值是1-4个水平半轴长度，后跟可选的1-4个垂直半轴长度，水平和垂直半轴之间以斜线/分隔。
如果没有提供垂直半轴长度，则其等于相应角的水平半轴长度。
对于水平半轴和垂直半轴，如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定左上、右上、右下、左下（顺时针）四个角的半轴长度。

为了让图像边框的宽度border-width值非0，即显示出边框，仍然需要将border-style设为none或hidden之外的任何值，这些样式在图像边框正常显示时没有作用，但图像边框无法显示时会作为后备边框样式。

border-image-slice属性用于将图像划分成多个区域，然后将图像的区域应用到对应边框的区域上。

  • 取值语法：
  • 初始值：100%
  • 适用于：除border-collapse属性值为collapse的表格内元素之外的所有元素，还可以应用到伪元素::first-letter上
  • 百分数：相对于图片的尺寸

数字值是距离图像边缘的坐标，它的隐含单位是像素。注意虽然实际含义都是像素，但该属性接受的是数字值而不是长度值，因此30像素应该写为30而不是30px。


图片通常被四条分割线划分成9个区域：

  • 1-4区是角区域。每个区域的图像只会使用一次来形成最终的图像边框的角。左上角区域的宽高由left和top决定，右上角区域的宽高由right和top决定，右下角区域的宽高由right和bottom决定，左下角的宽高由left和bottom决定。
  • 5-8区是边区域。每个区域的图像在最终的图像边框中会重复、缩放或修剪以匹配元素的尺寸。上边区域宽度是由图片宽度减去left和right得到，当算出的值为负时，实际值为0，其他边宽高类似。
  • 9区是中间区域，默认情况下不使用，如果在值最后添加了fill关键字，则会用该区域的图像作为背景图像，但它会在background设置的背景之上绘制。

当两条水平分割线left和right重合后，或者说当left与right之和大于等于图像宽度时，5、7两个边区域与9区中间区域被挤没了，因此最终的图像边框里没有上下边框。
当两条垂直分割线top和right重合后，或者说当top与right之和大于等于图像高度时，，6、8两个边区域与9区中间区域被挤没了，因此最终的图像边框里没有左右边框。
这就是为什么默认值100%的情况下，图片作为一个整体被重复四次放在了四个角。因为单一值100%经过扩展变成100% 100% 100% 100%，水平和垂直数值之和都是200%，大于图像宽度和高度，于是四个角是100%的图像，并且没有四条边的边框。

图像边框宽度

border-image-width属性用于设置图像边框的宽度，如果设置的值大于border-width，那么图像边框会扩展到内边距区域(padding area)或者内容区域(content area)。

  • 取值语法：
  • 初始值：1
  • 适用于：除border-collapse属性值为collapse的表格内元素之外的所有元素，还可以应用到伪元素::first-letter上
  • 百分数：相对于元素的 border box 的宽度或高度
  • 计算值：指定的值；但相对长度会转换为绝对长度
  • 继承性：否
  • 动画性：是

如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定距离 border box 外侧上边、右边、下边、左边特定长度的分割线。
  • 如果只有一个值，即只有距离 border box 外侧上边特定长度的分割线，则其他分割线的距离对应边的长度也是它。
  • 如果只有两个值，即只有距离 border box 外侧上边和右边特定长度的分割线，则距离下边、左边的分割线的长度是它对边分隔线距离对边的长度。
  • 如果只有三个值，即只有距离 border box 外侧上边、右边、下边特定长度的分割线，则距离左边的分割线距离左边的长度是它对边分割线距离对边的长度。

属性值的具体作用如下：

  • <length>：该值实际是从边框的外边界向内部的偏移，也就是说它可能会扩展到内边距区域和内容区域。
  • <percentage>：相对于元素的 border box 的宽度或高度，与border-width的值无关。
  • <number>：数字值表示的是border-width的倍数，不能是负数。
  • auto：如果定义了border-image-slice属性，则使用对应切片区域的宽高；如果没定义，则使用border-width的值。

外推宽度

border-image-outset属性用于设置图像边框的外推距离。外推边框将使得图片超出 border box并占用外边距区域甚至其他元素的区域。但不会产生滚动条，也不会捕获鼠标事件。取值不能为负。

  • 取值语法：
  • 初始值：0
  • 适用于：除border-collapse属性值为collapse的表格内元素之外的所有元素，还可以应用到伪元素::first-letter上
  • 计算值：指定的值；但相对长度会转换为绝对长度
  • 继承性：否
  • 动画性：是

如果提供的值的数量少于四个，值会自动扩展：

  • 如果提供了所有四个值，则四个值分别指定向上边、右边、下边、左边外推的距离。
  • 如果只有一个值，即上边外推的距离，则其他边外推的距离也是它。
  • 如果只有两个值，即只有上边和右边外推的距离，则下边、左边外推的距离是它对边外推的距离。
  • 如果只有三个值，即只有上边、右边、下边外推的距离，则左边外推的距离是它对边外推的距离。

属性值的具体作用如下：

  • <length>：在根据border-image-width等属性确定好位置之后，将图像边框向外推的距离。
  • <number>：数字值表示的是border-width的倍数。

图像重复方式

border-image-repeat属性用于设置作为边框时所划分区域的图像的重复方式，默认情况下各边图像是拉伸的。

  • 取值语法：
  • 初始值：stretch
  • 适用于：除border-collapse属性值为collapse的表格内元素之外的所有元素，还可以应用到伪元素::first-letter上
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

可以提供一个或两个值：

  • 如果只有一个值，即该值被用于四条边。
  • 如果提供了两个值，则第一个值被用于上边和下边，第二个值被用于左边和右边。

属性值的具体作用如下：

  • stretch：图像划分后的边区域被拉伸来填满边框区域的高度和宽度
  • repeat：平铺图像，先把图像放在每一边的边框区域中点，然后向两边平铺，直到边框左右边界，达到边界的图像可能被截断。
  • round：将各边的边框区域长度除以要平铺的图像尺寸，然后取整为最近的整数，平铺这一数量的图像，可能会拉伸图像以保证平铺的图像恰好互相接触。
  • space：将各边的边框区域长度除以要平铺的图像尺寸，然后取整为最近的整数，平铺这一数量的图像，但不会拉伸图像，如果不能使图像恰好互相接触，多出来的空间留白。

内容溢出

overflow属性用于设置元素溢出时的行为，即内容太多不能放在它的块格式化上下文内时如何处理。它是overflow-x和overflow-y属性的简写属性。

  • 取值语法：
  • 初始值：visible
  • 适用于：块级元素和置换元素
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

如果提供了两个值，则第一个值用于overflow-x，第二个值用于overflow-y。
如果只提供了一个值，则overflow-x和overflow-y都使用该值。

属性值的具体作用如下：

  • visible：内容不会被裁剪，渲染时可能会超出元素的 padding box。
  • hidden：内容被裁剪到 padding box 区域。不提供滚动条，也不支持用户滚动。但可以通过编程滚动。该属性值应该在高度固定时使用，否则元素一直被撑高。
  • scroll：内容被裁剪到 padding box 区域。浏览器总是显示滚动条。
  • auto：取决于用户代理。如果内容在 padding box 内，它看起来和 visible 一样，但仍然建立一个新的块格式化上下文。如果元素溢出，浏览器会提供滚动条。

可见性

visibility属性用于设置元素显示还是隐藏，同时不改变文档布局。

  • 取值语法：
  • 初始值：visible
  • 适用于：所有元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • visible：元素可见。
  • hidden：元素不可见，但依然像可见那样影响文档布局，元素还在那里，只是看不见，就像声明opacity: 0一样。元素的后代如果设置为visibility: visible则后代仍可见。元素无法获得焦点。和display: none的区别在于display: none除了使元素不可见外，还使元素从文档中移除，不影响布局。
  • collapse：用于表格，对于其他元素，作用与hidden相同。

计算width和margin

行内非替换元素

  • 计算值为margin-left: auto的应用值为0；计算值为margin-right: auto的应用值为0。
  • width属性不适用。

行内替换元素

  • 计算值为margin-left: auto的应用值为0；计算值为margin-right: auto的应用值为0。
    1. 如果同时有计算值height: auto和width: auto，且该元素具有固有宽度，则width的应用值为固有宽度。
    2. 如果同时有计算值height: auto和width: auto，且该元素没有固有宽度，但具有固有高度和固有比例。或者如果有计算值width: auto，height为非auto，且该元素有固有比例，则width的应用值为(高度的应用值) * (固有比例)
    3. 如果同时有计算值height: auto和width: auto，且该元素具有固有比例，但没有固有高度或者宽度，则width的应用值在CSS2.1未定义。建议如果包含块的宽度不依赖替换元素的宽度，那么width的应用值根据用于常规流中块级非替换元素的约束方程来计算
    4. 如果有计算值width: auto，并且该元素具有固有宽度，那么这个固有宽度就是width的应用值。
    5. 如果有计算值width: auto，但不满足上面的条件，那么width的应用值变为300px。如果300px对于适应设备来说太宽了，UA应该应用适应设备的最大的比例为2:1的矩形代替。

常规流中的块级非替换元素

应用值必须满足以下约束方程：
margin-left + border-left-width + padding-left + width + padding-right + border-right-width + margin-right = 包含块宽度

  • 如果width不是auto，且border-left-width + padding-left + width + padding-right + border-right-width（再加上不为auto的margin-left或者margin-right）大于包含块的宽度，那么所有margin-left或者margin-right的auto值在下列规则中会被当作0
  • 如果以上属性的计算值都不是auto，就说该值被过度限制了，其中一个应用值将与其计算值不同。如果包含块的direction属性值为ltr，margin-right的指定值会被忽略，并且重新计算该值，以便让等式成立。如果direction的值为rtl，就换成margin-left
  • 如果有且仅有一个值被指定为auto，其应用值由等式得出
  • 如果width被设置为auto，所有其它的auto值都变为0，然后width由产生的等式得出。
  • 如果margin-left和margin-right都是auto，那么它们的应用值相等，加上这个条件求解方程。这会让该元素相对于其包含块边界水平居中。

常规流中的块级替换元素

  • width的应用值由行内替换元素规则决定。
  • 然后，用常规流中的块级非替换元素规则确定外边距

常规流中的inline-block非替换元素

  • 如果width为auto，应用值就与浮动元素一样是自适应宽度
  • 计算值为auto的margin-left或者margin-right的应用值变为0

常规流中的inline-block替换元素

与行内替换元素规则完全一致

浮动非替换元素

  • 如果margin-left或者margin-right的计算值为auto，则其应用值为0
  • 如果width的计算值为auto，应用值就是“自适应（shrink-to-fit）”宽度。

自适应宽度的计算与用自动表格布局算法去算一个表格单元的宽度类似。粗略地讲：

  1. 通过计算不换行（显式换行除外）时的内容宽度得到首选（最大）宽度（preferred width）
  2. 然后计算首选最小宽度（preferred minimum width），例如，通过尝试所有可能的换行来得到首选最小宽度，CSS 2没有定义准确的算法。
  3. 接着算出可用宽度（available width）：这种情况下，就是包含块的宽度减去margin-left，border-left-width，padding-left，padding-right，border-right-width，margin-right以及所有相关滚动条的宽度的应用值。
  4. 最后得到自适应宽度就是：min(max(首选最小宽度, 可用宽度), 首选宽度)。该公式的意思是，首先要保证宽度不小于最小宽度，即尽可能多的换行时的宽度。如果包含块还有多余的宽度，就用上多余的宽度。但如果多余的宽度实在太多，以至于所有内容不换行后还有多余宽度，就只用所有内容不换行时的宽度。（首选最小宽度应该是小于或等于首选（最大）宽度的）

浮动替换元素

  • 如果margin-left或者margin-right的计算值为auto，它们的应用值就是0。
  • width的应用值由行内替换元素规则决定

相对定位的元素

对于上述情况中出现相对定位的情况，元素的left和right的值由下列规则确定：

  • left或right的应用值总是（满足）：left = -right
  • 如果left和right都是auto（各自的初始值），应用值为0（即盒待在原位置）
  • 如果left是auto，其应用值为负的right值（即盒向左移动right值）
  • 如果right是auto，其应用值为负的left值
  • 如果left和right都不是auto，位置就被过度约束（over-constrained）了，left和right其中有一个会被忽略。如果包含块的direction属性是ltr，那么left有效，right变成-left。如果包含块的direction属性是rtl，那么right有效，left被忽略

  • 计算值为margin-top: auto的应用值为0；计算值为margin-bottom: auto的应用值为0。
    1. 如果同时有计算值height: auto和width: auto，且该元素具有固有高度，则height的应用值为固有宽度。
    2. 如果有计算值height: auto，并且该元素具有固有比例，那么height的应用值为(宽度的应用值)/(固有比例)。
    3. 如果有计算值height: auto，并且该元素具有固有高度，那么height的应用值为固有高度。
    4. 如果有计算值height: auto，但不满足上面的条件，那么height的应用值必须设置为比例为2:1，高度不超过150px并且宽度不超过设备宽度的最大矩形的高度

常规流中的块级非替换元素(overflow计算值为visible时)

当overflow的计算值不为visible但已传播到视口时也适用。

  • 如果margin-top或margin-bottom为auto，则其应用值为0。
  • 如果height为auto，高度取决于该元素是否含有块级子元素，以及是否具有padding或border。元素的高度为 content box 上边界到下面第一个适用的（位置）的距离：
    1. 如果该盒为一行或多行建立了一个行内格式化上下文，则到最后一个行框的下边界。
    2. 如果最后一个流内子元素的下外边距没有和该元素的下外边距合并，则到最后一个流内子元素的下外边距边界
    3. 最后一个上边距没有和元素的下边距合并的流内子元素下边框边界
    4. 否则，0

注意：只考虑常规流中的子元素（即，浮动盒和绝对定位的盒会被忽略，并且相对定位的盒不考虑其偏移）。注意子级盒可以是一个匿名块盒。

常规流中的块级非替换元素(overflow计算值不为visible时)

inline-block的非替换元素

浮动非替换元素

  • 如果margin-top或margin-bottom为auto，应用值为0。
  • 如果height是auto，高度根据【块格式化上下文中的Auto高度】规则，取决于该元素的后代。
  • 对于inline-block元素，在计算行框的高度时，使用 margin box

块格式化上下文中的auto高度

特定情况下（例如，前面提到根据本规则的情况），一个建立了块格式化上下文的元素的高度按照如下规则计算（此时子元素考虑浮动子元素等情况）：

  • 如果它只含有行内子元素，高度就是最高的行框的顶端与最低的行框的底端之间的距离
  • 如果它只含有块级子元素，高度就是最高的块级子级盒的上外边距边界到最低的块级子级盒的下外边距边界之间的距离
  • 绝对定位的子级会被忽略，并且相对定位的盒不考虑其偏移。注意，子级盒可以是一个匿名块盒
  • 此外，如果该元素含有浮动后代，且该浮动后代的下外边距边界位于元素的内容下边界下方，那么高度增加至能够包含这些边界。只考虑参与此块级格式化上下文的浮动，例如，不考虑绝对定位的后代中的浮动或其它浮动

相对定位的元素

对于上述情况中出现相对定位的情况，元素的top和bottom的值由下列规则确定：

  • top或bottom的应用值总是（满足）：top = -bottom
  • 如果top和bottom都是auto（各自的初始值），应用值为0（即盒待在原位置）
  • 如果top是auto，其应用值为负的bottom值（即盒向左移动right值）
  • 如果bottom是auto，其应用值为负的top值
  • 如果top和bottom都不是auto，位置就被过度约束（over-constrained）了，忽略bottom（即bottom的值将是负的top）

绝对定位的非替换元素

  • 术语“（元素的）静态位置（static position）”，大致指的是元素在常规流中的位置。
  • 出于计算静态位置的目的，fixed定位元素的包含块为初始包含块，而不是视口

应用值必须满足以下约束方程：
top + margin-top + border-top-width + padding-top + height + padding-bottom + border-bottom-width + margin-bottom + bottom = 包含块高度

  • 如果top，height和bottom都是auto：把top设置为静态位置并应用下面的第3条规则
  • 如果这3个值都不是auto：
    • 如果margin-top和margin-bottom都是auto，解方程时要添上额外的约束：2个margin的值相等。
    • 如果margin-top或者margin-bottom有一个是auto，就解方程求这个值。
    • 如果该值被过度限制了，就忽略bottom再求出这个值（？？？只有一个是auto的情况怎么可能过渡限制？？？）
  • 否则，在下面6条规则中挑1条合适的应用：
    1. 如果top和height都是auto，bottom不是auto，则高度根据【块格式化上下文中的Auto高度】规则，由内容决定，把值为auto的margin-top和margin-bottom设置为0，再求出top。
    2. 如果top和bottom都是auto，height不是auto，把top设置为静态位置，把值为auto的margin-top和margin-bottom设置为0，再求出bottom。
    3. 如果height和bottom都是auto，top不是auto，则高度根据【块格式化上下文中的Auto高度】规则，由内容决定，把值为auto的margin-top和margin-bottom设置为0，再求出bottom。
    4. 如果top是auto，height和bottom都不是auto，则把值为auto的margin-top和margin-bottom设置为0，再求出top。
    5. 如果height是auto，top和bottom都不是auto，则把值为auto的margin-top和margin-bottom设置为0，再求出height。
    6. 如果bottom是auto，top和height都不是auto，则把值为auto的margin-top和margin-bottom设置为0，再求出bottom。

  2. 如果top和bottom都是auto，把top设为该元素的静态位置
