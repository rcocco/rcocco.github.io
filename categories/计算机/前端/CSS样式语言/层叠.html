指定值、计算值和实际值

属性值的最终值需要经过以下四步得到：

  1. 先通过指定来确定值（指定值）
  2. 处理后得到一个用于继承的值（计算值）
  3. 有必要的话转换为一个绝对值（应用值）
  4. 根据本地环境限制进行转换（实际值）

指定值

用户代理需要按以下优先顺序为每个属性赋予一个指定值：

  1. 如果层叠产生了一个值，使用它
  2. 否则，如果属性是继承的，且该元素不是根元素，则使用父元素的计算值
  3. 否则，使用属性的初始值。

计算值

指定值通过层叠被处理为计算值，例如URI被转换为绝对的，相对长度被转换为绝对长度。计算一个值不需要渲染文档。
一个属性的计算值由每个属性的Computed Value规定决定，当指定值为inherit时，计算值取决于继承。

应用值

处理计算值时，尽可能不要格式化文档。然而，有些值只能在文档布局完成时确定。例如，如果一个元素的宽度是其包含块的特定百分比，在包含块的宽度确定之前无法确定这个宽度。应用值是把计算值剩余的依赖（值）都处理成绝对值后的（计算）结果

实际值

原则上，应用值应该用于渲染，但用户代理可能无法在给定的环境中利用该值。例如，用户代理或许只能用整数像素宽度渲染边框，因此不得不对宽度的计算值做近似处理，或者用户代理可能被迫只能用黑白色调而不是全彩色。实际值是经过近似处理后的应用值

继承

继承是指把一个元素的某些属性值传给其后代的机制，即某些样式不仅应用到所指的元素上，还应用到元素的后代上。例如

h1 {color: gray}为元素<h1>的文本添加颜色，而<em>元素会从<h1>元素的样式中继承color属性，因而<em>元素中的文本也是灰色。

某些属性是不继承的，这通常是为了避免得到意外的结果。例如边框属性border就不继承。多数盒模型也不继承，包括外边距、内边距、背景和边框。

发生继承时，元素继承计算值，来自父元素的计算值同时作为指定值和其子级的计算值。

任何显式定义的直接作用于元素的CSS规则（例如选择器或者style属性定义的样式），都将覆盖继承得到的值。

确定应该把哪些值应用到元素上时，用户代理不仅要考虑继承，还要考虑声明的特指度，以及声明的来源。这个过程叫层叠。

层叠

来源

每一个样式规则都有一个层叠来源，意思是该样式规则是由谁编写的，CSS定义了三个核心来源：

  • 作者来源
    作者根据文档语言的约定为源文档指定样式表。例如，在HTML中，样式表可以包含在文档中或在外部链接。
  • 用户来源
    用户可能为特定文档指定样式信息。例如，用户可以指定一个包含样式表的文件，或者用户代理可以提供一个生成用户样式表的界面。
  • 用户代理来源
    合格的用户代理必须应用默认样式表。用户代理的默认样式表应以满足对文档语言的一般表示期望的方式来显示文档语言的元素（例如，对于可视浏览器，HTML中的EM元素使用斜体字体显示）。参见HTML用户代理样式表。

重要声明

CSS试图在作者样式表和用户样式表之间建立功能平衡。默认情况下，作者样式表中的规则将覆盖用户样式表中的规则，而用户样式表中的规则将覆盖用户代理的默认样式表中的规则。为了平衡这一点，可以将声明标记为重要，这会增加其在层叠中的权重并颠倒优先顺序。

重要声明是指一个声明在末尾的分号;前有一个!important标记。例如color: #333 !important;。所有其他声明都是普通声明（非重要声明）。

一个重要声明优先于普通声明。作者和用户样式表可能包含重要声明，而用户来源的重要声明将覆盖作者来源的重要声明。通过为用户提供特殊要求（大字体，颜色组合等）的控制，这项CSS特征提高了文档的可访问性。

任何来源的重要声明都优先于动画。这使作者在重要情况下可以覆盖动画值。（动画值通常会覆盖所有其他规则。）

用户代理样式表也可能包含重要的声明。这些将覆盖所有作者和用户声明。

层叠规则

CSS为每个声明计算出一个优先级，层叠规则如下：

  1. 找到匹配特定元素和目标属性的所有规则
  2. 按来源和重要性排序应用到特定元素上的所有声明，优先级从高到低如下：
    1. 过渡声明
    2. 用户代理的重要声明
    3. 用户的重要声明
    4. 作者的重要声明
    5. 动画声明
    6. 作者的普通声明
    7. 用户的普通声明
    8. 用户代理的普通声明
  3. 按声明所属的选择器的特指度排序，特指度高的优先级更高。在这一步里，style属性中的行内样式声明优先于所有选择器。
  4. 按出现顺序排序，最后出现的声明优先级更高。
    • @import导入的样式表中的声明，位置和@import语句的位置一样
    • 原文档独立链接的样式表中的声明，就好像它们是按照链接顺序（由主文档语言确定）那样串联在一起的。
    • Declarations from style attributes are ordered according to the document order of the element the style attribute appears on, and are all placed after any style sheets.
