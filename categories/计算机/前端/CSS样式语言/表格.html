表格
2020-10-19|计算机前端CSS样式语言

表格元素和表格内部元素是两个不同的概念。
表格内部元素生成矩形框，有内容、内边距和边框，但是没有外边距。所以单元格之间的间距不能通过外边距调整。
一列单元格有相同的宽度，一行单元格有相同的高度，但是不同行的高度未必相同，不同列的宽度未必相同。

表格元素的显示类型为：

CSS中的表格模型是“行主导”的。创作人员在使用标记时会显式声明行，而列是从行中单元格的布局中衍生而来。即第一列由每行中第一个单元格组成，第二列由每行中第二个单元格组成。

单元格可以同时属于两个上下文（行和列）。在CSS中，行和行组只能应用四个与表格无关的属性：border、background、width和visibility。
这四个属性应用于列上下文时有特殊规则：

  • border：仅当border-collapse属性的值为collapse时才能为列和列组设置边框。此时，根据折叠算法在列和列组中各单元格的边界处绘制边框。
  • background：仅当单元格及所在的行是透明背景时列和列组的背景才可见。
  • width：定义列和列组的最小宽度。列（或列组）中单元格的内容可能会迫使列变宽。
  • visibility：如果列或列组的visibility属性值为collapse，那么列或列组中的单元格不渲染。从折叠的列跨到其他列的单元格将被裁剪，好像从其他列跨到隐藏的列一样。此外，表格的总宽度将减去折叠的列所占的宽度。对列或列组而言，除了hidden之外，其他的visibility值将被忽略。

表格分六个图层渲染，从上到下分别是单元格、行、行组、列、列组、表格，上面的图层将盖住下面的图层，这就是为什么行的背景会覆盖列的背景，因为它的图层在上面。

表题

表题元素<caption>从<table>继承相关属性。
表格的边框外边界与表题的外边距外边界之间的间隔为1em，因为默认情况下表格的上外边距和标题的下外边距折叠了（对于caption-side: top的情况）。
表题的宽度由<table>元素中内容的宽度决定，<table>是表题元素的包含块。
多数情况下，表题可以像块级元素那样装饰。

caption-side属性用于设置表题元素<caption>放在表格上方还是下方。

  • 取值语法：
  • 初始值：top
  • 适用于：display: table-caption的元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

边框模型

单元格有两种完全不同的边框模型：

  • 分离边框模型(separated border model)：相邻的单元格各自有独立的边框。
  • 折叠边框模型(collapsed border model)：相邻单元格共享同一个边框。视觉上单元格彼此紧挨着。

border-collapse属性用于设置应用哪种边框模型。

  • 取值语法：
  • 初始值：separate
  • 适用于：display: table或display: table-inline的元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

分离边框模型

分离边框模型下，边框只能应用于单元格<th>或<td>，以及表格自身<table>。行，列，行组和列组没有边框（即，用户代理必须忽略这些元素的border属性）

单元格间距

使用分离边框模型时，可以通过border-spacing属性调整相邻单元格的 border box 之间、最外圈单元格与表格元素之间的间距。不论边框有多宽，间距总是这么大。这段间隔中，行，列，行组和列组的背景是不可见的，会让表格背景透出来。

  • 取值语法：
  • 初始值：0
  • 适用于：display: table或display: table-inline的元素
  • 计算值：两个绝对长度
  • 继承性：是
  • 动画性：否

当指定一个值时，这个值既是相邻单元格之间的横向间距，也是纵向间距。
当指定两个值时，第一个值是横向间距，第二个值是纵向间距。

注意适用元素，该属性要在<table>元素上声明，而不是在单元格上。

空单元格是否显示

使用分离边框模型时，可以通过empty-cells属性设置空单元格（没有内容，但空格、回车、制表符、换行除外）是否绘制边框和背景 。

  • 取值语法：
  • 初始值：show
  • 适用于：display: table-cell的元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • show：绘制边框和背景。
  • right：不绘制边框和背景。

折叠边框模型

在折叠边框模型中，边框可以应用于单元格、行、行组、列、列组和表格自身。
display: table和display: inline-table的元素不能有内边距，可以有外边距。因此表格四周的边框与最外层单元格之间没有间隔。
单元格之间没有间隔，只会在假想的中线(grid line)处绘制一条折叠的边框。
外侧的边框超出了表格的宽度，表格的一半含在宽度内，另一半在外边距中。

边框折叠规则：

  1. 如果折叠的边框中有一条边框样式为border-style: hidden，则该边框具有最高优先级，该位置的边框会被隐藏。
  2. 否则，较宽的边框比较窄的边框优先级高。
  3. 否则（宽度相同），样式优先级高的边框优先级高，样式优先级从大到小为：double、solid、dashed、dotted、ridge、outset、groove、inset、none
  4. 否则（宽度和样式相同），颜色取自下列元素，且优先级从大到小为：单元格、行、行组、列、列组、表格；如果同为行，例如上一行和下一行，则颜色取自离顶部和左边较远的边框。

表格尺寸

表格的宽度有两种确定方式：

  • 固定宽度布局
  • 自动宽度布局

表格的高度自动计算。

border-collapse属性用于设置应用哪种宽度确定方式。

  • 取值语法：
  • 初始值：auto
  • 适用于：display: table或display: table-inline的元素
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

固定宽度布局

  • 如果列元素的width属性值不是auto，设定的就是整列的宽度。
  • 如果列的宽度为auto，但是表格第一行中位于这一列的单元格的width属性值不是auto，则这个单元格的宽度就是整列的宽度。如果这个单元格跨多列，则宽度平分到各列中。
  • 如果列的宽度仍是自动确定的，则由浏览器自动确定宽度，尽可能使各列宽度相等。

表格的宽度将是表格width值与各列宽度之和中较大的那个。如果表格宽度比各列宽度之和大，则多余的宽度将增加到各列上。

这种布局渲染速度快，因为列的宽度由表格第一行决定，后续各行中的单元格都与第一行确定的列宽度保持一致。

为单元格设置的width值将被忽略，如果内容在单元格中放不下，可以用overflow属性决定内容可见、裁剪还是显示滚动条。

自动宽度布局

  1. 计算一列中每个单元格的最小宽度和最大宽度
    • 确定显示内容最少需要多少宽度。在这个过程中，内容可以换行，但不能超出单元格框。如果为单元格设置了width值，而且比最小宽度大，那么单元格的最小宽度将被设为width值。如果单元格的width值为auto，那么单元格的最小宽度将被设为内容的最小宽度。
    • 最大宽度即不换行（强制换行除外，例如<br>）显示内容所需的宽度。
  2. 计算各列的最小宽度和最大宽度
    • 列的最小宽度由那一列中单元格最小宽度值最大的那个确定。如果为列显式设定了width值，而且比列中每个单元格的最小宽度大，那么列的最小宽度被设为width
    • 列的最大宽度等于那一列中单元格最大宽度值最大的那个。如果为列显式设定了width值，而且比列中每个单元格的最大宽度都大，那么列的最大宽度被设为width值。这两个行为即列的宽度将增大到与最宽的单元格相等。
  3. 如果单元格横跨多列，那么所跨的那几列的最小宽度之和必须等于单元格的最小宽度；同样，列的最大宽度之和也要等于横跨多列的单元格的最大宽度。用户代理应该把列宽度的变化平均分到所跨的各列中。

对齐方式

横向对齐单元格中的内容，使用text-align属性，单元格相当于块级框。
纵向对齐单元格中的内容，使用vertical-align属性，但应用到单元格上的作用与平时不同。

  • top：单元格中内容的顶部与行的顶部对齐。在跨行的单元格中，内容的顶部与所跨的第一行的顶部对齐。
  • bottom：单元格中内容的底部与行的底部对齐。在跨行的单元格中，内容的底部与所跨的最后一行的底部对齐。
  • middle：单元格中内容的中线与行的中线对齐。在跨行的单元格中，内容的中线与所跨各行整体的中线对齐。
  • baseline：单元格的基线与行的基线对齐。在跨行的单元格中，单元格的基线与所跨的第一行的基线对齐。行的基线由第一行文本的基线定义。
