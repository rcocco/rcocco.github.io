浮动
2020-10-21|计算机前端CSS样式语言

浮动在过去是所有Web布局方案的基础（很大程度依赖于clear属性），但浮动并不是为布局而生的，把它作为布局工具是无奈之举，和使用表格布局一样，当时没有其他办法来实现布局效果。
浮动源于图像，它使得图像周围的元素围绕图像流动。

浮动

float属性将元素向左或者向右移动，直到碰到包含块或者另一个浮动元素的边界，使得文字或者行内元素将环绕它的 margin box。元素会从常规流中移除，尽管仍然在流中保留了一部分（和绝对定位相反）

  • 取值语法：
  • 初始值：none
  • 适用于：所有元素
  • 计算值：指定的值，但display:none或position为absolute或fixed时计算值为none
  • 继承性：否
  • 动画性：否

用户代理可以把根元素上的float当作none

  • 浮动隐含着使用块布局，即它会把display属性的计算值变为block，浮动元素周围的外边距不会折叠。
  • 浮动将元素向左边或者右边移动，一直移动到碰到包含块边界或者另一个浮动元素的边界，如果没有空间（浮动元素在文档中之前的元素、匿名文本也将占用空间），它会向下移动再尝试向左向右移动。如果存在行框，浮动元素的上外边界会与当前行框的上边界对齐，例如行内元素只是平行移动，如果元素不浮动时候在第二行，向左浮动时也是在第二行向左移动，即便第一行有空间它也不会上去的。
  • 因为浮动元素不在常规流内，在浮动元素之前或者之后创建的非定位块盒会垂直排列，就像浮动不存在一样，但是这些块盒会根据需要，缩短挨着浮动元素创建的当前行及后续行框，以便给浮动元素的外边距框让出空间。行框挨着浮动元素的条件是存在一个同时满足这4个条件的垂直位置：
    • (a)位于行框的顶端或顶端下方
    • (b)位于行框的底端或底端上方
    • ©在浮动元素上外边距边界下方
    • (d)在浮动元素下外边距边界上方
      换言之，如果存在一个垂直位置，既在一个行框顶端和底端之间，又在一个浮动元素上下外边距之间，则称行框挨着浮动元素。
      注意：也就是说0外高度（outer height）或者负外高度（盒）的浮动不会缩短行框
  • 如果一个缩短的行框小到无法容纳任何内容了，那么该行框会向下移动（并重新计算其宽度）直到有些内容能适应（它的空间）或者再没出现其它浮动了。当前行里浮动盒之前的所有内容都会在浮动方向另一侧的相同行重新排列（reflow）。换句话说，如果行内级盒被放在同一行中的左浮动元素之前，并且适应剩余的行框空间，左浮动（盒）就放在该行，与行框的顶端对齐，然后，相应地，该行中已经放好的这个内联级盒会被移动到浮动盒右边（右边是左浮动的另一侧），对于rtl和右浮动，反之亦然
  • 表格，块级替换元素或常规流中建立了新的块格式化上下文的元素（例如一个’overflow’不为’visible’的元素）不能和与元素自身处于同一块格式化上下文中的任何浮动（盒）的外边距框重叠。如果必要的话，实现应该把它放在所有之前出现的浮动（盒）下方，以清除（clear）该元素（受到的浮动影响），如果空间充足的话，可以放得与浮动（盒）相邻。它们甚至会让该元素的边框框比10.3.3小节中定义的更窄。CSS2没有定义UA什么情况下可以把该元素挨着浮动（盒）放以及该元素可以变得多窄
  • 浮动（盒）的内容会堆叠起来，就像浮动（盒）生成了新的堆叠上下文（stacking contexts）一样，除了所有定位元素和实际创建了新的堆叠上下文并参与了浮动（盒）的父级堆叠上下文的元素。浮动（盒）可以与常规流中的其它盒重叠（例如，当常规流盒挨着具有负外边距的浮动（盒）时），出现这种情况时，浮动（盒）会被渲染在非定位（non-positioned）流内块之上，流内行内（盒）之下；换句说话，行内盒与浮动元素重叠时，行内盒的边框、背景和内容都在浮动元素“之上”渲染。块级盒与浮动元素重叠时，块级盒的边框、背景在浮动元素“背后”渲染，而内容在浮动元素“之上”渲染。

浮动的九条规则如下：
注意：这些规则里涉及的其它元素仅仅是指与浮动（盒）处于同一个块格式化上下文的其它元素。

  1. 左浮动盒的左外边界不能位于其包含块的左边界的左边。向右浮动的元素也有类似的规则
    圆圈中的数字是元素在文档中的原始位置。
  2. 如果当前盒是向左浮动的，并且在这之前源文档中还有元素生成了左浮动盒，那么对于每一个之前的盒，要么当前盒的左外边界在之前的盒的右外边界的右边，要么其顶部要低于之前盒的底部。右浮动盒也有类似的规则。如下图，三个左浮动元素，3的左边界位于1的右边界右边，3的顶部低于2的底部，满足条件。
  3. 左浮动盒的右外边不能位于挨着它的任意右浮动盒的左外边界的右边。右浮动盒也有类似的规则
  4. 浮动盒的外顶端（outer top）不能高于其包含块的顶部。当浮动（盒）出现在两个折叠外边距（collapsing margins）之间时，浮动（盒）的定位就像它有个空的匿名块父级存在于（当前）流一样。这个父级的位置通过外边距合并章节中的规则来定义。例如，有三个段落，中间那个段落是浮动的。此时中间那个段落在浮动时好似有一个块级父元素，避免浮动的段落一直向上移动，超出三个段落共有的元素。

等价于

  1. 浮动盒的外顶端不能高于源文档中任何在该元素之前的元素生成的块盒或者浮动盒的外顶端
    这就是为什么第三个元素要对齐第二个元素，而不是第一个元素。
  2. 元素的浮动盒的外顶端不能高于任何含有源文档中在该元素之前的元素生成的盒的行框的顶端（浮动元素的顶边不能高于源代码中出现在浮动元素之前的元素生成的框体所在行框的顶边。）
    如果图像在1的位置，则它的顶边最高只能是该行行框的顶边。
  3. 左浮动元素的左边如果还有一个左浮动元素，那么它的右外边界不能在包含块右边界的右侧；类似地，右浮动元素的右边如果还有一个右浮动元素，那么它的右外边界不能在包含块左边界的左侧。
    这条规则说，浮动元素永远不会超过包含块边界，除非浮动元素太宽，包含块放不下。如果多个浮动元素排在一起时超出了包含块边界，则后面的浮动元素将放到前面浮动元素的下面的某个位置。
  4. 浮动元素必须放在尽可能高的位置上。
  5. 左浮动盒必须尽量往左放，右浮动盒尽量往右放。更高的位置要比更左/右的位置优先

清除浮动

clear属性说明了元素盒的哪一边不能与之前的浮动盒相邻，它令元素自身下移使得某一边没有浮动元素。clear属性不考虑元素自身的或者其它块格式化上下文中的浮动。

  • 取值语法：
  • 初始值：none
  • 适用于：块级元素
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

属性值含义如下：

  • left：要求该盒的上边框边界位于源文档中在此之前的元素形成的所有左浮动盒的下外边界下方

  • right：要求该盒的上边框边界位于源文档中在此之前的元素形成的所有右浮动盒的下外边界下方

  • both：要求该盒的上边框边界位于源文档中在该元素之前的元素形成的所有左浮动盒和右浮动盒的下外边界下方

  • none：对该盒相对于浮动（盒）的位置没有约束

  • 属性值不为none就隐含了要引入空隙（clearance）。空隙会阻止外边距合并，并作为元素margin-top上方的空间。用来在垂直方向上把元素推离浮动（元素）（It is used to push the element vertically past the float）

  • 想计算设置了clear的元素的空隙：

    1. 先要确定该元素的上边框边界的假定位置。该位置是当元素的clear属性为none时其上边框边界实际所在的位置。
    2. 如果该元素的上边框边界的假定位置没有越过（past）相关的浮动（盒），那么就得引入空隙，并根据规则合并外边距
    3. 然后，空隙的高度（amount）被设置为下面两者的较大值：
      (a)把块的边框边界放在最低的将被clear的浮动（盒）的下外边界，所必需的高度
      (b)把块的上边框边界放在其假定位置（clear为none时的位置），所必需的高度

    或者，空隙被精确地设置为把块的边框边界放在最低的将被clear的浮动（盒）的下外边界，所必需的高度
    注意：空隙可以为负或者为0

当该属性被设置在浮动元素上时，会改变浮动定位的规则。需要加上额外的约束：浮动（盒）的上外边界必须处于所有在此之前的左浮动盒（clear: left的情况）的下外边界下方，或者所有在此之前的右浮动盒（clear: right的情况）的下外边界下方，或者两者都要满足（clear: both的情况）

闭合浮动

根据规范，一般情况下，如果常规流中的块级非替换元素的height是auto，高度取决于该元素是否含有常规流中的块级子元素，以及是否具有padding或border。
因此如果该元素只有浮动后代，在计算高度时浮动后代会被忽略，就好像不存在一样。
但很多时候我们需要让高度计算浮动后代，称之为闭合浮动。

办法如下：

  • 使元素为内容建立新的块格式化上下文，从而采用【块格式化上下文中的auto高度】规则计算浮动后代高度。具体来说：
    • 使元素浮动
    • 使元素绝对定位
    • 使元素变为非块盒的块容器（例如inline-blocks，table-cells和table-captions）
    • 设置overflow不为visible
    • 设置display: flow-root
  • 利用清除浮动规则，通过伪元素::after或在浮动元素代码之后的其他显式元素，设置display: block与clear: both，令其放置在之前所有浮动元素之下，进而变相的增加了包含块高度。

浮动形状

shape-outside属性可以定义浮动形状，控制其他文本应该按照什么形状来环绕浮动元素。

  • 取值语法：
  • 初始值：none
  • 适用于：浮动元素
  • 计算值：basic-shape为定义的值，image是绝对URI，否则是指定的值
  • 继承性：否
  • 动画性：basic-shape时是，其他时候否

例如，将浮动形状设置为一个含有透明部分的图像，则内容会流入透明部分。
