文本属性

参考资料：https://www.zhangxinxu.com/wordpress/2015/11/diff-word-break-break-all-word-wrap-break-word/

  • 块级方向指当前书写模式放置块级元素的方向。以英语为例，块级方向从上到下，或者说是纵向的，例如一个段落（或其他文本元素）放在另一个段落下面。
  • 行内方向指块级元素中行内元素的书写方向。以英语为例，行内方向从左到右，或者说是横向的。对阿拉伯语和希伯来语等语言来说，行内方向从右到左。

缩进

text-indent属性用于指定段落第一行文本开头缩进的长度，长度可以是负值，负值可以实现悬挂缩进效果，即第一行突出。

  • 取值语法：
  • 初始值：0
  • 适用于：块级元素
  • 百分数：相对于所在块级元素的宽度
  • 计算值：百分数；长度值是一个绝对长度
  • 继承性：是
  • 动画性：是

无论怎么缩进第一行永远有文字，原第一行的文字可能会下移到其他行。非常大的缩进会使文字超出元素宽度。
同理，无论多大的负缩进，第一行永远有文字，后面的行的文字可能会上移到之前的行。即便所有字都向左缩进看不到了，这一行仍然有字。只不过超出了元素宽度。

Firefox65和Chrome72以前百分数是相对于父元素的内容宽度，后来修复了这个行为：Percentage on text-indent is now relative to containing block instead of the parent

水平对齐文本

text-align属性控制元素中各文本行的水平对齐方式。

  • 取值语法：
  • 初始值：start。如果浏览器不支持start，当direction是ltr时为left，当direction是rtl时为right。
  • 适用于：块级元素
  • 计算值：指定的值，match-parent除外
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • left：左对齐。对于纵向书写的语言，left对应于书写起边。
  • right：右对齐。对于纵向书写的语言，right对应于书写的终边。
  • center：居中对齐。
  • start：和起边对齐。对于从左到右书写的语言，等价于left；对于从右到左书写的语言，等价于right；
  • end：和终边对齐。对于从左到右书写的语言，等价于right；对于从右到左书写的语言，等价于left；
  • justify：两端对齐。使文本除了最后一行的两端分别与行框(line box)边缘对齐，然后调整单词之间的空白，使得每一行长度完全一致。
  • match-parent：类似于inherit，但如果父元素的值是start或end，match-parent的值会根据父元素的direction变成left或right。而inherit的值就是start或end。

text-align-last属性控制块元素中文本最后一行，或者有强制换行（例如<br>）的行的水平对齐方式。

  • 取值语法：
  • 初始值：auto
  • 适用于：块级元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • auto：当text-align为justify时，值为start。其他情况受影响的行取值跟随text-align属性的值。
  • left：左对齐。对于纵向书写的语言，left对应于书写起边。
  • right：右对齐。对于纵向书写的语言，right对应于书写的终边。
  • center：居中对齐。
  • start：和起边对齐。对于从左到右书写的语言，等价于left；对于从右到左书写的语言，等价于right；
  • end：和终边对齐。对于从左到右书写的语言，等价于right；对于从右到左书写的语言，等价于left；
  • justify：两端对齐。使文本的两端分别与行框(line box)边缘对齐，然后调整单词之间的空白，使得每一行长度完全一致。

如果元素既是第一行也是最后一行，text-align-last的优先级比text-align高。
两端对齐调整的是单词之间的空白，对于中文来说，每一个字被认为是一个单词。
如果只有一行字希望两端对齐，text-align属性设为justify是无效的，因为它明确说明了除了最后一行。应该使用text-align-last属性。

行高

line-height定义文本行的基线之间的最小距离（基线之间实际距离可能更大），与字号无关，决定着元素所在方框的高度是增还是减。line-height不等于行距，行距是它的一部分，它是除字体高度之外在文本行上方的额外空间。也就是说line-height的值与字体高度之差才是行距。

  • 取值语法：
  • 初始值：normal
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于元素自身的字号
  • 计算值：长度和百分数得到绝对长度；否则是指定的值。
  • 继承性：是
  • 动画性：是

文本行中的每一个元素构成一个内容区(content area)，其高度由字体的高度决定。
随内容区出现的是一个行内框(inline box)，如果不考虑其他因素，行内框的高度应该与内容区高度相同，但由于line-height的存在二者高度往往是不同的。
元素的行距等于line-height的值减去font-size的值。这个值是行距的总值，可能为负数。行距被分为两半，一半放到内容区上面，一半放到内容区下面。内容框高度加上行距，就是行内框的高度。
例如font-size: 14px; line-height: 18px;，则内容区高度也是14px，二者之差为4px，被分成两半放到内容区上面和下面，于是内容区上面多出2px，下面也多出2px。
确定了一行所有元素的行内框之后，行框(line box)也就确定了。行框的顶端就是最高的行内框的顶端，行框的底端就是最低的行内框的底端。
即公式：line-height = font-size + 行距，对于数字值，给定行距可以得到：
<number> = 行距 / font-size + 1

属性值的具体作用如下：

  • normal：取决于用户代理。一般是字体高度的1.2倍。
  • <number>：无单位数字，行高将是该数字乘以元素自己的font-size大小。这是首选方式。
  • <length>：长度值，即便给定绝对长度，浏览器实际使用的值也可能不同。相对长度em和ex是相对于元素的font-size值计算，在继承时可能产生意外结果。
  • <percentage>：百分值。相对于元素的font-size值计算，在继承时可能产生意外结果。

注意如果父元素设定的行高是百分比或长度值，子元素继承的是计算值，而非原来的百分比或长度值。这意味着子元素的行高是相对于父元素的字体计算的，因为继承的计算值是这么算的。这一点可能导致子元素的行高过大或过小，因为这种情况下的行高不是基于自身的字号算出的。
将行高设置为数字可以解决这个问题，因为数字值继承的仍然是数字，而数字又会乘以子元素自己的font-size。

纵向对齐文本

vertical-align属性用于设置行内元素和置换元素（例如图像和表单输入框）的纵向对齐方式。它不影响块级元素中文本的对齐方式。

  • 取值语法：
  • 初始值：baseline
  • 适用于：行内元素和单元格，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于元素自身的line-height计算
  • 计算值：长度和百分数得到绝对长度；否则是指定的值。
  • 继承性：否
  • 动画性：<length>、<percentage>
  • 备注：用在单元格上只能取baseline、top、middle和bottom

单词间距

word-spacing属性用于设置单词之间的距离在默认值基础上增加多少。可以为负值，表示减少。“单词”是指两侧具有某种空白的非空白字符构成的字符串，因此该属性不能用于中文。

  • 取值语法：
  • 初始值：normal
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：长度值得到绝对长度；否则是normal
  • 继承性：是
  • 动画性：是

属性值的具体作用如下：

  • normal：等价于长度值0，即默认的间距，通常由字体决定。
  • <length>：正长度值单词之间的距离增加给定长度，负长度值距离减少给定长度。

字符间距

letter-spacing属性用于设置字符之间的距离在默认值基础上增加多少。可以为负值，表示减少。

  • 取值语法：
  • 初始值：normal
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：长度值得到绝对长度；否则是normal
  • 继承性：是
  • 动画性：是

属性值的具体作用如下：

  • normal：默认的间距，通常与0相同，区别在于，如果元素设定了text-align: justify，为了让文本占满整行宽度，字符和单词之间的距离可能会被调整。
  • <length>：正长度值字符之间的距离增加给定长度，负长度值距离减少给定长度。长度值不会因为两端对齐而被调整。

继承得到的是计算值，因此可能遇到和行高一样的问题，计算值基于父元素的相关属性计算，然而这个值不一定适合后代元素。该属性没有行高那样的换算系数，因此你只能为每一个受影响的后代元素单独设定。

文本转换

text-transform属性用于改变文本的大小写。该属性的优点在于一个规则就能实现大小写转换，日后修改也方便。比font-variant先应用。
使用场景例如输入礼品卡号时让用户输入的文字自动大写，不需要用户按住Shift键来输入大写。

  • 取值语法：
  • 初始值：none
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • none：不对文本做任何修改。
  • uppercase：把文本转换成大写字母。
  • lowercase：把文本转换成小写字母。
  • capitalize：把各单词首字母大写。

文本装饰

text-decoration属性用于设置文本上的划线，例如下划线和删除线。它现在是四个属性text-decoration-line、text-decoration-color、text-decoration-style和text-decoration-thickness的简写属性。以前相当于text-decoration-line属性。

  • 取值语法：
  • 初始值：每个被简写的属性的初始值，即none solid currentColor auto
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

文本装饰线型

text-decoration-line属性用于设置文本装饰的种类。允许同时设置多种线型。

  • 取值语法：
  • 初始值：none
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

属性值的具体作用如下：

  • none：没有文本装饰。
  • underline：下划线。
  • overline：上划线。
  • line-throug：中划线（删除线）

文本装饰样式

text-decoration-style属性用于设置文本装饰的线的样式。

  • 取值语法：
  • 初始值：solid
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

属性值的具体作用如下：

  • solid：实线。
  • double：双实线。
  • dotted：点组成的线。
  • dashed：短线组成的线（虚线）
  • wavy：波浪线。

文本装饰颜色

text-decoration-style属性用于设置文本装饰的线的颜色。颜色适用于所有线型，因此你无法单独为下划线和删除线设置不同颜色，但可以通过使用嵌套元素，将不同的线型应用于每个元素，然后基于每个元素来指定线颜色来实现此效果。

  • 取值语法：
  • 初始值：currentColor（代表元素的color属性值）
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：计算的颜色值。
  • 继承性：否
  • 动画性：否

文本装饰粗细

text-decoration-style属性用于设置文本装饰的线的粗细。粗细适用于所有线型，因此你无法单独为下划线和删除线设置不同粗细，但可以通过使用嵌套元素，将不同的线型应用于每个元素，然后基于每个元素来指定线粗细来实现此效果。

  • 取值语法：
  • 初始值：auto
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于元素自身的font-size值
  • 计算值：指定的值
  • 继承性：否
  • 动画性：否

属性值的具体作用如下：

  • auto：浏览器自行选择适当的粗细。
  • from-font：如果字体文件包括关于合适的粗细的信息，使用该值。如果没有，等同于auto。
  • <length>：长度值。
  • <percentage>：将文本装饰线的粗细指定为当前字体的1em的百分值。百分值继承的时候是相对值，因此会随着字体的变化而缩放。浏览器必须至少使用1个设备像素。对于此属性的给定应用，即使存在具有不同字体大小的子元素，粗细在应用于它的整个盒子上也是恒定的。

文本阴影

text-shadow属性为文本添加阴影。

  • 取值语法：
  • 初始值：none
  • 适用于：所有元素，还可以应用到伪元素::first-letter和::first-line上
  • 计算值：一个颜色加三个绝对长度
  • 继承性：是
  • 动画性：是

属性值为none时没有阴影。
可以添加一个或多个阴影，彼此之间用逗号分隔。
每个阴影由一个可选的颜色和三个长度值定义，最后一个长度值也是可选的。

  • 颜色设定的是阴影的颜色，如果不设置将于文本颜色相同。
  • 前两个长度值是阴影离文本的距离，第一个长度值设定横向偏移，正的长度值向右偏，负的长度值向左偏；第二个长度值设定纵向偏移，正的长度值向下偏，负的长度值向上偏。
  • 可选的第三个长度值定义阴影的模糊半径。模糊半径是从阴影轮廓到模糊效果边界的距离。这个值越大越看不清阴影中的字，会扩散成模糊的一片。

处理空白

white-space属性用于指定对文档源码中的空格、换行符和制表符的处理方式。

  • 取值语法：
  • 初始值：normal
  • 适用于：所有元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • normal：换行符替换成空格，连续的空白符压缩成一个。允许文本自动换行。
  • pre：保留连续的空白符和换行符。禁止文本自动换行。
  • nowrap：换行符替换成空格，连续的空白符压缩成一个。禁止文本自动换行。
  • pre-wrap：保留连续的空白符和换行符。允许文本自动换行。
  • pre-line：保留换行符，连续的空白符被压缩成一个。允许文本自动换行。
  • break-spaces：与pre-line相同，除了：
    • 任何保留的空白序列总是占用空间，包括在行尾。
    • 每个保留的空格字符后都存在换行机会，包括空格字符之间。
    • 这样保留的空间占用空间而不会挂起，从而影响盒子的固有尺寸（最小内容大小和最大内容大小）。

见下表：

属性值 换行符 空格和制表符 文字换行 行尾空格
normal 合并 合并 换行 删除
pre 保留 保留 不换行 保留
nowrap 合并 合并 不换行 删除
pre-wrap 保留 保留 换行 挂起
pre-line 保留 合并 换行 删除
break-spaces 保留 保留 换行 换行

注意不论上面哪一种，遇到<br>元素都会换行。

制表符宽度

tab-size属性用于设置当显示制表符时，制表符显示的长度。注意，如果white-space的值导致空白被折叠了，这个属性虽然还会计算，但没有任何视觉效果。

  • 取值语法：
  • 初始值：8
  • 适用于：块级元素
  • 计算值：指定的数字或一个绝对长度
  • 继承性：是
  • 动画性：是

属性值的具体作用如下：

  • <integer>：制表符相当于<integer>个空格。
  • <length>：制表符将渲染成指定长度。

断字(连字符)

hyphens属性用于指定当单词在一行放不下可能跨越多行时是否用连字符将其断开，把一个单词分成两行写。可以在文本中手动插入换行符，也可以由浏览器自动插入换行符。
有两个Unicode字符用于指定潜在的换行时机：

  • 符号-，即U+2010，它是硬连字符，表示一个可见的潜在换行时机，即便该行并没有在这里打断单词，连字符仍然会显示。

  • 符号&shy;，即U+00AD，它是软连字符，如果没有打断单词，它并不会显示出来，如果有必要，浏览器应该在该处打断单词。

  • 取值语法：

  • 初始值：manual

  • 适用于：所有元素

  • 计算值：指定的值

  • 继承性：是

  • 动画性：否

属性值的具体作用如下：

  • manual：只有在文档中手动插入连字符（例如U+00AD或&shy;）处断字，否则不断字。
  • none：即便有手动插入的连字符，也不断字。U+00AD或&shy;将被忽略。
  • auto：即使没有手动插入连字符，浏览器也会在合适位置断字，如果有手动插入的连字符，浏览器一般会遵循。

所有语言的软换行

word-break属性用于控制不同语言处理文本软换行的方式，所谓软换行是为了避免文本溢出内容框而由用户代理自动进行的换行。

  • 取值语法：
  • 初始值：normal
  • 适用于：所有元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • normal：使用默认换行规则，文本会在单词之间换行，例如英语会在空格处换行，而CJK语言会在任何两个字符之间换行。
  • break-all：允许在任何两个字符（即便在单词内部）之间换行。使用这个值时，连字符不显示。
  • keep-all：CJK字符之间不换行。非CJK文本行为与normal相同。

CJK语言的软换行

line-breakk属性用于控制CJK语言处理文本软换行的方式，如果在一段英文中混杂部分CJK字符，该属性只作用于CJK字符。如果把内容声明为CJK语言，该属性会用于CJK字符，以及CJK文本中的非CJK字符，例如标点符号、货币符号和一些其他字符。

  • 取值语法：
  • 初始值：auto
  • 适用于：所有元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • auto：按正常行为软换行。用户代理可以根据情况变更换行方式，例如较短的行采用宽松的换行规则，较长的行使用严格的换行规则。其实它的作用是让用户代理根据需求在loose,normal和strict之间切换，甚至可以在一个元素的不同行之间切换。
  • loose：采用最宽松的换行规则，主要用于文本行较短的情况，例如报纸
  • normal：采用最常规的换行规则。
  • strict：采用最严格的换行规则。

文本换行

overflow-wrap属性用于行内元素，设置浏览器是否在不可断字的字符串中插入换行符，以防止文本溢出了行框。该属性以前叫word-wrap

注意：与相比word-break，overflow-wrap仅当无法将整个单词放在自己的行而不溢出的情况下，才会产生中断。
不溢出就无法将整个单词放在自己的行时，才会换行。

  • 取值语法：
  • 初始值：normal
  • 适用于：非替换行内元素
  • 计算值：指定的值
  • 继承性：是
  • 动画性：否

属性值的具体作用如下：

  • normal：按正常方式换行。即在单词之间换行，或者由所用语言决定。
  • break-word：可以在单词内部换行。与word-break: break-all相比，overflow-wrap会观察这一行是否有其他可以换行的字符，例如空格、CJK字符，然后在字符处换行，而不强行把英文单词打断。这会造成某些文本行后面留有很多空白。
