行框与对齐
2020-10-23|计算机前端CSS样式语言

参考资料：
https://stackoverflow.com/questions/36193355/what-is-the-definition-of-the-baseline-of-parent-box/51119730
https://christopheraue.net/design/vertical-align
《CSS权威指南 第四版》

行内元素的排版取决于行框，元素被放置在行框内，相邻两行的行框会紧挨着，即下一行的行框的顶边紧挨着上一行行框的底边。

行内框

行框取决于该行各元素的行内框，一行中的每个行内元素都会生成行内框，匿名文本会生成匿名行内框，空行内元素也会生成行内框。

行内非替换元素

对于行内非替换元素，行内框的高度就是line-height的值，行内框与内容区的垂直中线是对齐的，这是因为构造行内框时采用的办法是：将line-height减去内容区高度，得到的值平分给内容区上半部分和下半部分，如图：

行内框的高度可能等于、大于或小于内容区高度，如图：

内容区的高度由font-size属性控制，但不完全相等。在 Windows 上内容区高度可由公式Content-area=(Win Ascent + Win Descent)/Em Size * Font Size计算，其中Win Ascent、Win Descent和Em Size定义在字体文件中，如图：

字体文件中还定义了x-height，以及每个字形的基线位置等信息。

匿名行内框

根据规范，匿名行内框从它们的父级块盒继承了可继承的属性，不可继承的属性取其初始值。其中font-size和line-height都是可继承属性。于是匿名行内框的高度取决于父级块盒的line-height，内容区高度取决于父级块盒的font-size。

替换元素

行内框就是元素的 margin box，基线是 margin box 的下边界

行内块级元素

行内框就是元素的 margin box，基线分两种情况：

  • 如果有流内行框，且行内块级元素的overflow为默认值visible。例如文本，则基线是最后一个常规流中的行框的基线。
  • 如果有流内内容，但行内块级元素的overflow不是visible；又或者完全没有流内内容，则基线是 margin box 的下边界。

对齐

构造出了一行中所有行内元素的行内框后，就要进行对齐，对齐是根据vertical-align属性的值进行的。由于vertical-align是不可继承属性，因此匿名行内框会采用它的初始值baseline。
另外，每个行框还会以一个具有该元素的字体和行高属性的0宽假想行内盒开始。我们把这种假想盒叫做"strut"（这个名字是受TeX的启发）。strut盒也参与对齐。

对齐时，多数行内盒都要与 strut 盒的几个对齐线对齐，例如strut盒的基线、内容区顶部、内容区底部、半x-height（struct 盒中假想字母 x 的垂直中线，准确的定义为 struct 盒的基线加上半x-height高度）

  • 取值语法：
  • 初始值：baseline
  • 适用于：行内元素和单元格，还可以应用到伪元素::first-letter和::first-line上
  • 百分数：相对于元素自身的line-height计算
  • 计算值：长度和百分数得到绝对长度；否则是指定的值。
  • 继承性：否
  • 动画性：<length>、<percentage>
  • 备注：用在单元格上只能取baseline、top、middle和bottom

属性值具体作用如下：

  • baseline：把盒的基线与strut盒的基线对齐。
  • middle：把该盒的垂直中线与 strut 盒中假想字母 x 的垂直中线对齐
  • sub：把该盒的基线降低到合适的位置作为父级盒的下标（该值不影响该元素文本的字体大小）
  • super：把该盒的基线提升到合适的位置作为父级盒的上标（该值不影响该元素文本的字体大小）
  • text-top：把该盒的顶端和当前行的 strut 盒的内容区的顶端对齐
  • text-bottom：把该盒的底端和当前行的 strut 盒的内容区的底端对齐
  • <percentage>：相对于元素自身line-height计算得到绝对长度，将元素的基线与高于其 strut 盒基线给定长度的线对齐。允许为负值，表示低于 strut 盒基线给定长度。值0%表示与 strut 盒基线对齐，等同于baseline。
  • <length>：将元素的基线与高于其 struct 盒基线给定长度的线对齐。允许为负值，表示低于 struct 盒基线给定长度。
  • bottom：将元素行内框的底边与所在行框的底边对齐（如果有多个满足要求的位置，对齐必须让行框高度最小化）
  • top：将元素行内框的顶边与所在行框的顶边对齐（如果有多个满足要求的位置，对齐必须让行框高度最小化）

行框

在按照各元素vertical-align的要求对齐所有元素后，就需要生成行框。
行框需要同时包住一行中所有的行内框，行框的高度是从一行中所有行内框的最高点，到该行所有行内框的最低点，即行框的顶边经过最高的行内框的顶边，行框的底边经过最低的行内框的底边。

示例

示例：
黄色矩形是没有流内内容的行内块级元素。
粉色矩形是有流内内容，且元素自身overflow:visible的行内块级元素。
橘色矩形是有流内内容，但元素自身overflow:auto的行内块级元素。
红色矩形是图片的大小。

  1. 所有行内元素默认基线对齐，如图，将各对齐线摆好。绿色是行框。
  2. 所有元素默认基线对齐，可以看到图片<img>元素下方有空隙：这是显然的，空隙是stuct盒的基线到它内容区底部的高度。图片的vertical-align默认是baseline，替换元素的基线是 margin box 的下边界，根据要求它必须与stuct盒的基线对齐，在下面第二张图中，用黄色背景的x表示假想的 struct盒，可以看到图片底部确实与stuct盒的基线对齐了。对齐后生成行框，行框需要包括所有行内框，所以生成的行框就是粉色背景的范围。
