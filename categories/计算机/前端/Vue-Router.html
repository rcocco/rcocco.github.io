Vue-Router
2021-03-22|计算机前端

概述

Vue Router是Vue官方提供的前端路由插件。其基本思想是根据不同的（前端）请求路径，渲染对应的组件。

前端路由

后端路由：根据HTTP请求报文的请求路径和请求方法，返回对应的内容。
前端路由：始于Ajax，操作后不刷新页面，而是根据返回的结果修改DOM

在Ajax的基础上改网址，使其好像和后端路由一样，不同的URL对应不同的页面。
最早用井号实现，井号后修改不会发送请求。
后来为了美观不想出现井号，用新增的Push State API。
PJax，Push State+Ajax，前者用于改地址。

要么location.hash
要么history.pushState

前端路由更快，不需要和当前页面重复的HTML。

类似于Android程序

安装

对于不同的环境，有不同的安装方式：

  • NPM：使用npm install vue-router进行安装。
  • 浏览器：使用<script src="https://unpkg.com/vue-route"></script>引入。
  • Vue CLI：使用vue add router添加插件，注意它会覆盖App.vue的内容。

定义路由

要定义前端路由，需要实例化一个VueRouter对象，它的构造函数接受一个对象，该对象具有routes属性，属性值是一个数组，每个数组元素是具有path和component的路由记录，path指定前端路由的路径，component指定匹配该路由时渲染的组件，该组件会替换模板中的<router-view>元素。这种路由指定的组件也称为视图组件。

每当前端路由拿到一个地址，会按照数组中的顺序逐个检查是否匹配某个路由，如果匹配则渲染对应的组件。因此有通配符的路由需要放后面，否则可能永远也匹配不到它。

最后还需要将VueRouter实例添加到Vue实例的选项对象的router属性上。

动态路由匹配

VueRouter构造函数接收的参数对象的routes属性中，每个元素是一个路由。其中path是一个模式字符串（使用了path-to-regexp库），模式字符串类似于正则，但也有一些不同的规则：

  • 点.和连字符-只有字符串含义
  • 括号()用于分组。
  • 命名参数：即正则中的命名捕获组，它的语法为冒号后跟参数名称:参数名称，参数名称只能是[A-Za-z0-9_]中的字符。
    • 在参数名称后面加问号?可以使该命名参数变为可选的，例如/bar/:foo?匹配/bar和/bar/abc
    • 在参数名称后加*表示匹配零个或多个参数，例如/bar/:foo*匹配/bar和/bar/123/456
    • 在参数名称后加+表示匹配一个或多个参数。
    • 命名参数默认捕获除了点、连字符和斜杠以外的所有字符，即[^.-/]+，要改变捕获规则，可以在命名参数后面加一个括号，括号内写正则表达式，例如:id(\\d{4,4})。捕获的值可以在this.$route.params对象中通过参数名称获取到。
  • 未命名参数：例如/:foo/(.*)，注意单独的*等价于(.*)，因此也可以写成/:foo/*。星号*匹配任意个所有字符。捕获的值可以在this.$route.params.pathMatch中获取到。

使用了通配符或命名参数的path，将很多个地址对应到一个组件上，这种形式称为动态路由。

嵌套路由

通过children属性就可以定义嵌套路由，它是指先匹配path，然后渲染组件component（同时要求这个组件里具有<router-view>元素），接着再在当前path的基础上检查children中定义的嵌套路由，如果匹配，则用对应的组件替换父组件中的<router-view>元素。
定义一个path为空串的嵌套路由，可以在没有嵌套路由时，组件中<route-view>能够有默认渲染的组件。

可以用name属性为一条路由起一个名字，以便导航时指定此路由。

别名

在一条路由中使用alias可以创建路径的别名。
“重定向”的意思是，当用户访问 /a时，URL 将会被替换成 /b，然后匹配路由为 /b。
/a 的别名是 /b，意味着，当用户访问 /b 时，URL 会保持为 /b，但是路由匹配则为 /a，就像用户访问 /a 一样。

为视图组件传参

在一条路由中使用props可以为指定的视图组件传递参数。普通组件中，参数直接写为HTML属性，但对于视图组件，只有命名参数不同的URL指向同一个组件，你不可能把每种参数都当成属性写在模板上，这时候可以在定义路由时将命名参数作为组件参数传递。
当然你也可以通过this.$route.params来访问。

如果props是一个布尔值，则为true时，整个route.params都会作为组件参数。
如果props是一个对象， 则用来指定命名视图的props，属性键是命名视图的名字，属性值是一个布尔值，含义和前述相同。
如果props是一个函数，则它接受route作为参数，返回一个值作为组件参数。

router和route

VueRouter实例上有很多有用的方法，每个因为前端路由匹配而渲染的组件，都可以通过this.$router访问到对应的VueRouter实例，简称router。
this.$route是路由对象 (route object)，表示当前激活的路由的状态信息，包含了当前 URL 解析得到的信息，还有 URL 匹配到的路由记录 (route records)。

导航

链接元素

由于前端路由的地址与实际的URL中请求路径不完全相同，不方便像后端路由那样确定<a>元素中的链接地址，也不能去执行默认的点击事件来向服务器发送请求，因此Vue Router提供了<router-link>元素来添加链接，它的用法和<a>元素类似，最终也会被渲染为一个<a>元素，但其默认行为被修改了。

编程式导航

除了用户点击<route-link>来导航之外，也可以通过编程的方式导航。

前进到指定页面

router.push(location, onComplete?, onAbort?)
它会向浏览器的历史记录栈中添加一条记录，用户进入别的页面后再点击回退按钮时可以返回此页面。此方法在点击<route-link>元素后也会被隐式的调用，因此二者是等价的。
location参数可以是一个表示完整路径的字符串，也可以是一个对象，path属性是完结路径，name属性是命名路由的名字，params是和命名路由一起使用时的命名参数部分，query是查询字符串。
onComplete是可选参数，它是导航成功完成后 (在所有的异步钩子被解析之后) 执行的回调函数。
onAbort是可选参数，它是导航终止时 (导航到相同的路由、或在当前导航完成之前导航到另一个不同的路由) 执行的回调函数。

替换当前页面

router.replace(location, onComplete?, onAbort?)
类似router.push，可以导航到新页面，但它不会向浏览器的历史记录栈中添加新记录，而是替换掉当前记录。

前进或后退几步

router.go(n)
用于在浏览器的历史记录栈中前进或后退。

实现模式

Vue Router默认使用 hash 模式， 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载。
这种模式的缺点就是不美观，有一个#，为此可以使用 history 模式，这种模式充分利用 history.pushState API 来完成 URL 跳转而无须重新加载页面。
在创建VueRouter实例时通过添加mode: 'history'指定使用 history 模式。

在 history 模式下，需要后端配合，假设fe-route/往后都是前端路由的路径，那么后端针对任何以fe-route开头的请求都应该返回同一个HTML页面（包含了Vue和Vue Router的）。同时资源文件要么使用绝对路径（否则fe-route/aaa/bbb加载的JS相对路径是相对于这个地址，但实际文件在fe-route/a.js），要么使用相对路径加<base>标签修改相对根目录

这种模式下服务器就不再返回此路径下的404页面了，404页面必须由前端路由处理。

这种模式不利于缓存，因为井号模式每次HTTP请求路径都是完全相同的，因此可以做协商缓存。而History模式每次HTTP请求路径是不同的，只不过服务器返回的资源相同。

导航守卫

导航守卫用来告诉router是否应该允许导航到请求的路由。
守卫函数通常接受三个参数(to, from, next)，通过调用next()方法并传递不同的参数来允许导航或中断导航。执行守卫函数时必须保证执行期间有且只有一次next()调用（if-else中书写两次调用是可以的，因为只会执行一次），否则导航就被阻塞了，无法知道应该允许还是中断。

  • to是准备进入的路由对象。
  • from是准备离开的路由对象。
    路由对象在视图组件上是this.$route，它具有下列属性：
    • path：字符串，路由的绝对路径，例如/foo/bar
    • params：对象，路由匹配时捕获的参数，没有路由参数就为空对象。
    • query：对象，路由路径的查询参数，例如/foo?user=1，则对象是{user: 1}，若没有查询参数则是空对象。
    • hash：字符串，路由的hash值（片段标识符，包括#），如果没有 hash 则为空串。
    • fullPath：包括查询参数和hash的完整路径。
    • matched：数组，包含路由记录的副本，路由记录就是定义路由时形如{path:'/foo'}的对象。
    • name：路由的名称（如果有）
    • redirectedFrom：重定向来源的路由（那个路由重定向到了当前路由）的名字
  • next是一个函数，守卫函数内应该调用它以允许或中断导航。调用方式有以下几种：
    • next()：进入下一个守卫函数，如果没有，则导航状态变为confirmed，允许导航。
    • next(false)：中断导航，URL地址会被重置为from路由的地址。
    • next('/')：中断当前导航，开启到新路径的导航。它和router.push接受一样的参数。
    • next(error)：若参数是异常对象，则中断导航，并将错误传给router.onError()注册过的回调。

导航流程

  1. 触发导航
  2. 在deactivated的组件内调用beforeRouteLeave守卫函数
  3. 调用全局beforeEach守卫函数
  4. 在resued的组件内调用beforeRouteUpdate守卫函数
  5. 在route configs里调用beforeEnter
  6. resolve异步路由组件
  7. 在activated的组件里调用beforeRouteEnter
  8. 调用全局beforeResolve守卫函数
  9. 导航被确认，允许导航
  10. 调用全局afterEach钩子函数
  11. 触发DOM更新
  12. 调用beforeRouteEnter守卫函数中传给next的回调函数，创建好的组件实例会作为回调函数的参数传入。

注意，如果导航的旧地址和新地址只有路由参数或查询字符串不同，例如/user/123到/user/456，那么原来的组件实例会被复用，因为两个路由都渲染相同的组件。这也意味着组件的生命周期钩子不会再被调用。此时也就不会触发beforeRouteLeave和beforeRouteEnter守卫函数。
如果你需要让组件对路由参数的变化作出响应，有两种办法：

  1. 使用侦听器侦听$route对象。
  1. 使用beforeRouteUpdate导航守卫

beforeRouteLeave

beforeRouteLeave定义在视图组件的选项内。
当导航触发时，正离开的组件内的breforeRouteLeave被调用。
这个离开守卫通常用来禁止用户在还未保存修改前突然离开。该导航可以通过 next(false) 来取消。

beforeEach

使用router.beforeResolve函数注册一个全局前置守卫函数。
全局前置守卫函数可以有多个，会按照注册的顺序逐个调用。

beforeRouteUpdate

beforeRouteUpdate定义在视图组件的选项内。
这个函数会在导航到相同路由，但路由参数不同的情况下被调用。

beforeEnter

beforeEnter定义在路由记录上，即将进入该路由时会调用此路由记录上的beforeEnter函数。它的用法和beforeEach注册的守卫函数一样。

beforeRouteEnter

beforeRouteEnter定义在视图组件的选项内。
beforeRouteEnter方法不能访问 this，因为守卫在导航确认前被调用，因此即将登场的新组件还没被创建。但你可以给next方法传递一个回调函数，该回调函数接受一个新组件实例参数，将来新组件被创建出来后会传给该回调函数并调用。

beforeResolve

使用router.beforeResolve方法注册一个全局解析守卫函数。
全局解析守卫函数可以有多个，会按照注册的顺序逐个调用。

afterEach

使用router.afterEach方法注册一个全局后置钩子函数。
由于调用afterEach注册的钩子函数时，导航已经被确认，所以函数没有next参数。

获取数据

有时候，进入某个路由后，需要从服务器获取数据。例如，在渲染用户信息时，你需要从服务器获取用户的数据。我们可以通过两种方式来实现：

  • 导航完成之后获取：先完成导航，然后在接下来的组件生命周期钩子中获取数据。在数据获取期间显示“加载中”之类的指示。
  • 导航完成之前获取：导航完成前，在路由进入的守卫中获取数据，在数据获取成功后执行导航。

从技术角度讲，两种方式都不错 —— 就看你想要的用户体验是哪种。

导航完成后获取数据

当你使用这种方式时，我们会马上导航和渲染组件，然后在组件的 created 钩子中获取数据。这让我们有机会在数据获取期间展示一个 loading 状态，还可以在不同视图间展示不同的 loading 状态。

在导航完成前获取数据

通过这种方式，我们在导航转入新的路由前获取数据。我们可以在接下来的组件的 beforeRouteEnter 守卫中获取数据，当数据获取成功后只调用 next 方法。

在为后面的视图获取数据时，用户会停留在当前的界面，因此建议在数据获取期间，显示一些进度条或者别的指示。如果数据获取失败，同样有必要展示一些全局的错误提醒。

滚动行为

在导航到新路由时，你可以控制页面滚动到顶部，还是保持原来的滚动位置。
注意：这个功能只在支持 history.pushState 的浏览器中可用。
在实例化VueRouter时，通过scrollBehavior方法设置滚动行为。
scrollBehavior 方法接收 to 和 from 路由对象。第三个参数 savedPosition 当且仅当 popstate 导航 (通过浏览器的 前进/后退 按钮触发) 时才可用。
此方法应该返回一个滚动位置对象，例如{ x: number, y: number }或{ selector: string, offset? : { x: number, y: number }}(offset 只在 2.6.0+ 支持)。
如果返回一个空对象，或一个falsy（ToBoolean操作转换后为false），则不发生滚动。
