代理对象

和普通对象不同，代理对象只是一个媒介，对代理对象的任何访问都将变成对代理的目标对象的某种访问。
这种访问的实现可以是由系统默认提供的（就像普通对象一样），也可以是由用户提供的。
由于代理提供了由用户实现对象内部方法的机制，而ECMAScript提供的内置函数、运算符大量使用了内部方法，因此用户可以通过代理改变这些内置函数或运算符应用在特定对象上的行为。

内部槽

代理对象有两个内部槽：[[ProxyTarget]]和[[ProxyHandler]]

[[ProxyTarget]]内部槽通常称为代理的目标对象，其值要么是一个对象，要么是null（表明代理已被撤销）。对代理对象的任何访问都将变成对[[ProxyTarget]]对象的操作，如果[[ProxyTarget]]为null，则表明这个代理已经被撤销，对代理对象的任何访问都将抛出异常。

[[ProxyHandler]]内部槽通常称为代理的处理程序对象，其值要么是一个对象，要么是null（表明代理已被撤销）。对代理对象的任何访问，都会先在[[ProxyHandler]]对象中查找相应的处理程序，如果存在则会用目标对象作为参数调用处理程序，如果不存在再在目标对象上进行访问。

代理对象的内部槽在代理对象被创建时初始化，之后通常无法进行任何修改。
只有在创建可撤销代理时，允许调用撤销方法将代理对象的[[ProxyTarget]]或[[ProxyHandler]]设置为null。一旦设置为null就无法再修改，这个代理对象就没有任何作用了。

注意，目标对象也可能是一个代理对象。

创建代理

不可撤销代理对象

不可撤销代理对象只能通过Proxy构造函数创建。不作为构造函数调用会抛出异常。
target是要被设置给[[ProxyTarget]]的对象，target必须是一个对象，可以是一个普通对象，也可以是一个代理对象，但当它是代理对象时不能是一个已撤销的代理对象。
handler是要被设置给[[ProxyHandler]]的对象，handler必须是一个对象，可以是一个普通对象，也可以是一个代理对象，但当它是代理对象时不能是一个已撤销的代理对象。
不可撤销代理对象一旦创建，其[[ProxyTarget]]和[[ProxyHandler]]内部槽都无法再修改。

可撤销代理对象

可撤销代理对象只能通过Proxy.revocable函数创建。该函数返回一个对象{proxy: p, revoke: r}。
p是可撤销代理对象，r是一个函数，调用它将使得p的[[ProxyTarget]]和[[ProxyHandler]]内部槽被设置为null。r只能调用一次，多次调用没有任何效果。
在调用r之后，调用代理对象p的任何内部方法都将抛出异常，也没有任何办法将[[ProxyTarget]]和[[ProxyHandler]]内部槽改回去。

代理的处理程序对象

调用代理的内部方法时，会先检查代理的处理程序对象中是否有相应的属性键为【处理程序名】的函数：

  • 如果处理程序对象中不存在属性键为【处理程序名】的属性，或者存在但属性值为null，则直接调用目标对象上的相应的内部方法。
  • 如果处理程序对象中存在属性键为【处理程序名】的属性，但属性值不是函数，则抛出异常。

注意，例外是[[Call]]和[[Construct]]，如果目标对象没有这两个内部方法（例如目标对象不可调用，或者不可作为构造函数调用），那么代理对象也没有这两个内部方法，根本就不会去检查处理程序对象，因此即便处理程序对象中有相应的处理程序也没有任何意义。

由于ECMAScript的内置函数、操作符的内部逻辑依赖于调用对象的内部方法的结果，为了不使内置函数、操作符出现严重的错误（例如，本该返回布尔值的地方返回了字符串），内部方法必须遵守某些不变式。
对于普通对象，内部方法是由ECMAScript实现者提供的，因此很容易满足遵守不变式的要求。
但对于代理对象，内部方法是由用户编写的处理程序提供的，因此很可能打破这些不变式导致严重的错误，为此，ECMAScript规定了用户编写的处理程序所必须遵守的不变式，当违反这些不变式时会抛出异常。

[[GetPrototypeOf]] ( )

普通对象O的此内部方法返回[[Prototype]]内部槽。

普通对象O的此内部方法将[[Prototype]]内部槽设为V，同时会要求V只能是Object或Null，而且O不在V的原型链上（避免原型链循环）。

[[IsExtensible]] ( )

普通对象O的此内部方法返回[[Extensible]]内部槽。

下列常用内置函数、运算符调用了此内部方法：

[[PreventExtensions]] ( )

普通对象O的此内部方法将[[Extensible]]内部槽设为false，然后返回true。

下列常用内置函数、运算符调用了此内部方法：

[[GetOwnProperty]] ( P )

普通对象O的此内部方法：

  • 若O没有属性键为P的自有属性，则返回undefined
  • 若O具有属性键为P的自有属性，则返回这个属性的特性（规范术语为属性描述符）

下列常用内置函数、运算符调用了此内部方法：

[[DefineOwnProperty]] ( P, Desc )

普通对象O的此内部方法：

  • 若O没有属性键为P的自有属性，则根据特性Desc添加该属性
  • 若O具有属性键为P的自有属性，则根据特性Desc修改该属性的特性

下列常用内置函数、运算符调用了此内部方法：

[[HasProperty]] ( P )

普通对象O的此内部方法：

  1. 如果O具有自有属性P，则返回true
  2. 如果O.[[Prototype]]不为null，则返回O.[[Prototype]].[[HasProperty]](P)
  3. 否则返回false

下列常用内置函数、运算符调用了此内部方法：

[[Get]] ( P, Receiver )

initTarget是指最开始调用[[Get]]的对象，对于代理对象，initTarget是代理对象而不是target，对于普通对象，initTarget就是它自己。

普通对象O的此内部方法：

  1. 如果O具有自有属性P：
    • 若P是数据属性，则返回属性值
    • 若P是访问器属性且特性[[Get]]为undefined则返回undefined
    • 若P是访问器属性且特性[[Get]]不为undefined则返回getter(Receiver)
  2. 如果O没有自有属性P：
    1. 如果O.[[Prototype]]为null，则返回undefined
    2. 否则返回O.[[Prototype]].[[Get]](P, Receiver)

注意，当处理程序没有get属性时，它会原封不动的调用target.[[Get]](P, Receiver)，而Receiver是代理对象而不是target。这导致get访问器属性得到的this是代理对象而不是target对象，当get访问器内使用了Map内置对象的map.get(this)时，通常无法访问到想要的结果，因为Map中target作为键和代理对象作为键是不同的。
要解决这个问题，只需让它调用target.[[Get]](P)，省去Receiver，这样普通对象的[[Get]]内部方法就会用target作为Receiver。

下列常用内置函数、运算符调用了此内部方法：

[[Set]] ( P, V, Receiver )

initTarget是指最开始调用[[Set]]的对象，对于代理对象，initTarget是代理对象而不是target，对于普通对象，initTarget就是它自己。

普通对象O的此内部方法：

  • 如果对象O没有自有属性P，也没有继承属性P：
    • 为对象O添加一个属性键为P的自有属性，其特性为{ value: V, writable: true, enumerable: true, configurable: true}
  • 如果对象O没有自有属性P，但存在继承属性P：
    • 若继承属性P是数据属性：
      1. 若继承属性P的writable特性为false，则结束set访问，什么也不做。
      2. 否则，为对象O添加一个属性键为P的自有属性，其特性为{ value: V, writable: true, enumerable: true, configurable: true}
    • 若继承属性P是访问器属性：
      1. 若继承属性P没有set访问器函数，在结束set访问，什么也不做。
      2. 否则，使用对象O作为this值调用继承属性P的get访问器函数。
  • 如果对象O具有自有属性P：
    • 若自有属性P是数据属性：
      1. 若自有属性P的writable特性为false，则结束set访问，什么也不做。
      2. 否则，将自有属性P的属性值修改为V
    • 若自有属性P是访问器属性：
      1. 若自有属性P没有set访问器函数，在结束set访问，什么也不做。
      2. 否则，使用对象O作为this值调用自有属性P的get访问器函数。

下列常用内置函数、运算符调用了此内部方法：

[[Delete]] ( P )

普通对象O的此内部方法：

  1. 如果O没有自有属性P，返回true
  2. 如果O具有自有属性P，且P的[[Configurable]]特性为true，则删除O的P属性然后返回true
  3. 否则，返回false

常见用法

基于默认内部方法增强

在执行完自己需要的逻辑后，可以返回同名的反射方法，使其回到默认内部方法的逻辑。

隐藏属性

获取一个属性的键或值不可避免的用到[[Get]]、[[HasProperty]]、[[GetOwnProperty]]、[[OwnPropertyKeys]]等内部方法，只要创建一个代理对象，在处理程序中设置获取特定属性时返回不存在即可。

数据验证

添加/设置一个属性不可避免的用到[[Set]]和[[DefineOwnProperty]]内部方法，只要创建一个代理对象，在处理程序中进行验证，根据情况允许继续赋值或者不允许赋值。

函数参数验证

调用函数时不可避免的用到[[Call]]和[[Construct]]内部方法，只要创建一个代理对象，在处理程序中进行验证，根据情况允许调用或者抛出异常。
