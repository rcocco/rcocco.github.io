时间和日期
2021-04-06|计算机编程语言JavaScript语言

概述

ECMAScript中的日期对象Date在内部使用Number类型表示时间值。
时间值的范围比Number所能精确表示的整数范围略小，为-8,640,000,000,000,000到8,640,000,000,000,000毫秒。
规定UTC时间1970年1月1日午夜的起始为+0。负的毫秒数表示的时间相对于UTC 1970年1月1日午夜往前推，正的毫秒数往后推。因此能表示的时间是相对于1970年1月1日的-100,000,000天到100,000,000天，或-273,790到273,790年。
无效时间值在内部使用NaN表示

解析日期时间字符串

合法的日期时间字符串采用 ISO 8601 标准，有下面三种形式

当UTC偏移不存在时，只有日期的格式会被解释为UTC时间，而既有日期又有时间格式被解释为当地时间。

Date.parse( str )

返回Number类型。
首先使用抽象操作ToString(str)将参数转为字符串。
然后尝试按照指定格式解析该时间字符串，如果能解析则返回用Number类型表示的时间值。
如果不能解析则返回NaN。

new Date ( str )

返回Date类型对象。
按照指定格式解析该时间字符串，返回的Date对象表示解析出的时间值，如果无法解析会返回表示无效时间的Date对象。

手动设置日期时间

通常，月份范围是0到11，日期范围是1到31，小时范围是0到23，分钟和秒范围是0到59，毫秒范围是0到999。
所有参数可以超出范围，例如分钟范围是00到59，但可以超出范围例如-1或60，负值往前推时间，正值往后推时间。
当year在0到99之间时，令year变成year + 1900，因此公元0到99年无法通过传入year来直接表示，必须通过超出范围的其他参数得到。

Date对象

本地时区和UTC时区

Date对象上的很多方法分为：基于本地时区的版本，以及基于UTC时区的版本。

本地时区和UTC时区的区别在于：
如果Date对象表示东八区2021年1月1日00:00:00，则UTC时间（零时区）是2020年12月31日16:00:00。
此时getFullYear()返回2021，而getUTCFullYear()返回2020。
此时setDate(5)会得到东八区时间2021年1月5日00:00:00，而setUTCDate(5)会得到UTC时间2020年12月5日16:00:00（即东八区时间2020年12月6日00:00:00）

可执行的运算

当两个Date对象不是无效时间时，根据@@toPrimitive和valueOf方法的定义，可以直接执行下列运算

  1. 可以对两个Date对象执行减法-，得到两个Date对象所表示时间相差的毫秒数。
  2. 可以对两个Date对象执行比较运算<、>、<=、>=，得到两个Date对象所表示时间的前后关系。

注意：不能直接对两个Date对象执行相等==或严格相等===运算。你必须分别调用getTime或valueOf再进行相等性比较。
