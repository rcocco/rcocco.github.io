对象概述

对象是属性的集合，属性分为数据属性，以及 访问器属性。

数据属性由三部分组成：

  1. 一个属性键
  2. 一个ECMAScript语言类型（Undefined、Null、Boolean、String、Symbol、Number、Object）值
  3. 一个Boolean类型的特性(attributes)集合。

访问器属性由三部分组成：

  1. 一个属性键
  2. 一个（或两个）访问器函数。访问器函数用于获取或设置与属性关联的ECMAScript语言值。
  3. 一个Boolean类型的特性(attributes)集合组成。

属性使用属性键来标识，属性键可以是String类型（包括空串）或Symbol类型。
属性键用于访问属性及其值，对属性的访问有两种方法：get和set，get对应于获取属性值，set对应于设置属性值。
通过get和set可以访问对象的自有属性或它继承的属性，一个对象的自有属性的属性键，必须与该对象其他自有属性的属性键不同。

内部槽

对象有两个内部槽：[[Prototype]]和[[Extensible]]。
内部槽可以理解为内部属性，但程序员不能像普通属性一样看到和访问它。

[[Prototype]]内部槽通常称为对象的原型，其值要么是null，要么是一个对象，它用于实现继承。在get访问一个属性时，如果这个属性不是当前对象的自有属性，就会查找[[Prototype]]对象的同名自有属性，这样的属性称为继承的属性（继承属性就是子对象可以看见的非自有属性）。

[[Extensible]]内部槽通常称为对象的可扩展性，其值是一个布尔值。如果它为false，那么就不能再为当前对象添加任何属性。也不允许修改该对象的[[Prototype]]以及[[Extensible]]内部槽的值（也就是说改不回true了）

属性的特性

数据属性有四个特性：

特性名 值域 描述
[[Value]] 任何 ECMAScript语言类型 属性值的所在位置，通过属性的get访问来获取该值
[[Writable]] Boolean 如果为 false，试图通过 ECMAScript 代码 [[Set]] 去改变该属性的 [[Value]]将会失败。默认为true
[[Enumerable]] Boolean 如果为 true，则该属性可被 for-in 枚举出来，否则，该属性不可枚举。默认为true
[[Configurable]] Boolean 如果为 false，试图删除该属性或改变该属性为访问器属性，或改变它的特性（不包括修改 [[Value]]，或将[[Writable]]改为false的情况），都会失败。默认为true

访问器属性也有四个特性：

特性名 值域 描述
[[Get]] Object或Undefined 如果该值为一个 Object 对象，那么它必须是一个函数对象。每次有对属性进行 get 访问时，该函数的内部方法 [[Call]] 会伴随着一个空参数列表被调用，并取出该属性值。默认为undefined
[[Set]] Object或Undefined 如果该值为一个 Object 对象，那么它必须是一个函数对象。每次有对属性进行 set 访问时，该函数的内部方法 [[Call]] 会伴随着一个参数列表被调用，这个参数列表包含了指定值作为唯一的参数。属性的内部方法 [[Set]] 可能会对随后的 属性 内部方法 [[Get]] 的调用返回结果产生影响。默认为undefined
[[Enumerable]] Boolean 如果为 true，则该属性可被 for-in 枚举出来，否则，该属性不可枚举。默认为true
[[Configurable]] Boolean 如果为 false，试图删除该属性，改变该属性为数据属性，或改变它的特性，都会失败。默认为true

对 对象O的属性P进行get访问时，
先查找对象O的自有属性P，如果没有自有属性，再查找原型的自有属性。
当找到该属性时，如果是数据属性就返回属性值，如果是访问器属性，就返回用最初的对象O作为this值调用get访问器函数。
如果一直查找到原型为null仍没有找到，则返回undefined。

set访问

对 对象O的属性P使用参数V进行set访问时：

  • 如果对象O没有自有属性P，也没有继承属性P：
    • 为对象O添加一个属性键为P的自有属性，其特性为{ value: V, writable: true, enumerable: true, configurable: true}
  • 如果对象O没有自有属性P，但存在继承属性P：
    • 若继承属性P是数据属性：
      1. 若继承属性P的writable特性为false，则结束set访问，什么也不做。
      2. 否则，为对象O添加一个属性键为P的自有属性，其特性为{ value: V, writable: true, enumerable: true, configurable: true}
    • 若继承属性P是访问器属性：
      1. 若继承属性P没有set访问器函数，在结束set访问，什么也不做。
      2. 否则，使用对象O作为this值调用继承属性P的get访问器函数。
  • 如果对象O具有自有属性P：
    • 若自有属性P是数据属性：
      1. 若自有属性P的writable特性为false，则结束set访问，什么也不做。
      2. 否则，将自有属性P的属性值修改为V
    • 若自有属性P是访问器属性：
      1. 若自有属性P没有set访问器函数，在结束set访问，什么也不做。
      2. 否则，使用对象O作为this值调用自有属性P的get访问器函数。

创建同类对象

工厂模式

工厂模式通过普通的函数调用来创建具有特定属性的一类对象。在函数内它创建一个新的空对象，然后为该对象添加属性，最后返回该对象。

缺点是：

  • 创建的对象彼此无关
  • 每次调用都会创建新的函数对象

构造函数模式

构造函数模式通过new运算符调用构造函数来创建具有特定属性的一类对象。
由new运算符创建一个原型[[Prototype]]为构造函数的prototype属性的新对象，将该对象作为调用构造函数时的this值来调用函数，最后由new运算符自动返回创建的对象。

比起工厂模式

  • 优点是：
    • 通过使用new运算符减少了创建对象与返回对象的两行代码。
    • 在创建的对象与构造函数的prototype属性之间建立了明确的关联。
    • 创建的对象彼此之间具有共性，它们的原型[[Prototype]]都是同一个，所有对象都能访问到原型上的属性，使得继承属性能够动态的变化。
  • 缺点是：
    • 每次调用都会创建新的函数对象

原型模式

原型模式在构造函数模式的基础上，将所有可共享的属性放在构造函数的prototype属性上，让其作为创建对象的继承属性，使得这些属性的值只需要创建一遍就能被所有对象访问。

当为构造函数的prototype添加的属性较多时，这种点表示法或中括号表示法添加属性的方法略显麻烦，每次添加属性都要书写一遍Person.prototype.，为了减少重复代码，可以用一个对象字面值直接覆盖原有的prototype属性。
注意，原有的prototype对象具有一个数据属性，其属性键为"constructor"，属性值为构造函数，特性为{writable: true, enumerable: false, configurable: true}。
为了使通过对象字面值创建的新对象与原有的prototype保持一致，还需要在对象字面值中添加"constructor"属性，并将其特性修改为不可枚举的。

继承

继承的目的是使得子类对象可以访问父类对象的属性。

原型链

每个对象都有原型[[Prototoype]]内部槽。
在对 对象的某个属性进行get访问时，会依次检查对象的自有属性，对象的原型[[Prototype]]的自有属性，原型的原型的自有属性，以此类推。
查找过程所经过的这些对象就形成了一条链，称为原型链。
内置对象的原型[[Prototype]]几乎都是Object.prototype，而Object.prototype的原型为null，因此内置对象的原型链的顶端通常是Object.prototype。

实现方法

纯原型链继承

早期，实现继承的方法是使父类对象在子类对象的原型链上，这样使得子类对象可以访问父类对象的属性。具体而言，令子类构造函数的prototype属性为父类构造函数的实例对象。

  • 优点是：
    • 子类与父类之间有明确的联系，子类对象可以访问到父类对象的实例属性和原型属性。
  • 缺点是：
    • 父类对象的实例属性变成了子类对象的原型属性，所有子类对象共享相同的父类实例属性，当父类实例属性是一个可变对象（如数组）时，任何一个子类对象操作该属性时，都将导致所有子类对象的继承属性发生变化。
    • 每次创建子类对象时，无法为父类构造函数传递参数，仅能在最初设置子类构造函数的prototype属性时传递参数。

盗用(父类)构造函数继承

为了改进纯原型链继承的缺点，出现了一种在子类构造函数中调用父类构造函数来实现继承的办法，相当于子类中包含完整的父类构造函数代码。

  • 优点是：
    • 不需要使用原型链
  • 缺点是：
    • 父类对象和子类对象之间没有任何联系，子类对象无法访问父类的原型属性。
    • 父类只能使用构造函数模式，导致每次调用都会创建新的函数对象。

原型式继承

在纯原型链继承、盗用构造函数继承模式的基础上，又出现了一种结合这两种方法优点来实现继承的办法：使父类对象在子类对象的原型链上，同时让子类中包含完整的父类构造函数代码。

  • 缺点是：
    • 父类构造函数被多调用了一次，而这次调用是没有任何必要的
    • 子类对象的原型上有冗余的父类实例属性（因为子类对象的自有属性遮挡了子类原型上的自有属性）

寄生式继承

另一种实现继承的方法是：为已有的父类对象的副本添加子类属性。

缺点是：

  • 子类对象和父类对象共用相同的原型[[Prototype]]，无法仅为子类或父类对象共享属性
  • 每次调用都会创建新的函数对象

寄生式组合继承

结合原型式继承、寄生式继承的可以得到一种新的继承方法：使父类对象的原型在子类对象的原型链上，同时让子类中包含完整的父类构造函数代码。
这种办法完全避免了原型式继承的缺点，是最常用的继承模式。

    • 若extends子句不存在，则将构造函数constructor的原型[[Prototype]]设为Function.prototype，然后将constructor.prototype的原型[[Prototype]]设为Object.prototype
    • 若extends子句存在：
      • 当左值表达式的结果为null时，将构造函数constructor的原型[[Prototype]]设为Function.prototype，然后将constructor.prototype的原型[[Prototype]]设为null
      • 否则，左值表达式的结果为构造函数parent，将构造函数constructor的原型[[Prototype]]设为parent，然后将constructor.prototype的原型[[Prototype]]设为parent.prototype
  5. 在classScope词法环境创建名为标识符的const变量，并将其初始化为构造函数constructor
  6. 对于类体中除constructor之外的每个类元素：
    • 若类元素是方法定义，则在constructor.prototype对象上定义函数（该函数的[[Environment]]内部槽为classScope）
    • 若类元素是static 方法定义，则在constructor对象上定义函数（该函数的[[Environment]]内部槽为classScope）
  7. 如果类语法是类声明，则在包围class语法的词法环境中创建名为标识符的let变量，并将其初始化为构造函数constructor。否则，类语法是类表达式，返回构造函数constructor

类语法还有一些额外的内容：

  • 类语法的所有部分都采用严格模式，包括定义的函数。
  • 类语法定义的函数中可以使用类标识符来引用类中定义的constructor函数。
  • 属性名为constructor的类元素最多只能有一个
  • static定义的方法名不能是prototype
  • 类元素对应的函数体中可以有super.标识符或super[表达式]：
    • 当类元素为方法定义时（包括constructor的情况），super为对constructor.prototype.__proto__的引用。若该引用是函数，则在调用时用当前方法的this作为引用函数的this值。例如super.hello()相当于constructor.prototype.__proto__.hello.call(this)
    • 当类元素为static 方法定义时，super为对constructor.__proto__的引用。若该引用是函数，则在调用时用当前方法的this作为引用函数的this值。
  • 如果没有extends子句，constructor的函数体中不能有super(参数)调用。
  • 如果有extends子句，在constructor的函数体中：
    • 使用this之前必须通过super(参数)来调用父类构造函数并初始化子类的this，否则抛出异常。
    • 如果constructor中没有调用过super(参数)，那么constructor必须返回一个对象，否则抛出异常。
  • 无论如何，constructor以外的方法中不能有super(参数)调用。
  • 不通过new直接调用constructor将抛出异常。

简而言之，ES6开始定义构造函数应该尽可能使用类。

  • 将实例属性通过this定义在构造函数中；
  • 将方法定义在原型上；
  • 通过static将特殊方法定义为构造函数的属性。
  • 通过super来访问被子类原型遮盖的父类原型上的属性。

抽象类

抽象类是不希望直接实例化它，而仅希望其他类继承它的类。
通过检查new.target的值，可以禁止实例化抽象类。
子类一定是通过super(参数)调用抽象类的构造函数，而调用时的this是子类的this，因此在抽象类的构造函数中可以通过this去测试子类对象上是否存在特定方法，使得子类必须实现某些方法。

复制对象

复制一个对象，使新对象的所有属性值和特定，以及[[Prototype]]与[[Extensible]]内部槽与原对象完全相同。
当原对象的某个属性是数据属性、且属性值是对象时，复制该对象，而不是直接将该属性值（该对象的地址）拷贝。

输出字符串

该方法返回一个形如"[object TAG]"的字符串。
TAG是当前对象this的Symbol.toStringTag的字符串值。
如果没有该属性，又或者属性值不是字符串，则根据对象的种类采用不同的内置TAG，例如数组是Array，正则对象是RegExp等等，如果对象不是内置对象，则会采用Object作为TAG。
