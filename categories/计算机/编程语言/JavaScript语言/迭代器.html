迭代器
2021-05-26|计算机编程语言JavaScript语言

概述

迭代器是对遍历的抽象，比起常见的基于索引的循环，迭代器的优点是不需要了解被遍历对象的数据结构，只需要关心被遍历的值。

同步迭代器

可迭代对象

实现了Iterable接口的对象称为可迭代对象，可迭代对象类似于数学上的有限集合，同时有无歧义的遍历顺序。Iterable接口要求对象具有一个属性键为Symbol.iterator的属性，它的属性值是一个函数，该函数返回一个Iterator迭代器对象。

迭代器

实现了Iterator接口的对象称为迭代器对象，迭代器对象表示对可迭代对象的一次遍历，每个迭代器会关联到一个可迭代对象上。同一个可迭代对象可以创建多个Iterator迭代器对象，这些对象彼此之间没有任何关联，一个Iterator对象的遍历过程不应该影响另一个。所以在Symbol.iterator等返回迭代器的方法中应该使用闭包将当前可迭代对象的状态记录下来。

Iterator接口要求对象必须具有next方法，return和throw方法是可选的。

  1. next方法：
    • 用于获取本次遍历的下一个迭代元素，并作为IteratorResult迭代结果对象返回。
    • 参数是可选的，没有规定其含义，ECMAScript内置函数在调用该方法时不会传递参数。
    • 通常约定如果前一次调用next方法返回了done属性为true的IteratorResult迭代结果对象，那么之后所有对next方法的调用都应该返回done属性为true的IteratorResult迭代结果对象。
  2. return方法：
    • 用于通知迭代器，调用者之后不会再调用next方法。即迭代器应该结束掉。
    • 参数会作为迭代器的返回值。
    • 返回一个IteratorResult迭代结果对象，它的done属性为true，value属性为return方法的参数。
  3. throw方法：
    • 用于通知迭代器，调用者检测到了一个错误。
    • 参数通常是一个异常对象，throw方法内会抛出这个异常。如果不打算抛出异常，那么应该返回一个done为true的IteratorResult迭代结果对象

迭代结果对象

实现了IteratorResult接口的对象称为迭代结果对象。
IteratorResult接口要求对象具有两个属性：

  • done：属性值为布尔值，表示遍历是否结束，若为false表示遍历还没有结束，本次迭代结果value仍然有效，若为true表示遍历已经结束，本次迭代的结果value不表示迭代值。如果IteratorResult没有done属性，则默认它为false。
  • value：属性值是任意的ECMAScript类型值，如果done为false，则value为当前迭代元素值。 如果done为true，则此值为迭代器的返回值（如果提供了），如果不存在返回值，value可以为undefined或者不存在。

原型对象

内置迭代器对象（例如数组迭代器、字符串迭代器、Map迭代器、Set迭代器等）的原型是%IteratorPrototype%对象，这个原型对象并不是全局对象上的属性，因此不容易直接访问，可以通过表达式([][Symbol.iterator]()).__proto__.__proto__访问。

原型方法只有一个，其属性键为Symbol.iterator，属性值是一个函数，该函数简单的返回this值。

这意味着内置迭代器也是实现了Iterable接口的对象，由于约定了Symbol.iterator方法返回迭代器，当有多种不同的遍历方法时，Symbol.iterator也只能返回其中一种，for-of等内置语句调用Symbol.iterator的方法只能按这固定一种的遍历方法来遍历。而令迭代器对象同时实现Iterable接口的优点就是可以使一个可迭代对象拥有多种不同的遍历方法，for-of只要将返回的迭代器作为可迭代对象进行遍历即可。例如

异步迭代器

异步可迭代对象

实现了AsyncIterable接口的对象称为异步可迭代对象，异步可迭代对象类似于数学上的有限集合，同时有无歧义的遍历顺序。AsyncIterable接口要求对象具有一个属性键为Symbol.asyncIterator的属性，它的属性值是一个函数，该函数返回一个AsyncIterator迭代器对象。

异步迭代器

实现了AsyncIterator接口的对象称为异步迭代器对象，异步迭代器对象表示对异步可迭代对象的一次遍历，每个异步迭代器会关联到一个异步可迭代对象上。同一个异步可迭代对象可以创建多个异步AsyncIterator迭代器对象，这些对象彼此之间没有任何关联，一个AsyncIterator对象的遍历过程不应该影响另一个。所以在Symbol.asyncIterator等返回迭代器的方法中应该使用闭包将当前可迭代对象的状态记录下来。

AsyncIterator接口要求对象必须具有next方法，return和throw方法是可选的。

  1. next方法：
    • 用于获取本次遍历的下一个迭代元素，返回一个表示异步获取迭代元素的Promise：
      • 若Promise将来是已解决的，则这个Promise的结果值是IteratorResult迭代结果对象。
      • 若Promise将来是已拒绝的，则这个Promise的结果值是拒绝的理由(reason)
    • 参数是可选的，没有规定其含义，ECMAScript内置函数在调用该方法时不会传递参数。
    • 通常约定如果前一次调用next方法返回的Promise的结果值的done属性为true，那么之后所有对next方法的调用都应该返回一个Promise，其结果值的done属性为true。
  2. return方法：
    • 用于通知异步迭代器，调用者之后不会再调用next方法。即异步迭代器应该结束掉。
    • 返回一个Promise，
      • 若Promise是已解决的，这个Promise的结果值是IteratorResult迭代结果对象，它的done属性为true，value属性为return方法的参数（不应该是thenable）。
      • 若Promise是已拒绝的，则这个Promise的结果值是拒绝的理由(reason)
  3. throw方法：
    • 用于通知异步迭代器，调用者检测到了一个错误。
    • 通常返回一个已拒绝的Promise，结果值就是参数传递的值。
    • 如果返回了一个已解决的Promise，结果值应该是一个done为true的IteratorResult迭代结果对象，且value属性值不是thenable

迭代结果对象

实现了IteratorResult接口的对象称为迭代结果对象。
IteratorResult接口要求对象具有两个属性：

  • done：属性值为布尔值，表示遍历是否结束，若为false表示遍历还没有结束，本次迭代结果value仍然有效，若为true表示遍历已经结束，本次迭代的结果value不表示迭代值。如果IteratorResult没有done属性，则默认它为false。
  • value：属性值是任意的ECMAScript类型值，如果done为false，则value为当前迭代元素值。 如果done为true，则此值为迭代器的返回值（如果提供了），如果不存在返回值，value可以为undefined或者不存在。

原型对象

ECMAScript提供了一个可以用作异步迭代器原型的对象%AsyncIteratorPrototype%，这个原型对象并不是全局对象上的属性，因此不容易直接访问。

原型方法只有一个，其属性键为Symbol.asyncIterator，属性值是一个函数，该函数简单的返回this值。

也就是说该原型为异步迭代器实现了AsyncIterable接口，由于约定了Symbol.asyncIterator方法返回异步迭代器，当有多种不同的遍历方法时，Symbol.asyncIterator也只能返回其中一种。而令异步迭代器对象同时实现AsyncIterable接口的优点就是可以使一个异步可迭代对象拥有多种不同的遍历方法。

同步迭代器转异步迭代器

许多内置对象只有同步迭代器而没有异步迭代器，为了能够在for-await-of语句中正常遍历这样的对象，ECMAScript使用CreateAsyncFromSyncIterator抽象操作，基于同步迭代器创建了一种称为Async-from-Sync的异步迭代器。

Async-from-Sync异步迭代器

Async-from-Sync异步迭代器的原型是%AsyncFromSyncIteratorPrototype%对象，而这个对象的原型又是%AsyncIteratorPrototype%。
Async-from-Sync异步迭代器实例只有一个内部槽[[SyncIteratorRecord]]，保存用于创建Async-from-Sync对象的同步迭代器。

Async-from-Sync异步迭代器的原型%AsyncFromSyncIteratorPrototype%上有next、return和throw方法：

next方法

  • next方法调用同步迭代器的next方法得到IteratorResult迭代结果对象，然后通过Promise.resolve(迭代值)来包装迭代值。
    • 当这个Promise解决时，返回一个新的已解决的Promise，其结果值为{value: 旧Promise的结果值, done: 同步迭代结果的done}。
    • 当这个Promise拒绝时，返回一个与旧Promise相同结果值的新的已拒绝的Promise。

return方法

  • 如果同步迭代器没有return属性，或者属性值为null或undefined，则返回一个已解决的Promise，其结果值为{value: return的参数, done: true}
  • 如果同步迭代器有return属性，但属性值不是一个函数，则返回一个已拒绝的Promise，其结果值为TypeError异常对象。
  • 如果同步迭代器有return方法，则调用同步迭代器的return方法得到IteratorResult迭代结果对象，然后通过Promise.resolve(迭代值)来包装迭代值。
    • 当这个Promise解决时，返回一个新的已解决的Promise，其结果值为{value: 旧Promise的结果值, done: 同步迭代结果的done}。
    • 当这个Promise拒绝时，返回一个与旧Promise相同结果值的新的已拒绝的Promise。

throw方法

  • 如果同步迭代器没有throw属性，或者属性值为null或undefined，则返回一个已解决的Promise，其结果值为throw的参数
  • 如果同步迭代器有throw属性，但属性值不是一个函数，则返回一个已拒绝的Promise，其结果值为TypeError异常对象。
  • 如果同步迭代器有throw方法，则调用同步迭代器的throw方法
    • 如果抛出了异常，则返回一个已拒绝的Promise，结果值为异常对象。
    • 如果没有抛出异常，则得到IteratorResult迭代结果对象。通过Promise.resolve(迭代值)来包装迭代值。
      • 当这个Promise解决时，返回一个新的已解决的Promise，其结果值为{value: 旧Promise的结果值, done: 同步迭代结果的done}。
      • 当这个Promise拒绝时，返回一个与旧Promise相同结果值的新的已拒绝的Promise。
