14.1.20 运行时语义:计算

  1. 如果FunctionExpression的函数代码是严格模式代码，令strict为true。否则令strict为false
  2. 令runningContext为运行中的执行上下文的词法环境
  3. 令funcEnv为NewDeclarativeEnvironment(runningContext )
  4. 令envRec为funcEnv的EnvironmentRecord
  5. 令name为BindingIdentifier的StringValue
  6. 执行envRec.CreateImmutableBinding(name)
  7. 令closure为FunctionCreate(Normal, FormalParameters, FunctionBody, funcEnv, strict)
  8. 执行MakeConstructor(closure)
  9. 执行SetFunctionName(closure, name)
  10. 执行envRec.InitializeBinding(name, closure)
  11. 返回NormalCompletion(closure)

注2：可以从FunctionExpression的FunctionBody内部引用FunctionExpression中的BindingIdentifier，以允许该函数以递归方式调用其自身。 但是，与FunctionDeclaration中的功能不同，FunctionExpression中的BindingIdentifier不能从其包围FunctionExpression的作用域引用，也不会影响该作用域。

注3：将为使用FunctionDeclaration或FunctionExpression定义的每个函数自动创建一个prototype属性，以允许将该函数用作构造函数。

14.2.15 运行时语义:EvaluateBody

  1. 令exprRef为计算AssignmentExpression的结果
  2. 令exprValue为GetValue(exprRef)
  3. ReturnIfAbrupt(exprValue)
  4. 返回Completion{[[type]]: return, [[value]]: exprValue, [[target]]: empty}

14.2.16 运行时语义:计算

  1. 如果此ArrowFunction的代码是严格模式代码，令strict为true，否则令strict为false
  2. 令scope为运行中的执行上下文的LexicalEnvironment
  3. 令parameters为ArrowParameters的CoveredFormalsList
  4. 令closure为FunctionCreate(Arrow, parameters, ConciseBody, scope, strict)
  5. 返回closure

注：ArrowFunction没有为arguments，super，this或new.target定义本地绑定。 ArrowFunction中对arguments，super，this或new.target的任何引用都必须解析为词法包围环境（即包围ArrowFunction的词法环境）中的绑定。 通常，这将是立即封闭函数的“函数环境”。 即使ArrowFunction可能包含对super的引用，也无法通过执行MakeMethod将第4步中创建的函数对象变成方法。 引用super的ArrowFunction始终包含在非ArrowFunction中，并且可以通过ArrowFunction的函数对象捕获的范围来访问实现super的必要状态。

14.5.14 运行时语义:ClassDefinitionEvaluation

接受参数className

  1. 令lex为运行中的执行上下文的LexicalEnvironment
  2. 令classScope为NewDeclarativeEnvironment(lex)
  3. 令classScopeEnvRec为classScope的环境记录
  4. 如果className不为undefined，则执行classScopeEnvRec.CreateImmutableBinding(className, true)
  5. 如果ClassHeritage{opt}不存在，则
    1. 令protoParent为固有对象%ObjectPrototype%
    2. 令constructorParent为固有对象%FunctionPrototype%
  6. 否则：
    1. 将运行中的执行上下文的LexicalEnvironment设为classScope
    2. 令superclass为计算ClassHeritage的结果
    3. 将运行中的执行上下文的LexicalEnvironment设为lex
    4. ReturnIfAbrupt(superclass)
    5. 如果superclass为null，则
      1. 令protoParent为null
      2. 令constructorParent为固有对象%FunctionPrototype%
    6. 否则如果IsConstructor(superclass)为false，抛出 TypeError 异常
    7. 否则：
      1. 如果superclass有[[FunctionKind]]内部插槽，且其值为"generator"，则抛出 TypeError 异常
      2. 令protoParent为Get(superclass, "prototype")
      3. ReturnIfAbrupt(protoParent)
      4. 如果Type(protoParent)既不是Object也不是Null，抛出 TypeError 异常
      5. 令constructorParent为superclass
  7. 令proto为ObjectCreate(protoParent)
  8. 如果ClassBody{opt}不存在，令constructor为empty
  9. 否则，令constructor为ClassBody的ConstructorMethod
  10. 如果constructor为empty，则
    1. 如果ClassHeritage{opt}存在，则令constructor为使用目标符号MethodDefinition的句法解析源文本constructor(... args){ super (...args);}的结果
    2. 否则，令constructor为使用目标符号MethodDefinition的句法解析源文本constructor( ){ }的结果
  11. 将运行中的执行上下文的LexicalEnvironment设为classScope
  12. 令constructorInfo为使用参数proto和constructorParent作为可选的functionPrototype参数对constructor执行DefineMethod的结果。
  13. 断言：constructorInfo不是非常规完结
  14. 令F为constructorInfo.[[closure]]
  15. 如果ClassHeritage{opt}存在，将F的[[ConstructorKind]]内部插槽设为"derived"
  16. 执行MakeConstructor(F, false, proto)
  17. 执行MakeClassConstructor(F)
  18. 执行CreateMethodProperty(proto, "constructor", F)
  19. 如果ClassBody{opt}不存在，令methods为一个新的空列表
  20. 否则，令methods为ClassBody的NonConstructorMethodDefinitions
  21. 对methods中的每个ClassElement元素m，按照顺序：
    1. 如果m的IsStatic为false，则令status为使用参数proto和false对m执行PropertyDefinitionEvaluation的结果
    2. 否则，令status为使用参数F和false对m执行PropertyDefinitionEvaluation的结果
    3. 如果status是完结记录，则
      1. 将运行中的执行上下文的LexicalEnvironment设为lex
      2. 返回Completion(status)
  22. 将运行中的执行上下文的LexicalEnvironment设为lex
  23. 如果className不为undefined，则执行classScopeEnvRec.InitializeBinding(className, F)
  24. 返回F

14.5.15 运行时语义:BindingClassDeclarationEvaluation

  1. 令className为BindingIdentifier的StringValue
  2. 令value为使用参数className计算ClassTail的ClassDefinitionEvaluation的结果
  3. ReturnIfAbrupt(value)
  4. 令hasNameProperty为HasOwnProperty(value, "name")
  5. ReturnIfAbrupt(hasNameProperty)
  6. 如果hasNameProperty为false，则执行SetFunctionName(value, className)
  7. 令env为运行中的执行上下文的LexicalEnvironment
  8. 令status为InitializeBoundName(className, value, env)
  9. ReturnIfAbrupt(status)
  10. 返回value
  1. 返回使用参数undefined计算ClassTail的ClassDefinitionEvaluation

注：ClassDeclaration : class ClassTail只会作为ExportDeclaration的一部分出现，并且命名属性的设置及其建立的绑定将作为该产生式的计算操作的一部分进行处理。 见15.2.3.11。

14.5.16 运行时语义:计算

  1. 令status为计算此ClassDeclaration的BindingClassDeclarationEvaluation的结果
  2. ReturnIfAbrupt(status)
  3. 返回NormalCompletion(empty)

注1：ClassDeclaration : class ClassTail只会作为ExportDeclaration的一部分出现，永远不会直接计算

  1. 如果BindingIdentifier{opt}不存在，令className为undefined
  2. 否则，令className为BindingIdentifier的StringValue
  3. 令value为使用参数className计算ClassTail的ClassDefinitionEvaluation的结果
  4. ReturnIfAbrupt(value)
  5. 如果className不为undefined，则
    1. 令hasNameProperty为HasOwnProperty(value, "name")
    2. ReturnIfAbrupt(hasNameProperty)
    3. 如果hasNameProperty为false，则执行SetFunctionName(value, className)（备注value.name = className）
  6. 返回NormalCompletion(value)
