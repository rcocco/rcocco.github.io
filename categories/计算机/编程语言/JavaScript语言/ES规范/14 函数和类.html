语法

14.1.23 运行时语义: InstantiateFunctionObject

接受参数scope

  1. 令name为BindingIdentifier的StringValue
  2. 令F为OrdinaryFunctionCreate(%Function.prototype%, FormalParameters, FunctionBody, non-lexical-this, scope)
  3. 执行MakeConstructor(F)
  4. 执行SetFunctionName(F, name)
  5. 将F.[[SourceText]]设为与FunctionDeclaration匹配的源文本
  6. 返回F

14.1.25 运行时语义:计算

  1. 令scope为运行中的执行上下文的LexicalEnvironment
  2. 令funcEnv为NewDeclarativeEnvironment(scope)
  3. 令envRec为funcEnv的EnvironmentRecord
  4. 令name为BindingIdentifier的StringValue
  5. 执行envRec.CreateImmutableBinding(name, false)
  6. 令closure为OrdinaryFunctionCreate(%Function.prototype%, FormalParameters, FunctionBody, non-lexical-this, funcEnv)
  7. 执行MakeConstructor(closure)
  8. 执行SetFunctionName(closure, name)
  9. 将closure.[[SourceText]]设为与FunctionExpression匹配的源文本
  10. 执行envRec.InitializeBinding(name, closure)
  11. 返回closure

注2：FunctionExpression中的BindingIdentifier可以被FunctionExpression的FunctionBody引用，以允许函数递归地调用自己。然而，与FunctionDeclaration不同，FunctionExpression中的BindingIdentifier不能被包围FunctionExpression的作用域所引用，也不会影响该作用域。

注3：每个使用FunctionDeclaration或FunctionExpression定义的函数都会自动创建一个"prototype"属性，以允许将该函数用作构造函数。

14.2.14 运行时语义: IteratorBindingInitialization

接受参数iteratorRecord和environment

注：当接受的参数environment为undefined时，它表示应该使用PutValue操作来赋予初始值。对于非严格函数的形式参数列表就是这种情况。在这种情况下，形式参数的绑定被预先初始化，以处理可能出现的多个同名参数。

  1. 断言：iteratorRecord.[[Done]]为false

注：ArrowFunction没有为arguments，super，this或new.target定义本地绑定。 ArrowFunction中对arguments，super，this或new.target的任何引用都必须解析为词法包围环境（即包围ArrowFunction的词法环境）中的绑定。 通常，这将是立即封闭函数的“函数环境”。 即使ArrowFunction可能包含对super的引用，也无法通过执行MakeMethod将第3步中创建的函数对象变成方法。 引用super的ArrowFunction始终包含在非ArrowFunction中，并且可以通过ArrowFunction的函数对象捕获的范围来访问实现super的必要状态。

  1. 令exprRef为计算AssignmentExpression的结果
  2. 令exprValue为? GetValue(exprRef)
  3. 返回Completion { [[Type]]: return, [[Value]]: exprValue, [[Target]]: empty }

14.4.14 RS: 计算

  1. 令generatorKind为! GetGeneratorKind()
  2. 如果generatorKind为async，则返回? AsyncGeneratorYield(undefined)
  3. 否则，返回? GeneratorYield(CreateIterResultObject(undefined, false))
  1. 令generatorKind为! GetGeneratorKind()
  2. 令exprRef为计算AssignmentExpression的结果
  3. 令value为? GetValue(exprRef)
  4. 如果generatorKind为async，返回? AsyncGeneratorYield(value)

  2. 令exprRef为计算AssignmentExpression的结果
  3. 令value为? GetValue(exprRef)
  4. 令iteratorRecord为? GetIterator(value, generatorKind)
  5. 令iterator为iteratorRecord.[[Iterator]]
  6. 令received为NormalCompletion(undefined)
  7. 重复：
    1. 如果received.[[Type]]为normal，则
      1. 令innerResult为? Call(iteratorRecord.[[NextMethod]], iteratorRecord.[[Iterator]], « received.[[Value]] »)
      2. 如果generatorKind为async，则将innerResult设为? Await(innerResult)
      3. 如果Type(innerResult)不是Object，抛出 TypeError 异常
      4. 令done为? IteratorComplete(innerResult)
      5. 如果done为true，则返回? IteratorValue(innerResult)
      6. 如果generatorKind为async，则将received设为AsyncGeneratorYield(? IteratorValue(innerResult))
      7. 否则，将received设为GeneratorYield(innerResult)
    2. 否则如果received.[[Type]]为throw，则
      1. 令throw为? GetMethod(iterator, "throw")
      2. 如果throw不是undefined，则
        1. 令innerResult为? Call(throw, iterator, « received.[[Value]] »)
        2. 如果generatorKind为async，则将innerResult设为? Await(innerResult)
        3. 注：来自内部迭代器throw方法的异常会被传播（冒泡）。来自内部throw方法的常规完结与内部next方法的处理类似
        4. 如果Type(innerResult)不是Object，抛出 TypeError 异常
        5. 令done为? IteratorComplete(innerResult)
        6. 如果done为true，则返回? IteratorValue(innerResult)
        7. 如果generatorKind为async，则将received设为AsyncGeneratorYield(? IteratorValue(innerResult))
        8. 否则，将received设为GeneratorYield(innerResult)
      3. 否则：
        1. 注：如果iterator没有throw方法，则此throw会终止yield*循环。但首先我们需要给iterator一个机会来清理。
        2. 令closeCompletion为Completion { [[Type]]: normal, [[Value]]: empty, [[Target]]: empty }
        3. 如果generatorKind为async，执行? AsyncIteratorClose(iteratorRecord, closeCompletion)
        4. 否则，执行? IteratorClose(iteratorRecord, closeCompletion)
        5. 注：下一步会抛出一个TypeError，表示存在违反yield*协议的情况：iterator没有throw方法
        6. 抛出 TypeError 异常
    3. 否则：
      1. 断言：received.[[Type]]为return
      2. 令return为? GetMethod(iterator, "return")
      3. 如果return为undefined，则
        1. 如果generatorKind为async，则将received.[[Value]]设为? Await(received.[[Value]])
        2. 返回Completion(received)
      4. 令innerReturnResult为? Call(return, iterator, « received.[[Value]] »)
      5. 如果generatorKind为async，则将innerReturnResult设为? Await(innerReturnResult)
      6. 如果Type(innerReturnResult)不是Object，抛出 TypeError 异常
      7. 令done为? IteratorComplete(innerReturnResult)
      8. 如果done为true，则
        1. 令value为? IteratorValue(innerReturnResult)
        2. 返回Completion { [[Type]]: return, [[Value]]: value, [[Target]]: empty }
      9. 如果generatorKind为async，则将received设为AsyncGeneratorYield(? IteratorValue(innerReturnResult))
      10. 否则，将received设为GeneratorYield(innerReturnResult)

14.6.13 运行时语义:ClassDefinitionEvaluation

接受参数classBinding和className

  1. 令lex为运行中的执行上下文的LexicalEnvironment
  2. 令classScope为NewDeclarativeEnvironment(lex)
  3. 令classScopeEnvRec为classScope的EnvironmentRecord
  4. 如果className不为undefined，则执行classScopeEnvRec.CreateImmutableBinding(classBinding, true)
  5. 如果ClassHeritage{opt}不存在，则
    1. 令protoParent为固有对象%Object.Prototype%
    2. 令constructorParent为固有对象%Function.Prototype%
  6. 否则：
    1. 将运行中的执行上下文的LexicalEnvironment设为classScope
    2. 令superclassRef为计算ClassHeritage的结果
    3. 将运行中的执行上下文的LexicalEnvironment设为lex
    4. 令superclass为? GetValue(superclassRef)
    5. 如果superclass为null，则
      1. 令protoParent为null
      2. 令constructorParent为固有对象%Function.Prototype%
    6. 否则如果IsConstructor(superclass)为false，抛出 TypeError 异常
    7. 否则：
      1. 令protoParent为? Get(superclass, "prototype")
      2. 如果Type(protoParent)既不是Object也不是Null，抛出 TypeError 异常
      3. 令constructorParent为superclass
  7. 令proto为ObjectCreate(protoParent)
  8. 如果ClassBody{opt}不存在，令constructor为empty
  9. 否则，令constructor为ClassBody的ConstructorMethod
  10. 如果constructor为empty，则
    1. 如果ClassHeritage{opt}存在，则将constructor设为使用目标符号MethodDefinition的句法解析源文本constructor(...args){ super (...args);}的结果
    2. 否则，将constructor设为使用目标符号MethodDefinition的句法解析源文本constructor( ){ }的结果
  11. 将运行中的执行上下文的LexicalEnvironment设为classScope
  12. 令constructorInfo为使用参数proto和constructorParent执行constructor的! DefineMethod的结果。
  13. 令F为constructorInfo.[[Closure]]
  14. 执行MakeConstructor(F, false, proto)
  15. 如果ClassHeritage{opt}存在，将F.[[ConstructorKind]]设为derived
  16. 执行MakeClassConstructor(F)
  17. 如果className不是undefined，则执行SetFunctionName(F, className)
  18. 执行CreateMethodProperty(proto, "constructor", F)
  19. 如果ClassBody{opt}不存在，令methods为一个新的空列表
  20. 否则，令methods为ClassBody的NonConstructorMethodDefinitions
  21. 对methods中的每个ClassElement元素m，按照顺序：
    1. 如果m的IsStatic为false，则令status为使用参数proto和false执行m的PropertyDefinitionEvaluation的结果
    2. 否则，令status为使用参数F和false执行m的PropertyDefinitionEvaluation的结果
    3. 如果status是非常规完结，则
      1. 将运行中的执行上下文的LexicalEnvironment设为lex
      2. 返回Completion(status)
  22. 将运行中的执行上下文的LexicalEnvironment设为lex
  23. 如果classBinding不为undefined，则执行classScopeEnvRec.InitializeBinding(clasBinding, F)
  24. 返回F

14.7 异步函数定义

注1：当[Await]参数出现时，await被解析为一个AwaitExpression。在下列情况下，[Await]参数会出现。

  • 在AsyncFunctionBody里
  • 在AsyncFunctionDeclaration、AsyncFunctionExpression、AsyncGeneratorDeclaration或AsyncGeneratorExpression的FormalParameters中。在这个位置的AwaitExpression通过静态语义是一个语法错误。

当Module是句法目标符号并且[Await]参数不存在时，await被解析为一个关键字，并且会出现语法错误。当Script是句法目标符号并且[Await]参数不存在时，await可能被解析为一个标识符。这包括以下情况：

  • AsyncFunctionDeclaration、AsyncFunctionExpression、AsyncGeneratorDeclaration或AsyncGeneratorExpression的AsyncFunctionBody或FormalParameters之外的任何地方。
  • 在FunctionExpression、GeneratorExpression或AsyncGeneratorExpression的BindingIdentifier中

注2：与YieldExpression不同，省略AwaitExpression的操作数是一个语法错误。你必须await某个东西。

14.9 尾调用

14.9.1 静态语义: IsInTailPosition ( call )

抽象操作IsInTailPosition接受参数call并执行以下步骤：

  1. 断言：call是解析结点
  2. 如果与call匹配的源代码是非严格模式代码，返回false
  3. 如果call不在FunctionBody、ConciseBody、AsyncConciseBody里，返回false
  4. 令body为最近的包含call的FunctionBody、ConciseBody或AsyncConciseBody
  5. 如果body是GeneratorBody的FunctionBody，返回false
  6. 如果body是AsyncFunctionBody的FunctionBody，返回false
  7. 如果body是AsyncGeneratorBody的FunctionBody，返回false
  8. 如果body是AsyncConciseBody，返回false
  9. 返回使用参数call执行body的HasCallInTailPosition的结果

注：尾调用只定义在严格模式代码中，因为有一个常见的非标准语言扩展（见9.2.4），可以观察到调用上下文链

14.9.3 运行时语义: PrepareForTailCall ( )

抽象操作PrepareForTailCall执行以下步骤：

  1. 令leafContext为运行中的执行上下文
  2. 暂停leafContext
  3. 从执行上下文栈弹出leafContext。现在的栈顶上下文是运行中的执行上下文
  4. 断言：leafContext没有进一步用途，它永远不会再作为运行中的执行上下文

在调用目标函数之前，尾调用必须释放与当前执行的函数执行上下文相关的任何暂存的内部资源，或者重新使用这些资源来支持目标函数。

注：例如，一个尾调用应该只增长一个实现的激活记录堆栈，即目标函数的激活记录的大小超过了调用函数的激活记录的大小。如果目标函数的激活记录比较小，那么堆栈的总大小应该减少。
