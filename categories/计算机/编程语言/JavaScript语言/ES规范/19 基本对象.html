19 基本对象

19.1 Object对象

19.1.1 Object构造函数

Object构造函数

  • 是固有对象%Object%
  • 是全局对象的"Object"属性的初始值
  • 当作为构造函数调用时，它将创建一个新的普通对象(ordinary object)
  • 当作为函数而不是构造函数调用时，它将执行类型转换。
  • 被设计为可子类化的。 它可以用作类定义的extends子句的值。

19.1.1.1 Object ( [ value ] )

使用可选参数value调用Object函数时，将执行以下步骤：

  1. 如果NewTarget既不是undefined也不是激活的函数（即Object），则返回OrdinaryCreateFromConstructor(NewTarget, "%Object.Prototype%")
  2. 如果value为undefined或null，返回OrdinaryObjectCreate(%Object.prototype%)
  3. 返回! ToObject(value)

Object构造函数的"length"属性值为1。

19.1.2 Object构造函数的属性

Object构造函数

  • 具有[[Prototype]]内部槽，且其值是固有对象%Function.Prototype%。
  • 具有"length"属性
  • 还具有以下属性：

19.1.2.1 Object.assign ( target, …sources )

assign函数用于将所有可枚举的自有属性的值从一个或多个源对象复制到目标对象。 调用assign函数时，将执行以下步骤：

  1. 令to为? ToObject(target)
  2. 如果只传递了一个参数，返回to
  3. 令source为一个列表，列表中包含从第二个参数开始的所有参数值
  4. 对于source中的每个元素nextSource，按照索引的升序顺序，执行：
    1. 如果nextSource既不是undefined也不是null，则：
      1. 令from为! ToObject(nextSource)
      2. 令keys为? from.[[OwnPropertyKeys]]()
      3. 对于keys中的每个元素nextKey，按照列表顺序，执行
        1. 令desc为? from.[[GetOwnProperty]](nextKey)
        2. 如果desc不是undefined且desc.[[Enumerable]]为true，则
          1. 令propValue为? Get(from, nextKey)
          2. 执行? Set(to, nextKey, propValue, true)
  5. 返回to

assgin函数的"length"属性值为2

19.1.2.2 Object.create ( O, Properties )

create函数使用特定的原型创建一个新对象。当调用create函数时，执行以下步骤：

  1. 如果Type(O)不是Object也不是Null，抛出 TypeError 异常
  2. 令obj为OrdinaryObjectCreate(O)
  3. 如果Properties不是undefined，则返回? ObjectDefineProperties(obj, Properties)
  4. 返回obj

19.1.2.3 Object.defineProperties ( O, Properties )

defineProperties函数用于为对象添加自有属性，或者更新已存在的自有属性的特性。当调用defineProperties时，执行以下步骤：

  1. 返回? ObjectDefineProperties(O, Properties)
19.1.2.3.1 运行时语义: ObjectDefineProperties ( O, Properties )

抽象操作ObjectDefineProperties接受参数O和Properties，并执行以下步骤：

  1. 如果Type(O)不是Object，抛出 TypeError 异常
  2. 令props为? ToObject(Properties)
  3. 令keys为? props.[[OwnPropertyKeys]]()
  4. 令descriptors为一个新的空列表
  5. 对keys的每个元素nextKey，按照列表顺序：
    1. 令propDesc为? props.[[GetOwnProperty]](nextKey)
    2. 如果propDesc不是undefined且propDesc.[[Enumerable]]为true，则：
      1. 令descObj为? Get(props, nextKey)
      2. 令desc为? ToPropertyDescriptor(descObj)
      3. 将包含nextKey和desc的二元素列表添加到descriptors的末尾
  6. 对descriptors中的每个元素pair，按照列表顺序：
    1. 令P为pair的第一个元素
    2. 令desc为pair的第二个元素
    3. 执行? DefinePropertyOrThrow(O, P, desc)
  7. 返回O

19.1.2.4 Object.defineProperty ( O, P, Attributes )

defineProperty函数用于添加自有属性和/或更新对象已存在的自有属性的特性。调用defineProperty函数时，将执行以下步骤：

  1. 如果Type(O)不是Object，抛出 TypeError 异常
  2. 令key为? ToPropertyKey(P)
  3. 令desc为? ToPropertyDescriptor(Attributes)
  4. 执行? DefinePropertyOrThrow(O, key, desc)
  5. 返回O

19.1.2.5 Object.entries ( O )

使用参数O调用entries函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令nameList为? EnumerableOwnProperties(obj, key+value)
  3. 返回CreateArrayFromList(nameList)

19.1.2.6 Object.freeze ( O )

调用freeze函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回O
  2. 令status为? SetIntegrityLevel(O, frozen)
  3. 如果status为false，抛出 TypeError 异常
  4. 返回O

19.1.2.7 Object.fromEntries ( iterable )

当使用参数iterable调用fromEntries方法时，执行以下步骤：

  1. 执行? RequireObjectCoercible(iterable)
  2. 令obj为OrdinaryObjectCreate(%Object.prototype%)
  3. 断言：obj是一个没有自有属性的可扩展普通对象
  4. 令stepsDefine为CreateDataPropertyOnObject Functions中定义的算法步骤
  5. 令adder为! CreateBuiltinFunction(stepsDefine, « »)
  6. 返回? AddEntriesFromIterable(obj, iterable, adder)

注：此函数创建的adder永远无法被ECMAScript代码直接访问

19.1.2.7.1 CreateDataPropertyOnObject Functions

CreateDataPropertyOnObject函数是匿名内置函数。当使用参数key和value调用CreateDataPropertyOnObject函数时，执行以下步骤：

  1. 令O为this值
  2. 断言：Type(O)为Object
  3. 断言：O是可扩展的普通对象
  4. 令propertyKey为? ToPropertyKey(key)
  5. 执行! CreateDataPropertyOrThrow(O, propertyKey, value)
  6. 返回undefined

19.1.2.8 Object.getOwnPropertyDescriptor ( O, P )

当调用getOwnPropertyDescriptor函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令key为? ToPropertyKey(P)
  3. 令desc为? obj.[[GetOwnProperty]](key)
  4. 返回FromPropertyDescriptor(desc)

19.1.2.9 Object.getOwnPropertyDescriptors ( O )

当调用getOwnPropertyDescriptors函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令ownKeys为? obj.[[OwnPropertyKeys]]()
  3. 令descriptors为! OrdinaryObjectCreate(%Object.prototype%)
  4. 对ownKeys中的每个元素key，按照列表顺序：
    1. 令desc为? obj.[[GetOwnProperty]](key)
    2. 令descriptor为! FromPropertyDescriptor(desc)
    3. 如果descriptor不是undefined，执行! CreateDataPropertyOrThrow(descriptors, key, descriptor)
  5. 返回descriptors

19.1.2.10 Object.getOwnPropertyNames ( O )

当调用getOwnPropertyNames函数时，将执行以下步骤：

  1. 返回? GetOwnPropertyKeys(O, String)

19.1.2.11 Object.getOwnPropertySymbols ( O )

当使用参数O调用getOwnPropertySymbols函数时，将执行以下步骤：

  1. 返回? GetOwnPropertyKeys(O, Symbol)

19.1.2.11.1 运行时语义: GetOwnPropertyKeys ( O, type )

使用参数O和type调用抽象操作GetOwnPropertyKeys，其中O是对象，type是string或symbol，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令keys为? obj.[[OwnPropertyKeys]]()
  3. 令nameList为一个新的空列表
  4. 对于keys中的每个元素nextKey，按列表顺序执行：
    1. 如果Type(nextKey)为Symbol且type是symbol，又或者Type(nextKey)为String且type是string则：
      1. 将nextKey作为最后一个元素添加到nameList
  5. 返回CreateArrayFromList(nameList)

19.1.2.12 Object.getPrototypeOf ( O )

使用参数O调用getPrototypeOf函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 返回? obj.[[GetPrototypeOf]]()

19.1.2.13 Object.is ( value1, value2 )

当使用参数value1和value2调用is函数时，执行以下步骤：

  1. 返回SameValue(value1, value2)

19.1.2.14 Object.isExtensible ( O )

使用参数O调用isExtensible函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回false
  2. 返回? IsExtensible(O)

19.1.2.15 Object.isFrozen ( O )

使用参数O调用isFrozen函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回false
  2. 返回? TestIntegrityLevel(O, frozen)

19.1.2.16 Object.isSealed ( O )

使用参数O调用isSealed函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回false
  2. 返回? TestIntegrityLevel(O, sealed)

19.1.2.17 Object.keys ( O )

使用参数O调用keys函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令nameList为? EnumerableOwnProperties(obj, key)
  3. 返回CreateArrayFromList(nameList)

19.1.2.18 Object.preventExtensions ( O )

调用preventExtensions函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回O
  2. 令status为? O.[[PreventExtensions]]()
  3. 如果status为false，抛出 TypeError 异常
  4. 返回O

19.1.2.19 Object.prototype

Object.prototype的初始值为%Object.prototype%

此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.1.2.20 Object.seal ( O )

调用seal函数时，执行以下步骤：

  1. 如果Type(O)不是Object，返回O
  2. 令status为? SetIntegrityLevel(O, sealed)
  3. 如果status为false，抛出 TypeError 异常
  4. 返回O

19.1.2.21 Object.setPrototypeOf ( O, proto )

当使用参数O和proto调用setPrototypeOf函数时，执行以下步骤：

  1. 将O设为? RequireObjectCoercible(O)
  2. 如果Type(proto)既不是Object也不是Null，抛出 TypeError 异常
  3. 如果Type(O)不是Object，返回O
  4. 令status为? O.[[SetPrototypeOf]](proto)
  5. 如果status为false，抛出 TypeError 异常
  6. 返回O

19.1.2.22 Object.values ( O )

使用参数O调用values函数时，执行以下步骤：

  1. 令obj为? ToObject(O)
  2. 令nameList为? EnumerableOwnProperties(obj, value)
  3. 返回CreateArrayFromList(nameList)

19.1.3 Object原型对象的属性

Object原型对象

  • 是固有对象％ObjectPrototype％
  • 具有[[Extensible]]内部槽，且值为true
  • 具有为普通对象定义的内部方法，除了9.4.7.1中定义的[[SetPrototypeOf]]方法（因此，它是一个不可变的原型奇异对象）
  • 具有[[Prototype]]内部槽，且值为null

19.1.3.1 Object.prototype.constructor

Object.prototype.constructor的初始值为固有对象%Object%

19.1.3.2 Object.prototype.hasOwnProperty ( V )

当使用参数V调用hasOwnProperty方法时，将执行以下步骤：

  1. 令P为? ToPropertyKey(V)
  2. 令O为? ToObject(this值)
  3. 返回? HasOwnProperty(O, P)

注意：选择步骤1和2的顺序是为了确保即使this值为undefined或null，在本规范的先前版本中由步骤1引发的任何异常都将继续引发。

19.1.3.3 Object.prototype.isPrototypeOf ( V )

使用参数V调用isPrototypeOf方法时，将执行以下步骤：

  1. 如果Type(V)不是Object，返回false
  2. 令O为? ToObject(this值)
  3. 重复执行：
    1. 令V为? V.[[GetPrototypeOf]]()
    2. 如果V为null，返回false
    3. 如果SameValue(O, V)为true，返回true

注意：在V不是对象且this值为undefined或null的情况下，步骤1和2的顺序保留了本规范先前版本指定的行为。

19.1.3.4 Object.prototype.propertyIsEnumerable ( V )

当使用参数V调用propertyIsEnumerable方法时，将执行以下步骤：

  1. 令P为? ToPropertyKey(V)
  2. 令O为? ToObject(this值)
  3. 令desc为? O.[[GetOwnProperty]](P)
  4. 如果desc为undefined，返回false
  5. 返回desc.[[Enumerable]]的值

注意1：此方法不考虑原型链中的对象。
注意2：选择步骤1和2的顺序是为了确保即使this值undefined或null，在本规范的先前版本中由步骤1引发的任何异常都将继续引发。

19.1.3.5 Object.prototype.toLocaleString ( [ reserved1 [ , reserved2 ] ] )

调用toLocaleString方法时，将执行以下步骤：

  1. 令O为this值
  2. 返回Invoke(O, "toString")

此函数的可选参数未使用，但旨在与ECMA-402 toLocalString函数使用的参数模式相对应。 不包含ECMA-402支持的实现不得将这些参数位置用于其他目的。

toLocaleString的length属性为0

注意1：该函数为没有特定于语言环境的toString行为的对象提供了通用的toLocaleString实现。 Array，Number，Date和Typed Arrays提供了它们自己的语言环境敏感的toLocaleString方法。

注意2：ECMA-402故意不提供此默认实施的替代方法。

19.1.3.6 Object.prototype.toString ()

调用toString方法时，将执行以下步骤：

  1. 如果this值为undefined，返回"[object Undefined]"
  2. 如果this值为null，返回"[object Null]"
  3. 令O为! ToObject(this值)
  4. 令isArray为? IsArray(O)
  5. IfisArray为true，令builtinTag为"Array"
  6. Else if O有[[ParameterMap]]内部槽，令builtinTag为"Arguments"
  7. Else if O有[[Call]]内部方法，令builtinTag为"Function"
  8. Else if O有[[ErrorData]]内部槽，令builtinTag为"Error"
  9. Else if O有[[BooleanData]]内部槽，令builtinTag为"Boolean"
  10. Else if O有[[NumberData]]内部槽，令builtinTag为"Number"
  11. Else if O有[[StringData]]内部槽，令builtinTag为"String"
  12. Else if O有[[DateValue]]内部槽，令builtinTag为"Date"
  13. Else if O有[[RegExpMatcher]]内部槽，令builtinTag为"RegExp"
  14. Else 令builtinTag为"Object"
  15. 令tag为? Get(O, @@toStringTag)
  16. 如果Type(tag)不为String，将tag设为builtinTag
  17. 返回连接字符串"[object "，tag和"]"的结果。

此函数是固有对象%ObjProto_toString%

注意：从历史上看，此函数有时用于访问[[Class]]内部槽的String值，该值在本规范的先前版本中用作各种内置对象的标称类型标记。 上面对toString的定义保留了与使用toString来测试那些特定种类的内置对象的旧代码的兼容性。 它没有为其他类型的内置或程序定义的对象提供可靠的类型测试机制。 另外，程序可以使用@@toStringTag，这将使此类旧式测试的可靠性无效。

19.1.3.7 Object.prototype.valueOf ( )

调用valueOf方法时，将执行以下步骤：

  1. 返回? ToObject(this值)

此函数是固有对象%ObjProto_valueOf%

19.2 Function对象

19.2.1 Function构造函数

Function构造函数：

  • 是固有对象%Function%
  • 是全局对象的"Function"属性的初始值
  • 当作为函数而不是构造函数调用时，创建并初始化一个新的函数对象。函数调用Function(…)等价于使用相同参数的对象创建表达式new Function(…)
  • 被设计为可被子类化。它可以被用作类定义中的extends子句的值。打算继承指定Function行为的子类构造函数必须包括对Function构造函数的super调用，以创建并初始化一个具有内置函数行为所需内部槽的子类实例。所有定义函数对象的ECMAScript语法形式都会创建Function的实例。除了内置的GeneratorFunction、AsyncFunction和AsyncGeneratorFunction子类之外，没有任何语法方法可以创建Function子类的实例。

19.2.1.1 Function ( p1, p2, … , pn, body )

最后一个参数指定了一个函数的主体（可执行代码）；前面的所有参数都指定了形式参数。

当使用参数p1, p2, ..., pn, body（其中n可能是0，即没有 "p "参数，也可能没有提供body）调用Function函数时，会采取以下步骤：

  1. 令C为当前激活的函数对象
  2. 令args为通过[[Call]]或[[Constructor]]传递的argumentsList
  3. 返回? CreateDynamicFunction(C, NewTarget, normal, args)

注：允许但不需要为每个要指定的形式参数设置一个参数。例如，下面的三个表达式产生的结果都是一样的。

19.2.1.1.1 运行时语义: CreateDynamicFunction ( constructor, newTarget, kind, args )

使用参数constructor、newTarget、kind和args调用抽象操作CreateDynamicFunction。constructor是执行此操作的构造函数，newTarget是new最初应用到的构造函数，kind是normal、generator、async或asyncGenerator，args是一个包含传递给constructor的实际参数值的List。执行以下步骤：

  1. 断言：执行上下文栈至少有两个元素
  2. 令callerContext为执行上下文栈中栈顶之前的元素
  3. 令callerRealm为callerContext的Realm
  4. 令calleeRealm为当前Realm记录
  5. 执行? HostEnsureCanCompileStrings(callerRealm, calleeRealm)
  6. 如果newTarget为undefined，将newTarget设为constructor
  7. 如果kind为normal，则：
    1. 令goal为语法符号FunctionBody[~Yield, ~Await]
    2. 令parameterGoal为语法符号FormalParameters[~Yield, ~Await]
    3. 令fallbackProto为"%Function.prototype%"
  8. 否则如果kind为generator，则：
    1. 令goal为语法符号GeneratorBody
    2. 令parameterGoal为语法符号FormalParameters[+Yield, ~Await]
    3. 令fallbackProto为"%Generator%"
  9. 否则如果kind为async，则：
    1. 令goal为语法符号AsyncFunctionBody
    2. 令parameterGoal为语法符号FormalParameters[~Yield, +Await]
    3. 令fallbackProto为"%AsyncFunction.prototype%"
  10. 否则：
    1. 断言：kind为asyncGenerator
    2. 令goal为语法符号AsyncGeneratorBody
    3. 令parameterGoal为语法符号FormalParameters[+Yield, +Await]
    4. 令fallbackProto为"%AsyncGenerator%"
  11. 令argCount为args中的元素数量
  12. 令P为空字符串
  13. 如果argCount = 0，令bodyArg为空字符串
  14. 否则如果argCount = 1，令bodyArg为args[0]
  15. 否则：
    1. 断言：argCount > 1
    2. 令firstArg为args[0]
    3. 将P设为? ToString(firstArg)
    4. 令k为1
    5. 重复，只要k < argCount - 1：
      1. 令nextArg为args[k]
      2. 令nextArgString为? ToString(nextArg)
      3. 将P设为连接之前的P与","（逗号）和nextArgString得到的字符串
      4. 将k设为k + 1
    6. 令bodyArg为args[k]
  16. 令bodyString为连接字符串0x000A (LINE FEED)、? ToString(bodyArg)和0x000A (LINE FEED)的结果
  17. 按照与实现有关的顺序执行以下子步骤，可能会交错进行解析和错误检测：
    1. 令parameters为使用parameterGoal作为目标符号解析! UTF16DecodeString(P)的结果，当解析失败时抛出 SyntaxError 异常
    2. 令body为使用goal作为目标符号解析! UTF16DecodeString(bodyString)的结果，当解析失败时抛出 SyntaxError 异常
    3. 令strict为body的ContainsUseStrict
    4. 如果在parameters或body中检查到任何静态语义错误，抛出 SyntaxError 异常。如果strict为true，应用UniqueFormalParameters:FormalParameters的Early Error规则
    5. 如果strict为true且parameters的IsSimpleParameterList为false，抛出 SyntaxError 异常
    6. 如果parameters的BoundNames的任何元素出现在body的LexicallyDeclaredNames中，抛出 SyntaxError 异常
    7. 如果body包含SuperCall为true，抛出 SyntaxError 异常
    8. 如果parameters包含SuperCall为true，抛出 SyntaxError 异常.
    9. 如果body包含SuperProperty为true，抛出 SyntaxError 异常
    10. 如果parameters包含SuperProperty为true，抛出 SyntaxError 异常
    11. 如果kind为generator或asyncGenerator，则
      1. 如果parameters包含YieldExpression为true，抛出 SyntaxError 异常
    12. 如果kind为async或asyncGenerator，则
      1. 如果parameters包含AwaitExpression为true，抛出 SyntaxError 异常
    13. 如果strict为true，则：
      1. 如果parameters的BoundNames包含任何重复元素，抛出 SyntaxError 异常
  18. 令proto为? GetPrototypeFromConstructor(newTarget, fallbackProto)
  19. 令realmF为当前Realm记录
  20. 令scope为realmF.[[GlobalEnv]]
  21. 令F为! OrdinaryFunctionCreate(proto, parameters, body, non-lexical-this, scope)
  22. 如果kind为generator，则：
    1. 令prototype为OrdinaryObjectCreate(%Generator.prototype%)
    2. 执行DefinePropertyOrThrow(F, "prototype", PropertyDescriptor { [[Value]]: prototype, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: false })
  23. 否则如果kind为asyncGenerator，则：
    1. 令prototype为OrdinaryObjectCreate(%AsyncGenerator.prototype%)
    2. 执行DefinePropertyOrThrow(F, "prototype", PropertyDescriptor { [[Value]]: prototype, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: false })
  24. 否则如果kind为normal，执行 MakeConstructor(F)
  25. 注：Async函数不是可构造的，也没有[[Construct]]内部方法和"prototype"属性
  26. 执行SetFunctionName(F, "anonymous")
  27. 令prefix为表48中kind关联的前缀
  28. 令sourceString为连接字符串prefix, " anonymous("，P，0x000A (LINE FEED)，") {"，bodyString和"}"的结果
  29. 将F.[[SourceText]]设为! UTF16DecodeString(sourceString)
  30. 返回F

注：使用CreateDynamicFunction创建的每个非async函数都会创建一个 "prototype"属性，以提供该函数被用作构造函数的可能性。

表48：

Kind Prefix
normal "function"
generator "function*"
async "async function"
asyncGenerator "async function*"

19.2.2 Function构造函数的属性

Function构造函数

  • 它自己是内置的函数对象
  • 具有[[Prototype]]内部槽，且其值为%Function.prototype%
  • 具有以下属性：

19.2.2.1 Function.length

该属性是一个值为1的数据属性，此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

19.2.2.2 Function.prototype

Function.prototype的值是固有函数原型对象%Function.prototype%

此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.2.3 Function Prototype对象

Function Prototype对象：

  • 是%Function.prototype%
  • 它自己是内置函数对象
  • 调用时接受任意参数，返回undefined
  • 没有[[Construct]]内部方法，不能使用new运算符作为构造函数调用
  • 具有[[Prototype]]内部槽，且其值为%Object.prototype%
  • 没有prototype属性
  • 具有"length"属性，且其值为0
  • 具有"name"属性，且其值为空字符串

注：Function Protoype对象被指定为一个函数对象，以确保与ECMAScript 2015规范之前创建的ECMAScript代码兼容。

19.2.3.1 Function.prototype.apply ( thisArg, argArray )

当使用参数thisArg和argArray调用apply方法时，执行以下步骤：

  1. 令func为this值
  2. 如果IsCallable(func)为false，抛出 TypeError 异常
  3. 如果argArray为undefined或null，则：
    1. 执行PrepareForTailCall()
    2. 返回? Call(func, thisArg)
  4. 令argList为? CreateListFromArrayLike(argArray)
  5. 执行PrepareForTailCall()
  6. 返回? Call(func, thisArg, argList)

注1：thisArg值作为this值被不加修改地传递。这是对第3版的改变，在第3版中，undefined或null的thisArg被替换为全局对象，其他值则使用ToObject进行转换，结果作为this值被传递。即使thisArg被不加修改地传递，非严格的函数仍然在进入函数时执行这些转换。

注2：如果func是箭头函数，或者绑定函数奇异对象，则第6步的thisArg会被忽略。

19.2.3.2 Function.prototype.bind ( thisArg, …args )

当使用参数thisArg和零个或多个args调用bind方法时，执行以下步骤：

  1. 令Target为this值
  2. 如果IsCallable(Target)为false，抛出 TypeError 异常
  3. 令args为一个新的（可能是空的）列表，包含thisArg之后提供的所有参数值
  4. 令F为? BoundFunctionCreate(Target, thisArg, args)
  5. 令targetHasLength为? HasOwnProperty(Target, "length")
  6. 如果targetHasLength为true，则：
    1. 令targetLen为? Get(Target, "length")
    2. 如果Type(targetLen)不是Number，令L为0
    3. 否则：
      1. 将targetLen设为! ToInteger(targetLen)
      2. 令L为Max(0, targetLen - len(args))
  7. 否则，令L为0
  8. 执行! SetFunctionLength(F, L)
  9. 令targetName为? Get(Target, "name")
  10. 如果Type(targetName)不是String，将targetName设为空字符串
  11. 执行SetFunctionName(F, targetName, "bound")
  12. 返回F

注1：使用Function.prototype.bind创建的函数对象是奇异对象。它们没有prototype属性。
注2：如果Target是箭头函数或绑定函数奇异对象，则传递给此方法的thisArg在接下来对F的调用中不会被使用。

19.2.3.3 Function.prototype.call ( thisArg, …args )

当使用参数thisArg和零个或多个args调用call方法时，执行以下步骤：

  1. 令func为this值
  2. 如果IsCallable(func)为false，抛出 TypeError 异常
  3. 令argList为新的空列表
  4. 如果此方法被以超过一个参数调用，则从第二个参数开始，按照从左到右的顺序，将每个参数添加到argList中
  5. 执行PrepareForTailCall()
  6. 返回? Call(func, thisArg, argList)

注1：thisArg值作为this值被不加修改地传递。这是对第3版的改变，在第3版中，undefined或null的thisArg被替换为全局对象，其他值则使用ToObject进行转换，结果作为this值被传递。即使thisArg被不加修改地传递，非严格的函数仍然在进入函数时执行这些转换。

注2：如果func是箭头函数，或者绑定函数奇异对象，则第6步的thisArg会被忽略。

19.2.3.4 Function.prototype.constructor

Function.prototype.constructor的初始值为%Function%

19.2.3.5 Function.prototype.toString ( )

当调用toString方法时，执行以下步骤：

  1. 令func为this值
  2. 如果func是绑定函数奇异对象，或内置函数对象，则返回func的基于实现的表示源代码的字符串。此表示必须含有NativeFunction的语法。另外，如果func是知名固有对象，并且没有被识别为匿名函数，那么返回的字符串中被PropertyName匹配的部分必须是func的"name"属性的初始值
  3. 如果Type(func)是Object且func有[[SourceText]]内部槽，且func.[[SourceText]]是Unicode代码点序列，且! HostHasSourceTextAvailable(func)为true，则返回! UTF16Encode(func.[[SourceText]])
  4. 如果Type(func)是Object，且IsCallable(func)为true，则返回func的基于实现的表示源代码的字符串。此表示必须含有NativeFunction的语法。
  5. 抛出 TypeError 异常

19.2.3.6 Function.prototype [ @@hasInstance ] ( V )

当使用参数V调用对象F的@@hasInstance方法时，执行以下步骤：

  1. 令F为this值
  2. 返回? OrdinaryHasInstance(F, V)

此函数的"name"属性值为"[Symbol.hasInstance]"
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

这是大多数函数都会继承的@@hasInstance的默认实现。@@hasInstance被instanceof操作符调用，以确定一个值是否是一个特定构造函数的实例。表达式：v instanceof F会计算F[@@hasInstance](v)。一个构造函数可以通过在函数上暴露一个不同的@@hasInstance方法来控制哪些对象被识别为它的实例。

这个属性是不可写的，也是不可配置的，以防止被用来全局地暴露一个绑定函数的目标函数的篡改。

19.2.4 Function实例

每个Function实例都是ECMAScript的函数对象，具有表27中列出的内部槽。使用Function.prototype.bind方法（19.2.3.2）创建的Function对象具有表28中所列的内部槽。

Function实例具有下列属性：

19.2.4.1 length

"length"属性的值是一个整数，表示函数所期望的典型参数的数量。然而，语言允许用其他数量的参数来调用该函数。当一个函数被使用其"length"属性所指定的参数数量之外的其他参数数量上调用时，其行为取决于该函数。此属性具有特性：{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

19.2.4.2 name

"name"属性的值是一个描述该函数的字符串。该名称没有语义意义，但通常是一个变量或属性名称，用于在ECMAScript代码中的定义点引用该函数。此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

在本规范中没有关联上下文名称的匿名函数对象没有 "name"自有属性，而是继承%Function.prototype%的 "name"属性。

19.2.4.3 prototype

可以用作构造函数的函数实例有一个 "prototype"属性。每当这样的函数实例被创建时，另一个普通对象也被创建，并且是该函数的 "prototype"属性的初始值。除非另有规定，"prototype"属性的值用于初始化该函数作为构造函数被调用时创建的对象的[[Prototype]]内部槽。

此属性具有特性{ [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: false }

注：使用Function.prototype.bind，或计算MethodDefinition（不是GeneratorMethod或AsyncGeneratorMethod），或ArrowFunction都没有"prototype"属性

19.4 Symbol对象

19.4.1 Symbol构造函数

Symbol构造函数是固有对象％Symbol％和全局对象的Symbol属性的初始值。 当Symbol作为函数调用时，它将返回一个新的Symbol值。

Symbol构造函数不能与new运算符一起使用或被子类化。 它可以用作类定义的extends子句的值，但是对Symbol构造函数的super调用将导致异常。

19.4.1.1 Symbol ( [ description ] )

使用可选的参数description调用Symbol时，将执行以下步骤：

  1. 如果newTarget不是undefined，抛出TypeError异常
  2. 如果description是undefined，令descString为undefined
  3. 否则，令descString为ToString(description)
  4. ReturnIfAbrupt(descString)
  5. 返回一个新的唯一Symbol值，其[[Description]]值为descString

19.4.2 Symbol构造函数的属性

Symbol构造函数的[[Prototype]]内部槽的值是固有对象％FunctionPrototype％（19.2.3）。
除了length属性（其值为0）之外，Symbol构造函数还具有以下属性：

19.4.2.1 Symbol.for ( key )

当使用参数key调用Symbol.for时，它将执行以下步骤：

  1. 令stringKey为ToString(key)
  2. ReturnIfAbrupt(stringKey)
  3. 对GlobalSymbolRegistry列表中的每个元素e，执行：
    1. 如果SameValue(e.[[key]], stringKey)为true，返回e.[[symbol]]
  4. 断言：GlobalSymbolRegistry当前不包含条目stringKey。
  5. 令newSymbol为一个新的唯一Symbol值，其[[Description]]值为stringKey
  6. 将记录{ [[key]]: stringKey, [[symbol]]: newSymbol }添加到GlobalSymbolRegistry列表
  7. 返回newSymbol

GlobalSymbolRegistry是一个全局可用的列表。 所有Code Realms都共享它。 在计算任何ECMAScript代码之前，它会初始化为空列表。 GlobalSymbolRegistry的元素是具有表44中定义的结构的记录(Records)。

字段名 值 用途
[[key]] 一个String 一个字符串键，用来全局标识一个Symbol
[[symbol]] 一个Symbol 一个可以从任何Realm检索的Symbol

19.4.2.3 Symbol.isConcatSpreadable

Symbol.isConcatSpreadable的初始值是知名symbol@@isConcatSpreadable
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.4.2.4 Symbol.iterator

Symbol.iterator的初始值是知名symbol@@iterator
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.4.2.5 Symbol.keyFor ( sym )

当使用参数sym调用Symbol.keyFor时，它将执行以下步骤：

  1. 如果Type(sym)不是Symbol，抛出TypeError异常
  2. 对GlobalSymbolRegistry列表中的每个元素e，执行：
    1. 如果SameValue(e.[[symbol]], sym)为true，返回e.[[key]]
  3. 断言：GlobalSymbolRegistry当前不包含条目sym。
  4. 返回undefined

19.4.3.2 Symbol.prototype.toString ( )

采取以下步骤：

  1. 令s为this值
  2. 如果Type(s)为Symbol，令sym为s
  3. 否则：
    1. 如果Type(s)不是Object，抛出TypeError异常
    2. 如果s没有[[SymbolData]]内部槽，抛出TypeError异常
    3. 令sym为s的[[SymbolData]]内部槽的值
  4. 返回SymbolDescriptiveString(sym)
19.4.3.2.1 运行时语义:SymbolDescriptiveString ( sym )

当使用参数sym调用抽象操作SymbolDescriptiveString时，将执行以下步骤：

  1. 断言：Type(sym)为Symbol
  2. 令desc为sym的[[Description]]值
  3. 如果desc为undefined，令desc为空字符串
  4. 断言：Type(desc)为字符串
  5. 返回字符串连接的结果："Symbol("+desc+")"
