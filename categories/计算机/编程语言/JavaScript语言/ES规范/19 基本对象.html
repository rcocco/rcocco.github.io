19 基本对象

19.1 Object对象

19.1.1 Object构造函数

Object构造函数是固有对象%Object%，也是全局对象的Object属性的初始值。当作为构造函数调用时，它将创建一个新的普通对象(ordinary object)。当Object作为函数而不是构造函数调用时，它将执行类型转换。

Object构造函数被设计为可子类化的。 它可以用作类定义的extends子句的值。

19.1.1.1 Object ( [ value ] )

使用可选参数value调用Object函数时，将执行以下步骤：

  1. 如果NewTarget既不是undefined也不是激活的函数（运行中的执行上下文的函数组件被称为激活的函数对象），则返回OrdinaryCreateFromConstructor(NewTarget, "%ObjectPrototype%")
  2. 如果value为null或undefined或没有提供，返回ObjectCreate(%ObjectPrototype%)
  3. 返回ToObject(value)

19.1.2 Object构造函数的属性

Object构造函数的[[Prototype]]内部插槽的值是固有对象％FunctionPrototype％。
除了length属性（其值为1）之外，Object构造函数还具有以下属性：

19.1.2.1 Object.assign ( target, …sources )

assign函数用于将所有可枚举的自有属性的值从一个或多个源对象复制到目标对象。 调用assign函数时，将执行以下步骤：

  1. 令to为ToObject(target)
  2. ReturnIfAbrupt(to)
  3. 如果只传递了一个参数，返回to
  4. 令source为一个列表，列表中包含从第二个参数开始的所有参数值
  5. 对于source中的每个nextSource，按照索引的升序顺序，执行：
    1. 如果nextSource为undefined或null，令keys为一个空列表
    2. 否则：
      1. 令from为ToObject(nextSource)
      2. 令keys为from.[[OwnPropertyKeys]]()
      3. ReturnIfAbrupt(keys)
    3. 对于keys中的每个元素nextKey，按照列表顺序，执行
      1. 令desc为from.[[GetOwnProperty]](nextKey)
      2. ReturnIfAbrupt(desc)
      3. 如果desc不是undefined，并且desc.[[Enumerable]]为true，则
        1. 令propValue为Get(from, nextKey)
        2. ReturnIfAbrupt(propValue)
        3. 令status为Set(to, nextKey, propValue, true)
        4. ReturnIfAbrupt(status)
  6. 返回to

将每个源对象的自有可枚举属性（属性键依次为整数索引、字符串、Symbol）的值，复制到目标对象使其具有相同的属性键和值。

19.1.2.4 Object.defineProperty ( O, P, Attributes )

defineProperty函数用于添加自有属性和/或更新对象已存在的自有属性的特性。调用defineProperty函数时，将执行以下步骤：

  1. 如果Type(O)不是Object，抛出 TypeError 异常
  2. 令key为ToPropertyKey(P)
  3. ReturnIfAbrupt(key)
  4. 令desc为ToPropertyDescriptor(Attributes)
  5. ReturnIfAbrupt(desc)
  6. 令success为DefinePropertyOrThrow(O,key, desc)
  7. ReturnIfAbrupt(success)
  8. 返回O

ES8 19.1.2.5 Object.entries ( O )

使用参数O调用entries函数时，执行以下步骤：

  1. 令obj为ToObject(O)
  2. 令nameList为EnumerableOwnProperties(obj, "key+value")
  3. 返回CreateArrayFromList(nameList)

19.1.2.7 Object.getOwnPropertyNames ( O )

调用getOwnPropertyNames函数时，将执行以下步骤：

  1. 返回GetOwnPropertyKeys(O, String)

19.1.2.8 Object.getOwnPropertySymbols ( O )

当使用参数O调用getOwnPropertySymbols函数时，将执行以下步骤：

  1. 返回GetOwnPropertyKeys(O, Symbol)

19.1.2.8.1 运行时语义: GetOwnPropertyKeys ( O, Type )

使用参数O和Type调用抽象操作GetOwnPropertyKeys，其中O是对象，并且Type是ECMAScript规范类型String或Symbol之一。 采取以下步骤：

  1. 令obj为ToObject(O)
  2. ReturnIfAbrupt(obj)
  3. 令keys为obj.[[OwnPropertyKeys]]()
  4. ReturnIfAbrupt(keys)
  5. 令nameList为一个新的空列表
  6. 对于keys中的每个元素nextKey，按列表顺序执行：
    1. 如果Type(nextKey)为Type，则将nextKey作为最后一个元素添加到nameList
  7. 返回CreateArrayFromList(nameList)

19.1.2.14 Object.keys ( O )

当使用参数O调用函数keys时，执行以下步骤：

  1. 令obj为ToObject(O)
  2. ReturnIfAbrupt(obj)
  3. 令nameList为EnumerableOwnNames(obj)
  4. ReturnIfAbrupt(nameList)
  5. 返回CreateArrayFromList(nameList)

如果实现为for-in语句定义了枚举的特定顺序，则必须对步骤4中返回的数组元素使用相同的顺序。

ES8 19.1.2.21 Object.values ( O )

使用参数O调用values函数时，执行以下步骤：

  1. 令obj为ToObject(O)
  2. 令nameList为EnumerableOwnProperties(obj, "value")
  3. 返回CreateArrayFromList(nameList)

19.1.3 Object原型对象的属性

Object原型对象是固有对象％ObjectPrototype％。 Object原型对象是一个普通对象(ordinary object)

Object原型对象的[[Prototype]]内部插槽的值为null，[[Extensible]]内部插槽的初始值为true。

19.1.3.1 Object.prototype.constructor

Object.prototype.constructor的初始值为固有对象%Object%

19.1.3.2 Object.prototype.hasOwnProperty ( V )

当使用参数V调用hasOwnProperty方法时，将执行以下步骤：

  1. 令P为ToPropertyKey(V)
  2. ReturnIfAbrupt(P)
  3. 令O为ToObject(this值)
  4. ReturnIfAbrupt(O)
  5. 返回HasOwnProperty(O, P)

注意：选择步骤1和3的顺序是为了确保即使this值为undefined或null，在本规范的先前版本中由步骤1引发的任何异常都将继续引发。

19.1.3.3 Object.prototype.isPrototypeOf ( V )

使用参数V调用isPrototypeOf方法时，将执行以下步骤：

  1. 如果Type(V)不是Object，返回false
  2. 令O为ToObject(this值)
  3. ReturnIfAbrupt(O)
  4. 重复执行：
    1. 令V为V.[[GetPrototypeOf]]()
    2. 如果V为null，返回false
    3. 如果SameValue(O, V)为true，返回true

注意：在V不是对象且this值为undefined或null的情况下，步骤1和2的顺序保留了本规范先前版本指定的行为。

19.1.3.4 Object.prototype.propertyIsEnumerable ( V )

当使用参数V调用propertyIsEnumerable方法时，将执行以下步骤：

  1. 令P为ToPropertyKey(V)
  2. ReturnIfAbrupt(P)
  3. 令O为ToObject(this值)
  4. ReturnIfAbrupt(O)
  5. 令desc为O.[[GetOwnProperty]](P)
  6. ReturnIfAbrupt(desc)
  7. 如果desc为undefined，返回false
  8. 返回desc.[[Enumerable]]的值

注意1：此方法不考虑原型链中的对象。
注意2：选择步骤1和3的顺序是为了确保即使this值undefined或null，在本规范的先前版本中由步骤1引发的任何异常都将继续引发。

19.1.3.5 Object.prototype.toLocaleString ( [ reserved1 [ , reserved2 ] ] )

调用toLocaleString方法时，将执行以下步骤：

  1. 令O为this值
  2. 返回Invoke(O, "toString")

此函数的可选参数未使用，但旨在与ECMA-402 toLocalString函数使用的参数模式相对应。 不包含ECMA-402支持的实现不得将这些参数位置用于其他目的。

toLocaleString的length属性为0

注意1：该函数为没有特定于语言环境的toString行为的对象提供了通用的toLocaleString实现。 Array，Number，Date和Typed Arrays提供了它们自己的语言环境敏感的toLocaleString方法。

注意2：ECMA-402故意不提供此默认实施的替代方法。

19.1.3.6 Object.prototype.toString ()

调用toString方法时，将执行以下步骤：

  1. 如果this值为undefined，返回"[object Undefined]"
  2. 如果this值为null，返回"[object Null]"
  3. 令O为ToObject(this值)
  4. 令isArray为IsArray(O)
  5. ReturnIfAbrupt(isArray)
  6. IfisArray为true，令builtinTag为"Array"
  7. Else if O为奇异String对象，令builtinTag为"String"
  8. Else if O有[[ParameterMap]]内部插槽，令builtinTag为"Arguments"
  9. Else if O有[[Call]]内部方法，令builtinTag为"Function"
  10. Else if O有[[ErrorData]]内部插槽，令builtinTag为"Error"
  11. Else if O有[[BooleanData]]内部插槽，令builtinTag为"Boolean"
  12. Else if O有[[NumberData]]内部插槽，令builtinTag为"Number"
  13. Else if O有[[DateValue]]内部插槽，令builtinTag为"Date"
  14. Else if O有[[RegExpMatcher]]内部插槽，令builtinTag为"RegExp"
  15. Else 令builtinTag为"Object"
  16. 令tag为Get (O, @@toStringTag)
  17. ReturnIfAbrupt(tag)
  18. 如果Type(tag)不为String，令tag为builtinTag
  19. 返回字符串，该字符串是"[object "，tag和"]"连接的结果。

此函数是固有对象%ObjProto_toString%

注意：从历史上看，此函数有时用于访问[[Class]]内部插槽的String值，该值在本规范的先前版本中用作各种内置对象的标称类型标记。 上面对toString的定义保留了与使用toString来测试那些特定种类的内置对象的旧代码的兼容性。 它没有为其他类型的内置或程序定义的对象提供可靠的类型测试机制。 另外，程序可以使用@@toStringTag，这将使此类旧式测试的可靠性无效。

19.1.3.7 Object.prototype.valueOf ( )

调用valueOf方法时，将执行以下步骤：

  1. 返回ToObject(this值)

19.4 Symbol对象

19.4.1 Symbol构造函数

Symbol构造函数是固有对象％Symbol％和全局对象的Symbol属性的初始值。 当Symbol作为函数调用时，它将返回一个新的Symbol值。

Symbol构造函数不能与new运算符一起使用或被子类化。 它可以用作类定义的extends子句的值，但是对Symbol构造函数的super调用将导致异常。

19.4.1.1 Symbol ( [ description ] )

使用可选的参数description调用Symbol时，将执行以下步骤：

  1. 如果newTarget不是undefined，抛出TypeError异常
  2. 如果description是undefined，令descString为undefined
  3. 否则，令descString为ToString(description)
  4. ReturnIfAbrupt(descString)
  5. 返回一个新的唯一Symbol值，其[[Description]]值为descString

19.4.2 Symbol构造函数的属性

Symbol构造函数的[[Prototype]]内部插槽的值是固有对象％FunctionPrototype％（19.2.3）。
除了length属性（其值为0）之外，Symbol构造函数还具有以下属性：

19.4.2.1 Symbol.for ( key )

当使用参数key调用Symbol.for时，它将执行以下步骤：

  1. 令stringKey为ToString(key)
  2. ReturnIfAbrupt(stringKey)
  3. 对GlobalSymbolRegistry列表中的每个元素e，执行：
    1. 如果SameValue(e.[[key]], stringKey)为true，返回e.[[symbol]]
  4. 断言：GlobalSymbolRegistry当前不包含条目stringKey。
  5. 令newSymbol为一个新的唯一Symbol值，其[[Description]]值为stringKey
  6. 将记录{ [[key]]: stringKey, [[symbol]]: newSymbol }添加到GlobalSymbolRegistry列表
  7. 返回newSymbol

GlobalSymbolRegistry是一个全局可用的列表。 所有Code Realms都共享它。 在计算任何ECMAScript代码之前，它会初始化为空列表。 GlobalSymbolRegistry的元素是具有表44中定义的结构的记录(Records)。

字段名 值 用途
[[key]] 一个String 一个字符串键，用来全局标识一个Symbol
[[symbol]] 一个Symbol 一个可以从任何Realm检索的Symbol

19.4.2.3 Symbol.isConcatSpreadable

Symbol.isConcatSpreadable的初始值是知名symbol@@isConcatSpreadable
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.4.2.4 Symbol.iterator

Symbol.iterator的初始值是知名symbol@@iterator
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

19.4.2.5 Symbol.keyFor ( sym )

当使用参数sym调用Symbol.keyFor时，它将执行以下步骤：

  1. 如果Type(sym)不是Symbol，抛出TypeError异常
  2. 对GlobalSymbolRegistry列表中的每个元素e，执行：
    1. 如果SameValue(e.[[symbol]], sym)为true，返回e.[[key]]
  3. 断言：GlobalSymbolRegistry当前不包含条目sym。
  4. 返回undefined

19.4.3.2 Symbol.prototype.toString ( )

采取以下步骤：

  1. 令s为this值
  2. 如果Type(s)为Symbol，令sym为s
  3. 否则：
    1. 如果Type(s)不是Object，抛出TypeError异常
    2. 如果s没有[[SymbolData]]内部插槽，抛出TypeError异常
    3. 令sym为s的[[SymbolData]]内部插槽的值
  4. 返回SymbolDescriptiveString(sym)
19.4.3.2.1 运行时语义:SymbolDescriptiveString ( sym )

当使用参数sym调用抽象操作SymbolDescriptiveString时，将执行以下步骤：

  1. 断言：Type(sym)为Symbol
  2. 令desc为sym的[[Description]]值
  3. 如果desc为undefined，令desc为空字符串
  4. 断言：Type(desc)为字符串
  5. 返回字符串连接的结果："Symbol("+desc+")"
