25.1 迭代

25.1.1 公共迭代接口

接口是一组属性键，其关联的值与特定的规范相匹配。 提供接口规范所描述的所有属性的任何对象都符合该接口。 接口没有用不同的对象表示。 可能有许多单独实现的对象符合任何接口。 单个对象可能符合多个接口。

25.1.1.1 Iterable接口

Iterable接口包含表52中描述的属性：

表52：Iterable接口必需的属性

属性 值 要求
@@iterator 返回Iterator对象的函数。 返回的对象必须符合Iterator接口。

25.1.1.2 Iterator接口

实现Iterator接口的对象必须包括表53中的属性。此类对象还可以实现表54中的属性。

表53：Iterator接口必需的属性

属性 值 要求
next 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 如果对Iterator的next方法的上一次调用返回了done属性为true的IteratorResult对象，则对该对象的next方法的所有后续调用也应返回其done属性为true的IteratorResult对象。 但是，此要求不是强制的。

注：可以将参数传递给next函数，但是参数的解释和有效性取决于目标Iterator。 for-of语句和Iterators的其他普通用户不传递任何参数，因此，期望以这种方式使用的Iterator对象必须准备好处理不带参数的调用。

表54：Iterator接口可选的属性

属性 值 要求
return 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 调用此方法会通知Iterator对象，调用者无意再对Iterator进行next方法调用。 返回的IteratorResult对象通常具有一个done属性（值为true）和一个value属性（值为传递给return方法的参数）。 但是，此要求不是强制的。
throw 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 调用此方法会通知Iterator对象调用者已检测到错误情况。 参数可用于标识错误条件，并且通常是异常对象。 典型的响应是throw作为参数传递的值。 如果该方法没有throw，则返回的IteratorResult对象通常将具有一个值为true的done属性。

注2：通常，这些方法的调用者应在调用它们之前检查它们的存在。 某些ECMAScript语言功能（包括for-of，yield *和数组解构）在执行存在性检查后调用这些方法。 大多数接受Iterable对象作为参数的ECMAScript库函数也会有条件地调用它们。

25.1.1.3 IteratorResult接口

IteratorResult接口包含表55中列出的属性：
表55：IteratorResult接口属性

属性 值 要求
done true或false 这是迭代器next方法调用的结果状态。 如果到达迭代器的末尾，则done为true。 如果未达到末尾，则done为false，并且可以使用值。 如果done属性（自有的或继承的）不存在，则认为该属性的值为false。
value 任何ECMAScript语言值 如果done为false，则此值为当前迭代元素值。 如果done为true，则此值为迭代器的返回值（如果提供了）。 如果迭代器没有返回值，则value为undefined。 在那种情况下，如果符合属性的对象没有继承显式的value属性，则可能缺少value属性。

25.4 Promise对象

Promise是一个对象，用作延迟（可能还是异步）计算的最终结果的占位符。

任何Promise对象均处于以下三种互斥状态之一：已兑现(fulfilled)，已拒绝(rejected)和待定(pending)：

  • 如果p.then(f, r)立即使任务入队来调用函数f，则Promisep是已兑现的
  • 如果p.then(f, r)立即使任务入队来调用函数r，则Promisep是已拒绝的
  • 如果Promise既不是已兑现的，也不是已解决的，则它是待定的

如果Promise不是待定的，则被称为确定的(settled)，也就是要么已兑现要么已拒绝

如果一个Promise是确定的，或者已被“锁定(locked in)”以匹配另一个Promise的状态，则该Promise是已解决的(resolved)。尝试解决或拒绝一个已解决的Promise没有任何效果。如果Promise不是已解决的，则它是未解决的。一个未解决的Promises总是待定状态。一个已解决的Promise可能是待定的，也可能是已兑现的，还可能是已拒绝的

25.4.1 Promise抽象操作

25.4.1.1 PromiseCapability记录

PromiseCapability是一个Record值，用于将promise对象和能解决或拒绝此promise对象的函数封装在一起。PromiseCapability记录由NewPromiseCapability抽象操作生成。
PromiseCapability记录有表57列出的字段：
表57：PromiseCapability记录字段

字段名 值 含义
[[Promise]] 对象 用作promise的对象
[[Resolve]] 函数对象 用来确定给定promise对象的函数
[[Reject]] 函数对象 用来拒绝给定promise对象的函数
25.4.1.1.1 IfAbruptRejectPromise ( value, capability )

IfAbrupt Reject Promise是使用Promise Capability记录的一系列算法步骤的简写。
这种形式的的算法步骤：IfAbruptRejectPromise(value, capability)
意思是

  1. 如果value是非常规完结
    1. 令rejectResult为Call(capability.[[Reject]], undefined, «value.[[value]]»)
    2. ReturnIfAbrupt(rejectResult)
    3. Return capability.[[Promise]]
  2. 否则如果value是完结记录，令value为value.[[value]]

25.4.1.2 PromiseReaction记录

PromiseReaction是一个Record值，用于存储有关promise在使用给定值确定或拒绝时应如何反应的信息。 PromiseReaction记录是通过Promise原型的then方法创建的，并由PromiseReactionJob使用。

PromiseReaction记录有表58列出的字段：
表58：PromiseReaction记录字段

字段名 值 含义
[[Capabilities]] 一个PromiseCapability记录 该记录提供响应处理程序的promise的功能。
[[Handler]] 函数对象或字符串 应该应用于incoming值的函数，其返回值将控制派生的promise发生了什么。如果[[Handler]]为"Identity"，则等价于仅返回其第一个参数的函数。如果[[Handler]]为"Thrower"，则等价于将第一个参数作为异常抛出的函数。

25.4.1.3 CreateResolvingFunctions ( promise )

当使用参数promise执行CreateResolvingFunctions时，将执行以下步骤：

  1. 令alreadyResolved为一个新的记录{ [[value]]: false }
  2. 令resolve为Promise Resolve Functions(25.4.1.3.2)中定义的内置函数对象
  3. 将resolve的[[Promise]]内部插槽设为promise
  4. 将resolve的[[AlreadyResolved]]内部插槽设为alreadyResolved
  5. 令reject为Promise Reject Functions (25.4.1.3.1)中定义的内置函数对象
  6. 将reject的[[Promise]]内部插槽设为promise
  7. 将reject的[[AlreadyResolved]]内部插槽设为alreadyResolved
  8. 返回一个新记录{ [[Resolve]]: resolve, [[Reject]]: reject }
25.4.1.3.1 Promise Reject函数

promise reject函数是有[[Promise]]和[[AlreadyResolved]]内部插槽的匿名内置函数。
当使用参数reason调用promise reject函数F时，执行以下步骤：

  1. 断言：F有一个值为Object类型的[[Promise]]内部插槽
  2. 令promise为F的[[Promise]]内部插槽的值
  3. 令alreadyResolved为F的[[AlreadyResolved]]内部插槽的值
  4. 如果alreadyResolved.[[value]]为true，返回undefined
  5. 将alreadyResolved.[[value]]设为true
  6. 返回RejectPromise(promise, reason)

promise reject函数的length属性值为1

25.4.1.3.2 Promise Resolve函数

promise resolve函数是有[[Promise]]和[[AlreadyResolved]]内部插槽的匿名内置函数。
当使用参数resolution调用promise resolve函数F时，执行以下步骤：

  1. 断言：F有一个值为Object类型的[[Promise]]内部插槽
  2. 令promise为F的[[Promise]]内部插槽的值
  3. 令alreadyResolved为F的[[AlreadyResolved]]内部插槽的值
  4. 如果alreadyResolved.[[value]]为true，返回undefined
  5. 将alreadyResolved.[[value]]设为true
  6. 如果SameValue(resolution, promise)为true，则：
    1. 令selfResolutionError为新创建的 TypeError 对象
    2. 返回RejectPromise(promise, selfResolutionError)
  7. 如果Type(resolution)不是Object，则返回FulfillPromise(promise, resolution)
  8. 令then为Get(resolution, "then")
  9. 如果then是非常规完结，则返回RejectPromise(promise, then.[[value]])
  10. 令thenAction为then.[[value]]
  11. 如果IsCallable(thenAction)为false，则返回FulfillPromise(promise, resolution)
  12. 执行EnqueueJob ("PromiseJobs", PromiseResolveThenableJob, «‍promise, resolution, thenAction»)
  13. 返回undefined

promise resolve函数的length属性值为1

25.4.1.4 FulfillPromise ( promise, value)

当使用参数promise和value调用FulfillPromise抽象操作时，执行以下步骤：

  1. 断言：promise的[[PromiseState]]内部插槽的值为"pending"
  2. 令reactions为promise的[[PromiseFulfillReactions]]内部插槽的值
  3. 将promise的[[PromiseResult]]内部插槽的值设为value
  4. 将promise的[[PromiseFulfillReactions]]内部插槽的值设为undefined
  5. 将promise的[[PromiseRejectReactions]]内部插槽的值设为undefined
  6. 将promise的[[PromiseState]]内部插槽的值设为"fulfilled"
  7. 返回TriggerPromiseReactions(reactions, value)

25.4.1.5 NewPromiseCapability ( C )

抽象操作NewPromiseCapability接受构造函数，并尝试以内置Promise构造函数的方式使用该构造函数来创建Promise对象并提取其resolve和reject函数。 Promise加上resolve和reject函数用于初始化新的PromiseCapability记录，该记录将作为此抽象操作的值返回。

  1. 如果IsConstructor(C)为false，抛出 TypeError 异常
  2. 注：假定C是支持Promise构造函数的参数约定的构造函数 (见 25.4.3.1).
  3. 令promiseCapability为新的PromiseCapability { [[Promise]]: undefined, [[Resolve]]: undefined, [[Reject]]: undefined }
  4. 令executor为 GetCapabilitiesExecutor Functions (25.4.1.5.1) 中定义的内置函数对象
  5. 将executor的[[Capability]]内部插槽的值设为promiseCapability
  6. 令promise为Construct(C, «executor»)
  7. ReturnIfAbrupt(promise)
  8. 如果IsCallable(promiseCapability.[[Resolve]])为false，抛出 TypeError 异常
  9. 如果IsCallable(promiseCapability.[[Reject]])为false，抛出 TypeError 异常
  10. 将promiseCapability.[[Promise]]设为promise
  11. 返回 promiseCapability

注：此抽象操作支持Promise子类化，因为它在以与Promise构造函数相同的方式调用传递的执行程序函数参数的任何构造函数上都是通用的。 它用于将Promise构造函数的静态方法推广到任何子类。

25.4.1.5.1 GetCapabilitiesExecutor函数

GetCapabilitiesExecutor 函数是有[[Capability]]内部插槽的匿名内置函数。
当使用参数resolve和reject调用GetCapabilitiesExecutor 函数F时，执行以下步骤：

  1. 断言：F有一个值为PromiseCapability记录的[[Capability]]内部插槽
  2. 令promiseCapability为F的[[Capability]]内部插槽的值
  3. 如果promiseCapability.[[Resolve]]不为undefined，抛出 TypeError 异常
  4. 如果promiseCapability.[[Reject]]不为undefined，抛出 TypeError 异常
  5. 将promiseCapability.[[Resolve]]设为resolve
  6. 将promiseCapability.[[Reject]]设为reject
  7. 返回undefined

GetCapabilitiesExecutor函数的length属性值为2

25.4.1.6 IsPromise ( x )

抽象操作IsPromise在一个对象上检查其是否是promise类型

  1. 如果Type(x)不是Object，返回false
  2. 如果x没有[[PromiseState]]内部插槽，返回false
  3. 返回true

25.4.1.7 RejectPromise ( promise, reason)

当使用参数promise和reason调用抽象操作RejectPromise时，执行以下步骤：

  1. 断言：promise的[[PromiseState]]内部插槽的值为"pending"
  2. 令reactions为promise的[[PromiseRejectReactions]]内部插槽的值
  3. 将promise的[[PromiseResult]]内部插槽的值设为reason
  4. 将promise的[[PromiseFulfillReactions]]内部插槽的值设为undefined
  5. 将promise的[[PromiseRejectReactions]]内部插槽的值设为undefined
  6. 将promise的[[PromiseState]]内部插槽的值设为"rejected"
  7. 返回TriggerPromiseReactions(reactions, reason)

25.4.1.8 TriggerPromiseReactions ( reactions, argument )

抽象操作TriggerPromiseReactions接受PromiseReactionRecords的集合，并为每个记录入队一个新的Job。 每个这样的Job都会处理PromiseReactionRecord的[[Handler]]，如果[[Handler]]是一个函数，则调用它传递给定的参数。

  1. 对reactions中的每个reaction，按照原始插入顺序，执行：
    1. 执行EnqueueJob("PromiseJobs", PromiseReactionJob, «‍reaction, argument»)
  2. 返回undefined

25.4.2 Promise Jobs

25.4.2.1 PromiseReactionJob ( reaction, argument )

接受参数reaction和argument的任务PromiseReactionJob将适当的处理程序应用于incoming值，并使用该处理程序的返回值来确定或拒绝与该处理程序相关联的派生promise。

  1. 断言：reaction是PromiseReaction记录
  2. 令promiseCapability为reaction.[[Capabilities]]
  3. 令handler为reaction.[[Handler]]
  4. 如果handler是"Identity"，令handlerResult为NormalCompletion(argument)
  5. 否则如果 handler是"Thrower"，令handlerResult为Completion{[[type]]: throw, [[value]]: argument, [[target]]: empty}
  6. 否则，令handlerResult为Call(handler, undefined, «argument»)
  7. 如果handlerResult是非常规完结，则
    1. 令status为Call(promiseCapability.[[Reject]], undefined, «handlerResult.[[value]]»)
    2. NextJob Completion(status)
  8. 令status为Call(promiseCapability.[[Resolve]], undefined, «handlerResult.[[value]]»)
  9. NextJob Completion(status)

PromiseResolveThenableJob ( promiseToResolve, thenable, then)

接受参数promiseToResolve、thenable和then的任务PromiseResolveThenableJob执行以下步骤：

  1. 令resolvingFunctions为CreateResolvingFunctions(promiseToResolve)
  2. 令thenCallResult为Call(then, thenable, «resolvingFunctions.[[Resolve]], resolvingFunctions.[[Reject]]»)
  3. 如果thenCallResult是非常规完结
    1. 令status为Call(resolvingFunctions.[[Reject]], undefined, «thenCallResult.[[value]]»)
    2. NextJob Completion(status)
  4. NextJob Completion(thenCallResult)

注：此任务使用提供的thenable及其then方法来确定给定的promise。 此过程必须作为任务进行，以确保对所有周围代码的计算完成后，再对then方法进行计算。

25.4.3 Promise构造函数

Promise构造函数是固有对象%Promise%，也是全局对象的Promise属性的初始值。 当作为构造函数调用时，它将创建并初始化一个新的Promise对象。 Promise不能作为函数调用，以这种方式调用时会抛出异常。

Promise构造函数被设计为可子类化的。 它可以用作类定义的extends子句中的值。 打算继承指定Promise行为的子类构造函数必须包括对Promise构造函数的super调用，以便使用支持Promise和Promise.prototype内置方法所必需的内部状态来创建和初始化子类实例。

25.4.3.1 Promise ( executor )

使用参数executor来调用函数Promise时，执行以下步骤：

  1. 如果NewTarget为undefined，抛出 TypeError 异常
  2. 如果IsCallable(executor)为false，抛出 TypeError 异常
  3. 令promise为OrdinaryCreateFromConstructor(NewTarget, "%PromisePrototype%", «‍[[PromiseState]], [[PromiseResult]], [[PromiseFulfillReactions]], [[PromiseRejectReactions]]» )
  4. ReturnIfAbrupt(promise)
  5. 将promise的[[PromiseState]]内部插槽的值设为"pending"
  6. 将promise的[[PromiseFulfillReactions]]内部插槽的值设为一个新的空列表
  7. 将promise的[[PromiseRejectReactions]]内部插槽的值设为一个新的空列表
  8. 令resolvingFunctions为CreateResolvingFunctions(promise)
  9. 令completion为Call(executor, undefined, «resolvingFunctions.[[Resolve]], resolvingFunctions.[[Reject]]»)
  10. 如果completion是非常规完结，则
    1. 令status为Call(resolvingFunctions.[[Reject]], undefined, «completion.[[value]]»)
    2. ReturnIfAbrupt(status)
  11. 返回promise

注：executor参数必须是一个函数对象。 调用该函数以初始化和报告此Promise对象表示的可能延迟的操作的完成。 executor接受两个参数：resolve和reject。 这两个函数被executor函数使用，以便报告延迟计算将来的完成或者失败。从executor函数返回并不意味着延迟的动作已经完成，它仅表示接受了一个请求——未来会执行该延迟动作。

传递给executor函数的resolve函数接受一个参数。executor函数的代码最终可能会调用resolve函数来指示它希望解决关联的Promise对象。传递给resolve函数的参数表示延迟动作的最终值，可以是实际的兑现值，也可以是另一个Promise对象（当它兑现时则提供该值）

传递给executor函数的reject函数接受一个参数。executor函数的代码最终可能会调用reject函数来指示关联的Promise对象被拒绝了，并且永远不会兑现。传递给reject函数的参数用作promise的拒绝值，通常是一个Error对象。

Promise构造函数传递给executor函数的resolve和reject函数具有实际确定和拒绝关联的promise的能力。 子类可能具有不同的构造函数行为，这些构造函数将传入自定义值以进行确定和拒绝。

25.4.4 Promise构造函数的属性

Promise构造函数的[[Prototype]]内部插槽的值是固有对象%FunctionPrototype% (19.2.3)

除了值为1的length属性，Promise构造函数还有以下属性：

25.4.4.1 Promise.all ( iterable )

all函数返回一个新的promise，这个新的promise使用传递的若干promise组成的兑现值数组来兑现，又或者使用传递的若干promise中第一个拒绝的promise的reason来拒绝。当运行此算法时，它将传递的可迭代对象的所有元素解析为Promise。

  1. 令C为this值
  2. 如果Type(C)不为Object，抛出 TypeError 异常
  3. 令S为Get(C, @@species)
  4. ReturnIfAbrupt(S)
  5. 如果S既不是undefined也不是null，令C为S
  6. 令promiseCapability为NewPromiseCapability(C)
  7. ReturnIfAbrupt(promiseCapability)
  8. 令iterator为GetIterator(iterable)
  9. IfAbruptRejectPromise(iterator, promiseCapability)
  10. 令iteratorRecord为Record {[[iterator]]: iterator, [[done]]: false}
  11. 令result为PerformPromiseAll(iteratorRecord, C, promiseCapability)
  12. 如果result是非常规完结
    1. 如果iteratorRecord.[[done]]为false，令result为IteratorClose(iterator, result)
    2. IfAbruptRejectPromise(result, promiseCapability)
  13. 返回Completion(result)

注：all函数要求其this值是支持Promise构造函数的参数约定的构造函数。

25.4.4.1.1 运行时语义:PerformPromiseAll( iteratorRecord, constructor, resultCapability)

当使用参数iteratorRecord、constructor和resultCapability调用抽象操作PerformPromiseAll时，执行以下步骤：

  1. 断言：constructor是构造函数
  2. 断言：resultCapability是PromiseCapability记录
  3. 令values为一个新的空列表
  4. 令remainingElementsCount为新的记录Record { [[value]]: 1 }
  5. 令index为0
  6. 重复：
    1. 令next为IteratorStep(iteratorRecord.[[iterator]])
    2. 如果next是非常规完结，将iteratorRecord.[[done]]设为true
    3. ReturnIfAbrupt(next)
    4. 如果next为false
      1. 将iteratorRecord.[[done]]设为true
      2. 将remainingElementsCount.[[value]]设为remainingElementsCount.[[value]] − 1
      3. 如果remainingElementsCount.[[value]]为0
        1. 令valuesArray为CreateArrayFromList(values)
        2. 令resolveResult为Call(resultCapability.[[Resolve]], undefined, «valuesArray»)
        3. ReturnIfAbrupt(resolveResult)
      4. 返回resultCapability.[[Promise]]
    5. 令nextValue为IteratorValue(next)
    6. 如果nextValue是非常规完结，将iteratorRecord.[[done]]设为true
    7. ReturnIfAbrupt(nextValue)
    8. 将undefined添加到values
    9. 令nextPromise为Invoke(constructor, "resolve", «‍nextValue»). ReturnIfAbrupt(nextPromise )
    10. 令resolveElement为 Promise.all Resolve Element Functions中定义的内置函数
    11. 将resolveElement的[[AlreadyCalled]]内部插槽设为Record {[[value]]: false }
    12. 将resolveElement的[[Index]]内部插槽设为index
    13. 将resolveElement的[[Values]]内部插槽设为values
    14. 将resolveElement的[[Capabilities]]内部插槽设为resultCapability
    15. 将resolveElement的[[RemainingElements]]内部插槽设为remainingElementsCount
    16. 将remainingElementsCount.[[value]]设为remainingElementsCount.[[value]] + 1
    17. 令result为Invoke(nextPromise, "then", «‍resolveElement, resultCapability.[[Reject]]»)
    18. ReturnIfAbrupt(result)
    19. 将index设为index + 1
25.4.4.1.2 Promise.all Resolve Element Functions

Promise.all resolve element 函数是匿名内置函数，用来解决特定的 Promise.all 元素。每个 Promise.all resolve element 函数有[[Index]], [[Values]], [[Capabilities]], [[RemainingElements]], 和 [[AlreadyCalled]]内部插槽。

当使用参数x调用Promise.all resolve element函数F时，执行以下步骤：

  1. 令alreadyCalled为F的[[AlreadyCalled]]内部插槽的值
  2. 如果alreadyCalled.[[value]]为true，返回undefined
  3. 将alreadyCalled.[[value]]设为true
  4. 令index为F的[[Index]]内部插槽的值
  5. 令values为F的[[Values]]内部插槽的值
  6. 令promiseCapability为F的[[Capabilities]]内部插槽的值
  7. 令remainingElementsCount为F的[[RemainingElements]]内部插槽的值
  8. 将values[index]设为x
  9. 将remainingElementsCount.[[value]]设为remainingElementsCount.[[value]] - 1
  10. 如果remainingElementsCount.[[value]]为0
    1. 令valuesArray为CreateArrayFromList(values)
    2. 返回Call(promiseCapability.[[Resolve]], undefined, «valuesArray»)
  11. 返回undefined

Promise.all resolve element 函数的length属性值为1

25.4.4.2 Promise.prototype

Promise.prototype的初始值是固有对象%PromisePrototype%（25.4.5）

该属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

25.4.4.3 Promise.race ( iterable )

race函数返回一个新的promise，该新promise以和传递的第一个确定的Promise相同的方式确定。当运行此算法时，它将传递的可迭代对象的所有元素确定为Promise。

  1. 令C为this值
  2. 如果Type(C)不是Object，抛出 TypeError 异常
  3. 令S为Get(C, @@species)
  4. ReturnIfAbrupt(S)
  5. 如果S既不是undefined也不是null，令C为S
  6. 令promiseCapability为NewPromiseCapability(C)
  7. ReturnIfAbrupt(promiseCapability)
  8. 令iterator为GetIterator(iterable)
  9. IfAbruptRejectPromise(iterator, promiseCapability)
  10. 令iteratorRecord为Record {[[iterator]]: iterator, [[done]]: false}
  11. 令result为PerformPromiseRace(iteratorRecord, promiseCapability, C)
  12. 如果result是非常规完结，则：
    1. 如果iteratorRecord.[[done]]为false，令result为IteratorClose(iterator,result)
    2. IfAbruptRejectPromise(result, promiseCapability)
  13. 返回Completion(result)

注1：如果参数iterable为空，或者iterable中的所有promise均未确定，则此方法返回的待定的promise将永远不会被确定。
注2：race 函数期望其this值是一个支持Promise构造函数的参数约定的构造函数。它也期望它的this值提供resolve方法。

25.4.4.3.1 运行时语义:PerformPromiseRace ( iteratorRecord, promiseCapability, C )

当使用参数iteratorRecord，promiseCapability和C调用抽象操作PerformPromiseRace时，执行以下步骤：

  1. 重复：
    1. 令next为IteratorStep(iteratorRecord.[[iterator]])
    2. 如果next是非常规完结，将iteratorRecord.[[done]]设为true
    3. ReturnIfAbrupt(next)
    4. 如果next为false，则
      1. 将iteratorRecord.[[done]]设为true
      2. 返回promiseCapability.[[Promise]]
    5. 令nextValue为IteratorValue(next)
    6. 如果nextValue是非常规完结，将iteratorRecord.[[done]]设为true
    7. ReturnIfAbrupt(nextValue)
    8. 令nextPromise为Invoke(C, "resolve", «‍nextValue»)
    9. ReturnIfAbrupt(nextPromise)
    10. 令result为Invoke(nextPromise, "then", «‍promiseCapability.[[Resolve]], promiseCapability.[[Reject]]»)
    11. ReturnIfAbrupt(result)

25.4.4.4 Promise.reject ( r )

reject函数返回一个新的promise，该promise被传递的参数拒绝。

  1. 令C为this值
  2. 如果Type(C)不是Object，抛出 TypeError 异常
  3. 令promiseCapability为NewPromiseCapability(C)
  4. ReturnIfAbrupt(promiseCapability)
  5. 令rejectResult为Call(promiseCapability.[[Reject]], undefined, «r»)
  6. ReturnIfAbrupt(rejectResult)
  7. 返回promiseCapability.[[Promise]]

注：reject函数期望其this值是一个支持Promise构造函数的参数约定的构造函数

25.4.4.5 Promise.resolve ( x )

resolve函数返回使用传递的参数解决的新promise，或者，如果参数是此构造函数产生的promise，则返回参数本身。

  1. 令C为this值
  2. 如果Type(C)不为Object，抛出 TypeError 异常
  3. 如果IsPromise(x)为true：
    1. 令xConstructor为Get(x, "constructor")
    2. ReturnIfAbrupt(xConstructor)
    3. 如果SameValue(xConstructor, C)为true，返回x
  4. 令promiseCapability为NewPromiseCapability(C)
  5. ReturnIfAbrupt(promiseCapability)
  6. 令resolveResult为Call(promiseCapability.[[Resolve]], undefined, «x»)
  7. ReturnIfAbrupt(resolveResult)
  8. 返回promiseCapability.[[Promise]]

注：resolve 函数期望其this值是一个支持Promise构造函数的参数约定的构造函数

25.4.4.6 get Promise [ @@species ]

Promise[@@species]是一个访问器属性，其set访问器函数为undefined，它的get访问器函数执行以下步骤：

  1. 返回this值

此函数的name属性值为"get [Symbol.species]"

注：Promise原型方法通常使用其this对象的构造函数来创建派生对象。 但是，子类构造函数可以通过重新定义其@@species属性来覆盖该默认行为。

25.4.5 Promise原型对象的属性

Promise原型对象是固有对象%PromisePrototype%。Promise原型对象的[[Prototype]]内部插槽的值是固有对象%ObjectPrototype% (19.1.3)。Promise原型对象是一个普通对象。 它没有[[PromiseState]]内部插槽或Promise实例的任何其他内部插槽。

25.4.5.1 Promise.prototype.catch ( onRejected )

当使用参数onRejected调用catch方法时，执行以下步骤：

  1. 令promise为this值
  2. 返回Invoke(promise, "then", «‍undefined, onRejected»)

25.4.5.2 Promise.prototype.constructor

Promise.prototype.constructor的初始值是固有对象%Promise%

25.4.5.3 Promise.prototype.then ( onFulfilled , onRejected )

当使用参数onFulfilled和onRejected调用then方法时，执行以下步骤：

  1. 令promise为this值
  2. 如果IsPromise(promise)为false，抛出 TypeError 异常
  3. 令 C 为SpeciesConstructor(promise, %Promise%)
  4. ReturnIfAbrupt(C)
  5. 令resultCapability为NewPromiseCapability(C)
  6. ReturnIfAbrupt(resultCapability)
  7. 返回PerformPromiseThen(promise, onFulfilled, onRejected, resultCapability)
25.4.5.3.1 PerformPromiseThen ( promise, onFulfilled, onRejected, resultCapability )

抽象操作PerformPromiseThen使用onFulfilled和onRejected作为确定操作在promise上执行“ then”操作。 结果是resultCapability的promise。

  1. 断言：IsPromise(promise)为true
  2. 断言：resultCapability是PromiseCapability记录
  3. 如果IsCallable(onFulfilled)为false，则令onFulfilled为"Identity"
  4. 如果IsCallable(onRejected)为false，则令onRejected为"Thrower"
  5. 令fulfillReaction为PromiseReaction { [[Capabilities]]: resultCapability, [[Handler]]: onFulfilled }
  6. 令rejectReaction为PromiseReaction { [[Capabilities]]: resultCapability, [[Handler]]: onRejected}
  7. 如果promise的[[PromiseState]]内部插槽的值为"pending"：
    1. 将fulfillReaction添加到promise的[[PromiseFulfillReactions]]内部插槽（值为一个列表）的最后一个元素
    2. 将rejectReaction添加到promise的[[PromiseRejectReactions]]内部插槽（值为一个列表）的最后一个元素
  8. 否则如果promise的[[PromiseState]]内部插槽的值为"fulfilled"，则
    1. 令value为promise的[[PromiseResult]]内部插槽的值
    2. 执行EnqueueJob("PromiseJobs", PromiseReactionJob, «‍fulfillReaction, value»)
  9. 否则如果promise的[[PromiseState]]内部插槽的值为"rejected"，则
    1. 令reason为promise的[[PromiseResult]]内部插槽的值
    2. 执行EnqueueJob("PromiseJobs", PromiseReactionJob, «‍rejectReaction, reason»)
  10. 返回resultCapability.[[Promise]]

25.4.5.4 Promise.prototype [ @@toStringTag ]

@@toStringTag属性的初始值为字符串"Promise"
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

25.4.6 Promise实例的属性

Promise实例是从Promise原型对象（固有对象%PromisePrototype%）继承属性的普通对象。 最初使用表59中描述的内部插槽创建Promise实例。

表59：Promise实例的内部插槽

内部插槽 描述
[[PromiseState]] 一个字符串值，用于控制promise将如何响应其then方法的传入调用。可能的值为"pending"、"fulfilled"和"rejected"
[[PromiseResult]] promise的确定或拒绝值（如果有）。只在[[PromiseState]]不为pending时有意义。
[[PromiseFulfillReactions]] 一个PromiseReaction记录列表，当promise从"pending"状态转换为"fulfilled"状态时进行处理
[[PromiseRejectReactions]] 一个PromiseReaction记录列表，当promise从"pending"状态转换为"rejected"状态时进行处理
