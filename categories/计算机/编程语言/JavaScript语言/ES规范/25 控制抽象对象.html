25.1 迭代

25.1.1 公共迭代接口

接口是一组属性键，其对应的值符合特定的规范。 提供了接口规范描述的全部属性的对象，称为此对象符合该接口。一个接口不是由一个单独的对象来表示的。可能有许多单独实现的对象符合任何接口。一个单独的对象可能符合多个接口。

25.1.1.1 Iterable接口

Iterable接口包含表65中描述的属性：

表65：Iterable接口必需的属性

属性 值 要求
@@iterator 返回Iterator对象的函数。 返回的对象必须符合Iterator接口。

25.1.1.2 Iterator接口

实现Iterator接口的对象必须包括表66中的属性。此类对象还可以实现表67中的属性。

表66：Iterator接口必需的属性

属性 值 要求
next 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 如果对Iterator的next方法的前一次调用返回了done属性为true的IteratorResult对象，则对该对象的next方法的所有后续调用也应返回其done属性为true的IteratorResult对象。 但是，此要求不是强制的。

注：可以将参数传递给next函数，但是参数的解释和有效性取决于目标Iterator。 for-of语句和Iterators的其他普通用户不传递任何参数，因此，期望以这种方式使用的Iterator对象必须准备好应对不带参数的调用。

表67：Iterator接口可选的属性

属性 值 要求
return 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 调用此方法会通知Iterator对象，调用者无意再对Iterator进行next方法调用。 返回的IteratorResult对象通常具有一个done属性（值为true）和一个value属性（值为传递给return方法的参数）。 但是，此要求不是强制的。
throw 返回IteratorResult对象的函数。 返回的对象必须符合IteratorResult接口。 调用此方法会通知Iterator对象调用者已检测到错误情况。 参数可用于标识错误条件，并且通常是异常对象。 典型的响应是throw作为参数传递的值。 如果该方法没有throw，则返回的IteratorResult对象通常将具有一个值为true的done属性。

注2：通常，这些方法的调用者应在调用它们之前检查它们的存在。 某些ECMAScript语言功能（包括for-of，yield *和数组解构）在执行存在性检查后调用这些方法。 大多数接受Iterable对象作为参数的ECMAScript库函数也会有条件地调用它们。

25.1.1.3 IteratorResult接口

IteratorResult接口包含表55中列出的属性：
表55：IteratorResult接口属性

属性 值 要求
done true或false 这是迭代器next方法调用的结果状态。 如果到达迭代器的终点，则done为true。 如果未达到终点，则done为false，并且可以使用值。 如果done属性（自有的或继承的）不存在，则认为该属性的值为false。
value 任何ECMAScript语言值 如果done为false，则此值为当前迭代元素值。 如果done为true，则此值为迭代器的返回值（如果提供了）。 如果迭代器没有返回值，则value为undefined。 在那种情况下，如果符合属性的对象没有继承显式的value属性，则可能缺少value属性。

25.1.2 %IteratorPrototype%对象

%IteratorPrototype%对象：

  • 具有[[Prototype]]内部槽，且其值为%Object.prototype%
  • 是普通对象

本规范中定义的所有实现Iterator接口的对象也都继承自%IteratorPrototype%。ECMAScript代码也可以定义继承自%IteratorPrototype%的对象（但不能通过类定义的extends子句，因为%IteratorPrototype%不是构造函数）。%IteratorPrototype%对象提供了一个可以添加适用于所有迭代器对象的附加方法的地方。

以下表达式是ECMAScript代码访问%IteratorPrototype%对象的一种方式：
Object.getPrototypeOf(Object.getPrototypeOf([][Symbol.iterator]()))

25.2 生成器函数对象

生成器函数(GeneratorFunction)对象是通常通过计算产生式GeneratorDeclarations、GeneratorExpressions和GeneratorMethods创建的函数。它们也可以通过调用固有的%GeneratorFunction%来创建。

图5：生成器对象的关系

25.2.1 生成器函数的构造函数

生成器函数的构造函数：

  • 是固有对象%GeneratorFunction%
  • 当作为函数而不是构造函数调用时，将创建并初始化一个新的GeneratorFunction对象。因此，函数调用 GeneratorFunction (...) 等同于使用相同参数的对象创建表达式 new GeneratorFunction (...)
  • 被设计为可被子类化。它可以被用作类定义中的extends子句的值。打算继承指定的GeneratorFunction行为的子类构造函数必须包括对GeneratorFunction构造函数的super调用，以便用GeneratorFunction内置行为所需的内部槽来创建和初始化子类实例。所有用于定义生成器函数对象的ECMAScript语法形式都直接创建GeneratorFunction的实例。没有任何语法方法可以创建GeneratorFunction子类的实例。

25.2.1.1 GeneratorFunction ( p1, p2, … , pn, body )

最后一个参数指定了生成器函数的主体（可执行代码）；前面的所有参数都指定了形式参数。

当用参数p1, p2, …, pn, body（其中n可能是0，即没有 "p"参数，而且body也可能没有提供）调用GeneratorFunction函数时，会执行以下步骤：

  1. 令C为激活的函数对象
  2. 令args为通过[[Call]]或[[Construct]]传递给此函数的参数列表(argumentsList)
  3. 返回? CreateDynamicFunction(C, NewTarget, generator, args)

注：见 19.2.1.1 的注

25.4 生成器对象

一个生成器(Generator)对象是一个生成器函数的实例，它实现了Iterator和Iterable接口。
生成器实例直接从创建该实例的生成器函数的"prototype"属性值的对象继承属性。生成器实例间接地从生成器原型的固有属性%Generator.prototype%继承属性。

  • 是%Generator%（即GeneratorFunction.prototype）的"prototype"属性的初始值
  • 是普通对象
  • 不是生成器实例，没有[[GeneratorState]]内部槽

  • 具有被所有生成器实例间接继承的属性

25.4.3 生成器抽象操作

25.4.3.1 GeneratorStart ( generator, generatorBody )

接受参数generator和generatorBody的抽象操作GeneratorStart执行以下步骤：

  1. 断言：generator.[[GeneratorState]]的值为undefined
  2. 令genContext为运行中的执行上下文
  3. 将genContext的Generator组件设为generator
  4. 设置genContext的代码计算状态，使得恢复对该执行上下文的计算时，执行以下步骤：
    1. 令result为计算generatorBody的结果
    2. 断言：如果在这里返回，则generator要么抛出异常，要么执行一个隐式或显式的return
    3. 从执行上下文栈中删除genContext，并且将执行上下文栈的栈顶恢复成运行中的执行上下文
    4. 将generator.[[GeneratorState]]设为completed
    5. 一旦生成器进入completed状态，它就永远不会离开，其相关的执行环境也不会恢复。任何与generator相关的执行状态在此时都可以被丢弃。
    6. 如果result.[[Type]]为normal，令resultValue为undefined
    7. 否则如果result.[[Type]]为return，令resultValue为result.[[Value]]
    8. 否则：
      1. 断言：result.[[Type]]为throw
      2. 返回Completion(result)
    9. 返回CreateIterResultObject(resultValue, true)
  5. 将generator.[[GeneratorContext]]设为genContext
  6. 将generator.[[GeneratorState]]设为suspendedStart
  7. 返回NormalCompletion(undefined)

25.4.3.2 GeneratorValidate ( generator )

抽象操作GeneratorValidate接受参数generator并执行以下步骤：

  1. 执行? RequireInternalSlot(generator, [[GeneratorState]])
  2. 断言：generator也具有[[GeneratorContext]]内部槽
  3. 令state为generator.[[GeneratorState]]
  4. 如果state为executing，抛出 TypeError 异常
  5. 返回state

25.4.3.3 GeneratorResume ( generator, value )

抽象操作GeneratorResume接受参数generator和value并执行以下步骤：

  1. 令state为? GeneratorValidate(generator)
  2. 如果state为completed，返回CreateIterResultObject(undefined, true)
  3. 断言：state要么是suspendedStart，要么是suspendedYield
  4. 令genContext为generator.[[GeneratorContext]]
  5. 令methodContext为运行中的执行上下文
  6. 暂停methodContext
  7. 将generator.[[GeneratorState]]设为executing
  8. 将genContext推送到执行上下文栈。genContext现在是运行中的执行上下文
  9. 恢复暂停的genContext的计算，使用NormalCompletion(value)作为暂停该计算的操作的结果。令result为恢复的计算所返回的完结记录
  10. 断言：当在这里返回时，genContext已经被从执行上下文栈移除了，同时methodContext现在是运行中的执行上下文
  11. 返回Completion(result)

25.4.3.4 GeneratorResumeAbrupt ( generator, abruptCompletion )

抽象操作GeneratorResumeAbrupt接受参数generator和abruptCompletion并执行以下步骤：

  1. 令state为? GeneratorValidate(generator)
  2. 如果state为suspendedStart，则：
    1. 将generator.[[GeneratorState]]设为completed
    2. 一旦生成器进入completed状态，它就永远不会离开，其相关的执行环境也不会恢复。任何与generator相关的执行状态在此时都可以被丢弃。
    3. 将state设为completed
  3. 如果state为completed，则：
    1. 如果abruptCompletion.[[Type]]为return，则返回CreateIterResultObject(abruptCompletion.[[Value]], true)
    2. 返回Completion(abruptCompletion)
  4. 断言：state是suspendedYield
  5. 令genContext为generator.[[GeneratorContext]]
  6. 令methodContext为运行中的执行上下文
  7. 暂停methodContext
  8. 将generator.[[GeneratorState]]设为executing
  9. 将genContext推送到执行上下文栈。genContext现在是运行中的执行上下文
  10. 恢复暂停的genContext的计算，使用abruptCompletion作为暂停该计算的操作的结果。令result为恢复的计算所返回的完结记录。
  11. 断言：当在这里返回时，genContext已经被从执行上下文栈移除了，同时methodContext现在是运行中的执行上下文
  12. 返回Completion(result)

25.4.3.5 GetGeneratorKind ( )

  1. 令genContext为运行中的执行上下文
  2. 如果genContext没有Generator组件，返回non-generator
  3. 令generator为genContext的Generator组件
  4. 如果generator有[[AsyncGeneratorState]]内部槽，返回async
  5. 否则，返回sync

25.4.3.6 GeneratorYield ( iterNextObj )

抽象操作GeneratorYield接受参数iterNextObj并执行以下步骤：

  1. 断言：iterNextObj是一个实现了IteratorResult接口的对象
  2. 令genContext为运行中的执行上下文
  3. 断言：genContext是一个generator的执行上下文

25.6 Promise对象

Promise是一个对象，它被用作延迟（可能还是异步）计算的最终结果的占位符。

任何Promise对象均处于以下三种互斥状态之一：已兑现(fulfilled)，已拒绝(rejected)和待定(pending)：

  • 如果p.then(f, r)立即使Job入队来调用函数f，则Promisep是已兑现的
  • 如果p.then(f, r)立即使Job入队来调用函数r，则Promisep是已拒绝的
  • 如果Promise既不是已兑现的，也不是已解决的，则它是待定的

如果Promise不是待定的，则被称为确定的(settled)，也就是说它要么已兑现要么已拒绝

如果一个Promise是确定的，或者已被“锁定(locked in)”以匹配另一个Promise的状态，则该Promise是已解决的(resolved)。尝试resolve（解决）或reject（拒绝）一个已解决的Promise没有任何效果。如果Promise不是已解决的，则它是未解决的。一个未解决的Promises总是待定状态。一个已解决的Promise可能是待定的，也可能是已兑现的，还可能是已拒绝的

25.6.1.9 HostPromiseRejectionTracker ( promise, operation )

HostPromiseRejectionTracker是一个实现定义的抽象操作，它允许主机环境跟踪Promise的拒绝。

HostPromiseRejectionTracker的实现必须在所有情况下正常完结。HostPromiseRejectionTracker的默认实现是无条件地返回一个空的正常完结。

注1：HostPromiseRejectionTracker在两种情况下被调用：

  • 当一个promise在没有任何处理程序的情况下被拒绝时，使用"reject"作为参数operation来调用它。
  • 当一个处理程序第一次被添加到一个被拒绝的promise中时，使用"handle"作为参数operation来调用它。
    HostPromiseRejectionTracker的典型实现可能会尝试通知开发者未处理的拒绝，同时也会小心地通知他们，如果这种先前的通知后来因新的处理程序被附加而失效。

注2：如果operation是 "handle"，实现不应该以干扰垃圾回收的方式持有对promise的引用。如果operation是 "reject"，实现可以持有对promise的引用，因为预计拒绝将是罕见的，并且不在热代码路径上。

25.6.2 Promise Jobs

25.6.2.1 NewPromiseReactionJob ( reaction, argument )

抽象操作NewPromiseReactionJob接受两个参数：reaction和argument，返回一个新的Job抽象闭包，该闭包将适当的处理程序应用于incoming值，并使用处理程序的返回值来解决或拒绝与该处理程序关联的派生promise。它执行以下步骤：

  1. 令job为一个新的Job抽象闭包，该闭包不接受参数，捕获reaction和argument，并在调用时执行以下步骤：
    1. 断言：reaction为一个新的PromiseReaction记录
    2. 令promiseCapability为reaction.[[Capability]]
    3. 令type为reaction.[[Type]]
    4. 令handler为reaction.[[Handler]]
    5. 如果handler为undefined，则：
      1. 如果type是Fulfill，令handlerResult为NormalCompletion(argument)
      2. 否则，
        1. 断言：type为Reject
        2. 令handlerResult为ThrowCompletion(argument)
    6. 否则， 令handlerResult为Call(handler, undefined, « argument »)
    7. 如果promiseCapability为undefined，则：
      1. 断言：handlerResult不是非常规完结
      2. 返回NormalCompletion(empty)
    8. 如果handlerResult是非常规完结，则令status为Call(promiseCapability.[[Reject]], undefined, « handlerResult.[[Value]] »)
    9. 否则，令status为Call(promiseCapability.[[Resolve]], undefined, « handlerResult.[[Value]] »)
    10. 返回Completion(status)
  2. 令handlerRealm为null
  3. 如果reaction.[[Handler]]不是undefined，则：
    1. 令getHandlerRealmResult为GetFunctionRealm(reaction[[Handler]]).
    2. 如果getHandlerRealmResult是常规完结，则将handlerRealm设为getHandlerRealmResult.[[Value]]
  4. 返回记录{ [[Job]]: job, [[Realm]]: handlerRealm }

25.6.2.2 NewPromiseResolveThenableJob ( promiseToResolve, thenable, then)

抽象操作NewPromiseResolveThenableJob接受三个参数：promiseToResolve、thenable和then，并执行以下步骤：

  1. 令job为一个新的Job抽象闭包，该闭包不接受参数，捕获promiseToResolve、thenable和then，并在调用时执行以下步骤：
    1. 令resolvingFunctions为CreateResolvingFunctions(promiseToResolve)
    2. 令thenCallResult为Call(then, thenable, «resolvingFunctions.[[Resolve]], resolvingFunctions.[[Reject]]»)
    3. 如果thenCallResult是非常规完结，则
      1. 令status为Call(resolvingFunctions.[[Reject]], undefined, «thenCallResult.[[value]]»)
      2. 返回Completion(status)
    4. 返回Completion(thenCallResult)
  2. 令getThenRealmResult为GetFunctionRealm(then)
  3. 如果getThenRealmResult是常规完结，则令thenRealm为getThenRealmResult.[[Value]]
  4. 否则，令thenRealm为null
  5. 返回记录{ [[Job]]: job, [[Realm]]: thenRealm }

注：此Job使用提供的thenable及其then方法来解决给定的promise。 此过程必须作为Job进行，以确保对所有周围代码的计算完成后，再对then方法进行计算。

Promise构造函数：

  • 是固有对象%Promise%

  • 当作为构造函数调用时，它将创建并初始化一个新的Promise对象
  • 不能作为函数调用，以这种方式调用时会抛出异常。

使用参数executor来调用函数Promise时，执行以下步骤：

  3. 令promise为OrdinaryCreateFromConstructor(NewTarget, "%PromisePrototype%", «‍[[PromiseState]], [[PromiseResult]], [[PromiseFulfillReactions]], [[PromiseRejectReactions]]» )
  4. 将promise.[[PromiseState]]的值设为pending
  5. 将promise.[[PromiseFulfillReactions]]的值设为一个新的空列表
  6. 将promise.[[PromiseRejectReactions]]的值设为一个新的空列表
  7. 将promise.[[PromiseIsHandled]]的值设为false
  8. 令resolvingFunctions为CreateResolvingFunctions(promise)
  9. 令completion为Call(executor, undefined, «resolvingFunctions.[[Resolve]], resolvingFunctions.[[Reject]]»)

传递给executor函数的resolve函数接受一个参数。executor函数的代码最终可能会调用resolve函数来指示它希望解决关联的Promise对象。传递给resolve函数的参数表示延迟动作的最终值，可以是实际的兑现值，也可以是另一个Promise对象（当它兑现时则提供该值）

传递给executor函数的reject函数接受一个参数。executor函数的代码最终可能会调用reject函数来指示关联的Promise对象被拒绝了，并且永远不会兑现。传递给reject函数的参数用作promise的拒绝值，通常是一个Error对象。

Promise构造函数传递给executor函数的resolve和reject函数具有实际确定和拒绝关联的promise的能力。 子类可能具有不同的构造函数行为，这些构造函数将传入自定义值以进行确定和拒绝。

25.6.4.2 Promise.allSettled ( iterable )

allSettled函数返回一个使用promise状态快照的数组兑现的promise，但只有在所有的原始promise都被确定之后，也就是说，要么被兑现，要么被拒绝。在运行这个算法时，它将传递的可迭代对象的所有元素解析为promise。

  1. 令C为this值
  2. 令promiseCapability为? NewPromiseCapability(C)
  3. 令iteratorRecord为GetIterator(iterable)
  4. IfAbruptRejectPromise(iteratorRecord, promiseCapability)
  5. 令result为PerformPromiseAllSettled(iteratorRecord, C, promiseCapability)
  6. 如果result是非常规完结，则：
    1. 如果iteratorRecord.[[Done]]为false，将result设为IteratorClose(iteratorRecord, result)
    2. IfAbruptRejectPromise(result, promiseCapability)
  7. 返回Completion(result)

注：allSettled函数要求其this值是一个支持Promise构造函数的参数约定的构造函数。

25.6.4.2.1 运行时语义: PerformPromiseAllSettled ( iteratorRecord, constructor, resultCapability )

当使用参数iteratorRecord、constructor、resultCapability调用抽象操作PerformPromiseAllSettled时，执行以下步骤：

  1. 断言：! IsConstructor(constructor)为true
  2. 断言：resultCapability是PromiseCapability记录
  3. 令values为一个新的空列表
  4. 令remainingElementsCount为记录Record { [[value]]: 1 }
  5. 令index为0
  6. 令promiseResolve为? Get(constructor, "resolve")
  7. 如果! IsCallable(promiseResolve)为false，抛出 TypeError 异常
  8. 重复：
    1. 令next为IteratorStep(iteratorRecord)
    2. 如果next是非常规完结，将iteratorRecord.[[Done]]设为true
    3. ReturnIfAbrupt(next)
    4. 如果next为false
      1. 将iteratorRecord.[[Done]]设为true
      2. 将remainingElementsCount.[[Value]]设为remainingElementsCount.[[Value]] − 1
      3. 如果remainingElementsCount.[[Value]]为0，则：
        1. 令valuesArray为! CreateArrayFromList(values)
        2. 执行? Call(resultCapability.[[Resolve]], undefined, «valuesArray»)
      4. 返回resultCapability.[[Promise]]
    5. 令nextValue为IteratorValue(next)
    6. 如果nextValue是非常规完结，将iteratorRecord.[[Done]]设为true
    7. ReturnIfAbrupt(nextValue)
    8. 将undefined添加到values
    9. 令nextPromise为? Call(promiseResolve, constructor, « nextValue »)
    10. 令steps为Promise.allSettled Resolve Element函数 定义的算法步骤
    11. 令resolveElement为! CreateBuiltinFunction(steps, « [[AlreadyCalled]], [[Index]], [[Values]], [[Capability]], [[RemainingElements]] »)
    12. 令alreadyCalled为记录{ [[Value]]: false }
    13. 将resolveElement.[[AlreadyCalled]]设为alreadyCalled
    14. 将resolveElement.[[Index]]设为index
    15. 将resolveElement.[[Values]]设为values
    16. 将resolveElement.[[Capabilities]]设为resultCapability
    17. 将resolveElement.[[RemainingElements]]设为remainingElementsCount
    18. 令rejectSteps为Promise.allSettled Reject Element函数 定义的算法步骤
    19. 令rejectElement为! CreateBuiltinFunction(rejectSteps, « [[AlreadyCalled]], [[Index]], [[Values]], [[Capability]], [[RemainingElements]] »)
    20. 将rejectElement.[[AlreadyCalled]]设为alreadyCalled
    21. 将rejectElement.[[Index]]设为index
    22. 将rejectElement.[[Values]]设为values
    23. 将rejectElement.[[Capabilities]]设为resultCapability
    24. 将rejectElement.[[RemainingElements]]设为remainingElementsCount
    25. 将remainingElementsCount.[[value]]设为remainingElementsCount.[[value]] + 1
    26. 令result为? Invoke(nextPromise, "then", «‍resolveElement, rejectElement»)
    27. 将index设为index + 1
25.6.4.2.2 Promise.allSettled Resolve Element Functions

Promise.allSettled resolve element 函数是匿名内置函数，用来解决特定的 Promise.allSettled 元素。每个 Promise.allSettled resolve element 函数有[[Index]], [[Values]], [[Capabilities]], [[RemainingElements]], 和 [[AlreadyCalled]]内部槽。

当使用参数x调用Promise.allSettled resolve element函数F时，执行以下步骤：

  1. 令F为当前激活的函数对象
  2. 令alreadyCalled为F.[[AlreadyCalled]]
  3. 如果alreadyCalled.[[Value]]为true，返回undefined
  4. 将alreadyCalled.[[Value]]设为true
  5. 令index为F.[[Index]]
  6. 令values为F.[[Values]]
  7. 令promiseCapability为F.[[Capabilities]]
  8. 令remainingElementsCount为F.[[RemainingElements]]
  9. 令obj为! OrdinaryObjectCreate(%Object.prototype%)
  10. 执行! CreateDataPropertyOrThrow(obj, "status", "fulfilled")
  11. 执行! CreateDataPropertyOrThrow(obj, "value", x)
  12. 将values[index]设为obj
  13. 将remainingElementsCount.[[Value]]设为remainingElementsCount.[[Value]] - 1
  14. 如果remainingElementsCount.[[Value]]为0
    1. 令valuesArray为! CreateArrayFromList(values)
    2. 返回? Call(promiseCapability.[[Resolve]], undefined, «valuesArray»)
  15. 返回undefined

Promise.allSettled resolve element 函数的"length"属性值为1

25.6.4.2.3 Promise.allSettled Reject Element Functions

Promise.allSettled reject element 函数是匿名内置函数，用来拒绝特定的 Promise.allSettled 元素。每个 Promise.allSettled reject element 函数有[[Index]], [[Values]], [[Capabilities]], [[RemainingElements]], 和 [[AlreadyCalled]]内部槽。

当使用参数x调用Promise.allSettled reject element函数F时，执行以下步骤：

  1. 令F为当前激活的函数对象
  2. 令alreadyCalled为F.[[AlreadyCalled]]
  3. 如果alreadyCalled.[[Value]]为true，返回undefined
  4. 将alreadyCalled.[[Value]]设为true
  5. 令index为F.[[Index]]
  6. 令values为F.[[Values]]
  7. 令promiseCapability为F.[[Capabilities]]
  8. 令remainingElementsCount为F.[[RemainingElements]]
  9. 令obj为! OrdinaryObjectCreate(%Object.prototype%)
  10. 执行! CreateDataPropertyOrThrow(obj, "status", "rejected")
  11. 执行! CreateDataPropertyOrThrow(obj, "reason", x)
  12. 将values[index]设为obj
  13. 将remainingElementsCount.[[Value]]设为remainingElementsCount.[[Value]] - 1
  14. 如果remainingElementsCount.[[Value]]为0
    1. 令valuesArray为! CreateArrayFromList(values)
    2. 返回? Call(promiseCapability.[[Resolve]], undefined, «valuesArray»)
  15. 返回undefined

25.6.4.4 Promise.race ( iterable )

race函数返回一个新的promise，该新promise以和传递的第一个确定的Promise相同的方式确定。当运行此算法时，它将传递的可迭代对象的所有元素解析为Promise。

  1. 令C为this值
  2. 令promiseCapability为? NewPromiseCapability(C)
  3. 令iteratorRecord为GetIterator(iterable)
  4. IfAbruptRejectPromise(iteratorRecord, promiseCapability)
  5. 令result为PerformPromiseRace(iteratorRecord, C, promiseCapability)
  6. 如果result是非常规完结，则：
    1. 如果iteratorRecord.[[Done]]为false，将result设为IteratorClose(iterator,result)
    2. IfAbruptRejectPromise(result, promiseCapability)
  7. 返回Completion(result)

注1：如果参数iterable为空，或者iterable中的所有promise都不会被确定，则此方法返回的待定的promise将永远不会被确定。
注2：race 函数期望其this值是一个支持Promise构造函数的参数约定的构造函数。它也期望它的this值提供resolve方法。

25.6.4.4.1 运行时语义: PerformPromiseRace ( iteratorRecord, constructor, resultCapability )

当使用参数iteratorRecord，constructor和resultCapability调用抽象操作PerformPromiseRace时，执行以下步骤：

  1. 断言：IsConstructor(constructor)为true
  2. 断言：resultCapability是PromiseCapability记录
  3. 令promiseResolve为? Get(constructor, "resolve")
  4. 如果! IsCallable(promiseResolve)为false，抛出 TypeError 异常
  5. 重复：
    1. 令next为IteratorStep(iteratorRecord)
    2. 如果next是非常规完结，将iteratorRecord.[[Done]]设为true
    3. ReturnIfAbrupt(next)
    4. 如果next为false，则：
      1. 将iteratorRecord.[[Done]]设为true
      2. 返回resultCapability.[[Promise]]
    5. 令nextValue为IteratorValue(next)
    6. 如果nextValue是非常规完结，将iteratorRecord.[[Done]]设为true
    7. ReturnIfAbrupt(nextValue)
    8. 令nextPromise为? Call(promiseResolve, constructor, « nextValue »)
    9. 执行? Invoke(nextPromise, "then", « resultCapability.[[Resolve]], resultCapability.[[Reject]] »)

25.6.4.5 Promise.reject ( r )

reject函数返回一个新的promise，该promise被传递的参数拒绝。

  1. 令C为this值
  2. 令promiseCapability为? NewPromiseCapability(C)
  3. 执行Call(promiseCapability.[[Reject]], undefined, «r»)
  4. 返回promiseCapability.[[Promise]]

此函数是固有对象%Promise_reject%

注：reject 函数期望其this值是一个支持Promise构造函数的参数约定的构造函数

25.6.4.6 Promise.resolve ( x )

resolve函数要么返回一个使用传递的参数解决的新Promise，要么返回参数本身（如果参数是由这个构造函数产生的Promise）

  1. 令C为this值
  2. 如果Type(C)不为Object，抛出 TypeError 异常
  3. 返回? PromiseResolve(C, x)

此函数是固有对象%Promise_resolve%

注：resolve 函数期望其this值是一个支持Promise构造函数的参数约定的构造函数

25.6.4.6.1 PromiseResolve ( C, x )

抽象操作PromiseResolve，接受一个构造函数C、一个值x，返回一个使用x解决的新Promise

  1. 断言：Type(C)是Object
  2. 如果IsPromise(x)为true：
    1. 令xConstructor为? Get(x, "constructor")
    2. 如果SameValue(xConstructor, C)为true，返回x
  3. 令promiseCapability为? NewPromiseCapability(C)
  4. 执行? Call(promiseCapability.[[Resolve]], undefined, «x»)
  5. 返回promiseCapability.[[Promise]]

25.6.4.7 get Promise [ @@species ]

Promise[@@species]是一个访问器属性，其set访问器函数为undefined，它的get访问器函数执行以下步骤：

  1. 返回this值

此函数的name属性值为"get [Symbol.species]"

注：Promise原型方法通常使用其this对象的构造函数来创建派生对象。 但是，子类构造函数可以通过重新定义其@@species属性来覆盖该默认行为。

25.6.5.3 Promise.prototype.finally ( onFinally )

当使用参数onFinally调用finally方法时，执行以下步骤：

  1. 令promise为this值
  2. 如果Type(promise)不是Object，抛出 TypeError 异常
  3. 令C为? SpeciesConstructor(promise, %Promise%)
  4. 断言：IsConstructor(C)为true
  5. 如果IsCallable(onFinally)为false，则：
    1. 令thenFinally为onFinally
    2. 令catchFinally为onFinally
  6. 否则：
    1. 令stepsThenFinally为 Then Finally Functions 中定义的算法步骤
    2. 令thenFinally为! CreateBuiltinFunction(stepsThenFinally, « [[Constructor]], [[OnFinally]] »)
    3. 将thenFinally.[[Constructor]]设为C
    4. 将thenFinally.[[OnFinally]]设为onFinally
    5. 令stepsCatchFinally为 Catch Finally Functions 中定义的算法步骤
    6. 令catchFinally为! CreateBuiltinFunction(stepsCatchFinally, « [[Constructor]], [[OnFinally]] »)
    7. 将catchFinally.[[Constructor]]设为C
    8. 将catchFinally.[[OnFinally]]设为onFinally
  7. 返回? Invoke(promise, "then", « thenFinally, catchFinally »)
25.6.5.3.1 Then Finally Functions

Then Finally 函数是具有[[Constructor]]内部槽和[[OnFinally]]内部槽的匿名内置函数。[[Constructor]]内部槽的值是一个类Promise构造函数对象，[[OnFinally]]内部槽的值是一个函数对象。

当使用参数value调用 Then Finally 函数时，执行以下步骤：

  1. 令F为当前激活的函数对象
  2. 令onFinally为F.[[OnFinally]]
  3. 断言：IsCallable(onFinally)为true
  4. 令result为? Call(onFinally, undefined)
  5. 令C为F.[[Constructor]]
  6. 断言：IsConstructor(C)为true
  7. 令promise为? PromiseResolve(C, result)
  8. 令valueThunk等同于一个返回value的函数
  9. 返回? Invoke(promise, "then", « valueThunk »)

Then Finally函数的"length"属性为1

25.6.5.3.2 Catch Finally Functions

Catch Finally 函数是具有[[Constructor]]内部槽和[[OnFinally]]内部槽的匿名内置函数。[[Constructor]]内部槽的值是一个类Promise构造函数对象，[[OnFinally]]内部槽的值是一个函数对象。

当使用参数reason调用 Catch Finally 函数时，执行以下步骤：

  1. 令F为当前激活的函数对象
  2. 令onFinally为F.[[OnFinally]]
  3. 断言：IsCallable(onFinally)为true
  4. 令result为? Call(onFinally, undefined)
  5. 令C为F.[[Constructor]]
  6. 断言：IsConstructor(C)为true
  7. 令promise为? PromiseResolve(C, result)
  8. 令thrower等同于一个抛出reason的函数
  9. 返回? Invoke(promise, "then", « thrower »)

25.6.5.4 Promise.prototype.then ( onFulfilled , onRejected )

当使用参数onFulfilled和onRejected调用then方法时，执行以下步骤：

  1. 令promise为this值
  2. 如果IsPromise(promise)为false，抛出 TypeError 异常
  3. 令 C 为? SpeciesConstructor(promise, %Promise%)
  4. 令resultCapability为? NewPromiseCapability(C)
  5. 返回PerformPromiseThen(promise, onFulfilled, onRejected, resultCapability)

此函数是固有对象%PromiseProto_then%

25.6.5.4.1 PerformPromiseThen ( promise, onFulfilled, onRejected, resultCapability )

抽象操作PerformPromiseThen使用onFulfilled和onRejected作为解决动作在promise上执行“ then”操作。如果传递了resultCapability，那么结果将通过更新resultCapability的Promise来存储（如果没有传递，那么PerformPromiseThen就会被一个规范内部的操作所调用，其结果并不重要）

  1. 断言：IsPromise(promise)为true
  2. 如果resultCapability存在，则：
    1. 断言：resultCapability是PromiseCapability记录
  3. 否则：
    1. 将resultCapability设为undefined
  4. 如果IsCallable(onFulfilled)为false，则令onFulfilled为"undefined"
  5. 如果IsCallable(onRejected)为false，则令onRejected为"undefined"
  6. 令fulfillReaction为PromiseReaction { [[Capability]]: resultCapability, [[Type]]: Fulfill, [[Handler]]: onFulfilled }
  7. 令rejectReaction为PromiseReaction { [[Capability]]: resultCapability, [[Type]]: Reject, [[Handler]]: onRejected }
  8. 如果promise.[[PromiseState]]的值为pending，则：
    1. 将fulfillReaction作为最后一个元素添加到列表promise.[[PromiseFulfillReactions]]
    2. 将rejectReaction作为最后一个元素添加到列表promise.[[PromiseRejectReactions]]
  9. 否则如果promise.[[PromiseState]]的值为fulfilled，则
    1. 令value为promise.[[PromiseResult]]
    2. 令fulfillJob为NewPromiseReactionJob(fulfillReaction, value)
    3. 执行HostEnqueuePromiseJob(fulfillJob.[[Job]], fulfillJob.[[Realm]])
  10. 否则：
    1. 断言：promise.[[PromiseState]]的值为rejected
    2. 令reason为promise.[[PromiseResult]]
    3. 如果promise.[[PromiseIsHandled]]为false，执行HostPromiseRejectionTracker(promise, "handle")
    4. 令rejectJob为NewPromiseReactionJob(rejectReaction, reason)
    5. 执行HostEnqueuePromiseJob(rejectJob.[[Job]], rejectJob.[[Realm]])
  11. 如果resultCapability为undefined，则返回undefined
  12. 否则返回resultCapability.[[Promise]]

25.6.6 Promise实例的属性

Promise实例是从Promise原型对象（固有对象%Promise.prototype%）继承属性的普通对象。 Promise实例最初是以表78中描述的内部槽创建的。

表78：Promise实例的内部槽

内部槽 描述
[[PromiseState]] 可能的值为pending、fulfilled和rejected其中之一，用于控制promise将如何响应其将来的then方法的调用。
