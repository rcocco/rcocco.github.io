22.1 Array对象

数组对象是奇异对象，它对某一类属性名进行特殊处理。 有关特殊处理的定义，请参见9.4.2。

22.1.1 Array构造函数

Array构造函数：

  • 是固有对象%Array%
  • 是全局对象的"Array"属性的初始值
  • 作为构造函数调用时创建并初始化一个新的数组奇异对象
  • 当作为函数而不是构造函数调用时也创建并初始化了一个新的数组对象，因此接受相同参数的函数调用Array(…)等价于对象创建表达式new Array(…)
  • 被设计成可以被子类化。它可以被用作类定义的extends子句的值。打算继承奇异数组行为的子类构造函数必须包含对Array构造函数的super调用，以初始化为Array奇异对象的子类实例。然而，大多数Array.prototype方法都是通用方法，不依赖于它们的this值是一个Array奇异对象。
  • 具有"length"属性，其值为1

22.1.1.1 Array ( )

当且仅当Array构造函数被调用时没有参数，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs = 0
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 返回! ArrayCreate(0, proto)

22.1.1.2 Array ( len )

当且仅当Array构造函数接受1个参数，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs = 1
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 如果Type(len)不是Number，则
    1. 执行! CreateDataPropertyOrThrow(array, "0", len)
    2. 令intLen为1
  7. 否则：
    1. 令intLen为ToUint32(len)
    2. 如果intLen ≠ len，抛出 RangeError 异常
  8. 执行! Set(array, "length", intLen, true)
  9. 返回array

22.1.1.3 Array ( …items )

当且仅当Array构造函数接受2个参数及更多参数时，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs ≥ 2
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 令k为0
  7. 令items为起始索引为0的列表，包含按照顺序的参数元素
  8. 重复：只要k < numberOfArgs
    1. 令Pk为! ToString(k)
    2. 令itemK为items[k]
    3. 执行! CreateDataPropertyOrThrow(array, Pk, itemK)
    4. 将k设为k + 1
  9. 断言：array的"length"属性值为numberOfArgs
  10. 返回array

22.1.2.1 Array.from ( items [ , mapfn [ , thisArg ] ] )

当使用参数items以及可选参数mapfn和thisArg调用Array.from

  1. 令C为this值
  2. 如果mapfn为undefined，令mapping为false
  3. 否则：
    1. 如果IsCallable(mapfn)为false，抛出 TypeError 异常
    2. 令mapping为true
  4. 令usingIterator为? GetMethod(items, @@iterator)
  5. 如果usingIterator不是undefined，则
    1. 如果IsConstructor(C)为true，则令A为? Construct(C)
    2. 否则，令A为! ArrayCreate(0)
    3. 令iteratorRecord为? GetIterator(items, sync, usingIterator)
    4. 令k为0
    5. 重复：
      1. 如果k ≥ 2^53 - 1，则：
        1. 令error为ThrowCompletion(一个新创建的TypeError对象)
        2. 返回? IteratorClose(iteratorRecord, error)
      2. 令Pk为! ToString(k)
      3. 令next为? IteratorStep(iteratorRecord)
      4. 如果next为false，则
        1. 执行? Set(A, "length", k, true)
        2. 返回A
      5. 令nextValue为? IteratorValue(next)
      6. 如果mapping为true，则
        1. 令mappedValue为Call(mapfn, thisArg, « nextValue, k »)
        2. 如果mappedValue是非常规完结，返回? IteratorClose(iteratorRecord, mappedValue)
        3. 将mappedValue设为mappedValue.[[Value]]
      7. 否则，令mappedValue为nextValue
      8. 令defineStatus为CreateDataPropertyOrThrow(A, Pk, mappedValue)
      9. 如果defineStatus是非常规完结，返回? IteratorClose(iteratorRecord, defineStatus)
      10. 将k设为k + 1
  6. 注意：items不是可迭代对象，所以假设它是类数组对象
  7. 令arrayLike为! ToObject(items)
  8. 令len为? LengthOfArrayLike(arrayLike)
  9. 如果IsConstructor(C)为true，则令A为? Construct(C, « len »)
  10. 否则令A为? ArrayCreate(len)
  11. 令k为0
  12. 重复，只要k < len
    1. 令Pk为! ToString(k)
    2. 令kValue为? Get(arrayLike, Pk)
    3. 如果mapping为true，则令mappedValue为? Call(mapfn, thisArg, « kValue, k »)
    4. 否则，令mappedValue为kValue
    5. 执行? CreateDataPropertyOrThrow(A, Pk, mappedValue)
    6. 将k设为k + 1
  13. 执行? Set(A, "length", len, true)
  14. 返回A

注：from函数是一个有意通用的工厂方法；它不要求它的this值是Array构造函数。因此，它可以被任何其他构造函数转移或继承，这些构造函数可以用一个数字参数来调用。

22.1.2.3 Array.of ( …items )

当使用任意数量的参数调用of方法时，执行以下步骤：

  1. 令len为实际传递给此函数的参数数量
  2. 令items为传递给此函数的参数的列表
  3. 令C为this值
  4. 如果IsConstructor(C)为true，则令A为? Construct(C, « len »)
  5. 否则令A为? ArrayCreate(len)
  6. 令k为0
  7. 重复，只要k < len：
    1. 令kValue为items[k]
    2. 令Pk为! ToString(k)
    3. 执行? CreateDataPropertyOrThrow(A, Pk, kValue)
    4. 将k设为k + 1
  8. 执行? Set(A, "length", len, true)
  9. 返回A

注1：items参数被假定为一个符合语法规则的剩余参数值。
注2：of函数是一个有意通用工厂方法；它不要求它的this值是Array构造函数。因此，它可以转移到其他构造函数或由其他构造函数继承，这些构造函数可以用一个数字参数来调用。

22.1.3.1 Array.prototype.concat ( …arguments )

当使用零个或多个参数调用concat方法时，它将返回一个数组，该数组包含对象(this)的数组元素，后面跟着依次排列的每个参数的数组元素。

采取以下步骤：

  1. 令O为ToObject(this值)
  2. 令A为ArraySpeciesCreate(O, 0)
  3. 令n为0
  4. 令items为一个列表，它的第一个元素是O，之后的元素从左到右依次是传递给此函数参数。
  5. 只要items不为空，就重复执行：
    1. 从items中删除第一个元素，并令E为此元素的值
    2. 令spreadable为IsConcatSpreadable(E)
    3. 如果spreadable为true，则
      1. 令k为0
      2. 令len为LengthOfArrayLike(E)
      3. 如果n + len > 2^53-1，抛出TypeError异常
      4. 只要k < len，就重复执行：
        1. 令P为! ToString(k)
        2. 令exists为? HasProperty(E, P)
        3. 如果exists为true，则
          1. 令subElement为Get(E, P)
          2. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), subElement)
        4. 将n设为n + 1
        5. 将k设为k + 1
    4. 否则：
      1. 注：E被作为单个元素添加，而不是被展开后再添加
      2. 如果n≥2^53-1，抛出TypeError异常
      3. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), E)
      4. 将n设为n + 1
  6. 执行? Set(A, "length", n, true)
  7. 返回A

concat方法的length属性为1

注：必须在步骤6中显式设置length属性，以确保结果数组不存在尾随元素的情况下，其值正确。

注：concat函数是有意通用的； 它不需要其this值是Array对象。 因此，可以将其转移到其他类型的对象中作为方法。

22.1.3.3 Array.prototype.copyWithin ( target, start [ , end ] )

copyWithin方法接受三个参数：target、start、end。

注1：end参数是可选的，其默认值为this对象的长度。如果target为负值，则按length + target处理，其中length为数组的长度。如果start为负值，则按length + start处理。如果end为负值，则作为length + end处理。

执行以下步骤：

  1. 令O为? ToObject(this值)
  2. 令len为? LengthOfArrayLike(O)
  3. 令relativeTarget为? ToInteger(target)
  4. 如果relativeTarget < 0，令to为max((len + relativeTarget), 0)，否则令to为min(relativeTarget, len)
  5. 令relativeStart为? ToInteger(start)
  6. 如果relativeStart < 0，令from为max((len + relativeStart), 0)，否则令from为min(relativeStart, len)
  7. 如果end为undefined，令relativeEnd为len，否则令relativeEnd为? ToInteger(end)
  8. 如果relativeEnd < 0，令final为max((len + relativeEnd), 0)，否则令final为min(relativeEnd, len)
  9. 令count为min(final - from, len - to)
  10. 如果from < to且to < from + count，则
    1. 令direction为-1
    2. 将from设为from + count - 1
    3. 将to设为to + count - 1
  11. 否则，令direction为1
  12. 只要count > 0就重复：
    1. 令fromKey为! ToString(from)
    2. 令toKey为! ToString(to)
    3. 令fromPresent为? HasProperty(O, fromKey)
    4. 如果fromPresent为true，则
      1. 令fromVal为? Get(O, fromKey)
      2. 执行? Set(O, toKey, fromVal, true)
    5. 否则：
      1. 断言：fromPresent为false
      2. 执行? DeletePropertyOrThrow(O, toKey)
    6. 将from设为from + direction
    7. 将to设为to + direction
    8. 将count设为count - 1
  13. 返回O

注2：copyWithin函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。

22.1.3.5 Array.prototype.every ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 every对数组中的每一个存在的元素都调用一次callbackfn，按升序排列，直到找到一个callbackfn返回false的元素。如果找到这样的元素，every立即返回false。否则，如果callbackfn对所有元素都返回true，那么every将返回true。callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
every并不直接改变被调用的对象，但对象可能会因为调用callbackfn而改变。
every处理的元素范围是在第一次调用callbackfn之前设置的。在调用every开始后追加到数组的元素将不会被callbackfn访问。如果数组中现有的元素被改变，那么传递给 callbackfn 的值将是 every 访问它们时的值；在调用 every 开始后、被访问前被删除的元素不会被访问。 Every 的作用就像数学中的 "for all "量词。特别是，对于一个空数组，它返回true。

当使用一到两个参数调用every方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(callbackfn)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则
      1. 令kValue为? Get(O, Pk)
      2. 令testResult为! ToBoolean(? Call(callbackfn, thisArg, « kValue, k, O »))
      3. 如果testResult为false，返回false
    4. 将k设为k + 1
  6. 返回true

every函数是故意通用的，它不要求它的这个值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.6 Array.prototype.fill ( value [ , start [ , end ] ] )

fill方法接受三个参数value、start和end

注1：start和end参数是可选的，默认值分别为0和this对象的长度。如果start为负值，则按length+start处理，其中length是数组的长度。如果end为负值，则作为length + end处理。

执行以下步骤：

  1. 令O为? ToObject(this值)
  2. 令len为? LengthOfArrayLike(O)
  3. 令relativeStart为? ToInteger(start)
  4. 如果relativeStart < 0，令k为max((len + relativeStart), 0)，否则令k为min(relativeStart, len)
  5. 如果end为undefined，令relativeEnd为len，否则令relativeEnd为? ToInteger(end)
  6. 如果relativeEnd < 0，令final为max((len + relativeEnd), 0)，否则令final为min(relativeEnd, len)
  7. 重复，只要k < final：
    1. 令Pk为! ToString(k)
    2. 执行? Set(O, Pk, value, true)
    3. 将k设为k + 1
  8. 返回O

注2：fill函数是故意通用的，它不要求它的this值是一个Array对象，因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.7 Array.prototype.filter ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 filter对数组中的每一个元素都调用一次callbackfn，按升序排列，并构建一个新数组，元素为所有使得callbackfn返回true的值。callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
filter并不直接改变被调用的对象，但对象可能会因为调用callbackfn而改变。
filter处理的元素范围是在第一次调用callbackfn之前设置的。在调用filter开始后追加到数组的元素将不会被callbackfn访问。如果数组中现有的元素被改变，那么传递给 callbackfn 的值将是 filter 访问它们时的值；在调用 filter 开始后、被访问前被删除的元素不会被访问

当使用一到两个参数调用filter方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(callbackfn)为false，抛出 TypeError 异常
  4. 令A为ArraySpeciesCreate(O, 0)
  5. 令k为0
  6. 令to为0
  7. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则
      1. 令kValue为? Get(O, Pk)
      2. 令selected为! ToBoolean(? Call(callbackfn, thisArg, « kValue, k, O »))
      3. 如果selected为true，则：
        1. 执行? CreateDataPropertyOrThrow(A, ! ToString(to), kValue)
        2. 将to设为to + 1
    4. 将k设为k + 1
  8. 返回A

注2：filter函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.8 Array.prototype.find ( predicate [ , thisArg ] )

find方法接受一个或两个参数：predicate和thisArg

注1：predicate应该是一个接受三个参数的函数，并返回一个与布尔值相关的值。 find对数组中的每一个元素都调用一次predicate，按升序排列，直到找到一个使得predicate为true的元素。如果找到了这样的元素，find会立即返回这个元素。否则，find会返回undefined。
如果提供了thisArg参数，它将作为每次调用predicate时的this值。如果没有提供这个参数，则使用undefined代替。
predicate接受三个参数：元素的值、元素的索引和被遍历的对象。
find并不直接修改被调用的对象，但对象可能会因为调用predicate而改变。
find处理的元素范围是在第一次调用predicate之前设置的。在调用find开始后追加到数组的元素将不会被predicate访问。如果数组中存在的元素被改变，那么传递给 predicate 的值将是 find 访问它们时的值

当调用filter方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(predicate)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kValue为? Get(O, Pk)
    3. 令testResult为! ToBoolean(? Call(predicate, thisArg, « kValue, k, O »))
    4. 如果testResult为true，返回kValue
    5. 将k设为k + 1
  6. 返回undefined

注2：find函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.9 Array.prototype.findIndex ( predicate [ , thisArg ] )

注1：findIndex应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 findIndex对数组中的每一个元素都调用一次predicate，按升序排列，直到找到一个使得predicate为true的元素。如果找到了这样的元素，findIndex会立即返回这个元素的索引。否则，findIndex会返回-1。
如果提供了thisArg参数，它将作为每次调用predicate时的this值。如果没有提供这个参数，则使用undefined代替。
predicate接受三个参数：元素的值、元素的索引和被遍历的对象。
findIndex并不直接修改被调用的对象，但对象可能会因为调用predicate而改变。
findIndex处理的元素范围是在第一次调用predicate之前设置的。在调用findIndex开始后追加到数组的元素将不会被predicate访问。如果数组中存在的元素被改变，那么传递给 predicate 的值将是 findIndex 访问它们时的值

当使用一个或两个元素调用findIndex方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(predicate)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kValue为? Get(O, Pk)
    3. 令testResult为! ToBoolean(? Call(predicate, thisArg, « kValue, k, O »))
    4. 如果testResult为true，返回k
    5. 将k设为k + 1
  6. 返回-1

注2：findIndex函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.10 22.1.3.10 Array.prototype.flat ( [ depth ] )

当使用零个或一个参数调用flat方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令sourceLen为? LengthOfArrayLike(O)
  3. 令depthNum为1
  4. 如果depth不是undefined，则将depthNum设为? ToInteger(depth)
  5. 令A为? ArraySpeciesCreate(O, 0)
  6. 执行? FlattenIntoArray(A, O, sourceLen, 0, depthNum)
  7. 返回A
22.1.3.10.1 FlattenIntoArray ( target, source, sourceLen, start, depth [ , mapperFunction, thisArg ] )
  1. 断言：Type(target)是Object
  2. 断言：Type(source)是Object
  3. 断言：! IsNonNegativeInteger(sourceLen)为true
  4. 断言：! IsNonNegativeInteger(start)为true
  5. 断言：! IsInteger(depth)为true，或depth是+∞或-∞
  6. 断言：如果mapperFunction存在，则! IsCallable(mapperFunction)为true，thisArg存在，depth为1
  7. 令targetIndex为start
  8. 令sourceIndex为0
  9. 重复，只要sourceIndex < sourceLen
    1. 令P为! ToString(sourceIndex)
    2. 令exists为? HasProperty(source, P)
    3. 如果exists为true，则
      1. 令element为? Get(source, P)
      2. 如果mapperFunction存在，则将element设为? Call(mapperFunction, thisArg, « element, sourceIndex, source »)
      3. 令shouldFlatten为false
      4. 如果depth > 0，则将shouldFlatten设为? IsArray(element)
      5. 如果shouldFlatten为true，则
        1. 令elementLen为? LengthOfArrayLike(element)
        2. 将targetIndex设为? FlattenIntoArray(target, element, elementLen, targetIndex, depth - 1)
      6. 否则：
        1. 如果targetIndex ≥ 2^53 - 1，抛出 TypeError 异常
        2. 执行? CreateDataPropertyOrThrow(target, ! ToString(targetIndex), element)
        3. 将targetIndex设为targetIndex + 1
    4. 将sourceIndex设为sourceIndex + 1
  10. 返回targetIndex

当使用零个或一个参数调用flatMap方法时，执行以下步骤：

  3. 如果! IsCallable(mapperFunction)为false，抛出 TypeError 异常
  4. 令A为? ArraySpeciesCreate(O, 0)

  6. 返回A

22.1.3.12 Array.prototype.forEach ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数。 findIndex对数组中的每一个存在元素都调用一次predicate，遵循升序顺序，callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
forEach并不直接修改被调用的对象，但对象可能会因为调用callbackfn而改变。

当使用一个或两个元素调用forEach方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(predicate)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则：
      1. 令kValue为? Get(O, Pk)
      2. 执行? Call(callbackfn, thisArg, « kValue, k, O »)
    4. 将k设为k + 1
  6. 返回undefined

此函数是固有对象：%ArrayProto_forEach%
forEach函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.13 Array.prototype.includes ( searchElement [ , fromIndex ] )

includes使用SameValueZero算法，按升序将searchElement与数组中的元素进行比较，如果在任何位置找到，则返回true；否则返回false。
第二个参数fromIndex是可选的，默认为0（即搜索整个数组）。如果它大于或等于数组的长度，则返回false，即不搜索数组。如果它是负数，则作为从数组末端开始计算fromIndex的偏移量，如果计算出的索引小于0，则搜索整个数组。

当调用includes方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果len为0，返回false
  4. 令n为? ToInteger(fromIndex)
  5. 断言：如果fromIndex为undefined，则n为0
  6. 如果n ≥ 0，则令k为n
  7. 否则：
    1. 令k为len + n
    2. 如果k < 0，将k设为0
  8. 重复，只要k < len
    1. 令elementK为? Get(O, ! ToString(k))的结果
    2. 如果SameValueZero(searchElement, elementK)为true，返回true
    3. 将k设为k + 1
  9. 返回false

注2：includes函数是故意通用的；它不要求它的this值是一个Array对象。因此，它可以被转移到其他类型的对象上作为方法使用。
注3：includes方法有意在两个方面与类似的indexOf方法不同。首先，它使用了SameValueZero算法，而不是严格相等性比较，允许它检测NaN数组元素。第二，它不跳过缺失的数组元素，而是将它们作为undefined处理。

22.1.3.14 Array.prototype.indexOf ( searchElement [ , fromIndex ] )

indexOf使用严格相等运算符，按升序将searchElement与数组中的元素进行比较，如果找到一个或更多的索引，则返回最小的那个索引。否则返回-1。
第二个参数fromIndex是可选的，默认为0（即搜索整个数组）。如果它大于或等于数组的长度，则返回-1，即不搜索数组。如果它是负数，则作为从数组末端开始计算fromIndex的偏移量，如果计算出的索引小于0，则搜索整个数组。

当使用一个或两个参数调用indexOf方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果len为0，返回-1
  4. 令n为? ToInteger(fromIndex)
  5. 断言：如果fromIndex为undefined，则n为0
  6. 如果n ≥ len，返回-1
  7. 如果n ≥ 0，则令k为n
  8. 否则：
    1. 令k为len + n
    2. 如果k < 0，将k设为0
  9. 重复，只要k < len：
    1. 令kPresent为? HasProperty(O, ! ToString(k))
    2. 如果kPresent为true，则：
      1. 令elementK为? Get(O, ! ToString(k))
      2. 令same为执行严格相等性比较searchElement === elementK的结果
      3. 如果same为true，返回k
    3. 将k设为k + 1
  10. 返回-1

注2：indexOf函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。

22.1.3.15 Array.prototype.join ( separator )

注1：数组中的元素被转换为字符串，然后将这些字符串连接起来，用separator来分隔。如果没有提供分隔符，则使用一个逗号作为分隔符。

当使用一个参数separator调用join方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果separator是undefined，令sep为单个元素的字符串","
  4. 否则，令sep为? ToString(separator)
  5. 令R为空字符串
  6. 令k为0
  7. 重复，只要k < len：
    1. 如果k > 0，将R设为连接字符串R和sep的结果
    2. 令element为? Get(O, ! ToString(k))
    3. 如果element为undefined或null，令next空字符串，否则令next为? ToString(element)
    4. 将R设为连接字符串R和next的结果
    5. 将k设为k + 1
  8. 返回R

注2：join函数是故意通用的，它不要求它的this值是一个Array对象，因此，它可以转移到其他类型的对象上作为方法使用。

22.1.3.17 Array.prototype.lastIndexOf ( searchElement [ , fromIndex ] )

lastIndexOf使用严格相等运算符，按降序将searchElement与数组中的元素进行比较，如果找到一个或更多的索引，则返回最大的那个索引。否则返回-1。
第二个参数fromIndex是可选的，默认为数组长度减1（即搜索整个数组）。如果它大于或等于数组的长度，则搜索整个数组。如果它是负数，则作为从数组末端开始计算fromIndex的偏移量，如果计算出的索引小于0，则返回-1

当使用一个或两个参数调用lastIndexOf方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果len为0，返回-1
  4. 如果fromIndex存在，令n为? ToInteger(fromIndex)，否则令n为len - 1
  5. 如果n ≥ 0，则令k为min(n, len - 1)
  6. 否则，令k为len + n
  7. 重复，只要k >= 0：
    1. 令kPresent为? HasProperty(O, ! ToString(k))
    2. 如果kPresent为true，则：
      1. 令elementK为? Get(O, ! ToString(k))
      2. 令same为执行严格相等性比较searchElement === elementK的结果
      3. 如果same为true，返回k
    3. 将k设为k - 1
  8. 返回-1

注2：lastIndexOf函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。

22.1.5 Array迭代器对象

Array迭代器是一个对象，它代表了对某个特定Array实例对象的特定迭代。Array迭代器对象没有一个命名的构造函数，相反，Array迭代器对象是通过调用数组实例对象的某些方法来创建的。

22.1.5.1 CreateArrayIterator ( array, kind )

Array对象的几个方法会返回Iterator对象。抽象操作CreateArrayIterator接受参数array和kind以便创建这样的迭代器对象。它执行以下步骤：

  1. 断言：Type(array)是Object
  2. 断言：kind是key+value、key或value.
  3. 令iterator为OrdinaryObjectCreate(%ArrayIteratorPrototype%, « [[IteratedArrayLike]], [[ArrayLikeNextIndex]], [[ArrayLikeIterationKind]] »)
  4. 将iterator.[[IteratedArrayLike]]设为array
  5. 将iterator.[[ArrayLikeNextIndex]]设为0
  6. 将iterator.[[ArrayLikeIterationKind]]设为kind
  7. 返回iterator

22.1.5.2.1 %ArrayIteratorPrototype%.next ( )
  1. 令O为this值
