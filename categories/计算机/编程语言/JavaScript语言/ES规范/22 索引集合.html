22.1 Array对象

数组对象是奇异对象，它对某一类属性名进行特殊处理。 有关特殊处理的定义，请参见9.4.2。

22.1.1 Array构造函数

Array构造函数：

  • 是固有对象%Array%
  • 是全局对象的"Array"属性的初始值
  • 作为构造函数调用时创建并初始化一个新的数组奇异对象
  • 当作为函数而不是构造函数调用时也创建并初始化了一个新的数组对象，因此接受相同参数的函数调用Array(…)等价于对象创建表达式new Array(…)
  • 被设计成可以被子类化。它可以被用作类定义的extends子句的值。打算继承奇异数组行为的子类构造函数必须包含对Array构造函数的super调用，以初始化为Array奇异对象的子类实例。然而，大多数Array.prototype方法都是通用方法，不依赖于它们的this值是一个Array奇异对象。
  • 具有"length"属性，其值为1

当且仅当Array构造函数被调用时没有参数，这个描述才适用。

22.1.1.2 Array ( len )

当且仅当Array构造函数接受1个参数，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs = 1
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 如果Type(len)不是Number，则
    1. 执行! CreateDataPropertyOrThrow(array, "0", len)
    2. 令intLen为1
  7. 否则：
    1. 令intLen为ToUint32(len)
    2. 如果intLen ≠ len，抛出 RangeError 异常
  8. 执行! Set(array, "length", intLen, true)
  9. 返回array

22.1.1.3 Array ( …items )

当且仅当Array构造函数接受2个参数及更多参数时，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs ≥ 2
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 令k为0
  7. 令items为起始索引为0的列表，包含按照顺序的参数元素
  8. 重复：只要k < numberOfArgs
    1. 令Pk为! ToString(k)
    2. 令itemK为items[k]
    3. 执行! CreateDataPropertyOrThrow(array, Pk, itemK)
    4. 将k设为k + 1
  9. 断言：array的"length"属性值为numberOfArgs
  10. 返回array

22.1.2.1 Array.from ( items [ , mapfn [ , thisArg ] ] )

当使用参数items以及可选参数mapfn和thisArg调用Array.from

  1. 令C为this值
  2. 如果mapfn为undefined，令mapping为false
  3. 否则：
    1. 如果IsCallable(mapfn)为false，抛出 TypeError 异常
    2. 令mapping为true
  4. 令usingIterator为? GetMethod(items, @@iterator)
  5. 如果usingIterator不是undefined，则
    1. 如果IsConstructor(C)为true，则令A为? Construct(C)
    2. 否则，令A为! ArrayCreate(0)
    3. 令iteratorRecord为? GetIterator(items, sync, usingIterator)
    4. 令k为0
    5. 重复：
      1. 如果k ≥ 2^53 - 1，则：
        1. 令error为ThrowCompletion(一个新创建的TypeError对象)
        2. 返回? IteratorClose(iteratorRecord, error)
      2. 令Pk为! ToString(k)
      3. 令next为? IteratorStep(iteratorRecord)
      4. 如果next为false，则
        1. 执行? Set(A, "length", k, true)
        2. 返回A
      5. 令nextValue为? IteratorValue(next)
      6. 如果mapping为true，则
        1. 令mappedValue为Call(mapfn, thisArg, « nextValue, k »)
        2. 如果mappedValue是非常规完结，返回? IteratorClose(iteratorRecord, mappedValue)
        3. 将mappedValue设为mappedValue.[[Value]]
      7. 否则，令mappedValue为nextValue
      8. 令defineStatus为CreateDataPropertyOrThrow(A, Pk, mappedValue)
      9. 如果defineStatus是非常规完结，返回? IteratorClose(iteratorRecord, defineStatus)
      10. 将k设为k + 1
  6. 注意：items不是可迭代对象，所以假设它是类数组对象
  7. 令arrayLike为! ToObject(items)
  8. 令len为? LengthOfArrayLike(arrayLike)
  9. 如果IsConstructor(C)为true，则令A为? Construct(C, « len »)
  10. 否则令A为? ArrayCreate(len)
  11. 令k为0
  12. 重复，只要k < len
    1. 令Pk为! ToString(k)
    2. 令kValue为? Get(arrayLike, Pk)
    3. 如果mapping为true，则令mappedValue为? Call(mapfn, thisArg, « kValue, k »)
    4. 否则，令mappedValue为kValue
    5. 执行? CreateDataPropertyOrThrow(A, Pk, mappedValue)
    6. 将k设为k + 1
  13. 执行? Set(A, "length", len, true)
  14. 返回A

22.1.2.3 Array.of ( …items )

当使用任意数量的参数调用of方法时，执行以下步骤：

  1. 令len为实际传递给此函数的参数数量

  6. 令k为0
  7. 重复，只要k < len：
    1. 令kValue为items[k]
    2. 令Pk为! ToString(k)
    3. 执行? CreateDataPropertyOrThrow(A, Pk, kValue)
    4. 将k设为k + 1

注2：of函数是一个有意通用工厂方法；它不要求它的this值是Array构造函数。因此，它可以转移到其他构造函数或由其他构造函数继承，这些构造函数可以用一个数字参数来调用。

22.1.3.1 Array.prototype.concat ( …arguments )

当使用零个或多个参数调用concat方法时，它将返回一个数组，该数组包含对象(this)的数组元素，后面跟着依次排列的每个参数的数组元素。

采取以下步骤：

  1. 令O为ToObject(this值)
  2. 令A为ArraySpeciesCreate(O, 0)
  3. 令n为0
  4. 令items为一个列表，它的第一个元素是O，之后的元素从左到右依次是传递给此函数参数。
  5. 只要items不为空，就重复执行：
    1. 从items中删除第一个元素，并令E为此元素的值
    2. 令spreadable为IsConcatSpreadable(E)
    3. 如果spreadable为true，则
      1. 令k为0
      2. 令len为LengthOfArrayLike(E)
      3. 如果n + len > 2^53-1，抛出TypeError异常
      4. 只要k < len，就重复执行：
        1. 令P为! ToString(k)
        2. 令exists为? HasProperty(E, P)
        3. 如果exists为true，则
          1. 令subElement为Get(E, P)
          2. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), subElement)
        4. 将n设为n + 1
        5. 将k设为k + 1
    4. 否则：
      1. 注：E被作为单个元素添加，而不是被展开后再添加
      2. 如果n≥2^53-1，抛出TypeError异常
      3. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), E)
      4. 将n设为n + 1
  6. 执行? Set(A, "length", n, true)
  7. 返回A

concat方法的length属性为1

注：必须在步骤6中显式设置length属性，以确保结果数组不存在尾随元素的情况下，其值正确。

注：concat函数是有意通用的； 它不需要其this值是Array对象。 因此，可以将其转移到其他类型的对象中作为方法。

22.1.3.3 Array.prototype.copyWithin ( target, start [ , end ] )

copyWithin方法接受三个参数：target、start、end。

注1：end参数是可选的，其默认值为this对象的长度。如果target为负值，则按length + target处理，其中length为数组的长度。如果start为负值，则按length + start处理。如果end为负值，则作为length + end处理。

执行以下步骤：

  1. 令O为? ToObject(this值)
  2. 令len为? LengthOfArrayLike(O)
  3. 令relativeTarget为? ToInteger(target)
  4. 如果relativeTarget < 0，令to为max((len + relativeTarget), 0)，否则令to为min(relativeTarget, len)
  5. 令relativeStart为? ToInteger(start)
  6. 如果relativeStart < 0，令from为max((len + relativeStart), 0)，否则令from为min(relativeStart, len)
  7. 如果end为undefined，令relativeEnd为len，否则令relativeEnd为? ToInteger(end)
  8. 如果relativeEnd < 0，令final为max((len + relativeEnd), 0)，否则令final为min(relativeEnd, len)
  9. 令count为min(final - from, len - to)
  10. 如果from < to且to < from + count，则
    1. 令direction为-1
    2. 将from设为from + count - 1
    3. 将to设为to + count - 1
  11. 否则，令direction为1
  12. 只要count > 0就重复：
    1. 令fromKey为! ToString(from)
    2. 令toKey为! ToString(to)
    3. 令fromPresent为? HasProperty(O, fromKey)
    4. 如果fromPresent为true，则
      1. 令fromVal为? Get(O, fromKey)
      2. 执行? Set(O, toKey, fromVal, true)
    5. 否则：
      1. 断言：fromPresent为false
      2. 执行? DeletePropertyOrThrow(O, toKey)
    6. 将from设为from + direction
    7. 将to设为to + direction
    8. 将count设为count - 1
  13. 返回O

注2：copyWithin函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。

22.1.3.5 Array.prototype.every ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 every对数组中的每一个存在的元素都调用一次callbackfn，按升序顺序，直到找到一个callbackfn返回false的元素。如果找到这样的元素，every立即返回false。否则，如果callbackfn对所有元素都返回true，那么every将返回true。callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
every并不直接改变被调用的对象，但对象可能会因为调用callbackfn而改变。
every处理的元素范围是在第一次调用callbackfn之前设置的。在调用every开始后追加到数组的元素将不会被callbackfn访问。如果数组中现有的元素被改变，那么传递给 callbackfn 的值将是 every 访问它们时的值；在调用 every 开始后、被访问前被删除的元素不会被访问。 Every 的作用就像数学中的 "for all "量词。特别是，对于一个空数组，它返回true。

当使用一到两个参数调用every方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(callbackfn)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则
      1. 令kValue为? Get(O, Pk)
      2. 令testResult为! ToBoolean(? Call(callbackfn, thisArg, « kValue, k, O »))
      3. 如果testResult为false，返回false
    4. 将k设为k + 1
  6. 返回true

every函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.7 Array.prototype.filter ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 filter对数组中的每一个元素都调用一次callbackfn，按升序顺序，并构建一个新数组，元素为所有使得callbackfn返回true的值。callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
filter并不直接改变被调用的对象，但对象可能会因为调用callbackfn而改变。
filter处理的元素范围是在第一次调用callbackfn之前设置的。在调用filter开始后追加到数组的元素将不会被callbackfn访问。如果数组中现有的元素被改变，那么传递给 callbackfn 的值将是 filter 访问它们时的值；在调用 filter 开始后、被访问前被删除的元素不会被访问

当使用一到两个参数调用filter方法时，执行以下步骤：

  1. 令O为? ToObject(this value)

  8. 否则：
    1. 令kPresent为false
    2. 重复，只要kPresent为false并且k >= 0

    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则
      1. 令kValue为? Get(O, Pk)
      2. 将accumulator设为? Call(callbackfn, undefined, « accumulator, kValue, k, O »)
    4. 将k设为k - 1
  10. 返回accumulator

注2：reduceRight函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。

22.1.3.25 Array.prototype.slice ( start, end )

  9. 令n为0
  10. 重复，只要：k < final

    2. 令kPresent为? HasProperty(O, Pk)

      1. 令kValue为? Get(O, Pk)
      2. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), kValue)
    4. 将k设为k + 1
    5. 将n设为n + 1
  11. 执行? Set(A, "length", n, true)

22.1.3.26 Array.prototype.some ( callbackfn [ , thisArg ] )

注1：callbackfn应该是一个接受三个参数的函数，并返回一个与布尔值true或false相关的值。 some对数组中的每一个存在的元素都调用一次callbackfn，按升序顺序，直到找到一个callbackfn返回true的元素。如果找到这样的元素，some立即返回true。否则，some返回false。callbackfn只对数组中实际存在的元素调用，对数组中缺少的元素不调用。
如果提供了thisArg参数，它将作为每次调用callbackfn时的this值。如果没有提供这个参数，则使用undefined代替。
callbackfn接受三个参数：元素的值、元素的索引和被遍历的对象。
some并不直接改变被调用的对象，但对象可能会因为调用callbackfn而改变。
some处理的元素范围是在第一次调用callbackfn之前设置的。在调用some开始后追加到数组的元素将不会被callbackfn访问。如果数组中现有的元素被改变，那么传递给 callbackfn 的值将是 some 访问它们时的值；在调用 some 开始后、被访问前被删除的元素不会被访问。 some 的作用就像数学中的 "exists "量词。特别是，对于一个空数组，它返回false。

当使用一到两个参数调用some方法时，执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 如果IsCallable(callbackfn)为false，抛出 TypeError 异常
  4. 令k为0
  5. 重复，只要k < len：
    1. 令Pk为! ToString(k)
    2. 令kPresent为? HasProperty(O, Pk)
    3. 如果kPresent为true，则
      1. 令kValue为? Get(O, Pk)
      2. 令testResult为! ToBoolean(? Call(callbackfn, thisArg, « kValue, k, O »))
      3. 如果testResult为true，返回true
    4. 将k设为k + 1
  6. 返回false

some函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.27 Array.prototype.sort ( comparefn )

此数组的元素被排序。排序必须是稳定的（也就是说，比较相等的元素必须保持原来的顺序）。如果comparefn不是undefined的，它应该是一个接受两个参数x和y的函数，如果x<y，则返回负值，如果x=y，则返回0，如果x>y，则返回正值。

进入后，将执行以下步骤来初始化排序函数的计算。

  1. 如果comparefn不是undefined并且IsCallable(comparefn)为false，抛出 TypeError 异常
  2. 令obj为? ToObject(this值)
  3. 令len为? LengthOfArrayLike(obj)

在此规范内，对于sort方法，一个对象obj，如果下面的算法返回true，就说它是稀疏的：

  1. 对0 ≤ i < len范围内的每个整数i，执行：
    1. 令elem为obj.[[GetOwnProperty]](! ToString(i))
    2. 如果elem为undefined，返回true
  2. 返回false

排序顺序是此函数运行完后obj对象的小于len的整数索引属性的值的顺序。sort函数的结果按以下方式确定：
如果comparefn不是undefined，也不是这个数组元素的一致比较函数(见下文)，排序顺序是由实现定义的。如果comparefn是undefined的，并且SortCompare不作为一个一致的比较函数，那么排序顺序也是实现定义的。

令proto为obj.[[GetPrototypeOf]]()。如果proto不是null，并且存在一个整数j，使得下面的所有条件都得到满足，那么排序顺序是实现定义的：

  1. obj是稀疏的
  2. 0 ≤ j < len
  3. HasProperty(proto, ToString(j))为true

如果obj是稀疏的，并且以下任何一个条件为真，则排序顺序也是实现定义的：

  1. IsExtensible(obj)为false
  2. obj的任何小于len的非负整数索引是一个数据属性，且该属性的[[Configurable]]特性为false

如果以下任何一个条件为真，排序顺序也是实现定义的：

  • 如果obj是一个奇异对象（包括Proxy奇异对象），且其[[Get]]、[[Set]]、[[Delete]]和[[GetOwnProperty]]的行为不是这些内部方法的普通对象实现。
  • 如果obj的任何小于len的非负整数索引属性是一个访问器属性，或者是一个[[Writable]]特性为false的数据属性。
  • 如果comparefn是undefined的，并且ToString应用到任何作为参数传递给SortCompare的值时，修改了obj或obj的原型链上的任何对象。
  • 如果comparefn是undefined的，并且ToString应用到任何作为参数传递给SortCompare的值时不产生相同的结果。

执行以下步骤：

  1. 以obj为第一个参数，对Get、Set、DeletePropertyOrThrow和HasOwnProperty抽象操作，以及对SortCompare（如下所述）执行一个依赖于实现的调用序列：
    • 每次调用Get, Set, HasOwnProperty, 或DeletePropertyOrThrow的属性键参数是一个小于len的非负整数的字符串表示。
    • 每次调用Set的Throw参数是true
    • 调用SortCompare的参数是之前调用Get抽象操作所返回的值，除非根据HasOwnProperty，之前这些调用所访问的属性不存在。如果SortCompare的两个预期参数都对应不存在的属性，则使用+0代替调用SortCompare。如果只有第一个准参数是非存在的，使用+1。如果只有第二个预期参数是不存在的，使用-1。
    • 如果obj不是稀疏的，那么一定不要调用DeletePropertyOrThrow
    • 如果从这些操作中的任何一个操作中返回了一个非常规完结，则立即返回本函数的值。
  2. 返回obj

除非上面指定的排序顺序是由实现定义的，否则返回的对象必须具有以下两个特性：

  1. 小于len的非负整数一定存在某些数学排列π，使得对于每一个小于len的非负整数j，如果属性old[j]存在，那么new[π(j)]与old[j]的值完全相同。但如果属性 old[j]不存在，那么 new[π(j)]就不存在。
  2. 对于所有小于len的非负整数j和k，如果SortCompare(old[j]，old[k])<0(见下面的SortCompare)，那么new[π(j)]<new[π(k)]

这里用old[j]表示这个函数执行前调用Get(obj，j)的假设结果，用new[j]表示这个函数执行后调用Get(obj，j)的假设结果。

如果对集合S中的所有值a、b、c(可能是同一个值)满足以下所有要求，则函数comparefn是集合S的一致比较函数：记号a <CF b表示comparefn(a，b)<0；a =CF b表示comparefn(a，b)=0(任一符号)；a >CF b表示comparefn(a，b)>0。

  • 当给定一对特定的值a和b作为其两个参数时，调用comparefn(a，b)总是返回相同的值v。此外，Type(v)是Number，而且v不是NaN。请注意，这意味着对于给定的一对a和b，a <CF b，a =CF b，和a >CF b中的一个正好为真。
  • 调用comparefn(a, b)不改变obj或obj原型链上的对象。
  • a =CF a（自反性）
  • 如果a =CF b，则b =CF a (对称性)
  • 如果a =CF b并且b =CF c，则a =CF c (=CF的传递性)
  • 如果a <CF b且b <CF c，则a <CF c (<CF的传递性)
  • 如果a >CF b且b >CF c，则a >CF c (>CF的传递性)

注1：上述条件是必要的，也是充分的，可以保证comparefn将集合S划分为等价类，并且这些等价类是完全有序的。
注2：sort函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.27.1 Runtime Semantics: SortCompare ( x, y )

使用两个参数x和y调用抽象操作SortCompare。它还可以访问传递给当前调用的sort方法时的comparefn参数。具体步骤如下：

  1. 如果x和y都是undefined，返回+0
  2. 如果x是undefined，返回1
  3. 如果y是undefined，返回-1
  4. 如果comparefn不是undefined，则：
    1. 令v为? ToNumber(? Call(comparefn, undefined, « x, y »))
    2. 如果v是NaN，返回+0
    3. 返回v
  5. 令xString为? ToString(x)
  6. 令yString为? ToString(y)
  7. 令xSmaller为执行抽象关系比较xString < yString的结果
  8. 如果xSmaller为true，返回-1
  9. 令ySmaller为执行抽象关系比较yString < xString的结果
  10. 如果ySmaller为true，返回1
  11. 返回+0

注1：因为不存在的属性值总是比undefined的属性值大，而undefined总是比其他任何值大，所以undefined的属性值总是排序到结果的最后，然后是不存在的属性值。
注2：步骤5和步骤7中ToString抽象操作执行的方法调用有可能导致SortCompare不作为一个一致的比较函数。

22.1.3.28 Array.prototype.splice ( start, deleteCount, …items )

注1：当使用两个或更多的参数：start、deleteCount以及零个或多个items调用slice方法时，数组中从整数索引start开始的deleteCount个元素被参数items替换。返回一个包含被删除元素的Array对象（如果有的话）。

执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 令len为? LengthOfArrayLike(O)
  3. 令relativeStart为? ToInteger(start)
  4. 如果relativeStart < 0，令actualStart为max((len + relativeStart), 0) ，否则令actualStart为min(relativeStart, len)
  5. 如果实际的参数数量为0，则：
    1. 令insertCount为0
    2. 令actualDeleteCount为0
  6. 否则如果the number of actual arguments is 1，则：
    1. 令insertCount为0
    2. 令actualDeleteCount为len - actualStart
  7. 否则，
    1. 令insertCount为the number of actual arguments minus 2
    2. 令dc为? ToInteger(deleteCount)
    3. 令actualDeleteCount为min(max(dc, 0), len - actualStart)
  8. 如果len + insertCount - actualDeleteCount > 253 - 1，抛出 TypeError 异常
  9. 令A为? ArraySpeciesCreate(O, actualDeleteCount)
  10. 令k为0
  11. 只要k < actualDeleteCount就重复执行：
    1. 令from为! ToString(actualStart + k)

    3. 如果fromPresent为true，则：
      1. 令fromValue为? Get(O, from)
      2. 执行? CreateDataPropertyOrThrow(A, ! ToString(k), fromValue)

  12. 执行? Set(A, "length", actualDeleteCount, true)
  13. 令items为一个列表，其元素从左到右依次是实际的参数列表中从第三个参数开始的部分。如果传递的参数少于三个，则列表为空。
  14. 令itemCount为items中的元素数量
  15. 如果itemCount < actualDeleteCount，则：
    1. 将k设为actualStart
    2. 只要k < (len - actualDeleteCount)就重复执行：
      1. 令from为! ToString(k + actualDeleteCount)
      2. 令to为! ToString(k + itemCount)
      3. 令fromPresent为? HasProperty(O, from)
      4. 如果fromPresent is true，则：
        1. 令fromValue为? Get(O, from)
        2. 执行? Set(O, to, fromValue, true)
      5. 否则，
        1. 断言：fromPresent为false
        2. 执行? DeletePropertyOrThrow(O, to)
      6. 将k设为k + 1
    3. 将k设为len
    4. 只要k > (len - actualDeleteCount + itemCount)就重复执行：
      1. 执行? DeletePropertyOrThrow(O, ! ToString(k - 1))
      2. 将k设为k - 1
  16. 否则如果itemCount > actualDeleteCount，则：
    1. 将k设为(len - actualDeleteCount)
    2. 只要k > actualStart就重复执行：
      1. 令from为! ToString(k + actualDeleteCount - 1)
      2. 令to为! ToString(k + itemCount - 1)
      3. 令fromPresent为? HasProperty(O, from)
      4. 如果fromPresent is true，then
        1. 令fromValue为? Get(O, from)
        2. 执行? Set(O, to, fromValue, true)
      5. 否则，
        1. 断言：fromPresent为false
        2. 执行? DeletePropertyOrThrow(O, to)
      6. 将k设为k - 1
  17. 将k设为actualStart
  18. 只要items is not empty就重复执行：
    1. 从items中删除第一个元素并令E为该元素的值
    2. 执行? Set(O, ! ToString(k), E, true)
    3. 将k设为k + 1
  19. 执行? Set(O, "length", len - actualDeleteCount + itemCount, true)
  20. 返回A

注2：在步骤19中明确设置结果数组的 "length"属性在以前的ECMAScript版本中是必要的，以确保在结果数组的尾部元素不存在的情况下其长度是正确的。从ES2015开始，当结果Array被初始化为正确的长度而不是空Array时，设置 "length"就变得没有必要了，但为了保持向后的兼容性，还是沿用了这一设置。
注3：splice函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

    3. 只要k > 0,就重复执行：
      1. 令from为! ToString(k - 1)
      2. 令to为! ToString(k + argCount - 1)
      3. 令fromPresent为? HasProperty(O, from)
      4. 如果fromPresent为true，则：
        1. 令fromValue为? Get(O, from)
        2. 执行? Set(O, to, fromValue, true)
      5. 否则，
        1. 断言：fromPresent为false
        2. 执行? DeletePropertyOrThrow(O, to)
      6. 将k设为k - 1
    4. 令j为0
    5. 令items为一个列表，其元素从左到右依次是传给此函数的参数
    6. 只要items非空就重复执行：
      1. 从items中删除第一个元素并令E为此元素的值
      2. 执行? Set(O, ! ToString(j), E, true)
      3. 将j设为j + 1
  5. 执行? Set(O, "length", len + argCount, true)
  6. 返回len + argCount

unshift方法的"length"属性为1。
注2：unshift函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他类型的对象上作为方法使用。

22.1.3.32 Array.prototype.values ( )

执行以下步骤：

  1. 令O为? ToObject(this value)
  2. 返回CreateArrayIterator(O, value)

此函数是固有对象%ArrayProto_values%

22.1.3.33 Array.prototype [ @@iterator ] ( )

@@iterator属性的初始值是和Array.prototype.values的初始值相同的函数对象。

22.1.3.34 Array.prototype [ @@unscopables ]

@@unscopables数据属性的初始值是按以下步骤创建的对象：

  1. 令unscopableList为OrdinaryObjectCreate(null)
  2. 执行! CreateDataPropertyOrThrow(unscopableList, "copyWithin", true)
  3. 执行! CreateDataPropertyOrThrow(unscopableList, "entries", true)
  4. 执行! CreateDataPropertyOrThrow(unscopableList, "fill", true)
  5. 执行! CreateDataPropertyOrThrow(unscopableList, "find", true)
  6. 执行! CreateDataPropertyOrThrow(unscopableList, "findIndex", true)
  7. 执行! CreateDataPropertyOrThrow(unscopableList, "flat", true)
  8. 执行! CreateDataPropertyOrThrow(unscopableList, "flatMap", true)
  9. 执行! CreateDataPropertyOrThrow(unscopableList, "includes", true)
  10. 执行! CreateDataPropertyOrThrow(unscopableList, "keys", true)
  11. 执行! CreateDataPropertyOrThrow(unscopableList, "values", true)
  12. 返回unscopableList

此属性具有特性：{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

注：这个对象的自有属性名是在ECMAScript 2015规范之前没有作为Array.prototype的标准属性包含的属性名。这些名称在with语句绑定时被忽略，以便保留现有代码的行为，因为这些代码可能会使用这些名称之一作为外部作用域中的绑定，而该作用域被绑定对象是Array对象的with语句所遮蔽。

22.1.4 Array实例属性

Array实例是Array奇异对象，并具有为这种对象指定的内部方法。Array实例继承了Array原型对象的属性。

Array实例有一个 "length"属性，以及一组带有数组索引名的可枚举属性。

22.1.4.1 length

Array实例的 "length"属性是一个数据属性，它的值在数值上总是大于每一个可配置的自有属性的名称，而这些属性的名称是一个数组索引。

"length"属性最初具有特性{ [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: false }

注：减少 "length"属性的值有一个副作用，就是删除索引在新旧长度值之间的数组元素。然而，不可配置的属性不能被删除。试图将Array对象的 "length"属性设置为一个数值上小于或等于现有的不可配置的数组索引属性的最大数值自有属性名的值，将导致长度被设置为一个比该不可配置的数值自有属性名大一的数值。参见9.4.2.1。

22.1.5 Array迭代器对象

Array迭代器是一个对象，它代表了对某个特定Array实例对象的特定迭代。Array迭代器对象没有一个命名的构造函数，相反，Array迭代器对象是通过调用数组实例对象的某些方法来创建的。

22.1.5.1 CreateArrayIterator ( array, kind )

Array对象的几个方法会返回Iterator对象。抽象操作CreateArrayIterator接受参数array和kind以便创建这样的迭代器对象。它执行以下步骤：

  1. 断言：Type(array)是Object
  2. 断言：kind是key+value、key或value.
  3. 令iterator为OrdinaryObjectCreate(%ArrayIteratorPrototype%, « [[IteratedArrayLike]], [[ArrayLikeNextIndex]], [[ArrayLikeIterationKind]] »)
  4. 将iterator.[[IteratedArrayLike]]设为array
  5. 将iterator.[[ArrayLikeNextIndex]]设为0
  6. 将iterator.[[ArrayLikeIterationKind]]设为kind
  7. 返回iterator

22.1.5.2 %ArrayIteratorPrototype%对象

%ArrayIteratorPrototype%对象

  • 具有被所有Array迭代器对象继承的属性
  • 是普通对象
  • 具有[[Prototype]]内部槽，且其值为%IteratorPrototype%
  • 具有下列属性：
22.1.5.2.1 %ArrayIteratorPrototype%.next ( )
  1. 令O为this值
  2. 如果Type(O)不是Object，抛出TypeError异常
  3. 如果O不具有Array迭代器实例(22.1.5.3)的所有内部槽，抛出 TypeError 异常
  4. 令a为O.[[IteratedArrayLike]]
  5. 如果a是undefined，返回CreateIterResultObject(undefined, true)
  6. 令index为O.[[ArrayLikeNextIndex]]
  7. 令itemKind为O.[[ArrayLikeIterationKind]]
  8. 如果a具有[[TypedArrayName]]内部槽，则：
    1. 如果IsDetachedBuffer(a.[[ViewedArrayBuffer]])为true，抛出 TypeError 异常
    2. 令len为a.[[ArrayLength]]
  9. 否则，令len为? LengthOfArrayLike(a)
  10. 如果index ≥ len，则
    1. 将O.[[IteratedArrayLike]]设为undefined
    2. 返回CreateIterResultObject(undefined, true)
  11. 将O.[[ArrayLikeNextIndex]]设为index + 1
  12. 如果itemKind为key，返回CreateIterResultObject(index, false)
  13. 令elementKey为! ToString(index)
  14. 令elementValue为? Get(a, elementKey)
  15. 如果itemKind为value，令result为elementValue
  16. 否则：
    1. 断言：itemKind是key+value
    2. 令result为! CreateArrayFromList(« index, elementValue »)
  17. 返回CreateIterResultObject(result, false)
