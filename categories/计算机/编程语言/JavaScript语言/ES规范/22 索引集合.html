22.1 Array对象

数组对象是奇异对象，它对某一类属性名进行特殊处理。 有关特殊处理的定义，请参见9.4.2。

22.1.1 Array构造函数

Array构造函数：

  • 是固有对象%Array%
  • 是全局对象的"Array"属性的初始值
  • 作为构造函数调用时创建并初始化一个新的数组奇异对象
  • 当作为函数而不是构造函数调用时也创建并初始化了一个新的数组对象，因此接受相同参数的函数调用Array(…)等价于对象创建表达式new Array(…)
  • 被设计成可以被子类化。它可以被用作类定义的extends子句的值。打算继承奇异数组行为的子类构造函数必须包含对Array构造函数的super调用，以初始化为Array奇异对象的子类实例。然而，大多数Array.prototype方法都是通用方法，不依赖于它们的this值是一个Array奇异对象。
  • 具有"length"属性，其值为1

22.1.1.1 Array ( )

当且仅当Array构造函数被调用时没有参数，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs = 0
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 返回! ArrayCreate(0, proto)

22.1.1.2 Array ( len )

当且仅当Array构造函数接受1个参数，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs = 1
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 如果Type(len)不是Number，则
    1. 执行! CreateDataPropertyOrThrow(array, "0", len)
    2. 令intLen为1
  7. 否则：
    1. 令intLen为ToUint32(len)
    2. 如果intLen ≠ len，抛出 RangeError 异常
  8. 执行! Set(array, "length", intLen, true)
  9. 返回array

22.1.1.3 Array ( …items )

当且仅当Array构造函数接受2个参数及更多参数时，这个描述才适用。

  1. 令numberOfArgs为传递给此函数调用的参数数量
  2. 断言：numberOfArgs ≥ 2
  3. 如果NewTarget为undefined，令newTarget为当前激活的函数对象，否则令newTarget为NewTarget
  4. 令proto为? GetPrototypeFromConstructor(newTarget, "%Array.prototype%")
  5. 令array为! ArrayCreate(0, proto)
  6. 令k为0
  7. 令items为起始索引为0的列表，包含按照顺序的参数元素
  8. 重复：只要k < numberOfArgs
    1. 令Pk为! ToString(k)
    2. 令itemK为items[k]
    3. 执行! CreateDataPropertyOrThrow(array, Pk, itemK)
    4. 将k设为k + 1
  9. 断言：array的"length"属性值为numberOfArgs
  10. 返回array

22.1.2.1 Array.from ( items [ , mapfn [ , thisArg ] ] )

当使用参数items以及可选参数mapfn和thisArg调用Array.from

  1. 令C为this值
  2. 如果mapfn为undefined，令mapping为false
  3. 否则：
    1. 如果IsCallable(mapfn)为false，抛出 TypeError 异常
    2. 令mapping为true
  4. 令usingIterator为? GetMethod(items, @@iterator)
  5. 如果usingIterator不是undefined，则
    1. 如果IsConstructor(C)为true，则令A为? Construct(C)
    2. 否则，令A为! ArrayCreate(0)
    3. 令iteratorRecord为? GetIterator(items, sync, usingIterator)
    4. 令k为0
    5. 重复：
      1. 如果k ≥ 2^53 - 1，则：
        1. 令error为ThrowCompletion(一个新创建的TypeError对象)
        2. 返回? IteratorClose(iteratorRecord, error)
      2. 令Pk为! ToString(k)
      3. 令next为? IteratorStep(iteratorRecord)
      4. 如果next为false，则
        1. 执行? Set(A, "length", k, true)
        2. 返回A
      5. 令nextValue为? IteratorValue(next)
      6. 如果mapping为true，则
        1. 令mappedValue为Call(mapfn, thisArg, « nextValue, k »)
        2. 如果mappedValue是非常规完结，返回? IteratorClose(iteratorRecord, mappedValue)
        3. 将mappedValue设为mappedValue.[[Value]]
      7. 否则，令mappedValue为nextValue
      8. 令defineStatus为CreateDataPropertyOrThrow(A, Pk, mappedValue)
      9. 如果defineStatus是非常规完结，返回? IteratorClose(iteratorRecord, defineStatus)
      10. 将k设为k + 1
  6. 注意：items不是可迭代对象，所以假设它是类数组对象
  7. 令arrayLike为! ToObject(items)
  8. 令len为? LengthOfArrayLike(arrayLike)
  9. 如果IsConstructor(C)为true，则令A为? Construct(C, « len »)
  10. 否则令A为? ArrayCreate(len)
  11. 令k为0
  12. 重复，只要k < len
    1. 令Pk为! ToString(k)
    2. 令kValue为? Get(arrayLike, Pk)
    3. 如果mapping为true，则令mappedValue为? Call(mapfn, thisArg, « kValue, k »)
    4. 否则，令mappedValue为kValue
    5. 执行? CreateDataPropertyOrThrow(A, Pk, mappedValue)
    6. 将k设为k + 1
  13. 执行? Set(A, "length", len, true)
  14. 返回A

注：from函数是一个有意通用的工厂方法；它不要求它的this值是Array构造函数。因此，它可以被任何其他构造函数转移或继承，这些构造函数可以用一个数字参数来调用。

22.1.2.3 Array.of ( …items )

当使用任意数量的参数调用of方法时，执行以下步骤：

  1. 令len为实际传递给此函数的参数数量
  2. 令items为传递给此函数的参数的列表
  3. 令C为this值
  4. 如果IsConstructor(C)为true，则令A为? Construct(C, « len »)
  5. 否则令A为? ArrayCreate(len)
  6. 令k为0
  7. 重复，只要k < len：
    1. 令kValue为items[k]
    2. 令Pk为! ToString(k)
    3. 执行? CreateDataPropertyOrThrow(A, Pk, kValue)
    4. 将k设为k + 1
  8. 执行? Set(A, "length", len, true)
  9. 返回A

注1：items参数被假定为一个符合语法规则的剩余参数值。
注2：of函数是一个有意通用工厂方法；它不要求它的this值是Array构造函数。因此，它可以转移到其他构造函数或由其他构造函数继承，这些构造函数可以用一个数字参数来调用。

22.1.2.4 Array.prototype

Array.prototype的值是固有Array原型对象%Array.prototype%

此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

22.1.3 Array原型对象的属性

Array原型对象：

  • 是固有对象%ArrayPrototype%
  • 是Array奇异对象，且具有为这类对象规定的内部方法
  • 具有"length"属性，其值为0，特性为{ [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: false }
  • 具有[[Prototype]]内部槽，且其值为%Object.prototype%

注：Array原型对象被指定为Array奇异对象，以确保与ECMAScript 2015规范之前创建的ECMAScript代码兼容。

22.1.3.1 Array.prototype.concat ( …arguments )

当使用零个或多个参数调用concat方法时，它将返回一个数组，该数组包含对象(this)的数组元素，后面跟着依次排列的每个参数的数组元素。

采取以下步骤：

  1. 令O为ToObject(this值)
  2. 令A为ArraySpeciesCreate(O, 0)
  3. 令n为0
  4. 令items为一个列表，它的第一个元素是O，之后的元素从左到右依次是传递给此函数参数。
  5. 只要items不为空，就重复执行：
    1. 从items中删除第一个元素，并令E为此元素的值
    2. 令spreadable为IsConcatSpreadable(E)
    3. 如果spreadable为true，则
      1. 令k为0
      2. 令len为LengthOfArrayLike(E)
      3. 如果n + len > 2^53-1，抛出TypeError异常
      4. 只要k < len，就重复执行：
        1. 令P为! ToString(k)
        2. 令exists为? HasProperty(E, P)
        3. 如果exists为true，则
          1. 令subElement为Get(E, P)
          2. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), subElement)
        4. 将n设为n + 1
        5. 将k设为k + 1
    4. 否则：
      1. 注：E被作为单个元素添加，而不是被展开后再添加
      2. 如果n≥2^53-1，抛出TypeError异常
      3. 执行? CreateDataPropertyOrThrow(A, ! ToString(n), E)
      4. 将n设为n + 1
  6. 执行? Set(A, "length", n, true)
  7. 返回A

concat方法的length属性为1

注：必须在步骤6中显式设置length属性，以确保结果数组不存在尾随元素的情况下，其值正确。

注：concat函数是有意通用的； 它不需要其this值是Array对象。 因此，可以将其转移到其他类型的对象中作为方法。

22.1.3.1.1 运行时语义: IsConcatSpreadable ( O )

抽象操作IsConcatSpread接受参数O并执行以下步骤：

  1. 如果Type(O)不是Object，返回false
  2. 令spreadable为Get(O, @@isConcatSpreadable)
  3. 如果spreadable不为undefined，返回! ToBoolean(spreadable)
  4. 返回IsArray(O)

22.1.3.3 Array.prototype.copyWithin ( target, start [ , end ] )

copyWithin方法接受三个参数：target、start、end。

注1：end参数是可选的，其默认值为this对象的长度。如果target为负值，则按length + target处理，其中length为数组的长度。如果start为负值，则按length + start处理。如果end为负值，则作为length + end处理。

执行以下步骤：

  1. 令O为? ToObject(this值)
  2. 令len为? LengthOfArrayLike(O)
  3. 令relativeTarget为? ToInteger(target)
  4. 如果relativeTarget < 0，令to为max((len + relativeTarget), 0)，否则令to为min(relativeTarget, len)
  5. 令relativeStart为? ToInteger(start)
  6. 如果relativeStart < 0，令from为max((len + relativeStart), 0)，否则令from为min(relativeStart, len)
  7. 如果end为undefined，令relativeEnd为len，否则令relativeEnd为? ToInteger(end)
  8. 如果relativeEnd < 0，令final为max((len + relativeEnd), 0)，否则令final为min(relativeEnd, len)
  9. 令count为min(final - from, len - to)
  10. 如果from < to且to < from + count，则
    1. 令direction为-1
    2. 将from设为from + count - 1
    3. 将to设为to + count - 1
  11. 否则，令direction为1
  12. 只要count > 0就重复：
    1. 令fromKey为! ToString(from)
    2. 令toKey为! ToString(to)
    3. 令fromPresent为? HasProperty(O, fromKey)
    4. 如果fromPresent为true，则
      1. 令fromVal为? Get(O, fromKey)
      2. 执行? Set(O, toKey, fromVal, true)
    5. 否则：
      1. 断言：fromPresent为false
      2. 执行? DeletePropertyOrThrow(O, toKey)
    6. 将from设为from + direction
    7. 将to设为to + direction
    8. 将count设为count - 1
  13. 返回O

注2：copyWithin函数是故意通用的，它不要求它的this值是一个Array对象。因此它可以转移到其他种类的对象上作为方法使用。
