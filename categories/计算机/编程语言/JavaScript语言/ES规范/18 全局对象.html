18 全局对象

全局对象：

  • 是在控制流进入任何执行上下文之前创建的。
  • 没有[[Construct]]内部方法，不能在new运算符中将全局对象作为构造函数使用。
  • 没有[[Call]]内部方法，不能将全局对象作为函数调用。
  • 具有[[Prototype]]内部槽，其值取决于实现。
  • 除了本规范中定义的属性之外，全局对象还可以具有其他宿主定义的属性。 这可能包括一个值为全局对象本身的属性。 例如，在HTML文档对象模型中，全局对象的window属性是全局对象本身。

18.1 全局对象的值属性

18.1.1 globalThis

在Realm记录realm中，全局对象的"globalThis"属性的初始值是real.[[GlobalEnv]]'s EnvironmentRecord's [[GlobalThisValue]]

此属性具有特性{ [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: true }

18.1.1 Infinity

Infinity的值为+∞（参见 6.1.6.1）。此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

18.1.2 NaN

NaN的值为NaN（参见 6.1.6.1）。此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

18.1.3 undefined

undefined的值为undefined（参见 6.1.1）。此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }

18.2 全局对象的函数属性

18.2.1 eval (x)

eval函数是固有对象%eval%。当使用参数x调用eval函数时候，执行以下步骤：

  1. 断言：执行上下文栈至少有两个元素
  2. 令callerContext为执行上下文栈从栈顶数的第二个元素
  3. 令callerRealm为callerContext的Realm
  4. 返回? PerformEval(x, callerRealm, false, false)

注：如果调用上下文正在计算形式化参数初始化器，或者调用上下文的代码或eval代码是严格模式代码，那么eval代码就不能在调用eval的调用上下文的变量环境中实例化变量或函数绑定。相反，这样的绑定会被实例化在一个新的VariableEnvironment中，这个VariableEnvironment只对eval代码开放。由let、const或类声明引入的绑定总是被实例化在一个新的LexicalEnvironment中。

18.2.2 isFinite (number)

isFinite函数是固有对象％isFinite％。 当使用一个参数x调用isFinite函数时，将执行以下步骤：

  1. 令num为ToNumber(number)
  2. ReturnIfAbrupt(num)
  3. 如果num为NaN或+∞或−∞，返回false
  4. 否则，返回true

18.2.3 isNaN (number)

isNaN函数是固有对象％isNaN％。 当使用一个参数x调用isFinite函数时，将执行以下步骤：

  1. 令num为ToNumber(number)
  2. ReturnIfAbrupt(num)
  3. 如果num为NaN，返回true
  4. 否则，返回false

注意：ECMAScript代码测试值X是否为NaN的可靠方法是X！== X形式的表达式。当且仅当X为NaN时，结果为true。

18.2.4 parseFloat (string)

parseFloat函数产生一个Number值，该值由将string参数的内容解释为十进制字面值的结果确定。

parseFloat函数是固有对象％parseFloat％，当使用一个参数string调用parseFloat函数时，将执行以下步骤：

  1. 令inputString为ToString(string)
  2. 令trimmedString为! inputString.trimStart()
  3. 如果trimmedString和它的任何前缀都不满足 StrDecimalLiteral 的语法（请参见7.1.4.1），则返回NaN。
  4. 令numberString为trimmedString的最长前缀，该前缀满足 StrDecimalLiteral 语法。最长前缀也可能是trimmedString自己。
  5. 令mathFloat为numberString的数学值(MV)
  6. 如果mathFloat=0，则：
    1. 如果trimmedString的第一个代码单元为-，返回-0
    2. 返回+0
  7. 返回mathFloat的Number值

parseFloat可以将字符串的开头部分解释为Number值； 它忽略了不能解释为十进制字面值表示法的一部分的任何代码单元，并且没有给出任何此类代码单元被忽略的指示。

18.2.5 parseInt (string , radix)

parseInt函数产生一个整数值，该整数值由根据指定的radix来产生一个整数值，这个整数值由对string参数内容的解释决定。 string中的前导空白将被忽略。 如果radix为undefined或0，则假定为10，除非数字以代码单元对0x或0X开头（在这种情况下，基数为16）。 如果基数为16，则数字也可以选择以代码单元对0x或0X开头。

parseInt函数是固有对象%parseInt%，调用parseInt函数时，将执行以下步骤：

  1. 令inputString为ToString(string)
  2. 令S为! inputString.trimStart()
  3. 令sign为1
  4. 如果S不为空且S的第一个代码单元为0x002D(HYPHEN-MINUS)减号，令sign为-1
  5. 如果S不为空且S的第一个代码单元为0x002B(PLUS SIGN)加号或0x002D(HYPHEN-MINUS)减号，删除S的第一个代码单元
  6. 令R为ToInt32(radix)
  7. 令stripPrefix为true
  8. 如果R ≠ 0，则：
    1. 如果R < 2或者R > 36，返回NaN
    2. 如果R ≠ 16，令stripPrefix为false
  9. 否则令R = 10
  10. 如果stripPrefix为true，则：
    1. 如果S的长度大于等于2，且S的前两个代码单元为0x或0X，则删除S的前两个代码单元，并且令R = 16
  11. 如果S包含不是进制基数R的数字的代码单元，则令Z为S的子串，它由第一个这样的（不是进制基数R的数字的）代码单元之前的所有代码单元组成； 否则，令Z为S。
  12. 如果Z为空串，返回NaN
  13. 令mathInt为由Z表示的R进制下的数学整数值，使用字母A-Z和a-z表示从10到35的数字值（但是，如果R为10，并且Z包含20个以上有效数字，则根据实现方式的选择，第20位之后的每个有效数字都可以用0代替。如果R不是2、4、8、10、16或32，则mathInt可能是依赖于实现的由Z表示的R进制下的数学整数值的近似）
  14. 如果mathInt为0，则：
    1. 如果sign = −1，返回-0
    2. 返回+0
  15. 令number为mathInt的Number值
  16. 返回sign × number

注意：parseInt只能将字符串的开头部分解释为整数值； 它会忽略无法解释为整数符号的一部分的任何代码单元，并且不会给出任何此类代码单元被忽略的指示。

18.2.6 URI处理函数

统一资源标识符(Uniform Resource Identifiers)，或URIs是字符串，用来标识互联网上的资源（如网页或文件），以及访问它们的传输协议（如HTTP或FTP）。除了18.2.6.2、18.2.6.3、18.2.6.4和18.2.6.5中描述的对URI进行编码和解码的函数外，ECMAScript语言本身并没有提供任何使用URI的支持。

注：许多ECMAScript的实现提供了额外的功能和方法来操作网页；这些功能超出了本标准的范围。

18.2.6.1 URI语法和语义

一个URI是由一系列由组件分隔符分隔的组件组成。一般的形式是：
Scheme : First / Second ; Third ? Fourth

其中名称代表组件，而":"、"/"、";"和"?"则保留作为分隔符使用。encodeURI和decodeURI函数用于处理完整的URI；它们假定URI中任何保留的代码单元都有特殊的含义，因此不进行编码。encodeURIComponent和decodeURIComponent函数用于处理URI的单个组件部分；它们假定任何保留的代码单元都代表文本，因此必须进行编码，以便当组件是完整URI的一部分时，它们不会被解释为保留的代码单元。

18.2.6.1.1 运行时语义:Encode ( string, unescapedSet )

编码和转义过程由抽象操作Encode描述，接受两个字符串参数string和unescapedSet。

  1. 令strLen为string中代码单元的数量
  2. 令R为空字符串
  3. 令k为0
  4. 重复：
    1. 如果k等于strLen，返回R
    2. 令C为string中索引为k的代码单元
    3. 如果C在unescapedSet中，则
      1. 将k设为k + 1
      2. 将R设为连接字符串：先前的R值、C的结果
    4. 否则：
      1. 令cp为! CodePointAt(string, k)
      2. 如果cp.[[IsUnpairedSurrogate]]为true，抛出URIError 异常
      3. 将k设为k + cp.[[CodeUnitCount]]
      4. 令Octets为一个列表，元素是通过对cp.[[CodePoint]]进行UTF-8转换产生的8bit数
      5. 对Octets中的每个元素octet，按照列表顺序执行：
        1. 将R设为连接下列字符串的结果：
          • 先前的R值
          • "%"
          • octet的字符串表示，格式为两位大写16进制数，不足两位就在左侧补0。
18.2.6.1.2 运行时语义: Decode ( string, reservedSet )

解码和转义过程由抽象操作Decode描述，接受两个字符串参数string和reservedSet。

  1. 令strLen为string中代码单元的数量
  2. 令R为空字符串
  3. 令k为0
  4. 重复：
    1. 如果k等于strLen，返回R
    2. 令C为string中索引为k的代码单元
    3. 如果C不是代码单元0x0025 (PERCENT SIGN)，则令 S为只包含代码单元C的字符串值
    4. 否则：
      1. 令start为k
      2. 如果k+2 >= strlen，抛出 URIError 异常
      3. 如果string中索引为k + 1和k + 2的代码单元不表示十六进制数字，抛出 URIError 异常
      4. 令B为索引为k + 1和k + 2的代码单元表示的8bit数值
      5. 将k设为k+2
      6. 如果B的最高有效位是0，则
        1. 令C为值为B的代码单元
        2. 如果C不在reservedSet中，则令S为只包含代码单元C的字符串
        3. 否则令S为string从索引index到索引k（不包含）的子串
      7. 否则：
        1. 断言：B的最高有效位是1
        2. 令n为使得(B << n) & 0x80等于0的最小非负整数
        3. 如果n等于1或n大于等于4，抛出 URIError 异常
        4. 令Octets为大小为n的8bit整数的列表
        5. 将Octets[0]设为B
        6. 如果k + (3 × (n - 1))大于等于strLen，抛出 URIError 异常
        7. 令j为1
        8. 重复，只要j < n
          1. 将k设为k + 1
          2. 如果string中索引为k的代码单元不是0x0025 (PERCENT SIGN)，抛出 URIError 异常
          3. 如果string中索引为k+1和k+2的代码单元不表示十六进制数，抛出 URIError 异常
          4. 令B为索引k+1和k+2处的十六进制数字表示的8bit值
          5. 如果B的高两位不是10，抛出 URIError 异常
          6. 将k设为k + 2
          7. 将Octets[j]设为B
          8. 将j设为j + 1
        9. 如果Octets不包含一个有效的UTF-8编码的Unicode代码点，抛出 URIError 异常
        10. 令V为通过对Octets进行UTF-8转换产生的值，即从octets转为21-bit数
        11. 令S为字符串值，其编码单元依次是UTF16Encoding(V)中的元素。
      8. 将R设为连接字符串：R的先前值、S的结果
      9. 将k设为k + 1

18.2.6.2 decodeURI ( encodedURI )

decodeURI函数计算一个URI的新版本，其中每个转义序列和可能由encodeURI函数引入的UTF-8编码都被它所代表的码点的UTF-16编码所取代。不能由 encodeURI 引入的转义序列不会被替换。
decodeURI函数是固有对象%decodeURI%，当使用参数encodedURI调用函数decodeURI时，执行以下步骤：

  1. 令uriString为? ToString(encodedURI)
  2. 令reservedURISet为一个包含uriReserved中每个代码单元与#的字符串（";/?:@&=+$#"）
  3. 返回? Decode(uriString, reservedURISet)

18.2.6.3 decodeURIComponent ( encodedURIComponent )

decodeURIComponent函数计算了一个URI的新版本，在这个新版本中，每个转义序列和可能由encodeURIComponent函数引入的UTF-8编码都被它所代表的码点的UTF-16编码所取代。

decodeURIComponent函数是固有对象%decodeURIComponent%。当以一个参数encodedURIComponent调用decodeURIComponent函数时，会采取以下步骤。

  1. 令componentString为? ToString(encodedURIComponent)
  2. 令reservedURIComponentSet为空字符串
  3. 返回? Decode(componentString, reservedURIComponentSet)

18.2.6.4 encodeURI ( uri )

encodeURI函数计算了UTF-16编码（6.1.4）URI的新版本，其中某些码点的每一个实例都被代表码点UTF-8编码的一个、两个、三个或四个转义序列取代。
encodeURI函数是固有对象%encodeURI%。当使用参数uri调用encodeURI 函数时，执行以下步骤：

  1. 令uriString为? ToString(uri)
  2. 令unescapedURISet为一个包含uriReserved（;/?:@&=+$）和uriUnescaped（a-zA-Z0-9_.!~*'()-）中每个代码单元与#的字符串
  3. 返回? Decode(uriString, reservedURISet)

18.2.6.5 encodeURIComponent ( uriComponent )

encodeURIComponent函数计算UTF-16编码（6.1.4）URI的新版本，其中某些码点的每个实例都被代表码点UTF-8编码的一个、两个、三个或四个转义序列取代。

encodeURIComponent函数是固有对象%encodeURIComponent%。当使用参数uriComponent调用 encodeURIComponent 函数时，采取以下步骤。

  1. 令componentString为? ToString(uriComponent)
  2. 令unescapedURIComponentSet为一个包含uriUnescaped（a-zA-Z0-9_.!~*'()-）中代码单元的字符串
  3. 返回? Encode(componentString, unescapedURIComponentSet)
