13.3 声明和变量语句

13.3.1 Let和Const声明

注：let和const声明定义了范围为运行中的执行上下文的 LexicalEnvironment 上的变量。变量在包含它们的词法环境被实例化时创建，但在计算变量的 LexicalBinding 之前，不能以任何方式访问。
由接受Initializer的LexicalBinding定义的变量在计算 LexicalBinding 时（而不是在创建变量时）被赋予其Initializer的AssignmentExpression的值。如果let声明中的 LexicalBinding 没有 Initializer，则在计算 LexicalBinding 时变量会被赋予undefined值。

语法

  2. 返回InitializeReferencedBinding(lhs, undefined)

注：静态语义规则可确保这种形式的LexicalBinding不会出现在const声明中。

  1. 令bindingId为BindingIdentifier的字符串值
  2. 令lhs为ResolveBinding(bindingId)
  3. 如果IsAnonymousFunctionDefinition(Initializer)为true，则
    1. 令value为Initializer的接受参数bindingId的NamedEvaluation
  4. 否则：
    1. 令rhs为计算Initializer的结果
    2. 令value为? GetValue(rhs)

  4. 返回使用value和env作为参数对BindingPattern执行BindingInitialization的结果。

13.3.2 Variable语句

注：var语句声明了范围为运行中的执行上下文的VariableEnvironment上的变量。var变量在包含它们的词法环境被实例化时被创建，并且在创建时初始化为undefined。在任何 VariableEnvironment 的范围内，一个公共的 BindingIdentifier 可能会出现在多个VariableDeclaration中，但这些声明总共只定义一个变量。由接受参数Initializer的VariableDeclaration定义的变量在执行VariableDeclaration时（而不是在创建变量时）被赋予其Initializer的AssignmentExpression的值。

语法

13.3.2.4 运行时语义:计算

  1. 令next为计算VariableDeclarationList的结果
  2. ReturnIfAbrupt(next)
  3. 返回NormalCompletion(empty)

  2. ReturnIfAbrupt(next)
  3. 返回计算VariableDeclaration的结果

  2. 令lhs为? ResolveBinding(bindingId)
  3. 如果IsAnonymousFunctionDefinition(Initializer)为true，则：
    1. 令value为Initializer的使用参数bindingId的NamedEvaluation
  4. 否则：
    1. 令rhs为计算Initializer的结果
    2. 令value为? GetValue(rhs)
  5. 返回? PutValue(lhs, value)

  1. 令rhs为计算Initializer的结果
  2. 令rval为? GetValue(rhs)
  3. 返回使用rval和undefined作为参数执行BindingPattern的BindingInitialization的结果

  3. 使用参数value、environment和P执行BindingElement的? KeyedBindingInitialization

13.3.3.7 运行时语义: RestBindingInitialization

接受参数value、environment和excludedNames

  1. 令lhs为? ResolveBinding(BindingIdentifier的字符串值, environment)
  2. 令restObj为OrdinaryObjectCreate(%Object.prototype%)
  3. 执行? CopyDataProperties(restObj, value, excludedNames)
  4. 如果environment为undefined，返回PutValue(lhs, restObj)
  5. 返回InitializeReferencedBinding(lhs, restObj)

13.3.3.9 运行时语义: KeyedBindingInitialization

接受参数value、environment和propertyName

注：当environment为undefined时，它表示应该使用PutValue操作来赋予初始化值。对于非严格意义上的函数的形式参数列表就是这种情况。在这种情况下，形式参数的绑定被预初始化，以处理可能出现的多个同名参数。

  1. 令v为? GetV(value, propertyName)
  2. 如果Initializer存在且v为undefined，则：
    1. 令defaultValue为计算Initializer的结果
    2. 将v设为? GetValue(defaultValue)
  3. 返回使用参数v和environment执行BindingPattern的BindingInitialization的结果
  1. 令bindingId为BindingIdentifier字符串值
  2. 令lhs为? ResolveBinding(bindingId, environment)
  3. 令v为? GetV(value, propertyName)
  4. 如果Initializer存在且v为undefined，则
    1. 如果IsAnonymousFunctionDefinition(Initializer)为true，则将v设为使用参数bindingId执行Initializer的NamedEvaluation的结果
    2. 否则：
      1. 令defaultValue为计算Initializer的结果
      2. 将v设为? GetValue(defaultValue)
  5. 如果environment为undefined，返回? PutValue(lhs, v)
  6. 返回InitializeReferencedBinding(lhs, v)

13.7.4.7 运行时语义:LabelledEvaluation

接受参数labelSet

  1. 令varDcl为计算VariableDeclarationList的结果

13.7.4.8 运行时语义:ForBodyEvaluation( test, increment, stmt, perIterationBindings, labelSet )

抽象操作ForBodyEvaluation接受参数test, increment, stmt, perIterationBindings, 和 labelSet并执行以下操作：

  1. 令V=undefined
  2. 令status为CreatePerIterationEnvironment(perIterationBindings)
  3. ReturnIfAbrupt(status)
  4. 重复执行：
    1. 如果test不是[empty]，则
      1. 令testRef计算test的结果
      2. 令testValue为GetValue(testRef)
      3. ReturnIfAbrupt(testValue)
      4. 如果ToBoolean(testValue)为false，返回NormalCompletion(V)
    2. 令result为计算stmt的结果
    3. 如果LoopContinues(result, labelSet)为false，返回Completion(UpdateEmpty(result, V))
    4. 如果result.[[value]]不是empty，令V = result.[[value]]
    5. 令status为CreatePerIterationEnvironment(perIterationBindings)
    6. ReturnIfAbrupt(status)
    7. 如果increment不是[empty]，则
      1. 令incRef为计算increment的结果
      2. 令incValue为GetValue(incRef)
      3. ReturnIfAbrupt(incValue)

13.7.4.9 运行时语义:CreatePerIterationEnvironment( perIterationBindings )

抽象操作CreatePerIterationEnvironment接受参数perIterationBindings并执行以下步骤：

  1. 如果perIterationBindings有任何元素，则
    1. 令lastIterationEnv为运行中的执行上下文的LexicalEnvironment。
    2. 令outer为lastIterationEnv的外部环境引用
    3. 断言：outer不是null
    4. 令thisIterationEnv为NewDeclarativeEnvironment(outer)
    5. 对perIterationBindings中的每个元素bn，执行：
      1. 令status为thisIterationEnv.CreateMutableBinding(bn, false)
      2. 断言：status不可能是非常规完结
      3. 令lastValue为lastIterationEnv.GetBindingValue(bn, true)
      4. ReturnIfAbrupt(lastValue)
      5. 执行thisIterationEnv.InitializeBinding(bn, lastValue)
    6. 将运行中的执行上下文的LexicalEnvironment设为thisIterationEnv
  2. 返回undefined

13.7.5.12 运行时语义:ForIn/OfHeadEvaluation ( uninitializedBoundNames, expr, iterationKind )

使用参数uninitializedBoundNames、expr和iterationKind调用抽象操作ForIn/OfHeadEvaluation。iterationKind的值为enumerate或iterate或async-iterate

  1. 令oldEnv为运行中的执行上下文的LexicalEnvironment
  2. 如果uninitializedBoundNames不是空列表，则：
    1. 断言：uninitializedBoundNames没有重复的条目
    2. 令newEnv为NewDeclarativeEnvironment(oldEnv)
    3. 令newEnvRec为newEnv的EnvironmentRecord
    4. 对uninitializedBoundNames中的每个字符串name：
      1. 执行! newEnvRec.CreateMutableBinding(name, false)
    5. 将运行中的执行上下文的LexicalEnvironment设为newEnv
  3. 令exprRef为计算expr的结果
  4. 将运行中的执行上下文的LexicalEnvironment设为oldEnv
  5. 令exprValue为? GetValue(exprRef)
  6. 如果iterationKind为enumerate，则
    1. 如果exprValue为null或undefined，则返回Completion{[[type]]: break, [[value]]: empty, [[target]]: empty}
    2. 令obj为! ToObject(exprValue)
    3. 返回? EnumerateObjectProperties(obj)
  7. 否则：
    1. 断言：iterationKind为iterate或async-iterate
    2. 如果iterationKind为async-iterate，令iteratorHint为async
    3. 否则，令iteratorHint为sync
    4. 返回? GetIterator(exprValue, iteratorHint)

13.7.5.13 运行时语义:ForIn/OfBodyEvaluation ( lhs, stmt, iteratorRecord, iterationKind, lhsKind, labelSet [ , iteratorKind ] )

使用参数lhs、stmt、iteratorRecord、iterationKind、lhsKind和labelSet以及可选的iteratorKind调用抽象操作ForIn/OfBodyEvaluation。lhsKind的值为assignment或varBinding或lexicalBinding。iteratorKind的值为sync或async

  1. 如果iteratorKind不存在，将iteratorKind设为sync
  2. 令oldEnv为运行中的执行上下文的LexicalEnvironment
  3. 令V = undefined
  4. 令destructuring为lhs的IsDestructuring
  5. 如果destructuring为true且lhsKind为assignment，则
    1. 断言：lhs是一个LeftHandSideExpression
    2. 令assignmentPattern为被lhscovered的AssignmentPattern
  6. 重复：
    1. 令nextResult为? Call(iteratorRecord.[[NextMethod]], iteratorRecord.[[Iterator]])
    2. 如果iteratorKind为async，则将nextResult设为? Await(nextResult)
    3. 如果Type(nextResult)不为Object，抛出 TypeError 异常
    4. 令done为? IteratorComplete(nextResult)
    5. 如果done为true，返回NormalCompletion(V)
    6. 令nextValue为? IteratorValue(nextResult)
    7. 如果lhsKind为assignment或varBinding，则
      1. 如果destructuring为false，则
        1. 令lhsRef为计算lhs的结果 ( 它可能重复计算了)
    8. 否则：
      1. 断言：lhsKind是lexicalBinding
      2. 断言：lhs是ForDeclaration
      3. 令iterationEnv为NewDeclarativeEnvironment(oldEnv)
      4. 使用参数iterationEnv执行lhs的BindingInstantiation
      5. 将运行中的执行上下文的LexicalEnvironment设为iterationEnv
      6. 如果destructuring为false，则
        1. 断言：lhs绑定了单个名字
        2. 令lhsName 为lhs的BoundNames的唯一元素
        3. 令lhsRef为! ResolveBinding(lhsName)
    9. 如果destructuring为false，则
      1. 如果lhsRef是非常规完结，则令status为lhsRef
      2. 否则如果lhsKind是lexicalBinding，则令status为InitializeReferencedBinding(lhsRef, nextValue)
      3. 否则，令status为PutValue(lhsRef, nextValue)
    10. 否则，
      1. 如果lhsKind是assignment，则
        1. 令status为使用参数nextValue执行assignmentPattern的DestructuringAssignmentEvaluation的结果
      2. 否则如果lhsKind是varBinding，则
        1. 断言：lhs是ForBinding
        2. 令status为使用参数nextValue和undefined执行lhs的BindingInitialization的结果
      3. 否则：
        1. 断言：lhsKind是lexicalBinding
        2. 断言：lhs是一个ForDeclaration
        3. 令status为使用参数nextValue和iterationEnv执行lhs的BindingInitialization的结果
    11. 如果status是非常规完结，则
      1. 将运行中的执行上下文的LexicalEnvironment设为oldEnv
      2. 如果iteratorKind为async，返回? AsyncIteratorClose(iteratorRecord, status)
      3. 如果iterationKind为enumerate，返回status
      4. 否则：
        1. 断言：iterationKind为iterate
        2. 返回? IteratorClose(iteratorRecord, status)
    12. 令result为计算stmt的结果
    13. 将运行中的执行上下文的LexicalEnvironment设为oldEnv
    14. 如果LoopContinues(result, labelSet)为false，则：
      1. 如果iterationKind为enumerate，则返回Completion(UpdateEmpty(result, V))
      2. 否则：
        1. 断言：iterationKind为iterate
        2. 将status设为UpdateEmpty(result, V)
        3. 如果iteratorKind为async，返回? AsyncIteratorClose(iteratorRecord, status)
        4. 返回? IteratorClose(iteratorRecord, status)
    15. 如果result.[[value]]不为empty，将V设为result.[[value]]

13.7.5.15 EnumerateObjectProperties ( O )

当使用参数O调用抽象操作EnumerateObjectProperties时，执行以下步骤：

  1. 断言：Type(O)为Object
  2. 返回一个迭代器对象(25.1.1.2)，其next方法遍历O的可枚举的属性的字符串属性键。迭代器对象永远不会被ECMAScript代码直接访问。枚举属性的机制和顺序没有规定，但必须符合下面规定的规则。

迭代器的throw和return方法为null，永远不会被调用。迭代器的next方法处理对象属性，以确定属性键是否应该作为迭代值返回。返回的属性键不包括Symbol属性键。在枚举过程中，目标对象的属性可能被删除。在被迭代器的next方法处理之前被删除的属性会被忽略。如果在枚举期间向目标对象添加了新的属性，不保证新添加的属性在激活的枚举中被处理。一个属性名称在任何枚举中最多被迭代器的next方法返回一次。

枚举目标对象的属性包括枚举其原型的属性，以及原型的原型，以此类推；但是如果一个原型的属性与迭代器的next方法已经处理过的属性有相同的名称，则不会被处理。在确定原型对象的一个属性是否已经被处理时，不考虑[[Enumerable]]属性的值。原型对象的可枚举属性名称，必须使用原型对象作为参数来调用EnumerateObjectProperties来获得EnumerateObjectProperties必须通过调用其[[OwnPropertyKeys]]内部方法获得目标对象的自有属性键。目标对象的属性特性必须通过调用其[[GetOwnProperty]]内部方法获得。

此外，如果O或其原型链中的任何对象都不是代理奇异对象、整数索引奇异对象、模块命名空间奇异对象或实现提供的奇异对象，那么该迭代器的行为必须与CreateForInIterator(O)给出的迭代器一样，直到发生以下情况之一：

  • O或其原型链中的一个对象的[[Prototype]]内部槽的值发生变化。
  • 一个属性被从O或其原型链中的一个对象中移除。
  • 一个属性被添加到O的原型链中的一个对象
  • O或其原型链中的对象的属性的[[Enumerable]]特性的值发生变化。

注1：不要求主机直接实现13.7.5.16.2.1中的算法。他们可以选择任何行为不会偏离该算法的实现，除非违反了上一段中的某个约束。

下面是一个符合这些规则的ECMAScript生成器函数的信息性定义。

注2：不要求实现与CreateForInIterator相匹配的奇异对象的列表被选中，因为历史上实现在这些情况下的行为是不同的，而在所有其他情况下是一致的。

13.12 switch 语句

语法

13.12.9 运行时语义: CaseBlockEvaluation

接受参数input

  1. 令V为undefined
  2. 令A为包含CaseClauses中按照源文本顺序的CaseClause元素的列表，
  3. 令found为false
  4. 对A中的每个CaseClause元素C：
    1. 如果found为false，则将found设为? CaseClauseIsSelected(C, input)
    2. 如果found为true，则
      1. 令R为计算C的结果
      2. 如果R.[[Value]]不是empty，将V设为R.[[Value]]
      3. 如果R是非常规完结，返回Completion(UpdateEmpty(R, V))
  5. 返回NormalCompletion(V)

13.12.10 运行时语义: CaseClauseIsSelected ( C, input )

抽象操作CaseClauseIsSelected，给定CaseClause的C和input，确定C是否与input匹配。

  1. 断言：C是产生式CaseClause: case Expression:StatementList{opt}的实例.
  2. 令exprRef为计算C的Expression结果
  3. 令clauseSelector为? GetValue(exprRef)
  4. 返回执行严格相等性比较input === clauseSelector的结果

13.12.11 运行时语义:计算

  1. 令exprRef为计算Expression的结果
  2. 令switchValue为? GetValue(exprRef)
  3. 令oldEnv为正在运行中执行上下文的LexicalEnvironment
  4. 令blockEnv为NewDeclarativeEnvironment(oldEnv)
  5. 执行BlockDeclarationInstantiation(CaseBlock, blockEnv)
  6. 将运行中的执行上下文的LexicalEnvironment设为blockEnv
  7. 令R为使用参数switchValue执行CaseBlock的CaseBlockEvaluation的结果
  8. 将运行中的执行上下文的LexicalEnvironment设为oldEnv
  9. 返回R

注：不管控制流如何离开SwitchStatement，LexicalEnvironment总是恢复到它之前的状态。

语法

13.15.7 运行时语义: CatchClauseEvaluation

接受参数thrownValue

  1. 令oldEnv为运行中的执行上下文的LexicalEnvironment
  2. 令catchEnv为NewDeclarativeEnvironment(oldEnv)
  3. 令catchEnvRec为catchEnv的EnvironmentRecord

    1. 执行! catchEnvRec.CreateMutableBinding(argName, false)
  5. 将运行中的执行上下文的LexicalEnvironment设为catchEnv

  7. 如果status为非常规完结，则：

  8. 令B为计算Block的结果
  9. 将运行中的执行上下文的LexicalEnvironment设为oldEnv

  1. 令B为计算Block的结果

  3. 如果F.[[Type]]为normal，将F设为B
  4. 返回Completion(UpdateEmpty(F, undefined))
