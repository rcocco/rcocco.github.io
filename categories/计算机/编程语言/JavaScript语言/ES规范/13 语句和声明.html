13.3 声明和变量语句

13.3.1 Let和Const声明

注：let和const声明定义了范围为运行中的执行上下文的词法环境(LexicalEnvironment)的变量。变量在包含它们的词法环境被实例化时创建，但在计算变量的 LexicalBinding 之前，不能以任何方式访问。
由 LexicalBinding 和 Initializer 定义的变量在计算 LexicalBinding 时（而不是在创建变量时）被赋予其Initializer的AssignmentExpression的值。如果let声明中的 LexicalBinding 没有 Initializer，则在计算 LexicalBinding 时变量会被赋予undefined值。

语法

13.3.1.4 运行时语义:计算

  2. ReturnIfAbrupt(next)
  3. 返回NormalCompletion(empty)

  2. ReturnIfAbrupt(next)
  3. 返回计算LexicalBinding的结果
  1. 令lhs为ResolveBinding(StringValue of BindingIdentifier)
  2. 返回InitializeReferencedBinding(lhs, undefined)

注：静态语义规则可确保这种形式的LexicalBinding不会出现在const声明中。

  1. 令bindingId为BindingIdentifier的StringValue
  2. 令lhs为ResolveBinding(bindingId)
  3. 令rhs为计算Initializer的结果
  4. 令value为GetValue(rhs)
  5. ReturnIfAbrupt(value)
  6. 如果IsAnonymousFunctionDefinition(Initializer)为true，则
    1. 令hasNameProperty为HasOwnProperty(value, "name")
    2. ReturnIfAbrupt(hasNameProperty)
    3. 如果hasNameProperty为false，执行SetFunctionName(value, bindingId)
  7. 返回InitializeReferencedBinding(lhs, value)

  4. 令env为运行中的执行上下文的词法环境
  5. 返回使用value和env作为参数对BindingPattern执行BindingInitialization的结果。

13.3.2 Variable语句

注：var语句声明了范围为运行中的执行上下文的变量环境(VariableEnvironment)中的变量。var变量在包含它们的词法环境被实例化时被创建，并且在创建时初始化为undefined。在任何 VariableEnvironment 的范围内，一个公共的 BindingIdentifier 可能会出现在多个VariableDeclaration中，但这些声明只能共同定义一个变量。由VariableDeclaration和Initializer定义的变量在执行VariableDeclaration时（而不是在创建变量时）被赋予其Initializer的AssignmentExpression的值。

语法

13.3.2.4 运行时语义:计算

  1. 令next为计算VariableDeclarationList的结果
  2. ReturnIfAbrupt(next)
  3. 返回NormalCompletion(empty)
  1. 令next为计算VariableDeclarationList的结果
  2. ReturnIfAbrupt(next)
  3. 返回计算VariableDeclaration的结果
  1. 返回NormalCompletion(empty)
  1. 令bindingId为BindingIdentifier的StringValue
  2. 令lhs为ResolveBinding(bindingId)
  3. ReturnIfAbrupt(lhs)
  4. 令rhs为计算Initializer的结果
  5. 令value为GetValue(rhs)
  6. ReturnIfAbrupt(value)
  7. 如果IsAnonymousFunctionDefinition(Initializer)为true，则
    1. 令hasNameProperty为HasOwnProperty(value, "name")
    2. ReturnIfAbrupt(hasNameProperty)
    3. 如果hasNameProperty为false，执行SetFunctionName(value, bindingId)
  8. 返回PutValue(lhs, value)

注：如果VariableDeclaration嵌套在with语句中，并且VariableDeclaration中的BindingIdentifier与with语句的对象环境记录的绑定对象的属性名称相同，则第7步将为该属性赋值，而不是赋给Identifier的VariableEnvironment绑定。

    2. 令exprValue为GetValue(exprRef)

  2. 返回ForBodyEvaluation(第二个Expression, 第三个Expression, Statement, « », labelSet)
  1. 令varDcl为计算VariableDeclarationList的结果
  2. ReturnIfAbrupt(varDcl)
  3. 返回ForBodyEvaluation(第一个Expression, 第二个Expression, Statement, « », labelSet)
  1. 令oldEnv为运行中的执行上下文的词法环境(LexicalEnvironment)

  10. 令bodyResult为ForBodyEvaluation(第一个Expression, 第二个Expression, Statement, perIterationLets, labelSet)
  11. 将运行中的执行上下文的词法环境设为oldEnv
  12. 返回Completion(bodyResult)

13.7.4.8 运行时语义:ForBodyEvaluation( test, increment, stmt, perIterationBindings, labelSet )

抽象操作ForBodyEvaluation接受参数test, increment, stmt, perIterationBindings, 和 labelSet并执行以下操作：

  1. 令V=undefined
  2. 令status为CreatePerIterationEnvironment(perIterationBindings)
  3. ReturnIfAbrupt(status)
  4. 重复执行：
    1. 如果test不是[empty]，则
      1. 令testRef计算test的结果
      2. 令testValue为GetValue(testRef)
      3. ReturnIfAbrupt(testValue)
      4. 如果ToBoolean(testValue)为false，返回NormalCompletion(V)
    2. 令result为计算stmt的结果
    3. 如果LoopContinues(result, labelSet)为false，返回Completion(UpdateEmpty(result, V))
    4. 如果result.[[value]]不是empty，令V = result.[[value]]
    5. 令status为CreatePerIterationEnvironment(perIterationBindings)
    6. ReturnIfAbrupt(status)
    7. 如果increment不是[empty]，则
      1. 令incRef为计算increment的结果
      2. 令incValue为GetValue(incRef)
      3. ReturnIfAbrupt(incValue)

13.7.4.9 运行时语义:CreatePerIterationEnvironment( perIterationBindings )

抽象操作CreatePerIterationEnvironment接受参数perIterationBindings并执行以下步骤：

  1. 如果perIterationBindings有任何元素，则
    1. 令lastIterationEnv为运行中的执行上下文的词法环境。
    2. 令outer为lastIterationEnv的外部环境引用
    3. 断言：outer不是null
    4. 令thisIterationEnv为NewDeclarativeEnvironment(outer)
    5. 对perIterationBindings中的每个元素bn，执行：
      1. 令status为thisIterationEnv.CreateMutableBinding(bn, false)
      2. 断言：status不可能是非常规完结
      3. 令lastValue为lastIterationEnv.GetBindingValue(bn, true)
      4. ReturnIfAbrupt(lastValue)
      5. 执行thisIterationEnv.InitializeBinding(bn, lastValue)
    6. 将运行中的执行上下文的词法环境设为thisIterationEnv
  2. 返回undefined

13.7.5.12 运行时语义:ForIn/OfHeadEvaluation ( TDZnames, expr, iterationKind)

使用参数TDZnames、expr和iterationKind调用抽象操作ForIn/OfHeadEvaluation。iterationKind的值为enumerate或iterate

  1. 令oldEnv为运行中的执行上下文的词法环境。
  2. 如果TDZnames不是空列表，则：
    1. 断言：TDZnames没有重复的条目
    2. 令TDZ为NewDeclarativeEnvironment(oldEnv)
    3. 对TDZnames中的每个字符串name，执行：
      1. 令status为TDZ.CreateMutableBinding(name, false)
      2. 断言：status不可能是非常规完结
    4. 将运行中的执行上下文的词法环境设为TDZ
  3. 令exprRef为计算产生式expr的结果
  4. 将运行中的执行上下文的词法环境设为oldEnv
  5. 令exprValue为GetValue(exprRef)
  6. ReturnIfAbrupt(exprValue)
  7. 如果iterationKind为enumerate，则
    1. 如果exprValue.[[value]]为null或undefined，则返回Completion{[[type]]: break, [[value]]: empty, [[target]]: empty}
    2. 令obj为ToObject(exprValue)
    3. 返回obj.[[Enumerate]]()
  8. 否则：
    1. 断言：iterationKind为iterate
    2. 返回GetIterator(exprValue)

13.7.5.13 运行时语义:ForIn/OfBodyEvaluation ( lhs, stmt, iterator, lhsKind, labelSet )

使用参数lhs、stmt、iterator、lhsKind和labelSet调用抽象操作ForIn/OfBodyEvaluation。lhsKind的值为assignment或varBinding或lexicalBinding

  1. 令oldEnv为运行中的执行上下文的词法环境。
  2. 令V = undefined
  3. 令destructuring为lhs的IsDestructuring
  4. 如果destructuring为true且lhsKind为assignment，则
    1. 断言：lhs是一个LeftHandSideExpression
    2. 令assignmentPattern为使用AssignmentPattern作为目标符号解析lhs对应的源文本
  5. 重复：
    1. 令nextResult为IteratorStep(iterator)
    2. ReturnIfAbrupt(nextResult)
    3. 如果nextResult为false，返回NormalCompletion(V)
    4. 令nextValue为IteratorValue(nextResult)
    5. ReturnIfAbrupt(nextValue)
    6. 如果lhsKind为assignment或varBinding，则
      1. 如果destructuring为false，则
        1. 令lhsRef为计算lhs的结果 ( 它可能重复计算了)
    7. 否则：
      1. 断言：lhsKind是lexicalBinding
      2. 断言：lhs是一个ForDeclaration
      3. 令iterationEnv为NewDeclarativeEnvironment(oldEnv)
      4. 使用参数iterationEnv执行lhs的BindingInstantiation
      5. 将运行中的执行上下文的词法环境设为iterationEnv
      6. 如果destructuring为false，则
        1. 断言：lhs绑定了单个名字
        2. 令lhsName 为lhs的BoundNames的唯一元素
        3. 令lhsRef为ResolveBinding(lhsName)
        4. 断言：lhsRef不是非常规完结
    8. 如果destructuring为false，则
      1. 如果lhsRef是非常规完结，则令status为lhsRef
      2. 否则如果lhsKind是lexicalBinding，则令status为InitializeReferencedBinding(lhsRef, nextValue)
      3. 否则，令status为PutValue(lhsRef, nextValue)
    9. 否则，
      1. 如果lhsKind是assignment，则
        1. 令status为使用参数nextValue执行assignmentPattern的DestructuringAssignmentEvaluation的结果
      2. 否则如果lhsKind是varBinding，则
        1. 断言：lhs是ForBinding
        2. 令status为使用参数nextValue和undefined执行lhs的BindingInitialization的结果
      3. 否则：
        1. 断言：lhsKind是lexicalBinding
        2. 断言：lhs是一个ForDeclaration
        3. 令status为使用参数nextValue和iterationEnv执行lhs的BindingInitialization的结果
    10. 如果status是非常规完结，则
      1. 将运行中的执行上下文的词法环境设为oldEnv
      2. 返回IteratorClose(iterator, status)
    11. 令result为计算stmt的结果
    12. 将运行中的执行上下文的词法环境设为oldEnv
    13. 如果LoopContinues(result, labelSet)为false，返回IteratorClose(iterator, UpdateEmpty(result, V))
    14. 如果result.[[value]]不为empty，令V为result.[[value]]

13.11 with 语句

语法

with语句将已计算对象的对象环境记录添加到正在运行的执行上下文的词法环境中。 然后，它使用此扩充词法环境执行一条语句。 最后，它还原了原始词汇环境。

13.11.7 运行时语义:计算

  1. 令val为计算Expression的结果
  2. 令obj为ToObject(GetValue(val))
  3. ReturnIfAbrupt(obj)
  4. 令oldEnv为正在运行的执行上下文的词法环境
  5. 令newEnv为NewObjectEnvironment(obj, oldEnv)
  6. 将newEnv的环境记录的withEnvironment标志设为true
  7. 将正在运行的执行上下文的词法环境设为newEnv
  8. 令C为计算Statement的结果
  9. 将正在运行的执行上下文的词法环境设为oldEnv
  10. 如果C.[[type]]为normal并且C.[[value]]为empty，返回NormalCompletion(undefined)
  11. 返回Completion(C)
