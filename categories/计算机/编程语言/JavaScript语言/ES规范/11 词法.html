11.2 空白符

代码点 名称 缩写
U+0009 水平制表符 <TAB>
U+000B 垂直制表符 <VT>
U+000C 换页 <FF>
U+0020 空格 <SP>
U+00A0 不换行空格 <NBSP>
U+FEFF 零宽不换行空格 <ZWNBSP>
Zs类 任何"Separator, space"中的其他代码点 <USP>

ECMAScript实现必须将Unicode 5.1中“Separator, space”(Zs)类别中列出的识别为WhiteSpace代码点。 ECMAScript实现也可以将Unicode标准后续版本中的其他类别Zs代码点识别为WhiteSpace。

注意：除了表32中列出的代码点以外，ECMAScript中WhiteSpace有意排除了所有具有Unicode“White_Space”属性但未归入“ Zs”类别的代码点。

11.6 名字和关键字

标识符名(IdentifierName)和保留字(ReservedWord)是根据Unicode标准附录#31（UNICODE标识符和模式语法）的Default Identifier Syntax一节来解释、并作了一些小修改的 token。
保留字是标识符名的枚举子集。句法将标识符(Identifier)定义为不是保留字的标识符名（见11.6.2）。Unicode标识符语法基于Unicode标准指定的字符属性。 所有符合标准的ECMAScript实现都必须将Unicode标准5.1.0版中指定类别中的Unicode代码点视为这些类别中的Unicode代码点。 ECMAScript实现可以识别在Unicode标准的更高版本中定义的标识符代码点。

注意：本标准规定了额外的特定的代码点：U+0024 (DOLLAR SIGN) 和 U+005F (LOW LINE) 允许出现在标识符名的任意位置，代码点U+200C (ZERO WIDTH NON-JOINER) 和 U+200D (ZERO WIDTH JOINER)允许出现在标识符名第一个代码点之后的任意位置。

IdentifierName中允许使用Unicode转义序列，在该序列中，它们向IdentifierName贡献一个Unicode代码点。代码点由UnicodeEscapeSequence的HexDigits表示（请参见11.8.4）。UnicodeEscapeSequence之前的\以及u和{ }代码单元（如果出现）不会为IdentifierName提供代码点。UnicodeEscapequence 不能用于将代码点放入标识符名中，否则该标识符名将是非法的。换句话说，如果\ UnicodeEscapeSequence序列被其贡献的SourceCharacter替换，则结果必须仍然是有效的IdentifierName，其SourceCharacter元素序列与原始IdentifierName完全相同。本规范中对IdentifierName的所有解释均基于其实际代码点，无论是否使用转义序列来贡献任何特定代码点。

两个根据Unicode标准在规范上等价的IdentifierName不是相等的，除非在替换每个UnicodeEscapeSequence之后它们由完全相同的代码点表示。

11.8.4 字符串字面值

注意：字符串字面值是用单引号或双引号引起来的零个或多个Unicode代码点。Unicode代码点也可以由转义序列表示。除了用于闭合的引号、U+005C (反斜杠\), U+000D (回车), U+2028 (行分隔符), U+2029 (段分隔符)和U+000A (换行)之外，其他所有代码点都可能以其字面形式出现在字符串字面值中。任何代码点都可以转义序列的形式出现。字符串字面值的计算结果为ECMAScript String值。生成这些String值时，Unicode代码点按照10.1.1中的定义进行UTF-16编码。属于基本多语言平面的代码点被编码为字符串的单个代码单元元素。所有其他代码点都被编码为字符串的两个代码单元元素。

11.8.6.1 运行时语义: TV和TRV

模板字面值组件被解释为Unicode代码点的序列。 字面值组件的模板值(Template Value, TV)是根据模板字面值组件各个部分贡献的代码单元值(SV, 11.8.4)来描述的。作为此过程的一部分，模板组件中的某些Unicode代码点被解释为具有数学值(MV，11.8.3)。在确定模板值(TV)时，将转义序列替换为由转义序列表示的Unicode代码点的UTF-16代码单元。模板原始值(Template Raw Value, TRV)与模板值类似，不同之处在于TRV中的转义序列是按字面解释的。

  • 产生式NoSubstitutionTemplate :: ``的TV和TRV是空代码单元序列。
  • 产生式TemplateHead :: `${的TV和TRV是空代码单元序列。
  • 产生式TemplateMiddle :: }${的TV和TRV是空代码单元序列。
  • 产生式TemplateTail :: }`的TV和TRV是空代码单元序列。
  • 产生式NoSubstitutionTemplate :: `的TV是TemplateCharacters的TV
  • 产生式TemplateHead :: ` TemplateCharacters ${的TV是TemplateCharacters的TV
  • 产生式TemplateMiddle :: } TemplateCharacters ${的TV是TemplateCharacters的TV
  • 产生式TemplateTail :: } TemplateCharacters `的TV是TemplateCharacters的TV
  • 产生式TemplateCharacters :: TemplateCharacter的TV是TemplateCharacter的TV
  • 产生式TemplateCharacters :: TemplateCharacter TemplateCharacters的TV是一个序列，该序列由TemplateCharacter的TV中的代码单元后跟TemplateCharacters的TV中按照顺序的所有代码单元组成。
  • 产生式TemplateCharacter :: SourceCharacter 但不能是 ` 或 \ 或 $ 或 LineTerminator的TV是SourceCharacter的代码点值的UTF16Encoding(10.1.1)

The TV and TRV of NoSubstitutionTemplate :: `` is the empty code unit sequence.

The TV and TRV of TemplateHead :: `${ is the empty code unit sequence.

The TV and TRV of TemplateMiddle :: }${ is the empty code unit sequence.

The TV and TRV of TemplateTail :: }` is the empty code unit sequence.

注意：模板值不包括 LineContinuation 的代码单元，而TRV包括它们。 对于TV和TRV，<CR><LF>和<CR>LineTerminatorSequences序列均被标准化为<LF>。 需要显式的EscapeSequence来包含<CR>或<CR><LF>序列。
