20.1 Number对象

20.1.1 Number构造函数

Number构造函数是固有对象％Number％以及全局对象的Number属性的初始值。 当作为构造函数调用时，它将创建并初始化一个新的Number对象。 当Number作为函数而不是构造函数调用时，它将执行类型转换。

Number构造函数被设计为可子类化的。 它可以用作类定义的extends子句的值。 打算继承指定Number行为的子类构造函数必须包括对Number构造函数的super调用，以使用[[NumberData]]内部插槽创建和初始化子类实例。

20.1.1.1 Number ( [ value ] )

当使用参数value调用Number时，将执行以下步骤：

  1. 如果没有传递参数，令n为+0
  2. 否则，令n为ToNumber(value)
  3. ReturnIfAbrupt(n)
  4. 如果NewTarget为undefined，返回n
  5. 令O为OrdinaryCreateFromConstructor(NewTarget, "%NumberPrototype%", «[[NumberData]]» )
  6. ReturnIfAbrupt(O)
  7. 将O的[[NumberData]]内部插槽的值设为n
  8. 返回O

20.4.1.1 时间值和时间范围

ECMAScript中的时间计量类似于POSIX中的时间计量，特别是共享了外推格里历（公历）的定义：它开始于UTC时间的1970年1月1日午夜，每一天包含 86400 秒（每秒有1000毫秒）

一个ECMAScript时间值是一个Number，可以是有限整数，表示精确到毫秒的某个特定时刻，也可以是NaN，表示不特定时间。如果一个时间值s是（即等于，其中是某个整数）的倍数，它表示从UTC天开始的时刻往后d个UTC天（负d表示往前）这样的时刻。每个其他的有限时间值t都是相对于最大的前一个这样的时间值s定义的，表示和s相同的UTC天内但往后t-s毫秒的时刻。

时间值并没有考虑到UTC闰秒——没有表示正闰秒内的时刻的时间值，但有表示因为负闰秒而从UTC时间线上移除的时刻的时间值。但是，时间值的定义仍然可以与UTC分段对齐，仅在闰秒边界才是不连续的，在闰秒之外没有差异。

一个Number值可以精确表示从-9,007,199,254,740,992到9,007,199,254,740,992的所有整数（20.1.2.8 和 20.1.2.6）。一个时间值支持从-8,640,000,000,000,000到8,640,000,000,000,000毫秒这样的稍小的范围。这就产生了支持的时间值的范围：相对于UTC时间1970年1月1日午夜起始的-100,000,000天到100,000,000天。

UTC时间1970年1月1日午夜起始的确切时刻，使用时间值+0表示。

注：公历中每400年包含97个闰年，也就是说平均每年 365.2425 天，即31,556,952,000毫秒。 因此，Number可以精确表示的最大范围（以毫秒为单位）相对于1970年，约为-285,426至285,426年。本节中指定的时间值所支持的较小范围相对于1970年约为-273,790至273,790年。

20.4.1.7 LocalTZA ( t, isUTC )

LocalTZA( t, isUTC )是一个由实现定义的算法，它以毫秒为单位返回当地的时区调整或偏移。当地的规定的统一标准时间或夏令时对t的影响应该以本节中指定的方式来确定结果。

当isUTC为true时，LocalTZA( t_{UTC}, true )应该返回在时间值t_{UTC}所代表的时刻，以毫秒为单位的当地时区与 UTC 的偏移量。当这个结果被加到t_{UTC}后，它应该产生相应的Number值t_{local}。

当isUTC为false时，LocalTZA( t_{local}, false )应该返回在以Number值t_{local}代表的时刻，以毫秒为单位的当地时区与 UTC 的偏移量。当从t_{local}中减去这个结果后，它应该产生相应的时间值t_{UTC}。

输入t名义上是一个时间值，但可以是任何Number值。当isUTC为false，并且t_{local}代表的时间值已经在范围边界偏移到时间值范围之外时，就会出现这种情况。算法不能将t_{local}限制在时间值范围内，所以支持这种输入。

当t_local代表在负时区转换时(例如当夏令时结束或时区偏移量因时区规则改变而减少时)重复多次的本地时间，或在正时区转换时(例如当夏令时开始或时区偏移量因时区规则改变而增加时)跳过的本地时间时，必须使用转换前的时区偏移量来解释t_{local}。

如果一个实现不支持上面描述的转换，或者实现中没有时间t的政治规则，结果必须为0。

注：建议实现者使用IANA时区数据库的时区信息https://www.iana.org/time-zones/。

美国/纽约时间2017年11月5日 1:30 AM 重复了两次(倒退)，但必须解释为UTC-04的 1:30 AM，而不是UTC-05的1:30 AM。LocalTZA(TimeClip(MakeDate(MakeDay(2017, 10, 5), MakeTime(1, 30, 0, 0))), false)为-4 × msPerHour

美国/纽约时间2017年3月12日 2:30 AM不存在，但必须解释为UTC-05 2:30 AM（相当于UTC-04 3:30 AM）。LocalTZA(TimeClip(MakeDate(MakeDay(2017, 2, 12), MakeTime(2, 30, 0, 0))), false)为-5 × msPerHour

当地时区偏移值可以是正值或负值。

20.4.1.11 MakeTime ( hour, min, sec, ms )

抽象操作MakeTime根据其四个参数计算毫秒数，这些参数必须是ECMAScript的Number值。 该操作执行下列过程：

  1. 如果hour或min或sec或ms不是有限值，返回NaN
  2. 令h为! ToInteger(hour)
  3. 令m为! ToInteger(min)
  4. 令s为! ToInteger(sec)
  5. 令milli为! ToInteger(ms)
  6. 令t为h * msPerHour + m * msPerMinute + s * msPerSecond + milli，根据IEE754-2019规则执行运算（就像ECMAScript中的运算符*和+一样）
  7. 返回t

20.4.1.12 MakeDay ( year, month, date )

抽象操作MakeDay根据其三个参数计算天数，这些参数必须是ECMAScript的Number值。 该操作执行下列过程：

  1. 如果year或month或date不是有限值，返回NaN
  2. 令y为! ToInteger(year)
  3. 令m为! ToInteger(month)
  4. 令dt为! ToInteger(date)
  5. 令ym为! y + floor(m / 12)
  6. 令mn为m modulo 12
  7. 找到一个值t使得YearFromTime(t)为ym且MonthFromTime(t)为mn且DateFromTime(t)为1。如果找不到这样的值（由于某些参数超出范围），则返回NaN
  8. 返回Day(t) + dt - 1

20.4.1.13 MakeDate ( day, time )

抽象操作MakeDate根据其两个参数计算毫秒数，这些参数必须是ECMAScript的Number值。 该操作执行下列过程：

  1. 如果day或time不是有限值，返回NaN
  2. 返回day × msPerDay + time

20.4.1.14 TimeClip ( time )

抽象操作TimeClip根据其参数计算毫秒数，这些参数必须是ECMAScript的Number值。 该操作执行下列过程：

  1. 如果time不是有限值，返回NaN
  2. 如果abs(time) > 8.64 * 10^15，返回NaN
  3. 返回 ! ToInteger(time)

注：第4步的要点是，允许实现者选择时间值的内部表示方法，例如使用64位有符号整数或64位浮点数。根据不同的实现，这种内部表示法可以区分或不区分-0和+0。

20.4.1.15 日期时间字符串格式

ECMAScript在简化 ISO 8601日历日期扩展格式的基础上，定义了日期时间的字符串交换格式。其格式为：YYYY-MM-DDTHH:mm:ss.sssZ

  • YYYY表示在公历中，从0000到9999的四位十进制数，或者是"+"或"-"后跟紧跟六位十进制数的扩展年
  • -（连字符）作为字面值"-"在字符串中出现两次。
  • MM是一年中的月份，从01（1月）到12（12月）的两位十进制数。
  • DD 是指月份中的第几号，从 01 到 31 的两位十进制数字。
  • T 作为字面值"T"出现在字符串中，表示时间元素的开始。
  • HH是指从午夜到现在已经过去的完整小时数，从00到24的两位十进制数。
  • :（冒号）作为字面值":"在字符串中出现两次
  • mm是自小时开始后的经过的完整分钟数，从00到59的两位进制数字。
  • ss是自分钟开始后的经过的完整秒数，为00至59的两位十进制数字。
  • .（点）作为字面值"."出现在字符串中
  • sss是自秒开始后的经过的完整毫秒数，为三位十进制数字。
  • Z是UTC偏移量的表示方法，指定为 "Z"（对于没有偏移量的UTC），又或者是 "+"或"-"后面跟着一个时间表达式HH:mm（分别表示当地时间在UTC之前或之后）的偏移量。

此格式也包括只有日期的格式：

它还包括 "日期-时间 "格式，它由上面只有日期的格式之一后面紧跟着是以下时间格式之一组成，并附加了一个可选的UTC偏移表示。

含有越界或不符合要求的元素的字符串不是此格式的有效实例。

注1：由于每天都是以午夜为起点和终点，所以可以用 00:00 和 24:00 这两个记号来区分可以与一个日期相关联的两个午夜。这意味着以下两个记号所指的时间点完全相同：1995-02-04T24:00和1995-02-05T00:00。将后一种形式解释为 "一个日历日的结束 " 符合ISO 8601，尽管该规范将其保留用于描述时间间隔，不允许在单个时间点的表示中使用。

注2：目前还没有国际标准规定民用时区的缩写，如CET、EST等，有时甚至同一缩写用于两个截然不同的时区。为此，ISO 8601和这种格式都规定了时区偏移的数字表示。

20.4.1.15.1 扩展年

要涵盖从1970年1月1日(20.4.1.1)向前或向后约273,790年的全部时间值范围，需要表示0之前或9999之后的年份。ISO 8601允许扩大年份的表示范围，但只能由信息交换的伙伴双方同意。在简化的ECMAScript格式中，这种扩展的年份表示法应包含6位数字，并总是以+或-号为前缀。0年被认为是正数，因此前缀为+号。与【日期时间字符串格式】匹配的字符串，如果用扩展的年份表示时间值范围之外的时间实例，则会被Date.parse视为不可识别，并导致该函数返回NaN，而不会回到特定的实现行为或启发式方法。

20.4.2 Date构造函数

Date构造函数：

  • 是固有对象%Date%
  • 是全局对象的"Date"属性的初始值
  • 作为构造函数调用时创建并初始化一个新的Date对象
  • 作为函数而不是构造函数调用时，返回表示当前时间（UTC）的字符串
  • 是单个函数，但其行为是基于参数的数量和类型而重载的。
  • 被设计为可派生的。它可以被用作类定义的extends子句的值。打算继承指定的Date行为的子类构造函数必须包含对Date构造函数的super调用，以创建并初始化具有[[DateValue]]内部插槽的子类实例。
  • 有一个 "length"属性，其值为7

20.4.2.1 Date ( year, month [ , date [ , hours [ , minutes [ , seconds [ , ms ] ] ] ] ] )

仅当使用至少两个参数调用Date构造函数时，此描述才适用。
调用Date函数时，将执行以下步骤：

  1. 令numberOfArgs为传入的参数数量
  2. 断言：numberOfArgs ≥ 2
  3. 如果NewTarget为undefined，则
    1. 令now为表示当前时间值（UTC）的Number
    2. 返回ToDateString(now)
  4. 否则
    1. 令y为? ToNumber(year)
    2. 令m为? ToNumber(month)
    3. 如果date存在，令dt为? ToNumber(date)，否则令dt为1
    4. 如果hours存在，令h为? ToNumber(hours)，否则令h为0
    5. 如果minutes存在，令min为? ToNumber(minutes)，否则令min为0
    6. 如果seconds存在，令s为? ToNumber(seconds)，否则令s为0
    7. 如果ms存在，令milli为? ToNumber(ms)，否则令milli为0

20.4.3.2 Date.parse ( string )

parse函数对参数string应用ToString抽象操作。如果ToString的结果是非常规完结，则直接返回完结记录。否则，parse函数将结果的字符串解释为日期和时间；它返回一个对应于此日期和时间的UTC时间值（Number类型）。字符串可能被解释为当地时间、UTC时间或其他时区的时间，这取决于字符串的内容。该函数首先尝试根据【日期时间字符串格式】（20.4.1.15）中描述的格式解析字符串，包括扩展的年份。如果字符串不符合该格式，函数可能会返回到任何实现特定的启发式或实现特定的日期格式。无法识别或包含越界格式元素值的字符串应导致Date.parse返回NaN。

如果字符串符合【日期时间字符串格式】，则用替代值代替缺失的格式元素。当MM或DD元素缺失时，使用 "01"。当HH、mm或ss元素缺失时，使用 "00"。当ss元素缺失时，使用 "000"。当UTC偏移表示不存在时，只有日期的格式被解释为UTC时间，而日期-时间格式被解释为当地时间。

  8. 返回连接字符串offsetSign、offsetHour、offsetMin和tzName的结果
20.4.41.4 运行时语义: ToDateString ( tv )

执行以下步骤：

  1. 断言：Type(tv)为Number
  2. 如果tv是NaN，返回"Invalid Date"
  3. 令t为LocalTime(tv)
  4. 返回连接字符串DateString(t)、代码单元0x0020 (SPACE)、TimeString(t)和TimeZoneString(tv)的结果

20.4.4.42 Date.prototype.toTimeString ( )

执行以下步骤：

  1. 令O为此Date对象
  2. 令tv为? thisTimeValue(O)
  3. 如果tv是NaN，返回"Invalid Date"
  4. 令t为LocalTime(tv)
  5. 返回连接字符串TimeString(t)和TimeZoneString(tv)的结果

20.4.4.43 Date.prototype.toUTCString ( )

toUTCString方法返回一个字符串值，代表this时间值对应的时间实例。字符串的格式基于RFC 7231中的 “HTTP-date”，并进行了扩展，以支持ECMAScript Date对象所支持的全部时间范围。它执行以下步骤：

  1. 令O为此Date对象
  2. 令tv为? thisTimeValue(O)
  3. 如果tv是NaN，返回"Invalid Date"
  4. 令weekday为表50中数字WeekDay(tv)对应的名字
  5. 令month为表51中数字MonthFromTime(tv)对应的名字
  6. 令day为DateFromTime(tv)的字符串表示，格式为两位十进制数，在必要时左边用零填充
  7. 令yv为YearFromTime(tv)
  8. 如果yv ≥ 0，令yearSign为空字符串，否则令yearSign为"-"
  9. 令year为abs(yv)的字符串表示，格式为十进制数
  10. 令paddedYear为! StringPad(year, 4, "0", start)
  11. 返回连接字符串weekday、","、代码单元0x0020 (SPACE)、day、代码单元0x0020 (SPACE)、month、代码单元0x0020 (SPACE)、yearSign、paddedYear、代码单元0x0020 (SPACE)和TimeString(tv)的结果

20.4.4.44 Date.prototype.valueOf ( )

执行以下步骤：

  1. 返回? thisTimeValue(this值)

20.4.4.45 Date.prototype [ @@toPrimitive ] ( hint )

这个函数被ECMAScript语言的运算符调用，用于将Date对象转换为一个原始值。允许的hint值是 "default"、"number"和 "string"。Date对象，在内置ECMAScript对象中是独一无二的，因为它们将 "default"等同于 "string"，所有其他内置ECMAScript对象将 "default"等同于 "number"。

当使用参数hint调用方法@@toPrimitive时，执行以下步骤：

  1. 令O为this值
  2. 如果Type(O)不是Object，抛出 TypeError 异常
  3. 如果hint是字符串"string"或字符串"default"，则令tryFirst为"string"
  4. 否则如果hint是字符串"number"，则令tryFirst为"number"
  5. 否则，抛出 TypeError 异常
  6. 返回? OrdinaryToPrimitive(O, tryFirst)

此函数的"name"属性值为"[Symbol.toPrimitive]"
此属性具有特性{ [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }

20.4.5 Data实例的属性

Date实例是普通对象，它继承了Date原型对象的属性。Date实例也有一个[[DateValue]]内部插槽。[[DateValue]]内部插槽就是这个Date对象所代表的时间值。
