异步函数概述

异步函数是原型为%AsyncFunction.prototype%的函数对象，具有函数对象所拥有的内部槽和属性，但它不能作为构造函数调用，也没有prototype属性。
和普通函数的区别在于，异步函数代码中的await表达式将使异步函数暂停执行，并将await右侧的值传入Promise.resolve来得到一个Promise对象，在这个Promise对象被解决或拒绝后，将会恢复执行异步函数，并将await表达式的值设为Promise对象的结果值。

异步函数与生成器函数都能够暂停执行，它们的区别在于：

  • 生成器函数的恢复执行需要用户手动控制。
  • 异步函数的恢复执行是自动的，什么时候恢复取决于await表达式创建的Promise对象何时解决或拒绝。

创建异步函数

通过异步函数定义语法

异步函数定义语法执行下列过程：

  • 如果有标识符：
    1. 创建一个原型为%AsyncFunction.prototype%的异步函数对象F，将F.[[Environment]]设为运行中的执行上下文的词法环境（%AsyncFunction.prototype%的原型是Function.prototype）
    2. 注意：异步函数没有[[Construct]]内部槽，不能作为构造函数调用
    3. 注意：异步函数没有prototype属性
    4. 在当前VariableEnvironment词法环境创建一个名为标识符的var变量，其值为函数对象F
  • 如果有可选的标识符：
    1. 创建一个新词法环境funcEnv，其外部词法环境为运行中的执行上下文的词法环境
    2. 创建一个原型为%AsyncFunction.prototype%的异步函数对象F，将F.[[Environment]]设为funcEnv
    3. 注意：异步函数没有[[Construct]]内部槽，不能作为构造函数调用
    4. 注意：异步函数没有prototype属性
    5. 在词法环境funcEnv中创建const变量可选的标识符，将其初始化为函数对象F
    6. 返回异步函数对象F
  • 如果没有可选的标识符：
    1. 创建一个原型为%AsyncFunction.prototype%的异步函数对象F，将F.[[Environment]]设为运行中的执行上下文的词法环境
    2. 注意：异步函数没有[[Construct]]内部槽，不能作为构造函数调用
    3. 注意：异步函数没有prototype属性
    4. 返回函数对象F

通过箭头函数表达式

箭头函数表达式语法执行下列过程：

  1. 创建一个原型为%AsyncFunction.prototype%的异步函数对象F，将F.[[Environment]]设为运行中的执行上下文的词法环境
  2. 注意：箭头函数对象没有[[Construct]]内部槽，不能作为构造函数调用
  3. 注意：箭头函数对象没有prototype属性
  4. 返回函数对象F

注：

  • async，与标识符或(形式参数类表)之间不能有行结束符
  • 标识符或(形式参数列表)，与箭头=>之间不能有行结束符
  • 箭头=>左侧的标识符会被当做形式参数列表：(标识符)，也就是只有一个形式参数
  • 箭头=>右侧的表达式会被当做函数体：{ return 表达式 }
  • 箭头函数F中的arguments、super、this、new.target会被解析为F.[[Environment]]词法环境中的标识符，也就是箭头函数对象创建时的词法环境。

调用AsyncFunction函数会创建一个函数对象，执行过程如下：

  2. 获取要创建的函数的形式参数列表argList：

    • 否则，AsyncFunction的参数个数大于等于2，将除了最后一个参数以外的参数使用逗号连接起来，组成形式参数列表(argList)
  3. 获取要创建的函数的函数体body：
    • 如果AsyncFunction的参数个数为0，则body为空函数体{}
    • 否则，AsyncFunction的参数个数大于等于1，最后一个参数就是函数体的内容，即{最后一个参数}
  4. 创建一个原型为%AsyncFunction.prototype%的函数对象F，其形式参数列表为argList，函数体为body，将F.[[Environment]]设为全局词法环境
  5. 注意：异步函数没有[[Construct]]内部槽，不能作为构造函数调用
  6. 注意：异步函数没有prototype属性
  7. 返回F

调用异步函数

异步函数不能作为构造函数调用，调用异步函数会返回一个Promise，这个Promise表示本次调用异步函数的计算结果。

  • 在异步函数执行到显式或隐式的return，或抛出异常之前，这个Promise处于pending状态。
  • 当异步函数由于执行到显式或隐式的return而结束时，会使用return的返回值去resolve这个Promise。
  • 当异步函数由于未处理的异常而结束时，会使用这个异常去reject此Promise，使得Promise转变为rejected状态，其结果值就是未处理的异常。
  • 在异步函数执行到第一个await、return、未处理的异常时（即第一次暂停或结束异步函数执行上下文），会返回这个Promise。

从执行上下文的角度看，异步函数在执行时遇到await 表达式时会暂停函数执行上下文，并通过Promise.resolve(表达式)创建一个Promise对象，并为这个Promise对象添加响应函数：兑现时的响应是恢复执行异步函数的执行上下文，并将await 表达式的值设为Promise的兑现值；拒绝时的响应是恢复执行异步函数的执行上下文，并在await 表达式处throw出Promise的拒绝值。

函数体中的特殊表达式

与普通函数代码相比，异步函数代码中await表达式，以及return和throw语句具有特殊作用：

  • 在执行到await 表达式时，会先计算表达式的结果，然后通过Promise.resolve(表达式)创建一个Promise对象，并为这个Promise对象通过then添加响应：
    • 兑现时的响应是恢复执行异步函数，并将await 表达式的值设为这个Promise的兑现值；
    • 拒绝时的响应是恢复执行异步函数，并在await 表达式处throw出这个Promise的拒绝值。
  • 在执行到return 表达式语句时（包括执行完函数后隐式的return undefined），会结束执行异步函数，然后使用表达式值去resolve异步函数返回的Promise。
    • 当返回值不是thenable时，这个Promise的状态会变为fulfilled，其结果值为表达式值
    • 当返回值是thenable时，这个Promise的状态和结果值取决于thenable的状态和结果值
  • 在执行到throw语句时，如果异常没有处理，则会结束执行异步函数，然后使用throw的异常对象去reject异步函数返回的Promise，使得异步函数返回的Promise的状态变为rejected，其结果值为抛出的异常对象。如果异常处理了，则继续执行异步函数。

注意：函数的形式参数部分，不能使用await表达式。
