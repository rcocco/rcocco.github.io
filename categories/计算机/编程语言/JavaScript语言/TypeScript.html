TypeScript
2021-03-30|计算机编程语言JavaScript语言

TypeScript是JavaScript的超集，任何合法的JavaScript代码都是合法的TypeScript代码。
TypeScript的目的是为JavaScript提供静态类型检查，使得JavaScript更加清晰明确。
你可以将TypeScript当成静态类型版本的JavaScript，它的独有语法类似于Java和C#。
TypeScript的代码无法直接运行，必须要编译为JavaScript，编译通过后TypeScript中的独有语法例如类型声明等都会消失，因此TypeScript无法检查代码运行时的任何问题。

类型

类型是TypeScript最核心的部分，类型可以理解为所有可能值的集合，因为类型只是集合，所以一个特定的值可以同时属于多个集合。

TypeScript中具有下列类型，可以用于类型声明。

  • string
  • symbol
  • number
  • bigint
  • boolean
  • null
  • undefined
  • any：可以接受任何值的类型
  • unknown
  • never：不能接受任何值的类型
  • void
  • object：接受任何不是6种原始类型（string, number, boolean, symbol, null, undefined）的值
  • 对象类型：{ 属性类型声明 }
  • 数组类型：类型[]或Array<类型>
  • 函数类型：(参数类型声明) => 返回类型，还有其他声明方式。
  • 字面值类型：true、"Hello"、123等等。
  • 并类型：上述类型作并集得到的新类型

前几种都是JS中的原始类型，用法没什么需要特殊说明的，后几种的用法则相对复杂。

只读属性

在属性名前面加readonly可以将这个属性标记为可读属性，属性本身不能修改，但如果属性是一个引用（对象或数组），内部的值是可以修改的。它类似于const，但它只保证在类型检查期间不能为它赋值，而不会更改运行时的任何行为，因此编译为JavaScript后这个属性就变成了普通属性。

注意，readonly只保证不允许通过声明了readonly的类型来修改readonly属性，但如果将对象赋给另一个非readonly的类型，就可以修改这个属性。这种行为类似于C++中指向常量的指针，不能通过它来修改对象的值，但可以通过其他指针来修改对象的值。

对象字面值

将对象字面值作为右值赋值，或者作为参数传递时，如果类型是对象类型，则会进行额外的属性检查，此时会要求对象字面值完全匹配对象类型中的定义，而不仅仅是具有对象类型中的属性，即不能少也不能多。

类型别名

有些类型的声明很长，例如对象类型、函数类型和并类型等，每次书写一长串的声明十分麻烦，为此可以使用类型别名来引用它。
定义类型别名的语法是：type 别名 = 类型

在任何使用类型的地方，都可以使用类型别名。
一个别名只能定义一次，不允许出现同名的类型别名。

泛型类型别名

在类型别名后面可以通过尖括号来指定泛型参数，以此基于泛型来定义类型别名。

泛型类型别名在作为类型指定时，必须显式给出泛型参数。

接口

如果你只想为对象类型命名，也可以使用接口。
定义接口的语法是interface 接口名 对象类型
和类型别名不同，接口只支持为对象类型命名。
同名的接口可以多次定义，所有的定义会被自动合并，和将所有定义放到一个接口内是等价的，因此即便你在两个同名接口定义之间声明了该类型的变量，它也要满足下方接口的要求。

void类型

在函数类型中，如果返回类型是void，它通常表示函数不返回值。没有任何return语句（或者只有return没有值）的函数会被推断为返回void。

但如果返回void的函数是一个函数表达式，或者它是高阶函数中定义的回调函数的返回类型，则不强制要求函数不返回值，实际上返回任何东西都可以。此时void只是表示，函数调用者不会去关注或使用返回值。

never类型

函数返回类型为never表示，这个函数会抛出异常或者终止程序，从而不会存在一个返回值。

对于函数表达式，如果一定会抛出异常，则它的返回类型推断为never。
对于普通的函数，即使一定会抛出异常，返回类型也只会推断为void。

Function类型

Function类型也可以用于表示函数，但它只能表明它可以被调用，具有bind、call、apply等方法，而无法知道其他信息。编译器推断Function类型值的函数调用会返回any。如果函数不返回值，最好用()=>void来明确这一点，而不是用Function。

类型断言

很多浏览器内置方法，例如document.getElementById，TypeScript只能知道它返回的是HTMLElement类型的元素，然而实际返回的可能是它的一个子类，例如HTMLCanvasElement。
当你为一个声明为子类类型的变量，例如HTMLCanvasElement，赋予一个可能返回父类类型的函数返回值时，TypeScript会认为这是错的。

在这种情况下，你可以使用类型断言，将一个值的类型转为更具体的类型（例如子类），或者更不具体的类型（例如父类或any）。类型断言并不改变程序的行为，它只是为了通过TypeScript的编译检查，因此如果你传递了一个错误的类型，运行时仍然会出错。

类型断言有两种等价的语法：值 as 新类型或者<新类型>值

在调用函数时，必须传递必需参数，可以不传递可选参数，如果传递参数，必须是指定的类型或者undefined（可选参数的类型实际上是与undefined联合得到的类型，因为不传递参数时默认值就是undefined）。剩余参数用于收集所有传递给函数但没有命名参数来接收的参数，如果声明了剩余参数，则其类型必须是一个数组，默认是any[]。

我们的意图一般是：回调函数的编写者愿意接受几个参数就写几个参数，而不是回调函数的编写者必须考虑到调用者不传参数的情况。
因此回调函数的调用者应该始终传递所有参数，回调函数的编写者没有接受的参数会自动被忽略掉。

重载函数

使用重载签名可以进行定义重载函数，它的语法是若干个同名、参数不同、没有函数体的函数签名。
重载签名后面紧跟着实现签名，实现签名是和重载签名同名的函数实现，但参数和返回值必须和前面所有的重载签名兼容。这是因为JavaScript没有语言层面的重载，你只能把重载函数的实现写在同一个函数里。

重载签名和实现签名必须连续书写，中间可以有空行，但不能有其他语句。

实现签名看上去是一个函数，但它不能被直接调用，它只是前面所有重载签名的实现。

编写重载函数有三个建议：

  • 尽可能使用接受并类型的参数，而不是重载（因为TypeScript只能将函数调用解析为单个重载，如果传递的参数同时满足两种重载签名，那么就无法进行调用）

有区别的并类型

如果一个并类型中各类型有公共的属性（属性名相同的属性），且可以进行区分，那么通过判断这个公共属性，TypeScript就可以缩小类型。

如果你只使用一种类型，而将类型中的某个属性设为并类型，那么TypeScript是无法根据这个属性来区分其他属性的情况的。

类字段

TypeScript中可以直接在类体中声明实例属性，通常会根据默认值自动推断类型，推断不出的为any。等价于在构造函数中this.属性 = 默认值。没有提供默认值的实例属性，必须在构造函数中初始化。

TypeScript无法知道在构造函数中通过调用其他函数来初始化的属性，它会认为你没有初始化。如果你要执行这种操作，需要在属性名后加一个叹号，以要求TypeScript不检查该属性是否初始化过。

类字段前面可以加readonly使其变为只读的，这会禁止在构造函数以外的地方重新赋值。

构造函数

构造函数和函数相同，区别在于不能有泛型参数，也不能指定返回类型。
构造函数也可以重载。

实现接口

类可以通过implements子句实现接口，这要求类中必须具有接口所定义的实例属性或方法。语法为class C implements A, B。
注意implments并不会自动为类添加任何属性或方法，它只是要求类必须具有属性或方法。

重写继承方法

在TypeScript中，重写继承方法时，方法签名必须兼容基类的方法签名。以便在通过基类变量持有子类实例时，能够正常调用基类中的方法。

成员可见性

TypeScript支持使用public、protected、private声明成员的可访问性。
public成员可以在任何地方访问，不显式书写可访问性修饰符的成员，自动是public的。
protected成员只在类自己以及它的子类中可用，也可以通过子类调用。
private成员只能在自己的类中访问。派生类无法改变其可访问性。
派生类中重写成员会导致可访问性也被重写。

编译

凡是使用了TypeScript语法的代码，都无法直接运行，必须要通过编译器编译为JavaScript，在这个过程中，所有TypeScript语法例如类型注释都被删除了，如果可以通过编译，则说明你书写的TypeScript代码是正确的，如果不能通过编译，则说明存在错误。
同时，由于TypeScript编译器默认会将代码编译为ES3，因此模板字符串等新语法也会被转换，由于现今ES2015已经得到了广泛的支持，因此在编译时可以使用--target将TypeScript代码编译为更高版本的JavaScript，例如tsc --target es2015 input.ts

变量指向的值的类型可以在运行时发生变化的语言叫做动态类型，例如js，python

强类型 vs 弱类型
语言是否支持自动类型转换
