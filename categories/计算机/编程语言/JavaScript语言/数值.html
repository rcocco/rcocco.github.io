类型转换为数字值

使用ToPrimitive(v, hint Number)抽象操作获取参数v的原始值。
如果转换结果是BigInt，则将其进一步转换为能够用64位浮点数表示的数字。
如果转换结果不是BigInt，则使用ToNumber抽象操作将其转换为数字。
当不通过new直接调用时，返回转换后的数字。
当通过new调用时，根据转换后的数字创建Number对象并返回。

有限数字值

有限数字值指不是NaN也不是无穷的数字值，Number类型总共有2^64-2^53（18437736874454810624）个有限数字值，它们是阶码不全为1的所有浮点数，其中一半是负数，一半是正数。每个数字对应于实数集中相同数学值实数。

是否是有限数字值

如果num的类型是Number（BigInt不属于Number），且num不是NaN、+∞或-∞，则返回true。
否则返回 false

等价于Number.isFinite(ToNumber(numOrStr))

是否是整数值

如果num的类型是Number（BigInt不属于Number），且num不是NaN、+∞或-∞，且floor(abs(num)) == abs(num)，则返回true。
否则返回 false

特殊值

NaN

返回NaN

是否是NaN

如果num的类型是Number且num是NaN，则返回true。
否则返回false。

等价于Number.isNaN(ToNumber(numOrStrOrUndefined))，这导致undefined、字符串"NaN"、无法解析为数字的字符串都返回true，BigInt会抛出异常。

等价于Number.isNaN，对于严格相等运算符，只有NaN不等于它自身。

正无穷

安全整数

ECMAScript使用64位浮点数来表示Number值。
在浮点数中，最小的正整数为1，最大的正整数为2^1023 × (2-2^(-52))，但由于浮点数间距在随着阶码增大而增大，而阶码又是2的整数幂，当间距增大到2^1=2后，浮点数中整数集合就变成了实数域中的整数集合的子集，因为它缺少了间距内的整数。这导致实数域中的某些整数无法对应到浮点数中的整数，为此，只能用浮点数中的整数来近似表示这些无法对应的实数域中的整数，这使得一个浮点整数可能对应多个实数域中的整数。

安全整数就是指只对应一个实数域中的整数的浮点整数。即间距小于等于1的那些浮点整数。
为了满足这个条件，对于正整数，整数值必须小于等于2^(n+1)-1，对于负整数，整数值必须大于等于-(2^(n+1)-1)。
在ECMAScript中，n=52，于是安全整数的范围是[-2^53+1, 2^53-1]

是否是安全整数

若Number.IsInteger(num)为true，且abs(number) ≤ 2^53 - 1，则返回true。
否则返回false

最大安全整数

返回2^53-1

最小安全整数

返回-(2^53-1)

最值

最大值

返回Number类型最大的正有限值，其值为2^((2^11-1)-(2^10-1)-1) × (2-2^(-52))，即2^1023 × (2-2^(-52))，约为1.7976931348623157 × 10^308

最小值

返回Number类型最小的正有限值，其值为2^(-52) * 2^(1-(2^10-1))，即2^(-1074)，约为5 × 10^(-324)

Number.EPSILON

表示数字值2^(-52)，这个数是1与大于1的最小的浮点数之差，即阶码最高位为0，其他位为1时的间距。

解析数字字符串

解析整数

按radix进制规则解析string，返回一个数字值。radix可以是数字值，也可以是数字形式的字符串。

  1. radix首先被使用ToInt32抽象操作转为32位有符号整数。
  2. 如果radix属于(-oo, -1]+[1]+[37,+oo)，则返回NaN
  3. 如果radix为0或16，且string以0x或0X开头，则令radix为16。
  4. 如果radix仍为0，则令radix为10。
  5. string首先被调用trimStart()去除开头的空白符，也去掉表示正负的+或-
  6. 然后寻找一个只包含radix进制合法字符的前缀子串。
  7. 如果找不到这样的前缀就返回NaN。如果找到了就将字符串按照radix进制规则解析。由于数字值使用浮点数，因此2、4、8、10、16或32之外的进制可能无法精确表示，会被用近似值替代。

输出数字字符串

十进制科学计数法

返回当前Number值的十进制科学计数法形式，例如1.2e+2
NaN返回"NaN"，无穷返回"Infinity"。
fractionDigits可以是数字也可以是字符串，但必须介于[0, 100]之间，否则抛出异常。

输出字符串的小数点前有且只有一位数字，小数点后有fractionDigits位数字（0位数字会去掉小数点，不足就补0）。
如果没有传递fractionDigits或fractionDigits为undefined，则小数点后应该包含能使数字精确表示的尽可能少的位数。

十进制四舍五入定点小数

返回当前Number值的十进制定点表示法形式，形如123.45或123
NaN返回"NaN"，无穷返回"Infinity"，若当前Number值大于等于10^21，则返回Number.prototype.toString()的结果。
fractionDigits可以是数字也可以是字符串，但必须介于[0, 100]之间，否则抛出异常。

输出字符串小数点后有fractionDigits位数字（0位数字会去掉小数点，不足就补0），根据原数字（指数学值）小数部分的第fractionDigits+1位进行四舍五入。

注2：对于某些数值，toFixed的输出可能比toString更精确，因为toString只打印足够的有效数字（16-17位）来区分相邻的数值。例如
(1000000000000000128).toString()返回"1000000000000000100"
(1000000000000000128).toFixed(0)返回"1000000000000000128"

特定进制表示法

返回当前Number值的radix进制表示形式，形如123.45或7b.66。
radix可以是数字也可以是字符串，但必须介于[2, 36]之间，否则抛出异常。

对于十进制，输出不超过16或17位有效数字（有效数字是指从第一个非零位起的位数，超出位舍入）。如果当前Number值小于10^(-6)，则采用科学计数法输出。例如0.1、123.09999999999998、0.009000000000000001、9e-7、5e-324。

具体而言：

  1. 对于正浮点数x（指精确的数学值），计算满足10^(n-1) ≤ x < 10^n的整数n，可以基于下列方法得到：
    • 若x ≥ 1，则n是x的整数部分的位数，例如123.45的n=3
    • 若0 < x < 1，则n是x第一个非零位的索引（小数点后第一位索引是-1，第二位是-2）加1，例如0.13的n=0。
  2. 选择有效数字位数k（1 ≤ k ≤ 17），若浮点数有效数字大于16位，则根据情况进行舍入（浏览器等实现通常选择k=16或k=17），舍入后k值变小。令s为这k位有效数字组成的字符串。
  3. 按下列格式输出：
    1. 若k ≤ n ≤ 21（大于1的整数），返回${s}${"0".repeat(n-k)}
    2. 若0 < n ≤ 21（大于1的小数），返回${s.substr(0, n)}.${s.substr(n, k-n)}
    3. 若-6 < n ≤ 0（[0.000001, 1)），返回0.${"0".repeat(-n)}${s}
    4. 若k=1，返回${s}e${n-1>0?"+":"-"}${String(abs(n-1)).replace(/0/y), ""}
    5. 返回${s[0]}.${s.substr(1, k-1)}e${n-1>0?"+":"-"}${String(abs(n-1)).replace(/0/y), ""}

负浮点数只需要在前面加字符"-"

十进制给定有效数字的表示法

返回当前Number值的十进制科学计数法形式或十进制四舍五入定点小数形式。
precision可以是数字也可以是字符串，但必须介于[1, 100]之间，否则抛出异常。
NaN返回"NaN"，无穷返回"Infinity"。
若没有传递precision参数或precision为undefined，则返回Number.prototype.toString()

如果当前Number值小于10^(-6)，或大于等于10且整数部分位数大于precision，则采用科学计数法输出，形如1.25e+4，小数点有一位数字，小数点后有precision-1位数字。
否则，采用十进制四舍五入定点小数形式输出，保留原数字（指数学值）的precision位有效数字（按照第precision+1位进行四舍五入）。
