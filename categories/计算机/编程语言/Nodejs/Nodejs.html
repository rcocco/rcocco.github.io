简介

幂等：执行一次和执行多次效果一样

Node最初是用作高性能web服务器，但后来人们发现也可以用作前端构建/前端工程化，如转义(es6到es5，less/sass到css)、压缩与混淆（把易读的源代码转换成几乎不可读的，最主要的手段是改变形参的名字）、打包（把从文件入口开始用到的所有文件打包成一个文件）。
还可以用作爬虫、命令行工具等。
Electron将Node.js和浏览器环境集成到了一个环境

运行Nodejs脚本，使用node a.js命令
在Node控制台中有一些以点.开头的特殊命令：
.break Sometimes you get stuck, this gets you out（例如输错了一半，放弃现在输入）
.clear Alias for .break
.editor Enter editor mode
.exit Exit the REPL
.help Print this help message
.load Load JS from a file into the REPL session
.save 保存在控制台中输入过的所有代码到一个文件

Nodejs提供了一些全局变量：

process提供了多种方式来监听并操作当前程序
process.exit()可以结束进程
process.argv是一个字符串数组，用于表示传递给脚本的命令行参数，数组第一项是node，第二项是脚本的路径，实际参数从第三项即下标为2的项开始。
有一个叫做optmist的库可以用于解析命令行参数：
var args = require('potmist'); args.argv

全局对象global或者globalThis，相对于浏览器中的window

Node自带require函数用于获取模块，文件末尾的扩展名js可以省略。函数的参数是一个表示路径的字符串。
如果参数是一个路径（以.或/开头），则找到对应的文件，将该文件当模块加载并返回其导出
如果参数是一个内置模块的名字，则返回内置模块。
如果参数不是一个路径也不是内置模块的名字，则在./node_modules目录中寻找参数同名文件夹，如果此文件夹内的package.json存在，则根据它的main字段来查找文件。如果package.json不存在，则寻找index.js文件。
如果./node_modules目录没有同名文件夹，则不断向上级目录查找，即../node_modules、../../node_modules一直到根目录。

命令行工具node main.js testString也会使用导出值module.exports = function(string){}

Node代码中通常会编写异步代码，异步性具有传染性，如果一个函数调用了异步函数，那么这个函数自己也会变成异步函数。

文件系统模块

内建模块fs提供了处理文件和目录的函数。
大部分函数同时提供了异步版本和同步版本，异步版本需要将回调函数作为参数传递，同步版本则直接返回结果，同步版本的函数名比起异步版本多了一个Sync。
还有promiese版本

readdir函数返回某个目录下的所有文件和目录名字组成的数组。

stat函数返回某个文件的具体信息，包括文件大小、创建时间等等。

rename函数重命名文件
unlink函数用于删除文件

读取文件

读取文件使用readFile函数，第一个参数是文件路径，第二个参数是可选的编码，如果不指定则默认为读取二进制数据并返回字节数组，第三个参数是读取完成或出错时执行的回调函数。

写入文件

写入文件使用writeFile函数，第一个参数是要保存的文件路径，第二个参数是一个字符串或者一个字节数组，第三个参数是回调函数。
不需要制定编码，如果传递的是字符串则按UTF-8写入，如果传递的是字节数组则按二进制写入。

Buffer

ArrayBuffer表示一块内存，它不能直接读取内容，必须基于视图，即将ArrayBuffer看作什么来读
