fs模块

fs模块用于与文件系统交互。

相同函数的不同版本

同步

函数的同步版本在函数名后加Sync，同步版本会阻塞直到操作完成，异常会立刻抛出，可以用try-catch处理

异步回调

函数的异步回调版本需要传递一个回调函数作为最后一个参数，回调函数接受的第一个参数总是异常，部分函数有其他参数用来接受数据等。若任务成功完成，则第一个参数总是null或undefined

如果要在一个异步任务完成后执行另一个异步任务，需要把第二个异步任务放在第一个异步任务的回调函数内。

异步Promise

函数的异步Promise版本会返回Promise

如果要在一个异步任务完成后执行另一个异步任务，只需要用then串联。

特殊对象

fs.Dirent

该对象表示文件系统中的一个文件或目录，有一些方便的函数用于查看相关信息。
由fs.readdir()返回

判断是否是目录isDirectory()
判断是否是文件isFile()
返回文件的名字name

fs.Stats

该对象表示一个文件的信息，例如大小、时间、链接数量等等，有一些方便的函数用于查看相关信息。
由fs.stat()返回

判断是否是目录isDirectory()
判断是否是文件isFile()
文件大小（字节）size
文件权限：mode，通过(stat.mode & 0o777).toString(8)可以得到3位八进制数。每一位的权限是由可执行（1）、可写（2）、可读（4）相加得到的，类似于进制基数为2。
rwxrwxrwx前面三个表示所有者权限，中间三个表示同组用户权限，最后一组表示其他用户权限。drw-rw-rw-，-rw-rw-rw-，第一位是表示目录的d，没有权限用连字符表示。

列出目录文件

path为目录的路径
可选的options是一个字符串（指定返回的文件名的编码）或对象（指定编码和是否返回文件的fs.Dirent而非文件名的字符串），对象的encoding字段指定编码，对象的withFileTypes指定是否返回fs.Dirent数组。
callback是回调函数，它接受两个参数，第二个参数是目录中文件的名称的数组

创建目录

path为目录的路径
可选的 options 参数可以是整数（指定 mode（权限和粘滞位））、或对象（具有 mode 属性和 recursive 属性（指示是否要创建父目录））。 当 path 是已存在的目录时，调用 fs.mkdir() 仅在 recursive 为 false 时才会导致错误。
callback接受两个参数，第二个参数recursive 为 true 时才会传入创建的第一个目录的路径。

流式读取

fs.createReadStream创建一个可读流。
path是文件路径，options是参数对象，参数对象的highWaterMark字段是缓冲区大小，以字节为单位，设置2*1024*1024共2MB比较合适。

从 fd 指定的文件中读取数据。
fd是整数。
buffer 是数据（从 fd 读取）要被写入的 buffer。
offset 是 buffer 中开始写入的偏移量。
length 是整数，指定要读取的字节数。
position 参数指定从文件中开始读取的位置。 如果 position 为 null，则从当前文件位置读取数据，并更新文件位置。 如果 position 是整数，则文件位置会保持不变。
回调有三个参数 (err, bytesRead, buffer)。
如果文件没有被同时地修改，则当读取的字节数为零时，即到达文件的末尾。
