dependencies字段在使用npm i安装其他包时会自动写入

npm需要在命令行执行，而不是在node shell环境执行。

使用npm安装各种包：npm install package1 package2 ...可以简写为npm i package1 package2 ...默认安装在当前文件夹的node_modules里，以供当前项目require。
npm i -g、npm i --global安装在全局环境，这种方式主要是为了在命令行中提供一个可用的命令。对于require时只有在当前路径找不到依赖时才可能找到全局环境，

figlet包和figlet-cli包的区别：
figlet提供的是在代码中require用的库。
figlet-cli提供的是命令行工具，该工具内部调用了figlet。

安装时可以使用选项--registry=https://registry.npm.taobao.org来指定下载源。
为了免于每次指定选项，可以安装一个cnpm包，用于从淘宝源下载包：
npm install -g cnpm --registry=https://registry.npm.taobao.org
或者基于npm install创建一个带有选项的别名cnpm，来避免每次输入选项。
还有一个mirror-config-china包可以自动配置各个node模块的国内安装源
npm i -g mirror-config-china --registry=https://registry.npm.taobao.org

package.json文件包含了当前模块所依赖的所有其他模块，在安装当前模块时，会自动安装它依赖的所有模块。在当前模块文件夹执行npm install会安装所有依赖。
