stream模块

流（stream）是 Node.js 中处理流式数据的抽象接口。 stream 模块用于构建实现了流接口的对象。
流可以是可读的、可写的、或者可读可写的。 所有的流都是 EventEmitter 的实例。

stream 模块主要用于开发者创建新类型的流实例。 对于以消费流对象为主的开发者，极少需要直接使用 stream 模块。

流可以帮助调度，系统读入数据块，用户负责将其添加到队列

data事件

data事件当流将数据块传送给消费者后触发。
回调函数接受一个(chunk)参数，表示所读取到的数据块，对于非对象模式的流， chunk 可以是字符串或 Buffer。 对于对象模式的流， chunk 可以是任何 JavaScript 值，除了 null。

stream.Writable

drain事件

如果调用 stream.write(chunk) 返回 false，则当可以继续写入数据到流时会触发 ‘drain’ 事件。

向流中写入

返回一个布尔值，表示是否还可以向流缓冲区中写入。当返回false后，建议不要再进行写入（强行写入会导致缓冲区扩大直到内存占满）。之后一旦缓冲区被传输完了，则会触发drain事件，此时才可以继续写入。

对于非对象模式的流， chunk 必须是字符串、 Buffer 或 Uint8Array。 对于对象模式的流， chunk 可以是任何 JavaScript 值，除了 null。

encoding 是一个字符串值，如果 chunk 是字符串，则指定字符编码。默认值: ‘utf8’。
callback 是当数据块被输出到目标后的回调函数。
