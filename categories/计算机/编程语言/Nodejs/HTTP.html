http模块

http模块用于创建 HTTP 的服务器和客户端。

服务端

创建服务器

返回一个http.Server实例，表示一个HTTP服务器。
requestListener是一个函数，将被自动设置为request事件的监听器。监听器函数接受参数(request, response)，分别是http.IncomingMessage和http.ServerResponse的实例
request事件在一个新的请求时触发。对于Keep-Alive连接而言，每个连接可能有多个请求。

开始监听请求

port是要监听的端口，用数字表示。若省略或为0，则由系统随机分配。
host是要监听的地址，用字符串表示。若省略，则监听所有地址。
最后一个参数callback会被设置为listening事件的监听器，listening事件在开始监听后触发

请求对象

request事件的回调函数参数是(request, response)，其中request是表示请求的对象，它是http.IncomingMessage 的实例。

  • 请求方法：request.method
  • 请求路径：request.url
  • 请求HTTP版本：request.httpVersion
  • 请求头：request.headers（对象）

响应对象

request事件的回调函数参数是(request, response)，其中response是表示请求的对象，它是http.ServerResponse 的实例。

设置响应头

response.setHeader方法用于设置响应头，但没有发送。
在发送前都可以再次调用以修改之前设置的响应头。

如果没有设置Content-Length，则会自动设置Transfer-Encoding: chunked进行流式传输。Content-Length 以字节而非字符为单位。 使用 Buffer.byteLength() 来判断主体的长度（以字节为单位）。 Node.js 不检查 Content-Length 和已传输的主体的长度是否相等。

发送响应首部

response.writeHead方法用于发送响应首部，包括状态码、状态消息、响应头。
statusCode是一个3位整数，statusMessage是一个字符串，headers是一个对象。

这个方法在每个响应上只能调用一次。如果在调用此方法之前调用了 response.write() 或 response.end()，则首先会隐式的用已设置的状态码、消息和响应头调用此函数。

通过response.setHeader设置的响应头将与此方法写明的响应头合并，相同字段writeHead方法中的设置优先。

返回response，以便可以链式调用。

发送响应体

此方法发送响应报文的主体。 可以多次调用该方法以提供连续的响应主体片段。
根据规定，当请求是HEAD方法是应忽略响应体，因此不得调用该函数。同样，响应状态码为204或304时也不能包含响应体。

chunk可以是字符串或 buffer。 如果 chunk 是一个字符串，则第二个参数指定如何将其编码为字节流。
当数据块被刷新时将调用callback。

返回布尔值，true表示数据已经被刷新到内核缓冲区。false表示全部或部分数据正在内存中排队，如果缓冲区再次空闲则会触发drain事件。

结束发送

调用此方法表示已发送了所有响应头和主体，服务器应该结束连接。
每个响应最终都要调用此方法。
如果传递了callback，则在响应流完成时调用该回调。
如果传递了data参数，则相当于先调用response.write(data, encoding)再调用response.end(callback)

客户端

发送HTTP请求

url是要请求的链接。
options是请求的具体设置，包括方法、端口、域名、请求头、验证等等。
如果同时指定了 url 和 options，则对象会被合并，其中 options 属性优先。
可选的 callback 参数会作为单次监听器被添加到 ‘response’ 事件。
