求中间值

lo + (hi - lo) / 2，这个式子等价于(lo + hi) / 2，但可以避免lo + hi溢出的情况。
因为(hi - lo)/2 - 1 < Math.floor((hi - lo) / 2) <= (hi - lo)/2，它比(hi - lo)/2小，且小的值为0.5。因此Math.floor((hi - lo)/2) <= lo + (hi - lo)/2 = (lo + hi) / 2，结果是两个数的平均值，如果平均值不是整数，则结果是小于平均值的最大整数（靠左那个数）

最大公约数

计算两个非负整数p和q的最大公约数：
若q是0，则最大公约数为p。
否则，将p处以q得到余数r，q和r的最大公约数，就是p和q的最大公约数
即gcd(p,q)=gcd(q,p%q)

最简分数

两个数同除最大公约数后，它们就是互质的，即公约数为1。
假设p和q同除最大公约数a之后，公约数不为1，将这个公约数设为b。
则a*b也是p和q的公约数，且它比a更大，这与a是最大公约数矛盾。

平均值递推公式

已知前面有N个数，这N个数的平均值为m，又知道新数为x，求新的平均值。
前面N个数的和为N*m，于是新平均值为(N*m+x)/(N+1)=m*N/(N+1)+x/(N+1)，进一步简化得到m*(1-1/(N+1))+x/(N+1)=m-m/(N+1)+x/(N+1)，即m+(x-m)/(N+1)

方差递推公式

已知前面有N个数，这N个数的平均值为m，前n个数与平均值m偏差的平方和为s，又知道新数为x，求新的方差。
s = s + (N-1) / N * (x - m)^2
var = s / (N-1)
推导比较复杂。

双栈算术表达式求值

Dijkstra在60年代使用栈发明了一个简单的算法，用于计算使用括号表示优先级的算术表达式（即a+b*c必须写成(a+(b*c))）。用两个栈，一个保存运算符，一个保存操作数。

  • 将操作数压入操作数栈
  • 将运算符压入运算符栈
  • 忽略左括号
  • 在遇到右括号时，弹出一个运算符，弹出所需数量的操作数，然后将运算符和操作数的运算结果压入操作数栈。注意第一个弹出的操作数是运算符右侧的操作数。

最大子数组

给定数组中点mid，最大子数组要么位于中点及左侧A[low,mid]，要么位于中点右侧A[mid+1,high]，要么跨越中点。
位于中点及左侧，这是一个子问题。
位于中点右侧，这也是一个子问题。
子数组跨越中点，这不是子问题，需要单独的解法：
跨越中点的子数组由A[i,mid]和A[mid+1,j]组成，找出形如A[i,mid]的最大子数组，以及形如A[mid+1,j]的最大子数组，当它们都是最大子数组时，跨越中点的子数组也是最大的。
对于A[i,mid]，从mid开始，向前一个个累加，如果和变大就记录下标i，直到遍历完左侧所有元素，即向左遍历到A[low]。
类似，对于A[mid+1,j]，从mid+1开始，向后一个个累加，如果和变大就记录下标j直到遍历完右侧元素。

a和b其中一个一定小于等于根号N，反证法，假设a和b均大于根号N，则a*b > n，这与a*b=N矛盾。
若a <= sqrt(N)，则N/b <= sqrt(N)，即b >= sqrt(N)。

由算术基本定理可知，任何一个大于1的自然数N，如果N不为质数，那么N可以唯一分解成有限个质数的乘积。

  1. 删去所有2的倍数的数，下一个合数为9=3乘3（由分解定理，构造最小的合数，由于2被删不能参与构造，所以当前最小可用的参与构造的质数是3，因此3乘3是目前最小的合数）
  2. 删去所有3的倍数的数，下一个合数为25=5乘5
  3. 删去所有5的倍数的数（4的倍数包含在2的倍数），下一个合数为49=7乘7
  4. 删去所有7的倍数的数（6的倍数包含在2或3的倍数）

想选出所有质数，可以用:not。
上述筛选合数的一阶逻辑是：选出2的倍数，或3的倍数，或5的倍数，或7的倍数。
对命题否定用德摩根律，即选出非2的倍数且非3的倍数且非5的倍数且非7的倍数。

素数的必要条件是除了2和3之外，它总满足6n+1或6n+5，因为6n被2、3整除，6n+2被2整除，6n+3被3整除，6n+4被2整除。

给定年月日是星期几

1582年开始采用格里高利历，即现行公历。如果反推回公元1年1月1日则是星期一。
但是，教皇格里戈八世在1582年2月24日颁布法令，为了抵消儒略历的错误，永远抹去了1582年10月5日到1582年10月14日。历史上从来不曾有过这10天。1582年10月4日是星期四，它的第二天是1582年10月15日星期五。因此1年1月1日是星期六。
因此下列算法实际只适用于1582年10月15日及之后。

假设没有闰年，每一年都是365天。已知1月1日是周几。

  • 月偏移：一月有31=7*4+3天，每7天又回到和1月1日相同的weekday。还剩3天，所以2月1日的weekday和1月1日往后推3天的weekday一样。
    同理，一月和二月共有31+28=7*8+3天，所以3月1日的weekday和1月1日往后推3天一样。
    以此类推得到一个偏移数组，[0, 3, 3, 6, 1, 4, 6, 2, 5, 0, 3, 5]，表示每个月的第1天的weekday等价于1月1日往后推几天。由此就可以计算出月偏移。
  • 日偏移：偏移是相对于1日经过了几天，因此日偏移是day - 1
  • 年偏移：因为365=7*52+1，所以相同的日期每过一年，weekday会偏移1天，例如2020年11月18日是星期三，那么2021年11月18日是星期四。因此年偏移是相对于公元1年经过了多少年，即year - 1
  • 起始日期是周几：用1-6表示周一到周六，0表示周日。公元1年1月1日星期一，用数字1表示，因此前面所述偏移加1模7就得到所求的日期是星期几。

计算二进制1的个数

递归的思想，count_bits(n) = (n & 1) + count_bits(n >> 1)
n & 1得到最低位的值，最低位是1则是1，最低位是0则是0，然后加上它左边的位表示的1个数，这又回到了原来的问题，可以直接从此前的数组中得到。
通常我们可以把num设为256，构造低8位二进制串的1的个数表，然后每8位计算任意数n的二进制表示中1的个数

排序

冒泡排序

交换逆序对，直到没有逆序对

选择排序

从左到右对于每个位置，每次选出最小的元素放到该位置

插入排序

从索引1的元素开始，每个元素左侧子数组已排序，在已排序区间中从右到左将该元素插入到合适位置，在寻找过程中会将已排序区间中比该数大的元素右移一格。

归并排序

分成两个子数组，分别排序，然后将两个有序数组合并成一个有序数组

快速排序

naive：
在数组中随机选一个数，遍历一遍将数组分成小于、等于、大于该数的三个新数组。
将小于和大于该数的两个子数组分别排序，然后将三个数组直接合并即可。
