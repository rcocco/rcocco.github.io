B.2.3 String原型对象的额外属性

B.2.3.1 String.prototype.substr ( start, length )

substr方法接收两个参数：start和length，并返回将this对象转换为字符串后从索引start开始的length个代码单元的子串（如果length为undefined则一直到末尾）。如果start为负值，则按sourceLength+start处理，其中sourceLength是字符串的长度。结果是一个字符串值，而不是一个String对象。采取以下步骤。

  1. 令O为? RequireObjectCoercible(this值)
  2. 令S为? ToString(O)
  3. 令intStart为? ToInteger(start)
  4. 如果length为undefined，令end为+∞，否则令end为? ToInteger(length)
  5. 令size为S中的代码单元数量
  6. 如果intStart < 0，将intStart设为max(size + intStart, 0)
  7. 令resultLength为min(max(end, 0), size - intStart)
  8. 如果resultLength ≤ 0，返回空字符串
  9. 返回S的从索引intStart开始的连续resultLength个代码单元组成的字符串

注：substr函数是有意通用的，它不要求它的this值是一个String对象。因此，它可以转移到其他类型的对象上作为方法使用。

B.2.5 RegExp原型对象的额外属性

B.2.5.1 RegExp.prototype.compile ( pattern, flags )

使用参数pattern和flags调用compile方法，执行以下步骤：

  1. 令O为this值
  2. 如果Type(O)不是Object或Type(O)是Object但O没有[[RegExpMatcher]]内部插槽，则抛出 TypeError 异常
  3. 如果Type(pattern)为Object且pattern有[[RegExpMatcher]]内部插槽，则
    1. 如果flags不是undefined，抛出 TypeError 异常
    2. 令P为pattern.[[OriginalSource]]
    3. 令F为pattern.[[OriginalFlags]]
  4. 否则：
    1. 令P为pattern
    2. 令F为flags
    3. 返回? RegExpInitialize(O, P, F)

注：compile方法用新的模式和标志完全重新初始化这个对象RegExp。一个实现可能会把这个方法的使用解释为一个断言，即生成的RegExp对象将被多次使用，因此是一个额外优化的候选对象。
