视觉格式化模型 2021-03-26|计算机前端CSS样式语言

CSS中，视觉格式化模型(Visual Formatting Model)描述了用户代理如何处理文档树，以及如何在视觉媒体上显示。
在视觉格式化模型中，文档树中的每个元素根据盒模型生成零个或更多盒子（矩形框），这些盒子的布局受以下因素控制：

  • 盒子的尺寸和类型
  • 定位模式（常规流、浮动和绝对定位）
  • 文档树中元素之间的关系
  • 外部信息（例如视口大小、图片的固有尺寸等等）

HTML概述 2021-03-26|计算机前端HTML语言

元素基础

HTML元素通常包括开始标签、内容、结束标签，结束标签比开始标签多了一个反斜杠\，形如

HTML元素也可以嵌套，但嵌套后的标签顺序必须正确：即内层闭合标签必须在外层闭合标签的前面，例如

<p>今天的<strong>天气</strong>真好</p>

文档结构

HTML文档最基础的结构是以下形式：
<!DOCTYPE html>是文档声明，用来提醒浏览器这是HTML文档。
<html>元素是整个文档的根节点，其他元素要嵌套在它的内部，lang属性声明语言，这里是一个中文页面。
<head>元素用来提供一些页面的元数据信息，例如文件的编码，浏览器应该显示的标题，浏览器应该从何处下载样式表等等。
<body>元素是一切要显示给人看的内容的根节点，需要显示给人看的元素都应该放到<body>内。

<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8">
    <title>网页的标题</title>
  </head>
  <body>
  </body>
</html>

可访问性

使用语义化HTML（有时称为POSH或Plain Old Semantic HTML）十分重要，语义化是指尽可能将正确的HTML元素用于其预期目的，当想用一个元素表示某种目的时，首先考虑具有语义的HTML元素是否满足需求，只有当所有的语义元素都不符合目的时，才考虑使用无语义的<div>和<span>等元素。

虽然可以利用CSS和JavaScript来使几乎任何HTML元素以所希望的任何方式运行，例如一个按钮虽然有符合语义的<button>元素可用，但非要用<div>来实现一样的显示效果与功能也是完全可行的，但是相比起来语义化HTML更有优势。

语义化的优点在于：

  • 易于开发，HTML默认为元素提供了一些合适的样式，也提供了一些辅助功能，例如可以用Tab键在按钮之间导航，开发者不需要花大力气去实现一样的效果。
  • 易于维护，团队的后来者更容易理解原来的开发者想表达的目的
  • 在移动设备上更好，语义化代码通常比非语义化的代码小，可以更快的响应。
  • 有利于SEO，搜索引擎比起非语义化的<div>中包含的关键字，更重视标题或者链接里的关键字。

回退fallback

回退的设计有两种：

  1. 在元素的内容区放置回退内容
    前提是内容区不被使用，否则应该嵌套的元素与回退元素混在一起难以区分。
    这种情况元素主要内容都在属性里。
  2. 在元素外使用另一个元素放置回退内容
    如果内容区有需要嵌套的元素

为了回退不应该设计成自闭合标签，旧浏览器并不知道元素是否是自闭合，当读到一个不认识的开始标签时，它期望一个结束标签。如果是自闭合的，它会把后面所有元素当成该不认识标签的内容。

SVG 2021-03-26|计算机前端HTML语言

https://github.com/tutsplus/how-to-hand-code-svg

基本元素

  • <svg> 包裹并定义整个矢量图。<svg> 标签之于矢量图就如同 <html> 标签之于一个 web 页面。
  • <line> 创建一条直线。
  • <polyline> 创建折线。
  • <rect> 创建矩形。
  • <ellipse>创建圆和椭圆。
  • <polygon>创建多边形。
  • <path>通过指定点以及点和点之间的线来创建任意形状。
  • <defs>定义一个可复用的图形。初始情况下<defs>里面的内容是不可见的。<defs>标签之于矢量图就如同<head>标签之于一个 web 页面。
  • <g>将多种形状组合起来。将组合后的形状置于<defs>中可以让它能够被复用。
  • <symbol>类似于一个组合，但是拥有一些额外的特性。通常被置于 <defs>标签中便于复用。
  • <use>获取在<defs>中定义的复用对象并在 SVG 中显示出来。

React 2021-03-26|计算机前端

JSX

JSX是JavaScript的语法扩展，可以将创建React元素的函数调用，写成类似HTML代码的样子。
每个 JSX 元素只是调用 React.createElement(component, props, ...children) 的语法糖。component是字符串或者自定义元素的构造函数，props对象包括元素上的属性以及值，children是所有的子元素。
因此，使用 JSX 可以完成的任何事情都可以通过纯 JavaScript 完成，可以在Babel官网看到这种转换。
