数组是一个对象，其属性键

数组索引是整数索引，其数值i在+0 ≤ i <2^32-1范围内。
注意：一个字符串属性名P是一个数组索引，当且仅当ToString(ToUint32§)等于P，并且ToUint32§不等于2^32-1
每个Array对象都有一个length属性，该属性的值始终是一个小于2^32的非负整数。

类数组对象：类数组对象是具有"length"属性值的对象，该属性值一般是一个整数（但不能是BigInt类型）。类数组对象的整数索引属性对应的属性值，称为类数组对象的元素。

字面值

可以通过字面值[元素列表]创建数组。

元素列表由一个或多个表达式组成，每个表达式的结果都会作为数组元素。多个元素用逗号分隔，最后一个元素后面的逗号是可选的。
元素也可以留空，留空的元素将导致数组索引增加，但不会在数组中存储任何内容。最后一个元素后面的逗号不会导致数组索引增加。
每个表达式也可以是一个以...开头的返回可迭代对象的表达式，在构建数组的过程中会遍历可迭代对象，每个迭代值都会作为数组元素并递增数组索引。

UTF-8

编码

将字符串str中的某些字符转换为1到4个形如%XX（X是[0-9A-F]）的字符串。
其中，%XX里的XX是用大写16进制表示的该字符的UTF-8编码中的字节。
例如，字符"我"的UTF-8编码为 E4 BD A0，所以该字符会被转换为字符串"%E4%8D%A0"

encodeURI对于a-zA-Z0-9_.!~*'()-和;/?:@&=+$#以外的字符都会进行转换。
encodeURIComponent对于a-zA-Z0-9_.!~*'()-以外的字符都会进行转换。

解码

将字符串str中的1到4个连续的%XX（X是[0-9a-fA-F]）按UTF-8编码转换为对应字符。
其中，%XX里的XX是用16进制表示的该字符的UTF-8编码中的字节，每组连续的%XX必须是合法的UTF-8编码。

decodeURI会将%23、%24、%26、%2B、%2F、%3A、%3B、%3D、%3F、%40（#$&+/:;=?@）以外的%XX转换为对应字符。
decodeURIComponent会将所有%XX转换为对应字符。

代码单元值

编码

将字符串str中的某些字符，按照其代码单元值转换为形如%XX或%uXXXX（X是[0-9A-F]）的字符串。
其中，%XX里的XX，和%uXXXX里的XXXX是用十六进制表示的该字符的代码单元值。
如果该字符的代码单元值小于256，则转换为%XX的形式，不足2位在左侧补0。
如果该字符的代码单元值大于等于256，则转换为%uXXXX的形式，不足4位在左侧补0。

escape对于a-zA-Z0-9@*_+./-以外的字符都会进行转换。

解码

将字符串str中的%XX或%uXXXX（X是[0-9a-fA-F]）转换为代码单元值为XX或XXXX（用16进制表示）的字符。

base64

编码

将字符串str的代码单元值作为编码字节，进行base64编码，返回编码后的base64字符串。
要求str的代码单元值必须介于0x00到0xFF之间，否则抛出异常。

将字符串str的UTF-8编码作为编码字节，进行base64编码，返回编码后的base64字符串。
原理如下：

  1. encodeURI(str)：将str中a-zA-Z0-9_.!~*'()-和;/?:@&=+$#以外的字符转为1到4个表示其UTF-8编码字节的%XX。
  2. unescape(encodeURI(str))：将%XX转换为代码单元值为XX的字符。
  3. btoa(unescape(encodeURI(str)))：断言：此时每一个代码单元值都是UTF-8编码字节。然后将代码单元值进行base64编码。

第一步没有转换的a-zA-Z0-9_.!~*'()-和;/?:@&=+$#字符，其代码单元值都在0x00到0x7F之间，在UTF-8编码中，它的编码字节和代码单元值完全相同。因此在第三步的断言里，这些字符的代码单元值就是UTF-8编码字节。
第一步转换的字符，在第二步被转为1-4个字符，这些字符的代码单元值等于UTF-8编码字节，因此也满足第三步的断言。

解码

将字符串 str按base64规则进行解码，返回一个字符串，其每个代码单元值是解码后的字节。

将字符串str按base64规则进行解码，然后将解码结果当做UTF-8编码字节，转为对应的字符串。
原理如下：

  1. atob(str)：将str按base64规则解码，返回一个字符串，其每个代码单元值都是base64解码后得到的UTF-8编码字节。
  2. escape(atob(str))：将代码单元值对应字符为a-zA-Z0-9@*_+./-以外的字符，按照其代码单元值转换为%XX的形式，其中XX是代码单元值的16进制形式。
  3. decodeURI(escape(atob(str)))：将字符串中1-4个连续的%XX按UTF-8编码转换为对应字符。

第一步执行完后，代码单元为0x00到0x7F之间字符，已经是最终的字符。因为这些字符的UTF-8编码字节值与代码单元值完全相同，所以不需要进一步处理。
第二步中，没有处理的a-zA-Z0-9@*_+./-的代码单元值都在0x00到0x7F之间，不需要转换。其他的UTF-8编码字节被转换为1到4个%XX，其中XX是代码单元值的16进制形式。注意，尽管没有必要，但0x00到0x7F之间，a-zA-Z0-9@*_+./-以外的字符也被转换为了%XX的形式，这将在第三步再次处理。

B.2.1 全局对象的额外属性

B.2.1.1 escape ( string )

escape函数是全局对象的一个属性，它计算出一个字符串值的新版本，其中某些代码单元被十六进制转义序列所取代。

对于那些值为0x00FF或更低的被替换的代码单元，将使用%xx形式的两位数转义序列。对于那些被替换的代码单元值大于0x00FF的字符，使用四位数转义序列%uxxxx。

escape函数是固有对象%escape%。当使用一个参数string调用escape函数时，将采取以下步骤。

  1. 将string设为? ToString(string)
  2. 令length为string中代码单元的长度
  3. 令R为空字符串
  4. 令k为0
  5. 重复，只要k < length
    1. 令char为string中索引k处的代码单元（表示为16bit无符号整数）
    2. 如果char是下列字符串中的代码单元之一"ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789@*_+-./"，则令S为包含单个代码单元char的字符串值
    3. 否则如果char ≥ 256，则
      1. 令n为char的数字值
      2. 令S为连接下列字符串的结果：
        • "%u"
        • n的字符串表示，格式为4位大写16进制数，不足4位在左侧补0
    4. 否则：
      1. 断言：char < 256
      2. 令n为char的数字值
      3. 令S为连接下列字符串的结果：
        • "%"
        • n的字符串表示，格式为2位大写16进制数，不足4位在左侧补0
      4. 将R设为连接字符串：先前的R值和S的结果
      5. 将k设为k + 1
  6. 返回R

注：该编码部分基于RFC 1738中描述的编码，但本标准中规定的全部编码在上面描述，而不考虑RFC 1738的内容。该编码没有反映RFC 3986对RFC 1738的修改。

B.2.1.2 unescape ( string )

unescape函数是全局对象的一个属性。它计算一个字符串值的新版本，其中每个可能被转义函数引入的转义序列会被替换成它所代表的代码单元。

unescape函数是固有对象%unescape%。当用一个参数string调用unescape函数时，会采取以下步骤。

  1. 将string设为? ToString(string)
  2. 令length为string的代码单元数
  3. 令R为空字符串
  4. 令k为0
  5. 重复，只要k ≠ length
    1. 令c为string中索引为k的代码单元
    2. 如果c为代码单元0x0025 (PERCENT SIGN)，则
      1. 如果k ≤ length - 6并且string的索引为k+1的代码单元是0x0075 (LATIN SMALL LETTER U)，并且索引k + 2、k + 3、k + 4和k + 5处的四个代码单元都是十六进制数，则
        1. 将c设为一个代码单元，其值为string中索引为k + 2、k + 3、k + 4和k + 5处的四个十六进制数字表示的整数
        2. 将k设为k + 5
      2. 否则，如果k ≤ length - 3并且string的索引为k+1和k+2处的代码单元都是十六进制数，则
        1. 将c设为一个代码单元，其值为string中索引为k + 1、k + 2处的两个十六进制数字表示的整数
        2. 将k设为k + 2
    3. 将R设为连接字符串：R的先前值、c的结果
    4. 将k设为k + 1
  6. 返回R

B.2.3 String原型对象的额外属性

B.2.3.1 String.prototype.substr ( start, length )

substr方法接收两个参数：start和length，并返回将this对象转换为字符串后从索引start开始的length个代码单元的子串（如果length为undefined则一直到末尾）。如果start为负值，则按sourceLength+start处理，其中sourceLength是字符串的长度。结果是一个字符串值，而不是一个String对象。采取以下步骤。

  1. 令O为? RequireObjectCoercible(this值)
  2. 令S为? ToString(O)
  3. 令intStart为? ToInteger(start)
  4. 如果length为undefined，令end为+∞，否则令end为? ToInteger(length)
  5. 令size为S中的代码单元数量
  6. 如果intStart < 0，将intStart设为max(size + intStart, 0)
  7. 令resultLength为min(max(end, 0), size - intStart)
  8. 如果resultLength ≤ 0，返回空字符串
  9. 返回S的从索引intStart开始的连续resultLength个代码单元组成的字符串

注：substr函数是有意通用的，它不要求它的this值是一个String对象。因此，它可以转移到其他类型的对象上作为方法使用。

B.2.5 RegExp原型对象的额外属性

B.2.5.1 RegExp.prototype.compile ( pattern, flags )

使用参数pattern和flags调用compile方法，执行以下步骤：

  1. 令O为this值
  2. 如果Type(O)不是Object或Type(O)是Object但O没有[[RegExpMatcher]]内部槽，则抛出 TypeError 异常
  3. 如果Type(pattern)为Object且pattern有[[RegExpMatcher]]内部槽，则
    1. 如果flags不是undefined，抛出 TypeError 异常
    2. 令P为pattern.[[OriginalSource]]
    3. 令F为pattern.[[OriginalFlags]]
  4. 否则：
    1. 令P为pattern
    2. 令F为flags
    3. 返回? RegExpInitialize(O, P, F)

注：compile方法用新的模式和标志完全重新初始化这个对象RegExp。一个实现可能会把这个方法的使用解释为一个断言，即生成的RegExp对象将被多次使用，因此是一个额外优化的候选对象。

18 全局对象

全局对象：

  • 是在控制流进入任何执行上下文之前创建的。
